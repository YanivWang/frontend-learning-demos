实例 vm 与 _data 的关系

打开 DevTools → Console，刷新后在 mounted 日志中对比 this.message、this.$data 与 this._data（以及 this.data 为何不存在）；复习要点与面试答法见本区块。

知识点要点

  • Vue2 创建实例时会处理 data：先把返回对象转成响应式，再把每个 key 代理到实例上，所以 vm.message 实际读写的是 vm._data.message。
  • vm.$data 是公开实例属性，指向 Vue 观察过的数据对象；vm._data 是内部实现字段，调试时能看到，但业务代码不应依赖下划线内部属性。
  • vm.data 默认不存在，因为 data 是实例创建选项，不是自动暴露的实例属性。要看原始数据对象，用 this.$data。
  • data 字段不要和 methods、computed、props 等重名。重名会让实例代理含义冲突，轻则出现警告，重则覆盖预期访问路径。
  • mounted 表示当前实例已经挂载到 DOM，但状态变化后的 DOM 更新是异步批量执行的；修改数据后想读取最新 DOM，应放到 this.$nextTick。

面试考点

  • Vue2 为什么能用 this.message 访问 data 字段？ 结论是 Vue2 把 data key 代理到了组件实例上。原因是实例初始化时会为每个 data 字段定义代理 getter/setter，内部转发到 _data。项目里这让模板和 methods 写法简洁，但也要求 key 命名不能和其他选项冲突。
  • vm.$data 和 vm._data 有什么区别？ 结论是 $data 是公开 API，_data 是内部字段。二者通常指向同一个响应式数据对象，但业务代码应依赖公开 API。边界是调试源码或教学时可以观察 _data，生产逻辑不要依赖内部命名。
  • data 字段重名会带来什么问题？ 结论是 data key 若与 methods、computed、props 等同名，会冲突实例代理路径，轻则 Vue 警告，重则覆盖预期访问。项目里 data 字段命名应避开已有选项名。
  • mounted 中访问 DOM 和数据有什么边界？ 结论是首次挂载后的 DOM 可以读，但数据变化后的最新 DOM 要等 $nextTick。原因是 Vue 会异步批量刷新 DOM。项目里集成第三方 DOM 插件可在 mounted 初始化，后续依赖更新后的尺寸或节点时放进 $nextTick。
