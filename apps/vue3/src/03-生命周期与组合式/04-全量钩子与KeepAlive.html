Composition API 中的全部生命周期钩子

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • setup() 承担创建阶段初始化，Composition API 生命周期统一以 onXxx 函数注册，并且要在 setup 同步阶段调用。
  • 挂载阶段是 onBeforeMount / onMounted，更新阶段是 onBeforeUpdate / onUpdated，卸载阶段是 onBeforeUnmount / onUnmounted。
  • 示例中根组件和缓存子组件都会把钩子写入日志，点击切换能看到 DOM 挂载、响应式更新和组件隐藏之间的差异。
  • <KeepAlive> 缓存的组件被切走时不会立即销毁，而是触发 onDeactivated；再次显示时触发 onActivated。只有缓存被移除或容器卸载时才进入 unmount 阶段。
  • 常见坑包括把 onUnmounted 写成 onUnMounted、忘记从 Vue 解构钩子、在 onUpdated 中无条件改状态造成循环更新，以及把暂停逻辑误放到卸载清理里。

面试考点

  • Vue 3 生命周期如何按阶段记忆？ 结论是创建逻辑在 setup，之后按挂载、更新、卸载三组 before / after 钩子记忆。原因是 Composition API 用函数注册替代 Options API 的同名选项。项目里依赖 DOM 的逻辑放 mounted，资源释放放 unmounted。
  • KeepAlive 为什么触发 activated / deactivated？ 结论是缓存组件切换时实例被保留，不会每次销毁重建。原因是 KeepAlive 保存组件实例和状态，再次显示时激活缓存。项目里计时器、轮询、媒体播放等应在 deactivated 暂停，在 activated 恢复。
  • onUpdated 里能改状态吗？ 结论是不应无条件修改会触发本组件更新的状态。原因是 updated 已经在 DOM patch 之后，继续改同一依赖可能形成更新循环。需要响应 DOM 变化时，优先明确触发条件或使用 watch / nextTick。
  • onUnmounted 和 onDeactivated 怎么分工？ 结论是 unmounted 做最终释放，deactivated 做缓存组件临时停用。原因是 KeepAlive 组件停用后还可能回来。项目里 WebSocket 可在 unmounted 关闭，页面不可见时的轮询可在 deactivated 暂停。
