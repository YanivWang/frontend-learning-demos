默认插槽 / 具名插槽 / 作用域插槽

在浏览器中操作下方交互演示；复习要点与面试答法见本复习区。

知识点要点

  • 插槽是 Vue 的内容分发机制：子组件声明 <slot> 出口，父组件把模板片段传进去，从而让结构由子组件控制、具体内容由调用方决定。本页 BaseCard 同时演示三种插槽形态。
  • #header 具名插槽渲染「文章列表」标题；默认插槽（可写 #default 或省略）接收列表项模板； #footer 具名插槽显示「共 N 篇」。点「添加文章」后 footer 的 total 自动更新。
  • 作用域插槽通过 slot props 让子组件把 item 、 total 等内部数据交给父组件渲染。本页默认插槽使用 #default="{ item }" ，父组件决定每行显示 strong 标题 + summary，子组件只负责循环和数据。
  • 插槽内容在 父组件作用域 中编译，因此可以访问父组件变量（如 highlight ），但不能直接访问子组件内部变量；需要子组件主动通过 slot props 暴露。勾选「高亮模式」后标题变红，证明父级状态可参与插槽渲染。
  • 子组件可为插槽提供默认内容：本页 footer 若父组件未传 #footer ，会显示「（默认 footer：暂无统计）」；header 同理。组件库设计插槽时要稳定命名和 slot props 结构。
  • 动态组件与插槽组合见 02-动态组件与异步组件 ；列表数据通信见 04-props与emit 。

面试考点

  • 默认插槽、具名插槽和作用域插槽分别解决什么问题？ 默认插槽解决单出口内容组合，具名插槽解决多区域内容分发，作用域插槽解决「子组件提供数据、父组件决定渲染」。本页一张卡片同时包含 header / default / footer 三种出口。
  • 为什么插槽内容访问的是父组件作用域？ 结论是插槽内容由父组件模板编译，变量解析发生在父组件上下文。本页 highlight checkbox 在父组件定义，插槽内 :style 可直接使用。子组件数据须通过 :item="item" 等 slot props 暴露。
  • 作用域插槽和传组件 props 有什么区别？ 作用域插槽更适合组件库提供渲染扩展点，例如表格单元格、列表项。传 render props 或固定子组件类型适合结构固定的场景。项目里调用方需要完全控制一块模板时，作用域插槽通常比塞很多配置项更清楚。
  • 插槽 API 的常见坑是什么？ 结论是命名、默认内容和 slot props 契约要稳定。slot props 改名会直接破坏调用方模板。缺省插槽内容可避免调用方漏写时页面空白；本页取消 footer 传参可观察默认 footer。
  • 插槽与 scoped CSS 会冲突吗？ 插槽内容在父作用域编译，子组件 scoped 样式默认不影响插槽内容（除非 :deep）。组件库文档应说明样式责任边界，避免调用方假设子组件 scoped 会作用于插槽内元素。
