创建对象的多种方式

请打开 DevTools Console 查看输出。

知识点要点

  • 对象字面量优先： {} 是创建普通对象最常见、最清晰的写法； new Object() 行为相近，但在业务代码里通常没有必要。
  • 原生构造函数： Object 、 Array 等是函数， new Object() / new Array() 创建的对象 typeof 均为 "object" ；实例的 [[Prototype]] 分别链到 Object.prototype / Array.prototype 。
  • 自定义构造函数： 需要批量创建同类对象时用构造函数或 class ；实例属性写在构造函数中，共享方法挂到 prototype 。
  • constructor 边界： constructor 通常来自实例 [[Prototype]] （即构造函数的 prototype ），原型被替换或手动改写后可能不准；类型判断更常用 instanceof 、 Object.prototype.toString.call 或业务判别字段。
  • 属性删除： delete obj.key 会移除自有属性， obj.key = undefined 只是把值变成 undefined ， "key" in obj 仍为 true。
  • 动态扩展： 对象可随时用 . 或 [] 添加新属性； delete 对 var/let/const 声明的变量无效（严格模式下对变量使用 delete 会 SyntaxError）。

面试考点

  • 对象字面量、new Object、自定义构造函数如何取舍？ 普通配置/数据用对象字面量，同类实例用构造函数或 class 。 new Object() 更多是理解语言模型，不是项目首选；需要复用行为时要考虑原型方法共享。
  • constructor 为什么不适合作为唯一类型判断？ 因为 constructor 来自原型链，原型对象可以被替换或篡改。生产判断更依赖 instanceof 、 Array.isArray 或明确的业务类型字段。
  • delete 和 obj.key = undefined 有什么区别？ delete 移除属性本身， in 和 Object.keys 都会反映变化；赋值 undefined 只是清空值，键仍存在。需要表达「没有这个字段」时用 delete 或构造新对象更清晰。
  • 为什么构造函数中的方法不建议每个实例都创建一份？ 函数也是对象，每次 new 都创建新函数会浪费内存，也让方法引用不相等。共享行为应放到 prototype 或 class 方法上，实例只保存各自状态。
  • instanceof 与 constructor 各适合什么场景？ instanceof 沿 [[Prototype]] 链查找构造函数 prototype ，适合判断实例归属； constructor 可作辅助理解，但不可单独作为可靠依据。
