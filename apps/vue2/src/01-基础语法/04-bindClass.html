v-bind:class 三种写法（对象 / 对象引用 / 数组）

在 app1 / app2 / app3 分别点「点击」切换 active、text-danger 类名；观察静态 class="yellowBg" 与 :class 合并效果。

知识点要点

  • :class 是 v-bind:class 的简写。Vue 2 对 class 做了特殊增强，绑定表达式除了字符串，还可以返回对象或数组，最后统一合并成元素的 class 属性。
  • 对象语法 :class="{ active: isActive }" 的 key 是真正要输出的类名，value 按真值判断是否保留该类名；它可以和普通 class="yellowBg" 共存，Vue 会合并而不是覆盖。
  • 对象引用和 computed 返回对象适合条件较多的场景，例如 :class="classObj"。这样模板只描述绑定入口，具体条件集中在 JavaScript 中，后续加错误态、选中态、禁用态更容易维护。
  • 数组语法 :class="[activeClass, errorClass]" 适合组合多个类名变量；数组里也能写三元表达式或对象。注意 { active: isActive } 输出的是 active，而 { activeClass: isActive } 输出的是字面类名 activeClass。
  • Vue 2 的响应式依赖需要在实例创建前声明在 data 或 computed 依赖里；后续动态新增字段会遇到 Vue 2 响应式限制。维护旧项目时可以继续使用这些写法，新项目选型应优先 Vue 3。

面试考点

  • Vue 2 的 :class 对象语法和数组语法分别适合什么场景？ 结论是对象语法适合“某个类是否出现”，数组语法适合“组合多个类名来源”。原因是对象 key 直接对应类名，value 负责开关；数组元素可以是字符串、变量、对象，便于把基础类、状态类、错误类组合起来。项目里简单状态直接写对象，复杂状态通常抽成 computed 返回对象或数组。
  • 静态 class 和 :class 同时存在时 Vue 会覆盖还是合并？ 结论是会合并，不会用动态 class 覆盖静态 class。原因是 Vue 对 class 绑定做了特殊处理，会把普通属性和绑定结果归一成最终类名列表。项目里常把布局、基础样式写在静态 class，把 active、error、disabled 这类状态交给 :class。
  • 为什么复杂 class 条件通常放到 computed 里？ 结论是为了让模板保持声明式和可读。原因是模板里嵌套太多三元表达式或布尔判断会把展示结构和状态规则揉在一起，后期排查样式状态很困难。边界是只有一两个简单开关时直接写对象更清楚，不必为了抽象而抽象。
  • 数组语法里写 { active: isActive } 和 { activeClass: isActive } 有什么区别？ 结论是前者输出类名 active，后者输出类名 activeClass。原因是对象语法的 key 是类名字面量，不会再去 data 里取同名变量。项目里如果类名来自变量，应写成数组项 activeClass；如果是固定类名的条件开关，才写对象 key。
  • 维护 Vue 2 项目时如何说明 :class 与 Vue 3 选型边界？ 结论是 Vue 2/3 的 :class 对象/数组语法基本一致，差异在响应式底层与项目选型。原因是 Vue 2 已 EOL，新项目应优先 Vue 3；存量项目继续用 :class 维护样式状态即可。边界是 Vue 3 不再推荐 filters，复杂样式逻辑仍应抽 computed。
