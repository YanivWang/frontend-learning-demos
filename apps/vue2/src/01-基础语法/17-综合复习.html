综合复习（computed / methods / watch / 局部组件 / 简写）

打开 Console，修改下方 input 的 msg：观察 computed 多次读取只打印一次、methods 每次渲染都打印；对比 nowLabel （computed）与 nowMethod() 缓存差异。

知识点要点

  • computed 声明的是可缓存的派生状态，模板中写 {{ reverseMsg }} 即可读取；只有它依赖的响应式数据变化时才会重新求值。
  • methods 声明的是方法，模板里要写 {{ reversMsgMethod() }} 才会调用；每次渲染都会重新执行，所以不要把纯派生状态长期写成方法。
  • watch 观察某个响应式数据的变化，适合异步请求、日志、持久化、昂贵计算的调度等副作用；如果只是把 msg 反转给模板显示，优先用 computed 。
  • components: { componenta } 是 ES2015 对象属性简写，等价于 { componenta: componenta } ；局部注册只影响当前组件模板，子组件要使用另一个局部组件时必须在自己的 components 中再次声明。
  • 避免在同一个 Vue 2 选项对象里让 computed 与 methods 使用同名 key，否则模板读取时会出现覆盖或警告；Vue 2 已进入历史版本语境，维护旧项目可继续理解这些规则，新项目通常优先 Vue 3。

面试考点

  • computed、methods、watch 怎么选？ 结论是派生值用 computed ，用户动作或普通函数逻辑用 methods ，数据变化后的副作用用 watch 。原因是 computed 有依赖缓存， methods 每次渲染都会执行， watch 能拿到新旧值并承载异步或外部影响。项目里常见边界是：列表筛选结果用 computed，点击提交用 method，搜索词变化后防抖请求接口用 watch。
  • 为什么 computed 不加括号而 methods 要加括号？ 结论是 computed 暴露成实例上的属性， methods 暴露成实例上的函数。Vue 会为计算属性创建 getter 并在访问时按依赖缓存结果，而方法只有被调用才执行。易错点是把同一个名字同时放进 computed 和 methods ，会让模板含义变得不清楚。
  • 局部组件注册的作用域是什么？ 结论是局部组件只对注册它的那个组件模板可见。原因是 components 选项属于当前组件的解析上下文，不会自动向子组件继续传播。项目里如果 componentb 的模板要渲染 componenta ，就要在 componentb.components 再注册一次。
  • methods 和 computed 同名会带来什么问题？ 结论是模板读取时含义不清，可能触发覆盖或警告。原因是两者都挂载到实例上但访问方式不同（属性 vs 函数调用）。项目里应严格区分命名，派生值放 computed，动作放 methods。
  • 维护 Vue 2 时如何回答新项目选型？ 结论是 Vue 2 规则仍然值得维护旧系统时掌握，但新项目通常应优先 Vue 3 与 Vite。原因是 Vue 2 已进入 EOL 后的历史版本语境，生态与官方推荐都转向 Vue 3。边界是存量项目迁移要评估依赖、IE 兼容和构建链，而不是只因为语法熟悉就继续新开 Vue 2。
