Object.defineProperty 实现访问器属性（Vue2 响应式核心）

在下方输入新 name 并点「写入 name」观察 setter；点「读取 name」触发 getter；日志区会记录每次访问， 可对照为何 setter 内不能写 this.name（会递归触发自身）。

知识点要点

  • Object.defineProperty(obj, key, descriptor) 可以精确控制属性描述符。数据属性使用 value/writable，访问器属性使用 get/set，两组不能混用。
  • 访问器属性本身不自动存值，示例用闭包变量 _name 保存真实值。setter 中如果写 this.name = newValue，会再次触发同一个 setter，最终造成递归和栈溢出。
  • Vue2 的响应式就是在初始化时把 data 中已有字段转换为访问器属性：getter 收集依赖，setter 比较新旧值后通知 watcher 更新。
  • defineProperty 的边界是“只能拦截已经定义过的属性”。运行时新增对象字段、删除字段、直接改数组索引和 length 都需要 Vue2 额外 API 或数组方法配合。
  • 现代普通业务代码更常用对象字面量 get/set 或类访问器；只有需要改描述符、兼容旧框架原理或元编程时，才直接使用 Object.defineProperty。

面试考点

  • 数据属性和访问器属性有什么区别？ 结论是数据属性直接保存值，访问器属性通过 getter/setter 计算或拦截值。原因是二者使用不同的属性描述符字段，value/writable 不能和 get/set 同时出现。项目里做普通数据建模多用数据属性，做拦截、校验、响应式原理演示才用访问器属性。
  • getter/setter 里为什么不能直接读写 this.name？ 结论是会递归触发自身。setter 中执行 this.name = value 等于再次调用 name 的 setter，getter 中读 this.name 也会再次调用 getter。正确做法是把真实值放到闭包变量、Symbol key 或另一个不冲突的字段里。
  • Vue2 为什么能用 defineProperty 做响应式？ 结论是 Vue2 通过 getter 收集依赖，通过 setter 通知更新。原因是模板渲染或 computed 读取数据时会触发 getter，Vue 能记录当前 watcher；数据写入触发 setter 后，就能通知这些 watcher。边界是这要求属性在初始化时已经存在。
  • defineProperty 响应式有哪些天然限制？ 结论是只能拦截已有属性，无法天然感知新增/删除属性和数组索引变化。原因是每个 key 都要单独定义描述符。Vue2 因此需要 Vue.set 和数组变更方法；Vue3 Proxy 能从对象层面拦截更多操作。
