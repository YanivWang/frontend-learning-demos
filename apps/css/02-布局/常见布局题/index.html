常见布局题

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出布局尺寸实测值。

知识点要点

  • 单个盒子的水平垂直居中，现代写法优先 display: grid; place-items: center 或 flex 的 align-items + justify-content 。
  • absolute + transform 不依赖 flex/grid，适合维护旧项目或需要相对定位点的场景；代价是元素脱离普通流，周围布局不会为它预留空间。
  • 两栏/三栏布局常用 flex：侧栏 flex: 0 0 固定宽度 ，主栏 flex: 1 ；主栏需要 min-width: 0 ，否则长内容可能把容器撑破。
  • 粘性 footer 不是 position: fixed 。它通过页面容器 min-height: 100vh 、纵向 flex 和 main { flex: 1 } 在内容少时贴底、内容多时自然下推。

面试考点

  • 水平垂直居中有哪些现代写法？ 结论是单元素居中优先 Grid 的 place-items:center 或 Flex 的双轴对齐。原因是它们不需要知道子元素尺寸，也不会脱离普通流。边界是旧项目或特殊定位点场景仍可能使用 absolute + transform 。
  • 两栏/三栏布局为什么要给主栏 min-width:0 ？ 结论是为了允许 flex 主栏小于内容的默认最小宽度。原因是 flex item 默认 min-width:auto ，长文本、表格和代码块会撑破容器。项目里自适应内容区通常同时处理溢出省略、换行或横向滚动。
  • 粘性 footer 和 fixed footer 有什么区别？ 结论是粘性 footer 属于正常文档流，内容少时贴近视口底部，内容多时跟随页面被推下去；fixed footer 永远固定在视口底部。原因是粘性 footer 依靠 flex 分配剩余高度。移动端固定底栏要额外处理安全区和内容遮挡。
