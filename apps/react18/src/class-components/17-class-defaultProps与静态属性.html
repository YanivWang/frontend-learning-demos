defaultProps、静态属性与 displayName

请打开 DevTools Components 面板查看 displayName；对比三组 Greeting 的 name/level 输出（含 null 不触发 defaultProps）；下方 #root 为 React 挂载演示区。

知识点要点

  • Component.defaultProps 会在 prop 未传或显式为 undefined 时提供默认值。本 demo 中 <Greeting /> 没传 name 和 level，因此显示"访客"和 1。
  • null 不会触发 defaultProps，因为它表示调用方明确传入空值。业务上如果 null 也要兜底，应在渲染逻辑中显式处理。
  • static 属性属于类构造函数本身，例如 Greeting.species，不是每个组件实例自己的字段。它适合放组件级元信息、常量或 HOC 需要复制的静态成员。
  • displayName 影响 DevTools、错误栈和 HOC 包装后的调试名称。HOC 返回匿名 wrapper 时应设置类似 WithLoading(GreetingCard) 的名称。
  • React 18 中 class 组件仍可使用 defaultProps。函数组件新代码通常用 JavaScript 参数默认值或解构默认值。

面试考点

  • class 组件 defaultProps 在什么情况下生效？ 结论是 prop 缺失或值为 undefined 时生效。原因是 React 在把 props 交给组件前会补默认值；边界是传 null 不会触发默认值。
  • defaultProps 和参数默认值分别适合哪些代码？ 结论是 class 维护代码常见 defaultProps，函数组件新代码更常见参数默认值。原因是 class 没有函数参数解构位置；项目迁移时不要为了统一写法大范围改动稳定旧组件。
  • static 静态属性和实例属性有什么区别？ 结论是 static 挂在类上，实例属性挂在每个实例上。原因是它们的访问路径不同：Greeting.species 访问静态属性，this.state 访问实例状态。
  • displayName 为什么常和 HOC 一起使用？ 结论是它能保留调试可读性。原因是 HOC 会返回新的包装组件，不设置名称时 DevTools 里可能只看到匿名 wrapper；项目排查错误栈时会更难定位原组件。
  • props 传 null 时 defaultProps 会不会兜底？ 结论是不会。原因是 null 表示调用方明确传了空值，React 认为这不是"未提供"；若业务需要 null 也显示默认值，应在 render 中写 {name ?? "访客"}。
