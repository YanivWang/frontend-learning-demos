React 元素与渲染（React 18 createRoot）

请打开 DevTools Console 查看 props 与 element 结构输出；下方 #root 为 React 挂载演示区。

知识点要点

  • JSX 是 JavaScript 的语法扩展。Babel 转译有两种 runtime： classic 生成 React.createElement(type, props, ...children) ；现代工程默认的 automatic 生成 react/jsx-runtime 中的 jsx() 调用。浏览器不能直接理解 JSX；本 demo 用 babel.min.js 做运行时转译（classic），只适合学习演示。
  • React.createElement 返回 React element：描述 UI 的不可变普通对象（含 type 、 props 、 key 、 ref ），不是真实 DOM。ReactDOM 根据 element 树创建、对比并最小化更新浏览器 DOM。
  • React 18 客户端入口为 ReactDOM.createRoot(container).render(element) 。旧版 ReactDOM.render 已废弃，在 React 18 中会以 React 17 模式运行，无法使用 React 18 新根 API 相关特性。
  • 自定义组件名必须大写，例如 <App /> 。小写 <ele1 /> 等价于 createElement('ele1') ，会渲染为未知 HTML 标签， 不会 抛 ReferenceError。花括号表达式 {ele1} 在变量未定义时才会 ReferenceError 。
  • 挂载时须传入 React element： root.render(<App />) 正确； root.render(App) 传入函数引用会报 Functions are not valid as a React child 。

面试考点

  • JSX、React.createElement 和 React element 是什么关系？ 结论是 JSX 被 Babel 转译为 createElement （classic）或 jsx() （automatic），调用结果是 React element。原因是 React 用 element 描述 UI，渲染器（ReactDOM）负责落到具体平台；浏览器 DOM 只是目标之一。
  • React 18 为什么使用 createRoot 而不是 ReactDOM.render？ 结论是 createRoot 是 React 18 新客户端根 API， ReactDOM.render 已废弃。原因是 React 18 的新根特性（如更广范围的自动批处理）依赖 createRoot ；并发渲染需通过 startTransition 等 API 按需启用，不是调用 createRoot 就全面开启。
  • React element 和真实 DOM 节点有什么区别？ React element 是不可变描述对象，真实 DOM 是浏览器节点。原因是 React 先 diff element 树，再最小化更新 DOM；不要把 element 当 DOM 节点去 appendChild 。
  • 自定义组件为什么必须以大写字母开头？ 结论是 React 用大小写区分组件与内置标签。小写 <div> 的 type 是字符串 'div' ；大写 <App> 的 type 是组件函数本身。
  • JSX 表达式里引用未定义变量会发生什么？ 结论是渲染阶段求值 {变量名} 时若绑定不存在，会抛 ReferenceError ，当前渲染失败。注意与 <ele1 /> 区分：后者是合法 JSX，只是不会调用你的组件。
  • root.render(<App />) 和 root.render(App) 有什么区别？ 结论是前者传入 React element，后者传入函数引用。原因是 render 需要的是描述 UI 的 element/node，不是组件函数本身；传 App 会报函数不能作为 React 子节点。
