函数声明、函数表达式、Function 构造器、严格模式 this

请打开 DevTools Console 查看输出。

知识点要点

  • 函数声明：整体提升，可具名。函数表达式：const fn = function() {}，按变量规则提升。
  • Function 构造器：new Function('a','b','return a+b') 动态创建，函数体是字符串，无法闭包外层词法作用域。
  • 函数是对象，变量名只是指向函数对象的引用；devide = null 后 devideNew 仍可用。
  • fn.length 表示形参个数（不含剩余参数）；arguments 收集实参（见 arguments）。
  • 严格模式下，独立调用函数时 this === undefined；非严格模式为全局对象（浏览器中 window）。

面试考点

  • 三种创建函数的方式？ 声明（提升）、表达式（灵活/可匿名）、Function 构造器（动态但无词法闭包、有 eval 风险，少用）。
  • 严格模式下 this？ 独立调用为 undefined；作为对象方法调用仍指向该对象；call/apply/bind 可显式指定。
  • Function 构造器为何不推荐？ 无法访问定义处的外层变量、性能差、等同 eval 有安全风险。动态逻辑优先用普通函数 + 闭包。
