CSS 单位与函数：px、em、rem、calc、clamp、var()

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出实际测量值。

知识点要点

  • px 是 CSS 绝对长度单位；em 用在 font-size 上时参考父元素字体大小，用在其他长度属性上通常参考当前元素已计算字体大小；rem 始终参考根元素（<html>）字体大小。
  • 百分比不是永远参考父元素宽度，它的参考对象由具体属性定义；本例卡片宽度的百分比来自包含块宽度，而 vw/vh 来自视口尺寸，适合视口相关区域但在移动端地址栏变化时要谨慎。
  • calc() 允许不同单位在同一个属性值里按表达式计算，例如 calc(50% - var(--gap) / 2)；表达式结果必须适合当前属性，单位不兼容或语法无效会导致声明失效。
  • clamp(min, preferred, max) 会把首选值限制在最小值和最大值之间，常用于流式字号、容器宽度和间距，让布局随视口变化但不突破可读性边界。
  • CSS 自定义属性以 --name 定义，用 var(--name) 读取，默认会继承；实践中适合沉淀颜色、间距、圆角等设计 token，但要提供 fallback 或保证作用域内有定义。

面试考点

  • em 和 rem 的计算基准有什么区别？ 结论是 em 跟当前字体环境走，rem 跟根元素字体走。原因是 em 用在 font-size 上时参考父级，嵌套组件里可能层层放大；rem 更稳定，适合全局字号和间距体系。项目里组件内部相对当前文字的 padding 可用 em，全站设计 token 更常用 rem。
  • 百分比、vw/vh 和容器宽度在响应式布局里如何选？ 结论是跟随父容器用百分比，跟随浏览器视口用 vw/vh，跟随组件容器则优先考虑容器查询或基于布局上下文计算。原因是百分比的参考对象由属性决定，而视口单位不关心父容器。边界是移动端浏览器 UI 会改变可视视口，满屏高度场景要测试 vh 行为或使用现代动态视口单位（dvh 等）。
  • calc() 与 clamp() 分别适合解决什么问题？ 结论是 calc() 解决混合单位和变量运算，clamp() 解决响应式值的上下限。原因是 calc() 返回一个表达式计算结果，clamp() 等价于 max(min, min(preferred, max))。项目里两列卡片可用 calc(50% - gap)，流式标题可用 clamp(1.25rem, 2vw + 1rem, 2rem)。
  • CSS 变量在主题系统里有哪些继承和失效边界？ 结论是 CSS 变量适合主题切换和局部覆盖，但要管理作用域和 fallback。原因是自定义属性会继承，并在 var() 替换时才校验最终属性值；变量未定义或替换后类型不合法会让声明失效。项目里通常把基础 token 放在 :root，组件内覆盖局部 token，并用 var(--brand, #2f80ed) 兜底关键值。
