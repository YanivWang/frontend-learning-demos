5 条竖条横向并排呼吸 loading

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出各竖条 animation-delay 与 transform 实测值。

知识点要点

  • 这个 loading 由 5 个竖条组成，所有竖条复用同一个 @keyframes loadAni ，再用不同 animation-delay 让动画依次开始。
  • animation-delay 的正值表示等待一段时间再播放；如果使用负值，动画会立即开始但从周期中间位置切入，常用于让首屏不从静止状态统一起跑。
  • 持续循环的 loading 不建议反复动画 height 、 margin 这类影响布局的属性；本页用 transform: scaleY() 模拟伸缩，视觉相近且更符合现代 CSS 动画实践。
  • transform-origin: center 让竖条从中心缩放，适合呼吸感；如果要从底部向上伸缩，可改成 transform-origin: bottom 。
  • loading 是状态提示，不应无限阻塞用户；真实项目还需要处理请求失败、超时、重复触发和 prefers-reduced-motion 等边界。

面试考点

  • 多个竖条怎样只写一套关键帧？ 结论是把形变和颜色变化写进同一个 @keyframes ，每个竖条引用同名动画，再用递增的 animation-delay 错开节奏。原因是 delay 改变的是播放起点时间，不需要复制多份关键帧。项目里常把间隔抽成 CSS 变量或用预处理器生成，避免手写大量 nth-child。
  • 为什么持续 loading 不推荐动画 height ？ 结论是优先用 transform 模拟伸缩。原因是 height 、 margin 会影响盒模型和周围布局，循环执行时更容易带来重排和重绘压力。边界是如果动画本身必须改变真实布局，就要控制元素数量和频率，不能把它当纯装饰动画处理。
  • animation-delay 的正负值怎么理解？ 结论是正值延后开始，负值立即开始但跳到动画周期中对应进度。原因是 delay 是从动画应用到元素那一刻计算的时间偏移。项目里骨架屏或 loading 想避免所有元素同步起步时，可以用递增 delay 或负 delay 做自然错峰。
  • loading 动画有哪些工程边界？ 结论是 loading 只能表达“正在处理”，不能替代错误处理。原因是网络请求、异步任务和用户取消都可能失败或超时。项目里要配合超时提示、重试按钮、禁用重复提交，并用 prefers-reduced-motion 降低非必要动效。
