全局组件 vs 局部组件 + 子组件嵌套使用

对比两节演示：第一节 load-page-b 内嵌全局 load-page-a ；第二节 load-page-d 在自身 components 中声明后才可使用局部 load-page-c 。

知识点要点

  • Vue.component 是全局注册。本页的 load-page-a 与 load-page-b 在根实例创建前注册，所以 load-page-b 的模板里可以直接使用 load-page-a 。
  • 局部注册写在某个组件或根实例的 components 选项里，只对当前模板可见。本页根实例注册了 load-page-d ，因此根模板能用它。
  • 局部组件不会自动向子组件树透传。 loadPageD 想在自己的模板中使用 load-page-c ，必须在 loadPageD.components 中声明。
  • 全局注册必须早于模板编译，否则 Vue 在解析自定义标签时找不到组件定义。实际项目里基础按钮、图标可全局注册，业务组件建议就近局部注册。
  • Vue 2 的 DOM 模板会受 HTML 大小写规则影响，组件名推荐 kebab-case；在构建型 SFC 中也要保持命名清晰，避免注册名和使用名不一致。

面试考点

  • 全局注册和局部注册的可见范围有什么区别？ 结论：全局注册对之后创建的根实例和子组件树可见，局部注册只对声明它的当前组件模板可见。原因是两者进入不同注册表。项目里全局注册适合基础组件，局部注册适合业务组件。
  • 为什么局部组件不会自动透传给子组件树？ 结论：局部注册不是依赖注入，只声明当前模板可解析哪些标签。原因是每个组件有自己的编译上下文。嵌套使用时应在真正使用该标签的组件里注册。
  • 组件嵌套时在哪里注册被嵌套组件？ 结论：在使用它的组件的 components 选项中声明。原因是 Vue 只有在编译当前模板时能查到注册名才会当组件解析。边界是 DOM 模板建议使用 kebab-case，避免大小写被浏览器改写。
  • 大型项目里为什么不建议把所有业务组件都全局注册？ 结论：会让依赖关系隐式化。原因是任意模板都能用全局组件，读代码时难判断来源，也不利于按需拆分。旧 Vue 2 项目可保留基础组件全局注册，新代码应偏向显式局部依赖。
