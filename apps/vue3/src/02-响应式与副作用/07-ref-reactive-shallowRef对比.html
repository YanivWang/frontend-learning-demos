ref / shallowRef / reactive 三种响应式 API 对比

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • ref() 在 JS 中读写必须用 .value，模板顶层自动解包，本页 {{ message }} 即示例。点“修改 message”会整体替换 .value，控制台打印 [ref]。
  • ref 包裹对象时默认深层响应：改 objRef.value.form.name 会更新视图，本页“切换 objRef name”按钮即验证。
  • reactive() 返回深层 Proxy，访问字段无需 .value；直接解构会丢响应性，见 01-ref-reactive与解构 与 03-toRef-toRefs。
  • shallowRef() 只追踪 .value 引用替换：改嵌套 form.name 通常不更新模板，整体替换 .value 才会触发；与 shallowReactive 的“浅”对比见 04-readonly-shallow-markRaw。
  • 已知内部字段变更但仍用 shallowRef 时，可调用 triggerRef(shallowObjRef) 手动通知依赖更新；本页“triggerRef 刷新”按钮演示该逃生舱，生产仍应优先整体替换或改建模方式。
  • 选型口诀：可替换整块数据用 ref，稳定对象模型用 reactive，大对象/外部实例/手动刷新用 shallowRef。深层 watch/computed 见 02-computed-watch-watchEffect。

面试考点

  • ref 和 reactive 最大差异是什么？ 结论是 ref 是值容器，reactive 是对象代理。项目里基本类型、需整体替换用 ref，稳定对象模型用 reactive。
  • shallowRef 为什么改内部字段不更新？ 结论是只追踪 .value 引用变化，内部对象不递归代理。应整体替换、triggerRef 或额外版本号。本页三个按钮分别演示三种结果。
  • reactive 解构后如何保留响应式？ 结论是使用 toRef / toRefs。直接解构得到的是快照，不再经过代理 get/set。
  • 业务状态如何选型？ 结论先看状态形态和更新方式：替换用 ref，聚合对象用 reactive，避免深层代理用 shallowRef。边界是 shallow API 牺牲自动深层更新。
  • triggerRef 何时需要？ 结论是 shallowRef 内部已变更但需强制刷新视图时。原因是 shallow 不追踪深层。应优先设计为整体替换，triggerRef 作例外手段。
