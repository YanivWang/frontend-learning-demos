class 组件 constructor 必须调用 super()

请打开 DevTools Console 查看 constructor 内 this.props 输出；点击下方按钮验证 bind 后的方法；下方 #root 为 React 挂载演示区。

知识点要点

  • React class 组件本质上是继承 React.Component 的 ES class；只要显式声明 constructor ，就必须先调用 super() ，否则在访问 this 前会触发 JavaScript 子类初始化错误（ReferenceError）。
  • React 官方要求：constructor 内须先调用 super(props) ，否则 this.props 在 constructor 运行期间可能为 undefined 。因此即使不读取 props，也应写 super(props) ，不要刻意省略参数。
  • constructor 只适合同步初始化 this.state 或绑定实例方法；网络请求、订阅、定时器等副作用应放在 componentDidMount ，并在 componentWillUnmount 中清理。
  • 若不需要初始化 state 且不需要手动绑定方法，可以完全省略 constructor ；现代写法可用类字段 state = {} 和箭头函数替代 constructor 中的 state 赋值与 .bind(this) 。
  • class 组件必须实现 render() ，它是 React 调用组件并产出 UI 的唯一必需方法； render 应是 props、state、context 的纯函数，不应包含副作用或直接操作 DOM。

面试考点

  • 为什么 class 组件 constructor 里要先调用 super(props)？ 结论是：它既满足 ES class 子类必须先初始化父类的规则，也让 React 组件在 constructor 内能正确读取 this.props 。原因是子类在 super() 之前不能访问 this ；react.dev 还指出若不先 super(props) ， this.props 在 constructor 中可能为 undefined 。
  • 什么时候可以不写 constructor？ 结论是：没有初始化 state、没有手动绑定实例方法时就不要写。原因是 React 会正常构造组件实例，空 constructor 只增加样板代码；现代项目可用类字段 state = {} 和箭头函数进一步省略。
  • super() 与 super(props) 有什么区别？ 结论是：两者都能调用父类构造函数，但 React class 中应始终写 super(props) 。原因是 constructor 内若读取 this.props ，只有传入 props 才能保证值正确；仅写 super() 时 react.dev 明确警告 this.props 可能为 undefined 。
  • constructor 里适合做哪些事？ 结论是：只做同步初始化，例如设置初始 state、 this.method = this.method.bind(this) 。原因是 constructor 发生在挂载前，不能依赖 DOM，也不适合请求数据；副作用放到 componentDidMount ，清理放到 componentWillUnmount 。
  • class 组件为什么必须实现 render()？render 里能做什么？ 结论是： render() 是 class 组件描述 UI 的必需方法，漏写会报 No render method found 。原因是 React 需要调用它得到 React element；边界是 render 应写为纯函数，可返回 JSX、字符串、数字、 null 等，但不应放订阅、请求等副作用。
