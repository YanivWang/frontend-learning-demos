计算属性 vs 方法（缓存差异）

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • computed 适合声明“由响应式数据派生出的值”。本页中 computedMsg 依赖 msg，模板多次读取它时，只要 msg 没变，Vue 会直接复用缓存结果。
  • methods 是普通函数。模板中写 {{ reverseMsg() }} 时，只要组件重新渲染并执行到这个表达式，方法就会重新调用；即使返回结果和 computed 一样，也没有 computed 的依赖缓存。
  • computed 的缓存只跟响应式依赖有关。依赖 this.msg 会在 msg 变化后重新求值；如果 getter 只读 Date.now() 这类非响应式值，它不会因为时间流逝自动更新。
  • computed getter 应尽量保持纯粹：只根据依赖返回值，不做请求、计时器、DOM 操作或修改别的状态。需要传参、不需要缓存或要响应点击的动作，用 methods；需要异步副作用或监听变化过程，用 watch。
  • Vue 2 已于 2023-12-31 进入 EOL。维护旧项目时仍应理解 computed 的缓存模型；新项目或大版本升级应优先按 Vue 3 文档选型。

面试考点

  • computed 和 methods 返回结果一样时为什么还要区分？ 结论是 computed 适合可缓存的派生值，methods 适合每次都要执行的动作或计算。原因是 computed 会基于响应式依赖缓存，依赖不变时多次读取不会重复运行 getter；methods 在模板重新渲染并被调用时会再次执行。项目里大量列表过滤、格式化汇总优先 computed，点击提交、需要参数的逻辑用 methods。
  • computed 依赖 Date.now() 为什么不会自动更新？ 结论是它不会随时间自动重新计算。原因是 Date.now() 不是 Vue 2 响应式依赖，Vue 没有可追踪的数据变化来让 computed 失效。边界是如果同时读取了响应式字段，则该字段变化会触发重新求值，但这不等于时间本身被追踪。
  • computed、methods、watch 在项目里如何选型？ 结论是派生状态用 computed，事件动作或传参逻辑用 methods，异步请求、节流防抖、同步外部系统这类副作用用 watch。原因是三者表达的意图不同：值、动作、变化后的副作用。项目里表单全名、过滤列表是 computed；按钮点击是 methods；输入变化后请求接口是 watch。
  • computed getter 里为什么不建议写副作用？ 结论是 computed getter 应保持可预测和可缓存。原因是 Vue 可能因为依赖缓存而跳过 getter，也可能在渲染时读取多次；如果里面修改状态或发请求，会让执行次数和时机变得难以推断。边界是教学 demo 可以用 console.log 观察执行次数，真实业务不要把副作用放在 getter 里。
