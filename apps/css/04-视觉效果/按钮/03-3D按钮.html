渐变 + 下边框 + box-shadow 的 3D 按钮

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • 这个按钮把 linear-gradient 当作光照：默认上亮下暗，表示顶部受光、底部较深，从而产生凸起感。
  • border-bottom 模拟按钮厚度，box-shadow 模拟投影；两者叠加后，比单独阴影更像有实体厚度的按钮。
  • :active 状态反转渐变、缩短下边框和阴影，并用 transform: translateY() 下移，让点击反馈更像被按下。
  • 示例使用 <button type="button">，避免 <a href="#"> 点击后改变 URL hash 或滚动到顶部；如果是导航才应该用链接。
  • 动效只服务于反馈，不应影响可用性：要保留焦点样式，尊重 prefers-reduced-motion，并避免在大量按钮上同时做高成本阴影动画。

面试考点

  • 渐变、下边框和阴影分别承担什么视觉职责？ 结论是渐变表示光照方向，下边框表示实体厚度，阴影表示按钮离开背景。原因是人眼会把亮暗变化和投影解释为空间层次。项目里三者要统一光源方向，否则按钮会显得假或突兀。
  • :active 反转渐变为什么能产生按下感？ 结论是它改变了受光方向，并同步缩短厚度和投影。原因是按下后按钮更接近平面，底部厚度和投影都应该变小。边界是不要只反转颜色而不调整位置，否则反馈会像闪烁而不是按压。
  • 什么时候用 button，什么时候用 a？ 结论是触发页面动作使用 button，跳转到资源或路由使用 a。原因是两者语义、键盘行为和默认行为不同。像本页这种提交/点击反馈演示不需要跳转，使用 href="#" 会带来无意义 hash 和滚动副作用。
  • 3D 按钮动效如何降级？ 结论是保留基本按钮语义和可见状态，减少非必要动画。原因是阴影和渐变只是增强反馈，不应成为唯一状态表达。低端设备或开启减少动效时，可以关闭 transition，只保留颜色、边框或文本状态变化。
