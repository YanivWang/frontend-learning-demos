Symbol 原始类型与典型用法

请打开 DevTools Console 查看输出。

知识点要点

  • Symbol(description) 创建唯一原始值；不能用 new；typeof 为 'symbol'。
  • Symbol.for(key) 全局注册表复用；Symbol.keyFor 反查 key。
  • 作对象键：不会被 for...in / Object.keys 枚举；需 Object.getOwnPropertySymbols。
  • 内置 Well-known symbols：iterator、toStringTag、hasInstance 等定义语言协议。
  • JSON.stringify 会忽略 Symbol 属性。

面试考点

  • Symbol 有什么用？ 结论是它提供唯一的原始值，常用作不易冲突的对象键和语言协议入口。原因是即使描述相同，两个 Symbol() 也不相等；边界是它不是强私有，拿到 symbol 引用仍可访问属性。
  • Symbol() 和 Symbol.for() 区别是什么？ Symbol() 每次都创建新值，Symbol.for(key) 会在全局 symbol 注册表中按 key 复用。跨模块约定同一个扩展点时可用 Symbol.for，局部避免冲突时用普通 Symbol()。
  • 如何遍历 Symbol 属性？ 结论是 Object.keys、for...in 不会返回 symbol 键，要用 Object.getOwnPropertySymbols() 或 Reflect.ownKeys()。项目里写序列化、复制或调试工具时要明确是否包含 symbol 键。
