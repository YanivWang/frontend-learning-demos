用静态私有标志位模拟「私有构造函数」

请打开 DevTools Console 查看输出。

知识点要点

  • JavaScript class 没有 TypeScript 那种 private constructor 语法；这里用静态私有字段和静态工厂方法做运行时模拟。
  • 静态私有字段 #isInternalConstructing 只能在类体内部访问，类外既不能读取也不能改写，因此适合作为构造许可标志。
  • 工厂方法先把许可标志设为 true，再调用 new PrivateConstructor()；constructor 校验通过后必须立刻复位，避免后续直接 new 被误放行。
  • 如果构造过程可能抛错，工厂方法应使用 try...finally 复位标志；本页示例在 constructor 内复位，适合演示基本机制，但真实项目还要覆盖失败路径。
  • 这种模式常用于单例、受控工厂、不可直接创建的值对象；如果只是想限制模块外创建，更简单的做法是不要导出 class，只导出创建函数。

面试考点

  • JavaScript 能不能声明真正的私有构造函数？ 结论是不能，当前 JS class 的 constructor 没有 private 修饰符。原因是 # 私有元素适用于字段、方法和访问器，不是 constructor 可见性控制；需要限制创建时只能用运行时检查、模块封装或 TypeScript 编译期约束。
  • 静态私有字段为什么能隐藏构造许可标志？ 因为 #private 名称只在声明它的类体内部可访问，外部代码无法读取或赋值。项目里把许可标志放成静态私有字段，可以避免调用方伪造状态直接绕过工厂方法。
  • 工厂方法模拟私有构造有什么坑？ 最大风险是构造失败后没有复位标志，导致后续直接 new 误通过。真实代码应把“打开标志、构造、复位”放进 try...finally，并保证 constructor 里不要把内部标志暴露到实例上。
  • 什么时候不值得使用这种模式？ 如果只是模块内部不想暴露构造能力，直接不导出 class 更简单；如果只是 TypeScript 项目内部约束，private constructor 编译期提示更直观。运行时模拟适合发布给纯 JS 使用者或需要强制走工厂方法的库。
