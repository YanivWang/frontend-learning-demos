IIFE（Immediately Invoked Function Expression）

请打开 DevTools Console 查看输出。

知识点要点

  • IIFE = 函数表达式 + 紧跟的 () ； function 开头会被当声明，需括号或一元运算符逼成表达式。
  • 常见写法： (function(){})() 、 !function(){}() 、具名 IIFE、带参 IIFE。
  • 可返回对象/函数，实现揭示模块模式（Revealing Module），隐藏私有变量。
  • 经典题：用 IIFE 为每次循环创建独立作用域，修复 var+for 闭包；现代用 let。
  • 现代 ES Module 取代 IIFE 做模块隔离；IIFE 仍见于 legacy bundle、书签脚本、立即初始化和隔离临时变量的场景。
  • IIFE 是正常函数调用，能闭包外层变量； new Function 从字符串创建函数，通常只能访问全局作用域和自身参数，且有类似 eval 的风险。

面试考点

  • IIFE 是什么？ 结论是「函数表达式创建后立即调用」。因为函数声明不能直接跟调用括号，所以常用外层括号或一元运算符把 function 转成表达式；它会创建一个函数作用域，避免临时变量泄漏到全局。
  • var+for 闭包怎么修？ 结论是 ES5 可把当前 i 作为参数传入 IIFE，ES6 直接用 let 。原因是 var 只有函数作用域，循环结束后闭包读到同一个变量；IIFE 每轮创建新的参数绑定。
  • IIFE 和模块化的关系？ 结论是 IIFE 是 ES Module 普及前常见的模块隔离方式。它可以返回公开 API、隐藏私有变量；现代项目优先用 import / export ，但 UMD、旧 bundle 和一次性初始化代码里仍能看到 IIFE。
  • IIFE 与 new Function 的作用域差异？ 结论是 IIFE 能访问外层词法作用域， new Function 不能闭包局部变量。原因是前者是普通函数表达式，后者从字符串编译并在全局语境创建；涉及动态字符串时还要考虑安全和 CSP。
