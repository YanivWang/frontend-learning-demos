类型检测三板斧

请打开 DevTools Console 查看输出。

知识点要点

  • typeof 适合快速区分原始类型与 object/function；数组、Date、null 等都会得到 'object'（function 除外）。
  • typeof null === 'object' 是 ES 第一版遗留 bug，不要用 typeof 判 null。
  • instanceof 沿原型链查找 constructor.prototype；字面量原始值不是包装对象实例，"abc" instanceof String 为 false。
  • instanceof 不能跨 iframe/window（不同全局对象的原型链隔离）。
  • Object.prototype.toString.call(value) 返回 [object Type]，可区分 Array、Date、RegExp、Null 等所有内置类型。
  • 工程上常封装 type(value)：取 toString 结果 slice(8,-1).toLowerCase()。

面试考点

  • typeof null 为什么是 object？ 结论是这是历史遗留行为，不能用 typeof 判断 null。项目里直接用 value === null，或在通用类型工具里用 Object.prototype.toString.call(value) 兜底。
  • instanceof 的原理与局限是什么？ 它检查右侧构造函数的 prototype 是否出现在左侧对象原型链上。边界是跨 iframe/window、原型被改写、原始值包装对象等场景会影响判断。
  • 如何精准判断数组、null、Date？ 数组优先用 Array.isArray()，null 用严格相等，Date 可用 value instanceof Date 并结合有效性检查；写通用工具时可用 Object.prototype.toString.call()。
  • 三种方式怎么选？ typeof 适合判断原始类型和函数，instanceof 适合自定义类实例，toString.call 适合跨内置类型的通用检测。项目里不要为了“统一”牺牲语义清晰度。
