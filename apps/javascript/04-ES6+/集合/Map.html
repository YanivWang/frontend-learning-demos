Map 键值对集合

请打开 DevTools Console 查看输出。

知识点要点

  • Map 存储键值对，键和值都可以是任意 JavaScript 值；对象键按引用身份比较，不按对象内容深比较。
  • Map 维护键值对的首次插入顺序，for...of、entries()、keys()、values() 都按该顺序迭代。
  • 常用 API 包括 set、get、has、delete、clear、size、forEach、entries、keys、values。
  • 与普通对象相比，Map 没有原型默认键，更适合用户输入键、频繁增删、需要直接迭代或键不是字符串/Symbol 的场景。
  • 不要用 map.foo = value 存储 Map 数据，那只是给 Map 对象本身加普通属性，get/has/delete 都不会识别。

面试考点

  • Map 和 Object 用作字典时如何选择？ 动态键、非字符串键、频繁增删和需要稳定迭代顺序时优先 Map；固定字段的数据模型、需要 JSON 序列化时用对象。原因是对象有原型和属性枚举语义，Map 是专门的键值集合。
  • Map 的对象键为什么两个字面量对象不相等？ 对象键按引用身份比较，{} 和另一个 {} 是两个不同对象。项目里如果要按业务 ID 查找，应使用 ID 字符串/数字做键，而不是临时对象字面量。
  • Map 的迭代顺序和 entries 结果是什么？ Map 按首次插入顺序迭代，默认迭代器和 entries() 都产生 [key, value]。这让它适合缓存、菜单配置、按插入顺序输出统计结果等场景。
  • 为什么不要用 map.foo = value？ 这会设置普通对象属性，不会进入 Map 的键值存储。后续 map.has("foo") 会是 false，容易造成看似有值但 API 读不到的 bug。
