泛型基础

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • 泛型把类型作为参数传入函数、接口或类，用来表达“输入是什么类型，输出仍和它有关”；本页的 first<T> 能保留数组元素类型。
  • extends 在泛型里表示类型约束，不等同于 class 继承；它要求类型参数至少满足某种结构，编译器才允许安全访问相关属性。
  • keyof T 取对象键名联合，T[K] 取对应属性类型；pluck<T, K extends keyof T> 因此能限制 key 必须存在，并返回精确字段类型。
  • 泛型默认值、多个类型参数和约束组合常用于组件 props、API 响应、表格列配置等场景；类型参数太多会降低可读性，应优先让编译器从参数推断。
  • 泛型在编译后会被擦除，运行时没有 T 这个值；需要运行时分支时必须传入真实值、构造函数、标签字段或校验函数。

面试考点

  • 泛型解决的核心问题是什么？ 结论是泛型让类型关系可复用，而不是把参数都写成 any。原因是它能保留输入和输出之间的关联，例如数组元素类型、接口响应数据类型。项目中封装请求、表格、表单和工具函数时非常常见。
  • extends 约束和继承有什么区别？ 结论是泛型约束只发生在类型检查阶段，class 继承会产生运行时原型关系。原因是 T extends { id: number } 只是要求传入类型具备 id。边界是它不会给对象自动添加字段，也不会改变运行时代码。
  • keyof 与 T[K] 为什么能保证 pluck 的返回类型？ 结论是 K extends keyof T 限定 key 必须来自对象，T[K] 精确取出该 key 对应的值类型。原因是 TypeScript 在类型层面建立了 key 与 value 的映射。项目中读取配置、表格列字段和表单字段时常用这种写法。
  • 泛型在运行时是否存在，为什么不能根据 T 做判断？ 结论是不存在，泛型会被编译擦除。原因是 TypeScript 不携带运行时类型元数据。需要运行时判断时应显式传入类型标签、构造函数或解析器，而不是写不存在的 if (T === ...)。

上方为 TypeScript 源码（在 .ts / IDE 中检查），下方为等价的可运行 JavaScript + 断言输出。

TypeScript

// —— 泛型函数保留输入输出关系 ——
function first<T>(arr: T[]): T | undefined {
  return arr[0];
}

// —— extends 约束 + keyof 索引访问 ——
function pluck<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}
// pluck(user, "age");  // ❌ 编译错误：age 不在 keyof User 中
