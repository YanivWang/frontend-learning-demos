eval 与 Global 对象（window）

请打开 DevTools Console 查看输出。

知识点要点

  • 浏览器全局对象是 window；Node 是 global；ES2020 标准 globalThis 跨环境统一。
  • 顶层 var 和 function 声明会成为 window 属性；let/const 不会。
  • eval(str) 在运行时把字符串当 JS 执行，能访问外层词法变量。
  • 严格模式 + 直接 eval 有独立词法环境；indirect eval（如 (0,eval)(s)）在全局执行。
  • 风险：XSS 注入、CSP 的 unsafe-eval 限制、引擎难以优化、作用域难追踪和调试困难。
  • 现代替代：解析数据用 JSON.parse，表达式白名单用受控解析器或配置 DSL，动态行为优先函数映射表。

面试考点

  • window、globalThis 与顶层变量有什么关系？ 结论是浏览器里 window 是全局对象，globalThis 是跨环境访问全局对象的标准方式。普通 script 顶层 var / function 会成为 window 属性，let / const 和 ES module 顶层绑定不会。
  • 直接 eval 和间接 eval 有什么区别？ 结论是直接 eval(code) 在调用处语境执行，间接调用如 (0, eval)(code) 在全局语境执行。原因是规范把直接 eval 作为特殊语法处理；边界是严格模式会让 eval 内部声明隔离得更明显。
  • eval 有哪些风险？ 结论是生产代码应默认避免。它会执行字符串，接入用户输入时容易变成 XSS；还会触发 CSP 限制、阻碍优化，并让静态分析和调试变难。
  • eval 和 Function 构造器如何取舍？ 结论是两者都慎用。eval 可访问调用处闭包，new Function 主要访问全局与参数，隔离稍强但仍会执行字符串；项目里优先用 JSON、映射表或专用解析器。
