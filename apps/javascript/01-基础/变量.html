变量声明、提升、null vs undefined、constructor

请打开 DevTools Console 查看输出。

知识点要点

  • var 在函数内为函数作用域、脚本顶层为全局作用域；声明会提升，可重复声明； for (var i…) 的 i 会泄漏到循环外。
  • let / const 是块级作用域，不可重复声明；脚本顶层的 let / const 不会成为 globalThis 的属性（与顶层 var 不同）。
  • 声明提升： var 声明前访问得 undefined ； let / const 在暂时性死区（TDZ）内访问或对其使用 typeof 均抛 ReferenceError （进阶见 变量进阶 ）。
  • const 必须初始化；绑定不可重新赋值，但对象/数组的内部属性仍可修改（MDN 称「immutable binding」而非 immutable value）。
  • undefined 表示缺省/无值（未初始化、缺省返回、不存在的属性等）； null 表示刻意赋的「无对象」； null == undefined 为 true， === 为 false。
  • ECMAScript 共 8 种类型：7 种 primitive + Object ； typeof null === 'object' 是历史遗留；不加声明直接赋值在非严格模式下会污染 globalThis 。

面试考点

  • var / let / const 区别？ var 函数/全局作用域、可重复声明、提升后值为 undefined；let/const 块级、不可重复声明、有 TDZ。顶层 var 挂 globalThis，let/const 不挂。业务代码优先 let/const。
  • 什么是 TDZ？TDZ 内 typeof 会怎样？ 从块开始到 let/const 声明执行前，绑定已存在但不可访问；访问或对未声明完成的绑定使用 typeof 均 ReferenceError。var 提升后可直接读到 undefined。
  • null 和 undefined 区别？ undefined 是缺省/无值；null 是刻意表示无对象。判断空值推荐 value == null 或 value === null || value === undefined 。
  • typeof null 为什么是 object？ 早期实现中 null 的 type tag 与 object 相同。精准检测用 Object.prototype.toString.call 或 value === null （见 typeof ）。
  • const 能否修改对象？constructor 可靠吗？ 不能重新赋值绑定，但可改对象属性。primitive 访问 .constructor 会经自动装箱；MDN 不建议用 constructor 做类型检测，优先 typeof / instanceof / Symbol.toStringTag。
