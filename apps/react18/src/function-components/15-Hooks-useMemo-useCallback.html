useMemo 与 useCallback

请打开 DevTools Console 查看过滤/总价重算与 Toolbar 渲染日志；下方 #root 为 React 挂载/交互演示区。

知识点要点

  • useMemo 缓存的是计算结果；示例中商品过滤和总价计算只在 keyword 或 count 相关依赖变化时重新计算。
  • useCallback 缓存的是函数引用，本质上可以理解为返回一个稳定函数；当子组件用 memo 做浅比较时，稳定引用才有实际价值。
  • useMemo / useCallback 本身也有依赖比较和额外代码成本，普通轻量计算不需要默认包一层。
  • 依赖数组必须写全：计算中读取了哪个 state、props 或组件内变量，就应加入依赖；漏依赖会让缓存值或回调停留在旧闭包。
  • 它们只能减少渲染中的部分计算或引用变化，不能阻止父组件渲染；是否优化应结合性能瓶颈、子组件成本和可读性判断。

面试考点

  • useMemo 和 useCallback 分别缓存什么？ 结论是 useMemo 缓存值， useCallback 缓存函数引用。原因是 React 会在依赖不变时复用上一次结果。
  • 为什么说它们不是越多越好？ 结论是只在有明确性能收益时使用。原因是依赖比较、闭包维护和代码复杂度都有成本，轻量计算包一层反而降低可读性。
  • useCallback 为什么常和 React.memo 一起出现？ 结论是稳定函数引用只有在接收方比较 props 时才明显有用。原因是 memo 可以利用引用相等跳过渲染。
  • 依赖数组漏写会造成什么问题？ 结论是既可能显示旧数据，也可能让优化结果不可信。原因是缓存值或回调会闭包住旧变量。
