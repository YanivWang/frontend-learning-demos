显式绑定 this：call / apply / bind

请打开 DevTools Console 查看输出。

知识点要点

  • 显式绑定：call、apply、bind 都用于指定普通函数调用时的 this，优先级高于「谁调用函数 this 指向谁」的隐式绑定。
  • 立即调用 vs 延迟调用：call / apply 会马上执行函数；bind 返回一个新函数，适合事件回调、定时器回调或提前固定部分参数。
  • 传参差异：call(thisArg, a, b) 逐个传参，apply(thisArg, [a, b]) 接收数组或类数组；现代代码里 fn.call(obj, ...args) 或 fn(...args) 能替代很多 apply 场景。
  • 严格模式边界：非严格函数收到 null / undefined 作为 thisArg 时会替换为全局对象；严格函数会保留原值。
  • 绑定不可逆：bind 产生的函数已经固定 this，再用 call / apply 调它也不能改掉已绑定的 this。

面试考点

  • call、apply、bind 的共同点和区别是什么？ 共同点是都能显式指定函数调用时的 this。区别是 call 和 apply 立即执行，传参形式不同；bind 返回新函数，适合延迟执行。项目里看「现在执行」还是「稍后执行」来选。
  • apply 为什么常和数组参数一起出现？ 因为 apply 第二参接受数组或类数组，能把集合展开成实参列表。现代 JS 中多数场景可用展开语法 fn(...args)，但借用数组方法处理类数组时仍可能见到 call。
  • bind 返回的新函数还能被 call 改 this 吗？ 结论是不能。原因是绑定函数内部已经记录了目标函数、绑定的 this 和预置参数；后续再用 call 只会影响外层调用，不会覆盖绑定值。
  • null/undefined 作为 thisArg 有什么差异？ 非严格函数会把它替换为全局对象，严格函数会原样保留。边界是模块、class 方法和严格脚本下不要依赖非严格兜底行为。
  • 现代代码中什么时候还需要 call/apply/bind？ 常见于借用方法、固定回调 this、函数柯里化或兼容旧代码。框架代码里更常用箭头函数、闭包或类字段来避免手动绑定。
