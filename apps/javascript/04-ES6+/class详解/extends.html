extends 继承与 super 调用

请打开 DevTools Console 查看输出。

知识点要点

  • extends 会建立两条关系：Object.getPrototypeOf(Child) === Parent 让静态成员可继承，Object.getPrototypeOf(Child.prototype) === Parent.prototype 让实例方法可继承。
  • 派生类如果显式声明 constructor，必须先调用 super(...args) 才能使用 this；因为实例的 this 需要由父类构造器先完成初始化。
  • super() 只用于派生类构造器调用父类构造器；super.method() 可在实例方法中调用父类原型方法，也可在静态方法/静态块中调用父类静态方法。
  • 实例方法挂在 prototype 上，由所有实例共享；实例字段和私有字段在每次 new 时初始化，每个实例各有自己的状态。
  • 读取私有字段必须写 this.#field，不能写成具体实例名；私有字段也不能在类体外新增或访问。
  • 继承适合稳定的 “is-a” 关系；如果只是复用能力或包装内建对象，组合通常更清晰，也能避开内建对象子类化的性能和语义风险。

面试考点

  • extends 会建立哪两条原型链？ 结论是子类构造器继承父类构造器，子类原型继承父类原型。前者让静态成员可沿构造器链查找，后者让实例方法可沿实例原型链查找；排查继承问题时要分清静态方法和实例方法放在哪里。
  • 为什么派生类 constructor 必须先调用 super()？ 因为派生类实例的 this 由父类构造器初始化，在 super() 返回前访问 this 会抛 ReferenceError。项目中常见错误是在 super() 前设置实例字段或调用依赖 this 的方法。
  • super() 和 super.method() 有什么区别？ super() 是构造器调用，只能在派生类 constructor 中使用；super.method() 是属性查找，可在实例方法或静态方法中调用父级同名方法。边界是静态上下文查父类构造器，实例上下文查父类 prototype。
  • 什么时候应该用组合替代继承？ 当关系不是稳定的 “is-a”，或只是复用一组能力时，组合更合适。原因是继承会默认继承父类所有行为，扩展内建对象还可能遇到性能、安全和新方法语义变化；业务封装 Map、Array、Date 时尤其要谨慎。
