事件绑定 / 表单 v-model

在浏览器中操作下方交互演示；复习要点与面试答法见本复习区。

知识点要点

  • @click 是 v-on:click 缩写，处理函数运行在组件上下文。本页「提交」按钮直接调用 submit ；「记录坐标」通过 $event 读取原生 MouseEvent ，日志写入下方 pre 。
  • v-model 在原生表单上是语法糖：文本框绑定 value 并监听 input ，复选框处理布尔值，选择框同步选中项。下方 <pre>{{ form }}</pre> 实时反映绑定结果；组件侧 v-model 见 04-组件v-model 。
  • v-model.trim 同步时去掉首尾空白：在用户名输入框首尾加空格后失焦， form.name 不含空白。 v-model.number 尝试转数字：清空年龄后 typeof form.age 显示为 number 或空值时的校验失败。
  • v-model.lazy 改为 change 时同步（失焦或 Enter）。备注 textarea 输入过程中上方「当前 lazyNote」不变，失焦后才更新；与上方表单实时同步形成对比。
  • 提交时必须处理失败路径：清空用户名、输入非法年龄或未勾选协议，点「提交」分别出现不同错误文案；成功时显示「已提交」。实践中服务端仍须校验，前端状态可被绕过。
  • 用户输入用插值或 v-text 展示时 Vue 会转义 HTML；若把用户输入交给 v-html 则有 XSS 风险，见 01-应用实例与模板语法 安全说明。
  • 事件修饰符： @click.stop 阻止冒泡， @click.prevent 阻止默认行为。本页内外层各有一个点击计数，内层带 .stop 时外层计数不增加。

面试考点

  • v-model 在原生表单上本质是什么？ 结论是语法糖，把表单值绑定到状态并在用户输入时更新。原因是不同控件的值来源和事件不同。边界是自定义组件依赖 modelValue + update:modelValue 约定，见 04 章。
  • .trim、.number、.lazy 分别改变了什么行为？ 结论是 .trim 去首尾空白、 .number 尝试转数字、 .lazy 改为 change 时同步。本页备注框与上方实时表单即 lazy 对比入口。
  • 为什么表单不能只依赖前端 v-model 类型转换？ 结论是前端转换只改善交互，不是可信校验。空输入、NaN、篡改请求都可能绕过。本页提交校验演示失败路径，生产须加服务端校验。
  • 用户输入展示时如何避免 XSS 风险？ 结论是默认插值/ v-text ，不用 v-html 渲染用户输入。本页 pre 展示表单 JSON 为文本转义。
  • 事件修饰符 .stop 与 .prevent 各解决什么问题？ 结论是 .stop 阻止冒泡， .prevent 阻止默认行为（如表单 submit、链接跳转）。本页内外层点击计数可验证 .stop 。
