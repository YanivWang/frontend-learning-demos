箭头函数与普通函数差异

请打开 DevTools Console 查看输出。

知识点要点

  • 箭头函数是函数表达式，不能具名声明；语法简写： (a,b) => a+b 、单参数可省略括号。
  • 没有自己的 this ，继承定义处外层非箭头函数的 this （词法 this）。
  • 没有 arguments 、没有 prototype 、不能 new （不是构造函数）。
  • 普通函数有 prototype ，作为构造函数时实例的 __proto__ 指向它。
  • 收集可变参数用剩余参数 ...args ，是真数组。

面试考点

  • 箭头函数 vs 普通函数？ 箭头：无 own this/arguments/prototype，不可 new，语法简洁。普通：动态 this，有 prototype，可作构造函数。
  • 箭头函数 this 指向？ 定义时外层（词法）作用域的 this，call/apply/bind 无法改变。对象方法、DOM 事件、需要动态 this 时勿用箭头。
  • 什么时候不能用箭头函数？ 构造函数、需要 arguments、对象字面量方法（若需 this 指向对象）、Vue2 选项式 methods、类的方法需要 super 时等。
