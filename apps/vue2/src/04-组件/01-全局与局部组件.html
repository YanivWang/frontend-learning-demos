Vue.component 全局组件 + data 与 vm 代理同步

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Vue 2 组件是可复用的 Vue 实例。Vue.component('todo-item', options) 会做全局注册，当前根实例及其子组件树在注册之后编译模板时都能识别这个组件。
  • props 是父组件向子组件传值的声明入口；本页的 todo-item 接收 todo，并在模板里读取 todo.name 与 todo.age。子组件不应直接修改 prop，而应通过事件请求父组件更新。
  • new Vue({ data }) 会把 data 对象上的既有字段代理到实例上，所以示例里 app_2.a 与 data.a 互相同步，且 app_2.$data === data。
  • Vue 2 响应式初始化发生在实例创建阶段；创建后给根级 data 或实例动态添加新字段，不会自动变成响应式。维护旧项目时用 Vue.set 或预声明字段，新项目应优先使用 Vue 3。
  • 同一元素上重复写同名事件监听没有教学价值且容易误导维护者；实际项目应合并处理逻辑或拆成语义明确的方法。

面试考点

  • Vue.component 注册后为什么所有子组件都能用？ 结论：全局注册会把组件挂到 Vue 构造器的全局注册表，之后创建的根实例及其子组件模板都能解析到它。原因是模板编译需要先知道标签对应的组件定义。边界是注册顺序要早于实例模板编译，过多全局组件也会让依赖关系变得不清晰。
  • props 和 data 代理分别解决什么问题？ 结论：props 解决父到子的外部输入，data 代理解决实例内部状态访问。原因是 Vue 会把 data 中已有字段代理到 vm，同时把 props 暴露给组件模板。项目里应避免子组件直接改 prop，状态来源放在拥有它的父组件或状态层。
  • 为什么 Vue 2 里动态新增根级 data 字段不推荐？ 结论：实例创建后新增的根级字段不会自动具备响应式能力。原因是 Vue 2 基于 Object.defineProperty 初始化拦截已有属性。边界是嵌套对象可用 Vue.set，但根级字段最好提前声明。
  • 全局组件和局部组件怎么选？ 结论：跨很多页面复用的基础组件可以全局注册，业务组件优先局部注册。原因是局部注册让依赖更显式，也更利于维护和按需加载。Vue 2 已是历史版本语境，新项目通常应直接使用 Vue 3 的组件组织方式。
