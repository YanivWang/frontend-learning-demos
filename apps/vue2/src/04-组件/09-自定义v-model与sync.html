自定义组件 v-model 与 .sync

在 text-editor 输入框编辑观察 title 联动；点 switch-button 切换 enabled；点「打开弹窗」再点「关闭」验证 :visible.sync。

知识点要点

  • Vue 2 自定义组件上的 v-model 默认等价于父组件传入 :value，并监听子组件触发的 input 事件。
  • 本页 text-editor 把输入框的 value 绑定到 prop，再在原生 input 事件中 $emit('input', value)，父组件的 title 因此更新。
  • model 选项可以改写默认 prop/event。本页 switch-button 使用 checked/change，避免把 value 占用在复选、开关等组件语义中。
  • .sync 是 :visible 加 @update:visible 的语法糖，表达“子组件请求父组件同步某个 prop”。本页弹窗点击关闭时发出 update:visible。
  • v-model 和 .sync 都不代表子组件可以直接修改 prop；它们只是约定事件名，让父组件仍然拥有最终状态。

面试考点

  • Vue 2 自定义组件 v-model 默认等价于什么？ 结论：等价于 :value 加 @input。原因是 Vue 2 用固定 prop 和事件名约定双向绑定语法糖。项目里封装输入组件时必须接收 value 并 emit input。
  • model 选项解决什么问题？ 结论：解决组件不想使用 value/input 作为 v-model 通道的问题。原因是 checkbox、switch 等组件的 value 可能已有其他语义。边界是团队要统一文档，否则使用方不容易猜到事件名。
  • .sync 和 v-model 有什么区别？ 结论：v-model 通常表达组件的主值，.sync 表达某个 prop 的同步请求。原因是 .sync 展开为 @update:prop。项目里弹窗 visible、分页 pageNo 这类单个 prop 同步常见。
  • 为什么不能直接改 prop？ 结论：prop 属于父组件传入的数据，子组件直接改会破坏单向数据流。原因是父组件重新渲染时会覆盖子组件修改，也会让数据来源不清晰。正确做法是 emit 事件让父组件更新。
