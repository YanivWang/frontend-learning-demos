状态更新后读取 DOM — flushSync 与 useEffect

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • setState 后立即读 state 仍是旧值；class 组件可用 setState 第二参数 callback。
  • 函数组件等价方案: useEffect 监听 state 变化后执行副作用。
  • 需要同步刷 DOM 时用 ReactDOM.flushSync（慎用，会打断批处理）。
  • 大多数场景优先 useEffect，flushSync 仅用于必须同步读 DOM 的少数情况。
