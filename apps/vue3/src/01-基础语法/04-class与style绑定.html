class / style 绑定

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • :class 可以绑定字符串、对象或数组；对象写法适合“某个状态是否启用某个类”，数组写法适合组合多个来源的类名。
  • :style 可以绑定对象或数组，样式属性既可写 fontSize，也可写 'font-size'；本页用对象表达字号，用数组合并基础样式和强调色。
  • 静态视觉规则优先写在 CSS 类里，动态绑定只表达状态差异，例如选中、危险、强调、尺寸等；这样更容易复用、覆盖和排查样式来源。
  • 复杂业务判断不要直接塞进模板里的 :class 或 :style，应抽成 computed、方法或语义化状态字段，避免模板变成规则引擎。
  • 绑定到组件上的 class 和 style 会作为透传 attribute 合并到组件根元素；多根组件或显式控制透传时，需要额外确认落点。

面试考点

  • :class 的对象写法和数组写法各适合什么场景？ 结论是对象写法适合按布尔状态开关类名，数组写法适合组合基础类、条件类和外部传入类。原因是对象强调“是否启用”，数组强调“按顺序合并多个来源”。项目里常用数组包对象，例如 [baseClass, { active: isActive }]。
  • :style 与 CSS class 应该如何分工？ 先结论：可复用和可维护的视觉规则用 CSS class，真正依赖运行时变量的少量值用 :style。原因是内联样式优先级高、复用差，也不方便媒体查询和伪类。动态尺寸、坐标、主题变量适合内联绑定，状态样式更适合类名。
  • 动态样式为什么不适合承载复杂业务规则？ 结论是模板可读性和可测试性会迅速下降。原因是复杂条件混在 HTML 属性里不易复用，也容易和 CSS 优先级互相影响。项目中应把规则沉到计算属性或状态模型里，让模板只表达结果。
  • 组件上绑定 class 和 style 时会如何合并？ 结论是默认会作为透传 attribute 合并到子组件根元素。原因是 Vue 会保留父组件传入的普通 attribute，包括 class 和 style。边界是多根组件、禁用继承或手动绑定 $attrs 时，需要明确这些样式落在哪个元素上。
