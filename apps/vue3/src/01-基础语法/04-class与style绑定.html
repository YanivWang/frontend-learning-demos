class / style 绑定

在浏览器中操作下方交互演示；复习要点与面试答法见本复习区。

知识点要点

  • :class 对象写法按布尔状态开关类名。本页点「切换选中」时卡片出现蓝色边框与浅蓝底（ active ）；点「次数 +1」到 3 次及以上时文字变红（ danger ）。
  • :style 对象写法绑定运行时数值。点「字号 +2」时卡片内文字随 fontSize 增大；静态视觉规则（圆角、过渡）仍写在 CSS .card 类里。
  • 数组写法适合合并多个来源： [baseClass, { highlight: useHighlight }] 。点「切换 highlight」观察内阴影； baseClass 固定为 large-text 加粗。
  • :style 也可绑数组： [baseStyle, emphasisStyle] 。点「切换强调色」在默认与红色间切换 emphasisStyle.color ，与 CSS 类分工：可复用规则用 class，少量运行时值用 style。
  • 复杂业务判断应沉到 computed 。本页「状态标签」区块的 statusClass 由 computed 根据 count 返回 status-ok 或 status-warn ，模板只写 :class="statusClass" 。
  • 绑定到组件上的 class / style 默认作为透传 attribute 合并到子组件根元素；多根组件与手动 $attrs 落点见 04-attrs与透传 。

面试考点

  • :class 的对象写法和数组写法各适合什么场景？ 结论是对象适合布尔开关，数组适合合并多来源。本页第一块用对象、第二块用数组包对象。实践中常用 [base, { active: isActive }] 。
  • :style 与 CSS class 应该如何分工？ 结论是复用规则用 class，依赖变量的少量值用 :style 。本页字号用 style、选中/危险态用 class。内联样式优先级高，不宜承载整套主题。
  • 动态样式为什么不适合承载复杂业务规则？ 结论是模板可读性与可测试性下降。本页 statusClass computed 即正例：规则在 JS，模板只绑定结果。
  • 组件上绑定 class 和 style 时会如何合并？ 结论是默认透传到子组件根元素并与自身 class/style 合并。多根或禁用 inherit 时需明确落点，见 04-attrs 页。
