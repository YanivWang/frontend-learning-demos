box-shadow vs filter: drop-shadow 对比

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • box-shadow 是盒模型阴影，默认围绕元素边框盒绘制；即使内容是透明 PNG 或虚线边框，它也主要表现为矩形盒子的投影。
  • filter: drop-shadow() 是滤镜函数，基于元素渲染后的 alpha 轮廓生成阴影，所以本页虚线边框的阴影会跟随可见线段。
  • drop-shadow() 参数类似 box-shadow 的水平偏移、垂直偏移、模糊半径和颜色，但没有 spread 扩张半径，也不支持 inset 内阴影。
  • 选型上，普通卡片、按钮、面板用 box-shadow 更直观；透明图标、SVG、裁剪形状或非矩形图形需要沿可见轮廓投影时用 drop-shadow()。
  • 性能边界是 filter 需要额外滤镜处理，频繁动画、大面积模糊或长列表批量使用时要实测，必要时降低模糊半径或改为静态资源。

面试考点

  • box-shadow 和 drop-shadow() 为什么画出的阴影不同？ 结论是 box-shadow 看盒模型，drop-shadow() 看渲染结果的 alpha 轮廓。原因是前者是盒阴影属性，后者是滤镜函数，会对输入图像做投影。本页虚线边框能明显看出差异：矩形盒子阴影和线段轮廓阴影不是同一种模型。
  • 透明 PNG 或 SVG 图标应该用哪种阴影？ 结论是通常优先用 filter: drop-shadow()。原因是图标透明区域不会参与 alpha 轮廓，阴影更贴合可见图形。边界是如果只是卡片容器、按钮背景或需要 spread / inset，仍然应该用 box-shadow。
  • drop-shadow() 有哪些能力限制？ 结论是它适合轮廓投影，但不是 box-shadow 的完整替代。原因是 CSS 滤镜的 drop-shadow() 不提供扩张半径和内阴影语义，多个阴影也要通过多个 filter 函数组合。项目里需要复杂层次阴影时，盒阴影或设计切图可能更稳定。
  • 阴影动画为什么要谨慎？ 结论是大量模糊阴影和滤镜动画都可能增加渲染成本。原因是浏览器需要重新计算阴影或滤镜输出，面积越大、模糊半径越高越昂贵。真实列表、瀑布流或低端设备上应减少同时动画的阴影数量，并用性能面板验证。
