var / let / const 变量提升与暂时性死区

请打开 DevTools Console 查看输出。

知识点要点

  • var 在函数内提升到函数顶部、脚本顶层为全局作用域；声明前访问得到 undefined ，不会抛错。
  • let / const 在词法上也会提升，但从块开始到声明语句之间是 暂时性死区（TDZ） ，访问会 ReferenceError ；TDZ 内 typeof 同样报错（与未声明变量不同）。
  • const 必须在声明时初始化；绑定不可改，但对象/数组的内容仍可变（见 变量 ）。
  • 函数声明会被 整体提升 ，函数名在声明前就是可用的函数对象（见下方 typeof example2 ）。
  • 块级作用域对 if 、 for 、 {} 均生效； var 在 if 块内声明仍属于外层函数作用域， let / const 不泄漏。

面试考点

  • 什么是暂时性死区？ 从块级作用域开始到 let/const 声明执行前，变量已存在于词法环境但不可访问。这是规范行为，不是「没提升」。
  • let 真的不提升吗？ 会提升，只是 TDZ 内访问报错。与 var 的区别是：var 提升后值为 undefined；let/const 在 TDZ 内访问直接 ReferenceError。
  • TDZ 内 typeof 会怎样？ 与访问变量一样抛 ReferenceError ；未声明的变量 typeof x 才返回 "undefined" 。这是 TDZ 与「真正未声明」的重要区别。
  • 函数声明 vs 变量声明的提升？ 同作用域内函数声明优先于 var；函数声明整体提升，var 只提升声明不赋值。经典题见 var 与 function 同名提升 。
  • 块级作用域踩坑？ for (var i…) 的 i 泄漏到外层； for (let i…) 每次迭代有独立绑定。闭包题见 let for 循环闭包 。
