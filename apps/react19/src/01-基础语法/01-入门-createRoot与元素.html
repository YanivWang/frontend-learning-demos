React 19 元素与 createRoot 渲染

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • JSX 是 JavaScript 的语法扩展，本页用 Babel 在浏览器中转译；转译后会创建 React 元素，元素本质上是描述 UI 的不可变对象。
  • ReactDOM.createRoot(container) 创建并管理一个 React 根，root.render(element) 把元素树交给 React 协调并提交到容器。
  • 更新界面不是修改已有元素对象，而是根据新 state / props 重新返回一棵元素树；React 再比较前后树并更新必要 DOM。
  • 本页同时展示 React.createElement 与 JSX，说明两者最终都表达同一类 React 元素；项目代码通常使用 JSX 可读性更好。
  • React 19 官方移除了 UMD 构建，本仓库通过本地 React 19 运行时文件教学演示；真实项目应优先使用构建工具或 ESM CDN，并和 React 18 demo 区分版本。

面试考点

  • React 元素是什么？ 结论是 React 元素是描述 UI 的不可变对象，不是真实 DOM。原因是 React 需要用元素树表达“此刻 UI 应该长什么样”，再由协调过程决定 DOM 更新。项目里通常用 JSX 创建元素，而不是手写大量 React.createElement。
  • createRoot 和 render 分别做什么？ 结论是 createRoot 创建 React 根，render 提交元素树。原因是根对象负责后续调度、协调和卸载。边界是一个 DOM 容器通常只创建一个 root，不要在每次状态变化时重复 createRoot。
  • 为什么更新时要返回新元素树？ 结论是 React 不修改旧元素，而是根据最新 state / props 重新描述 UI。原因是不可变描述能让 React 比较前后结果并最小化 DOM 操作。本页点击“改名”就是通过 state 触发新 props 和新元素树。
  • React 19 的浏览器脚本引入有什么版本边界？ 结论是官方不再发布 UMD 构建，真实项目应使用构建工具或 ESM CDN。原因是 React 19 发布策略转向现代模块生态。本仓库为教学保留本地运行时文件，不代表生产项目推荐继续用 script 标签直连。
