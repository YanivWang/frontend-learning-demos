display 与文档流

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出尺寸与布局实测值。

知识点要点

  • display 不只是“是否换行”，它定义外部显示类型和内部布局类型：外部类型决定元素如何参与普通文档流，内部类型决定子元素按 flow、flex、grid 等哪种格式化上下文布局。
  • block 在普通文档流中生成块级盒，通常独占一行并可设置宽高；inline 生成行内盒，和文本一起排在行内，宽高不按块盒方式生效。
  • inline-block 的外部表现像行内元素，可以和文字排在同一行；内部表现像块级盒，可以设置宽高、padding 和垂直方向尺寸，适合标签、按钮旁的小块布局。
  • display: none 不生成盒，元素和后代从布局中消失；visibility: hidden 仍保留占位但不绘制，元素不能获得焦点，且通常也不会暴露给辅助技术。
  • 实际项目隐藏元素时要按目标选型：需要移除布局用 display: none，需要保留占位用 visibility: hidden，需要仅视觉隐藏但仍供读屏访问则应使用专门的 visually-hidden 样式。

面试考点

  • block、inline 和 inline-block 的布局差异是什么？ 结论是 block 生成块级盒，inline 生成行内盒，inline-block 外部按行内排布但内部像块级盒。原因是 display 同时影响外部参与文档流的方式和内部布局上下文。项目里按钮、徽标、标签常用 inline-block 或现代的 inline flex/grid，正文中的强调文本仍应保持 inline。
  • display:none 和 visibility:hidden 如何选？ 结论是要移除布局用 display: none，要保留原位置用 visibility: hidden。原因是前者不生成盒，页面当它不存在；后者仍生成盒，只是不绘制。边界是两者都不适合作为“只给读屏器保留内容”的方案，辅助技术可见性要单独用语义和 visually-hidden 样式设计。
  • 为什么 inline 元素设置 width 和 height 经常不生效？ 结论是非替换行内元素的尺寸主要由行盒和内容决定，不能像块盒那样直接用 width/height 控制。原因是它参与的是行内格式化上下文，水平和垂直布局规则与块级盒不同。项目里如果需要文字旁边有固定尺寸的图标、标签或按钮，应改成 inline-block、inline-flex 或 inline-grid。
  • 隐藏元素时有哪些可访问性和交互边界？ 结论是隐藏策略要同时考虑视觉、布局、焦点和辅助技术。原因是 display: none 会让后代也不参与布局和可访问性树，visibility: hidden 保留布局但元素不可见且不能获得焦点。项目里做折叠面板、弹窗和菜单时，还要同步 ARIA 状态、键盘焦点和过渡动画，不能只切一个 CSS 属性。
