文字逐字淡出的 loading 动画

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出各字符 animation-delay 与 color 实测值。

知识点要点

  • 这个 demo 把 “Loading...” 拆成多个字符元素，每个字符复用同一个 @keyframes ，通过递增 animation-delay 做逐字淡出。
  • 动画只改变 color ，不会改变布局尺寸；如果需要更明显的淡入淡出，也可以动画 opacity ，但要保证文字在任一时刻仍能被理解。
  • rgba() 在规范和 MDN 中是 rgb() 的别名；现代 CSS 推荐 rgb(120 120 120 / 35%) 这种空格加斜杠的 alpha 写法。
  • 示例改用 inline-block 横排字符，避免用 float 做纯视觉排版；真实项目中更常用 flex、inline 内容拆分或框架循环渲染。
  • 文字 loading 不能只依赖颜色闪动传达状态，异步失败、超时和无障碍读屏文案都要单独处理。

面试考点

  • 逐字 loading 怎样控制每个字符的节奏？ 结论是所有字符复用同一个关键帧，只给每个字符不同的 animation-delay 。原因是逐字效果本质是时间相位错开，而不是动画逻辑不同。项目里字符数量动态时可用循环生成元素和 delay，避免手写十几个 id。
  • rgba() 和 rgb(... / alpha) 有什么关系？ 结论是 rgba() 是 rgb() 的别名，现代写法推荐在 rgb() 中用斜杠写 alpha。原因是 CSS Color 新语法统一了颜色函数的透明度表达。边界是老代码和浏览器序列化结果仍常见逗号形式，维护时要能识别两种写法等价。
  • 为什么 loading 文案不能只靠颜色动画？ 结论是颜色变化只能作为视觉增强，状态含义要由文本、ARIA 或明确 UI 状态表达。原因是用户可能开启 reduced motion、看不到低对比颜色，或使用屏幕阅读器。项目里需要可读的 “Loading” 文案，并在失败或超时时替换为错误提示和重试入口。
  • 文字淡出动效有什么边界？ 结论是要控制闪烁频率、对比度和循环时长。原因是持续闪烁可能干扰阅读，低透明度文字也可能达不到可读要求。项目里可在 prefers-reduced-motion 下停用动画，保留静态加载文案。
