HTTP 强缓存 vs 协商缓存

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • 强缓存：Cache-Control / Expires，命中时不发请求（200 from disk cache）。
  • 协商缓存：ETag / Last-Modified，发条件请求 304 Not Modified。
  • 面试常问：no-cache vs no-store；private vs public。
  • 实践：HTML 入口短缓存 + 带 hash 静态资源 immutable 长缓存。

面试考点

  • HTTP 强缓存与协商缓存的核心概念是什么？ 强缓存：Cache-Control / Expires，命中时不发请求（200 from disk cache）；协商缓存：ETag / Last-Modified，发条件请求 304 Not Modified
  • HTTP 强缓存与协商缓存在实际项目里怎么用？ 强缓存：Cache-Control / Expires，命中时不发请求（200 from disk cache）；协商缓存：ETag / Last-Modified，发条件请求 304 Not Modified
  • HTTP 强缓存与协商缓存有哪些常见坑或易错点？ 强缓存：Cache-Control / Expires，命中时不发请求（200 from disk cache）；协商缓存：ETag / Last-Modified，发条件请求 304 Not Modified
类型关键响应头行为
强缓存Cache-Control: max-age=3600未过期直接用本地副本，不发网络请求
协商缓存ETag + 请求 If-None-Match过期后问服务器，未变则 304，用本地副本
协商缓存Last-Modified + If-Modified-Since精度到秒，ETag 更精确

常见 Cache-Control

  • no-store：完全不缓存（敏感数据）
  • no-cache：可存但必须协商验证后才能用
  • max-age=31536000, immutable：带 hash 的静态资源长缓存
