类型转换、相等比较与浮点精度

下方为 console.log 同步输出；也可打开 DevTools Console 查看。

知识点要点

  • === 严格相等，不做类型转换；业务比较默认用 ===。
  • == 会按规范做 ToNumber / ToString 等隐式转换，产生反直觉结果（如 [] == false）。
  • 对象转原始值走 ToPrimitive：先 valueOf 再 toString；也可自定义 Symbol.toPrimitive。
  • NaN === NaN 为 false；用 Number.isNaN 或 Object.is(NaN, NaN) 判断。
  • 浮点二进制无法精确表示 0.1、0.2，相加后与 0.3 比较需用 Number.EPSILON 或定点化（toFixed）。
  • ?? 仅对 null/undefined 取默认值；|| 对所有假值生效，0 和 '' 会被替换。

面试考点

  • == 和 === 区别？ === 类型和值都相同才 true；== 会类型转换后比较。除 null == undefined 等少数场景，一律推荐 ===。
  • [] == false 为什么 true？ [] 先 ToPrimitive 得 ''，再 ToNumber 得 0；false 转数字也是 0，故相等。
  • 如何判断 NaN？ Number.isNaN(x) 不做隐式转换；全局 isNaN('abc') 会先转数字。Object.is 可判 NaN 且区分 +0/-0。
  • 0.1 + 0.2 !== 0.3？ IEEE 754 浮点精度问题。金额用整数分存储；比较用 Math.abs(a-b) < Number.EPSILON 或 decimal 库。
