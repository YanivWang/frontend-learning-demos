Vue 2 基础指令与响应式绑定合集

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Vue 2 模板把声明式 DOM 和实例数据连接起来：{{ msg }} 渲染文本，v-bind 绑定 HTML 属性，v-on 绑定事件，v-model 在表单控件和状态之间建立双向绑定。
  • v-if 根据表达式结果决定节点是否渲染，v-for 根据数组渲染列表。本页列表给每个 li 增加 :key，让 Vue 在更新列表时能稳定复用或移动节点。
  • Vue 2 实例通过 el 挂载到一个 DOM 容器，data 中的字段会被代理到实例上并转换为响应式数据。事件方法里修改 this.msg 后，依赖它的模板会自动更新。
  • 本页为了教学把六个知识点拆成六个根实例，避免演示互相影响。真实项目里更常见的做法是一个根实例挂载应用，再用组件拆分功能区域。
  • 常见坑包括：插值只能用于文本内容，属性要用 v-bind；列表不要省略稳定 key；v-model 只负责表单值同步，不等于表单校验或提交逻辑。

面试考点

  • Vue 2 模板里的插值、指令和响应式数据是什么关系？ 结论是模板声明“数据如何映射到 DOM”，指令声明特殊绑定行为，响应式系统负责在数据变化后更新视图。原因是 Vue 会代理 data 字段并追踪模板依赖。项目里应修改状态而不是手写 DOM 操作。
  • v-bind、v-on、v-model 分别解决哪类绑定问题？ 结论是 v-bind 绑定属性，v-on 监听事件，v-model 封装表单值和输入事件的双向同步。原因是文本、属性、事件和表单输入属于不同 DOM 通道。项目里普通属性用 :，事件用 @，表单控件再考虑 v-model。
  • v-if 和 v-for 在基础 demo 中各自承担什么职责？ 结论是 v-if 做条件渲染，v-for 做列表渲染。原因是前者根据真假创建或销毁节点，后者根据集合生成重复节点。项目里两者同用在复杂列表时要注意优先级和可读性，通常先过滤数据再渲染。
  • 为什么 v-for 列表项建议提供稳定 key？ 结论是稳定 key 能帮助 Vue 识别每个节点的身份。原因是列表更新时框架需要判断复用、移动还是销毁节点，缺少 key 可能导致局部状态或表单输入错位。项目里优先使用业务唯一 id，不要用会变化的索引当 key。
  • 多个 Vue 根实例演示和真实组件化项目有什么区别？ 结论是多个根实例适合隔离教学，真实项目通常一个根实例加组件树。原因是组件树更利于状态流、复用和调试。项目里只有在迁移老页面或局部增强静态页面时，才会在同页挂多个独立实例。
