fetch 基础与 CORS 跨域

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • fetch 只有网络错误才会 reject；HTTP 404/500 需要检查 response.ok。
  • 跨域由浏览器同源策略限制，CORS 需要服务端返回 Access-Control-* 响应头。
  • 简单请求直接发出；非简单请求会先发送 OPTIONS 预检请求。
  • fetch 只有网络错误才会 reject；HTTP 404/500 需要检查 response.ok（补充）。

预检请求流程

  
浏览器
  |-- OPTIONS /api/resource
  |   Origin: https://a.com
  |   Access-Control-Request-Method: PUT
  |   Access-Control-Request-Headers: Content-Type
服务端
  |-- 204
  |   Access-Control-Allow-Origin: https://a.com
  |   Access-Control-Allow-Methods: PUT
  |   Access-Control-Allow-Headers: Content-Type
浏览器
  |-- PUT /api/resource
