函数也是对象

请打开 DevTools Console 查看输出。

知识点要点

  • 可调用对象：函数既有 [[Call]] 能力可以执行，也能像对象一样挂属性，所以 typeof fn 是 "function"，但它仍参与对象原型链。
  • 函数自身原型链：普通函数对象的 [[Prototype]] 指向 Function.prototype，而 Function.prototype 又链接到 Object.prototype。
  • 两个 prototype 概念：fn.__proto__ 描述函数对象自己从哪里继承；fn.prototype 是函数作为构造函数时给实例链接的原型对象。
  • instanceof 结果：fn instanceof Function 和 fn instanceof Object 都为 true，是因为原型链上分别能找到 Function.prototype 和 Object.prototype。
  • 项目意义：理解函数是对象，才能解释给函数挂静态属性、构造函数原型方法、手写 bind/call 以及 class 静态方法的本质。

面试考点

  • 为什么说函数也是对象？ 结论是函数是一种特殊对象：能被调用，也能拥有属性并沿原型链继承。原因是 JS 中函数对象有内部调用能力，同时仍链接到 Function.prototype 和 Object.prototype。
  • fn.__proto__ 和 fn.prototype 有什么区别？ fn.__proto__ 是函数对象自己的原型，通常等于 Function.prototype；fn.prototype 是它作为构造函数时给实例使用的原型对象。两者方向不同，不能混淆。
  • Function instanceof Object 为什么是 true？ 因为 Function 本身也是一个函数对象，它的原型链最终会到达 Object.prototype。这体现了“函数是对象”的统一模型。
  • 理解函数对象对原型链有什么帮助？ 它能解释构造函数为什么能挂静态属性、实例为什么能共享原型方法，以及 class 静态方法和实例方法为什么落在不同对象上。
