用正则识别 URL 与图片格式

请打开 DevTools Console 查看输出。

知识点要点

  • URL 匹配通常分：协议 https?、主机、路径、query；复杂 URL 用 URL API 更可靠。
  • 图片扩展名：\.(jpg|jpeg|png|gif|webp)(?=\?|$)；i 标志忽略大小写。
  • 贪婪 .* 匹配长 query 时可能大量回溯；改用 [^?#]+ 或非贪婪 .*?。
  • 生产环境 URL 解析优先 new URL(str)，正则仅做轻量格式提示。

面试考点

  • URL 校验正则？ 可写基础 pattern，但 RFC 复杂；推荐 URL 构造函数 + try/catch，正则辅助图片后缀等。
  • 贪婪量词风险？ (.*)* 类嵌套可能导致 catastrophic backtracking；收紧字符集、用占有量词或 RE2 风格。
  • 正则局限？ 难覆盖所有合法 URL、国际化域名、IPv6；安全校验必须配合 URL API 与白名单。
