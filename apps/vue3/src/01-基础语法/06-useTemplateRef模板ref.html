useTemplateRef（Vue 3.5+）

在浏览器中操作下方交互演示；复习要点与面试答法见本复习区。

知识点要点

  • useTemplateRef() 是 Vue 3.5+ API；本仓库 vue.global.js 为 3.5.x，示例可直接运行。Vue 3.4 及更早须用 ref(null) + 模板 ref 字符串配对。
  • 旧写法：const legacyInput = ref(null) 与 ref="legacyInput" 手动对齐。新写法：useTemplateRef("myInput") 与 ref="myInput" 同名绑定，减少命名分散。
  • 挂载后下方日志首两行显示 legacyInput=INPUT 与 myInput=INPUT，证明两种方式都拿到 DOM。点「聚焦 legacyInput」或「聚焦 myInput」会聚焦对应输入框并追加日志。
  • useTemplateRef 必须在 setup 同步阶段调用，以便登记到当前组件实例；异步回调中调用会缺少组件上下文。
  • 模板 ref 挂载前为 null；v-if 未渲染时同样为空。DOM 访问时机与 nextTick 见 05-模板ref与nextTick。
  • 指向子组件时拿到组件公开实例；<script setup> 默认私有，需 defineExpose 暴露方法。DOM vs 响应式 ref 区分见 02-ref-reactive与解构。

面试考点

  • useTemplateRef 相比 ref(null) 解决了什么问题？ 结论是把模板 ref 名称与 setup 中的 ref 显式关联，减少手动配对错位。边界：不改变 DOM 访问时机与空值风险。本页两种聚焦按钮行为一致。
  • useTemplateRef 的版本边界是什么？ 结论是 Vue 3.5+；3.4 及更早不能直接使用。项目须先确认运行时版本。
  • useTemplateRef 为什么要在 setup 同步阶段调用？ 结论是需要当前组件实例登记模板 ref。异步中调用会警告且无法关联。
  • 模板 ref 指向 DOM 和子组件时分别要注意什么？ DOM 适合聚焦/测量；子组件须 defineExpose 暴露可调用能力。
