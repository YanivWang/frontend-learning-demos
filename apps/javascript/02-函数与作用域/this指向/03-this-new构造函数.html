new 调用构造函数时 this 的行为

请打开 DevTools Console 查看输出。

知识点要点

  • new 的四步：创建空对象，把对象的 [[Prototype]] 链到构造函数 prototype，用新对象作为 this 执行构造函数，最后返回实例。
  • 返回值规则：构造函数显式返回对象时，new 的结果会变成该对象；返回原始值或不返回时，结果仍是新创建的实例。
  • 漏写 new 风险：传统构造函数直接调用时按普通函数绑定 this，非严格模式可能写到全局对象，严格模式下可能因为 this 是 undefined 报错。
  • 现代写法：新代码优先用 class 或工厂函数表达意图；class 构造器不能不带 new 调用，能减少漏写风险。
  • 类型关系：实例能通过原型链访问 Person.prototype 上的方法，因此 instanceof Person 为 true。

面试考点

  • new 操作符内部做了哪几步？ 结论是创建对象、链接原型、绑定 this 执行构造函数、返回对象。原因是 JS 的“实例”本质依赖原型链；项目里理解这点才能解释 instanceof、原型方法共享和手写 new。
  • 构造函数显式 return 对 new 的结果有什么影响？ 如果返回对象，最终结果就是该对象；如果返回原始值或没有返回，结果仍是新实例。边界是构造函数里随意 return 对象会破坏实例原型关系。
  • 漏写 new 为什么危险？ 传统构造函数会变成普通函数调用，this 不再指向新实例。非严格模式可能污染 window，严格模式通常报错；项目里用 class、工厂函数或命名规范降低风险。
  • class 和传统构造函数有什么差异？ class 语法仍基于原型，但构造器必须用 new 调用，并默认运行在严格模式。维护旧代码时要理解两者的共同原型模型和调用边界。
  • 手写 new 要注意哪些边界？ 要用 Object.create(Constructor.prototype) 建立原型关系，并处理构造函数返回对象的情况。还要拒绝不可构造的目标，例如箭头函数没有 prototype，不能被 new。
