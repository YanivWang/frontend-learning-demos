useTemplateRef（Vue 3.5+）

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • useTemplateRef() 是 Vue 3.5+ 提供的模板 ref 辅助 API；本仓库当前 vue.global.js 是 3.5.x，所以示例可以直接运行。
  • 它通过 useTemplateRef("myInput") 与模板中的 ref="myInput" 建立关联，减少 const myInput = ref(null) 与字符串 ref 手动配对时的命名分散。
  • useTemplateRef() 必须在 setup() 同步执行期间调用，用来绑定当前组件实例；异步回调或普通工具函数里调用会缺少当前组件上下文。
  • 返回值由模板渲染结果赋值，挂载前通常为 null；访问 DOM 要放在 onMounted 后或用户事件中，并做好条件渲染导致为空的边界处理。
  • 如果项目仍在 Vue 3.4 或更早版本，继续使用 ref(null) + 模板 ref 的旧写法；不要为了一个窄 API 强行升级整个项目。

面试考点

  • useTemplateRef 相比 ref(null) 解决了什么问题？ 结论是它把“模板 ref 名称”和“setup 中的 ref 对象”显式关联起来，减少手动初始化和命名错位。原因是旧写法需要同时维护 ref(null) 变量和模板字符串。边界是它仍然只是模板 ref，不改变 DOM 访问时机和空值风险。
  • useTemplateRef 的版本边界是什么？ 先结论：它是 Vue 3.5+ API，Vue 3.4 及更早版本不能直接使用。原因是该函数在 3.5 才进入官方 API。项目里要先确认运行时版本；如果库或业务仍锁在旧版本，应保留 ref(null) 写法。
  • useTemplateRef 为什么要在 setup 同步阶段调用？ 结论是它需要当前组件实例来登记模板 ref。原因是模板 ref 的归属和赋值都依赖正在创建的组件上下文。异步回调中调用时当前实例可能不存在，Vue 会警告且无法关联模板。
  • 模板 ref 指向 DOM 和子组件时分别要注意什么？ 指向 DOM 时拿到的是原生元素，适合聚焦、测量和滚动；指向子组件时拿到组件公开实例。使用 <script setup> 的子组件默认私有，需要通过 defineExpose 暴露可调用能力。
