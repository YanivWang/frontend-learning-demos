template 编译到 render 函数

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Vue 2 的 template 不是直接变成真实 DOM，而是先编译成 render 函数；render 执行后返回 vnode，再由 patch 更新真实 DOM。
  • 完整版 Vue 包含模板编译器，可以在浏览器里编译字符串模板；runtime-only 构建体积更小，但要求模板在构建阶段已经编译成 render。
  • 单文件组件中的 <template> 通常由 vue-loader 预编译，这样浏览器运行时不需要携带编译器，也减少启动时编译成本。
  • 手写 render(h) 可以表达高度动态的标签、插槽或渲染逻辑；普通业务页面优先 template，因为更直观，也更容易被团队维护。
  • 示例同时注册 template-demo 和 render-demo，点击按钮后两者都会随同一个响应式 message 更新，说明它们最终都走 vnode 更新链路。

面试考点

  • template 到真实 DOM 的过程是什么？ 结论是 template 编译成 render，render 返回 vnode，patch 把 vnode 更新到真实 DOM。原因是 Vue 用 vnode 统一描述 UI 更新。项目里看到 render 函数和 template，本质上都是生成 vnode 的不同写法。
  • runtime-only 和完整版 Vue 有什么差别？ 结论是完整版带模板编译器，runtime-only 不带。原因是编译器会增加包体积和运行时成本。项目里如果使用 SFC + 构建工具，通常选 runtime-only；直接在 HTML 写 template 字符串则需要完整版。
  • 为什么工程项目要预编译模板？ 结论是减少浏览器端编译成本和包体积。原因是构建阶段已经能把模板转成 render 函数，运行时只需执行。边界是动态从服务器下发模板字符串的低频场景才可能需要运行时编译器，并要额外注意安全风险。
  • template 和 render 怎么选？ 结论是普通页面用 template，高度动态结构或封装底层组件时考虑 render。原因是 template 可读性高，render 表达能力更强。项目里 render 常见于表格列、插槽代理、动态表单等抽象组件。
