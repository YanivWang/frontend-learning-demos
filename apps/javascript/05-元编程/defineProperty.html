Object.defineProperty 数据属性

请打开 DevTools Console 查看输出。

知识点要点

  • 数据属性由 value、writable、enumerable、configurable 描述：值本身、能否赋值、能否被枚举、能否删除或重新配置。
  • Object.defineProperty 新增属性时，未显式写出的布尔描述符默认都是 false；对象字面量或普通赋值创建的属性通常三者都是 true。
  • writable: false 会阻止重新赋值；非严格模式下赋值失败可能静默，严格模式下会抛出 TypeError。
  • configurable: false 是最容易踩坑的开关：属性不能删除，描述符大多不能再改，只允许把 writable 从 true 改成 false。
  • 调试属性行为时优先用 Object.getOwnPropertyDescriptor 查看真实描述符，不要只看 console.log 展示出的对象外观。

面试考点

  • defineProperty 创建的数据属性默认为什么不可写、不可枚举、不可配置？ 结论是规范默认值如此：省略的布尔描述符会按 false 处理。原因是 defineProperty 面向精确描述符定义，而不是普通业务赋值。项目里如果希望表现得像普通属性，要显式写出 writable/enumerable/configurable: true。
  • writable、enumerable、configurable 分别影响哪些对象操作？ 结论是 writable 管赋值，enumerable 管 Object.keys、for...in、对象展开等枚举入口，configurable 管删除和重新配置。原因是它们分别对应数据属性的不同内部特性。边界是 JSON.stringify 也只序列化可枚举自有属性。
  • configurable: false 之后还能修改哪些描述符？ 结论是几乎不能改，通常只能把仍为 true 的 writable 降为 false。原因是不可配置属性要保证对象形状稳定。封装库里一旦把公共属性设成不可配置，后续热更新、测试 mock 或兼容补丁都会更难做。
  • defineProperty 和对象字面量创建属性的默认特性有什么区别？ 结论是字面量/赋值更偏日常可变属性，defineProperty 更偏精确控制。原因是普通创建方式默认可写、可枚举、可配置，而 defineProperty 默认收紧。需要隐藏内部状态、定义只读 API 或兼容 Vue 2 响应式拦截时才更常用它。
