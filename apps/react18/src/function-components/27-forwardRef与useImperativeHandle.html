forwardRef 与 useImperativeHandle

请打开 DevTools Console 查看命令式 ref 调用日志；下方 #root 为 React 挂载/交互演示区。

知识点要点

  • 在 React 18 中，普通函数组件不会自动接收 ref ；需要用 forwardRef 把父组件传入的 ref 转发到内部 DOM 或自定义句柄。
  • useImperativeHandle(ref, createHandle, deps) 可以隐藏真实 DOM 节点，只暴露父组件需要的少量方法，例如本 demo 的 focus() 与 clear() 。
  • 依赖数组应覆盖暴露方法读取的 props、state 或函数引用；如果句柄只调用稳定的 DOM ref 与 state setter，空依赖通常可接受。
  • 命令式 ref API 适合聚焦、滚动、文本选择、媒体播放、第三方组件实例桥接；业务数据流仍优先通过 props、state、Context 或事件回调声明式表达。
  • 版本边界：React 19 开始支持把 ref 作为 prop 传给函数组件；本仓库 React 18 demo 仍应使用 forwardRef 。

面试考点

  • React 18 函数组件为什么不能像 DOM 节点一样直接拿 ref？ 结论是 React 18 中 ref 不是普通 prop，函数组件没有实例可挂。原因是函数组件只是渲染函数。需要让父级操作内部 DOM 时，用 forwardRef 明确声明这个组件支持 ref 转发。
  • useImperativeHandle 相比直接暴露 DOM 节点有什么价值？ 结论是它把内部实现封装成稳定、受控的命令式接口。原因是父组件只拿到 focus 、 clear 这类方法，不能随意改内部 DOM。项目里封装输入框、播放器、弹窗实例时更容易维护边界。
  • 命令式 ref API 适合哪些场景？ 适合 DOM 能力或第三方实例能力，例如聚焦、滚动、选中文本、播放暂停。原因是这些行为本身不是纯 UI 状态。边界是业务开关、表单值、权限状态应优先走声明式数据流，否则父子组件会强耦合。
  • React 19 的 ref as prop 与 React 18 有什么边界？ React 19 中函数组件可把 ref 当 prop 接收，减少 forwardRef 包装。React 18 当前推荐仍是 forwardRef ；面试回答要明确版本，不能把 React 19 能力写成 React 18 现状。
