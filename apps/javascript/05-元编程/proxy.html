Proxy 代理与基本拦截器

请打开 DevTools Console 查看输出。

知识点要点

  • Proxy 会在目标对象外包一层代理，handler 里的 trap 可以拦截读取、写入、in、删除、枚举、函数调用等基本对象操作。
  • get(target, property, receiver) 用于读取属性；set(target, property, value, receiver) 用于写属性，并且必须返回布尔值表示写入是否成功。
  • 在 trap 内优先用 Reflect.get、Reflect.set 做默认转发，能减少手写 target[property] 带来的继承、访问器 this 和返回值问题。
  • Proxy 本身没有可用于 instanceof Proxy 的 prototype，代理对象通常仍表现得像目标对象；需要识别时应使用业务标记、WeakMap 或能力检测。
  • 与 Object.defineProperty 相比，Proxy 能覆盖新增属性、删除属性和更多元操作，但它不能被完整 polyfill，旧浏览器兼容性要单独评估。

面试考点

  • Proxy 的 get/set trap 分别能拦截什么，参数含义是什么？ 结论是 get 拦截属性读取，set 拦截属性写入。原因是它们分别对应对象内部的取值和赋值操作，参数里的 target 是原对象，property 是属性键，receiver 是本次访问的接收者，set 还多一个新值。项目里常用它们做响应式依赖收集、校验、日志和权限边界。
  • 为什么 set trap 必须返回 true 或 Reflect.set 的结果？ 结论是 set trap 的返回值表示写入是否成功。原因是规范会把返回值转成布尔值，严格模式下返回 false 会抛出 TypeError。实际写法通常直接 return Reflect.set(target, property, value, receiver)，避免遗漏。
  • Proxy 和 Reflect 为什么经常一起使用？ 结论是 Proxy 负责拦截，Reflect 负责按默认语义转发。原因是 Reflect 的方法名和许多 trap 对应，参数也能保留 receiver。在有 getter、继承链或不可写属性时，Reflect 比手写 target[key] 更接近语言本身行为。
  • Proxy 与 Object.defineProperty 在响应式实现上有什么区别？ 结论是 defineProperty 针对具体属性，Proxy 针对整个对象的元操作。原因是 defineProperty 需要逐个属性改写 getter/setter，而 Proxy 可以天然感知新增、删除、in、枚举等操作。边界是 Proxy 无法完整降级到旧环境，兼容性要求高时要保留替代方案。
