Vite 工程化结构与面试要点

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Vite 开发服务器把源码作为原生 ESM 按需提供给浏览器，不需要冷启动时打包完整应用；第三方依赖会用 esbuild 预构建成更适合浏览器加载的模块。
  • 生产构建默认使用 Rollup，是因为生产环境仍需要 tree-shaking、代码分割、资源哈希、压缩和兼容性处理；所以面试中不能简单说“Vite 等于不打包”。
  • import.meta.env 暴露构建时环境信息，只有 VITE_ 前缀变量会进入客户端 bundle；API 密钥、数据库密码等服务端秘密不能放在这里。
  • Vue 工程常见目录是 views、components、composables、stores、router、assets、services，目录约定应服务于依赖边界和团队协作。
  • 路由懒加载使用 () => import('@/views/Home.vue')，组件级异步加载可用 defineAsyncComponent；拆包后要关注加载失败、重复 chunk、首屏关键路径和预加载策略。

面试考点

  • Vite 为什么开发启动通常更快？ 结论是 Vite 开发期不先打完整应用，而是按浏览器请求即时转换源码。原因是现代浏览器支持原生 ESM，Vite 只需预构建依赖并按需处理业务模块；边界是首次访问某个页面仍会触发该模块转换，超大项目还要关注依赖预构建缓存和插件成本。
  • Vite 开发期和生产构建为什么不是同一套实现？ 结论是开发期优先反馈速度，生产期优先产物质量。原因是开发服务器面向按需加载和 HMR，生产构建需要 Rollup 做 tree-shaking、chunk、hash 和压缩；项目里要分别调试 dev server 配置和 build 配置，不能只看开发环境表现。
  • import.meta.env.VITE_ 前缀解决了什么边界？ 结论是它明确哪些变量允许暴露给客户端。原因是前端环境变量会被打进浏览器代码，不具备保密性；边界是 VITE_API_BASE 这类公开配置可以放，私钥、数据库密码和服务端 token 不能放。
  • 路由懒加载和异步组件如何配合？ 结论是页面级拆包优先用路由动态 import，局部重组件再用 defineAsyncComponent。原因是路由天然是用户导航边界，适合拆成独立 chunk；边界是首屏关键组件不要过度懒加载，异步组件要提供 loading/error 体验。
