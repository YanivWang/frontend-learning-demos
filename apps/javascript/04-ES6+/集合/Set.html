Set 值的集合

请打开 DevTools Console 查看输出。

知识点要点

  • Set 存储唯一值，使用 SameValueZero 判断相等：NaN 可被识别为同一个值，0 和 -0 视为相同。
  • Set 保留成功插入的顺序，for...of、values()、keys() 和 entries() 都按插入顺序迭代。
  • [...new Set(arr)] 是数组去重常见写法，但对象按引用身份比较，两个内容相同的对象字面量仍会被视为不同值。
  • 大量成员存在性判断时，set.has(value) 通常比反复 array.includes(value) 更适合；但如果需要索引、排序或重复次数，仍应使用数组或 Map 计数。
  • Set 只保存值；如果要给每个键关联额外数据，应使用 Map。

面试考点

  • Set 如何判断两个值是否重复？ 它使用 SameValueZero，相比 === 的重要差异是 NaN 等于 NaN。对象仍按引用判断，所以对象内容相同不代表 Set 里是同一个值。
  • Set 数组去重为什么不能按对象内容去重？ 因为对象值比较的是引用身份，{ id: 1 } 和另一个 { id: 1 } 是两个对象。项目里按业务字段去重时应先用 Map 以 ID 为键，或手写归一化逻辑。
  • Set 和 Array 在成员查询上如何选择？ 如果只是少量数据或需要保持数组操作，includes 足够；大量查询、频繁判断是否已处理时用 Set.has 更清晰。边界是 Set 不保存重复项和索引信息。
  • Set、Map、WeakSet 分别适合什么场景？ Set 表达唯一值集合，Map 表达键值映射，WeakSet 表达对对象的临时弱标记。需要可枚举和 size 用 Set/Map，需要不阻止对象回收用 WeakSet/WeakMap。
