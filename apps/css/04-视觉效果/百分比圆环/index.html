旧式纯 CSS 百分比圆环组件

下方组件来自 Pure CSS Percentage Circle：通过百分比类名、半圆裁剪和旋转展示 12%、25%、50% 等静态进度。

知识点要点

  • 这个组件把进度写成类名：.p25、.p50、.p75 等分别对应百分比，样式表中按 百分比 * 3.6deg 生成 transform: rotate(...)。
  • 圆环由两个半圆拼出：.slice 负责裁出可见区域，.bar 旋转显示当前半圆进度，超过 50% 时再让 .fill 显示另一半。
  • dark、green、orange、big、small 都只是修饰类，改变颜色或字号尺寸，不改变百分比计算逻辑。
  • 配套样式使用了已废弃的 clip: rect() 和大量预生成选择器，适合理解早期 CSS 技巧；新项目通常更推荐 conic-gradient()、SVG stroke-dasharray 或组件库进度环。
  • 可访问性边界是圆环视觉本身不等于进度语义。真实进度组件应提供文本数值，必要时补 role="progressbar"、aria-valuenow、aria-valuemin 和 aria-valuemax。

面试考点

  • 纯 CSS 圆环如何用两个半圆表示百分比？ 结论是把圆环拆成左右两个可裁剪半圆，再用旋转角度映射进度。原因是一个完整圆是 360 度，每 1% 对应 3.6 度，.p25 就让半圆旋转到 90 度。这个 demo 的算法写在 css/circle.css 的 .p1 到 .p100 选择器里。
  • 为什么 50% 以上需要 fill 半圆？ 结论是单个半圆最多只能覆盖半个圆环。原因是前 0-50% 只需要旋转一个被裁剪的半圆，超过 180 度后必须解除或改变裁剪，并显示另一个固定半圆补足剩余区域。项目里如果用这种旧方案，要特别检查 50% 临界点是否跳变。
  • clip: rect() 有什么维护风险？ 结论是它能解释旧组件机制，但不适合作为新代码首选。原因是 clip 已废弃，且只适用于绝对定位元素；大量百分比类也让动态进度更新不灵活。现代项目通常用 conic-gradient() 写静态圆环，或用 SVG 更精确地控制动画、圆角和可访问性。
  • CSS、SVG、Canvas 圆环如何选型？ 结论是静态装饰圆环用 CSS，数据型进度环优先 SVG，高频复杂绘制再考虑 Canvas。原因是 CSS 轻量但语义和精细控制有限，SVG 有图形语义、描边和动画能力，Canvas 适合大量绘制但需要自己补可访问性。后台仪表盘或上传进度通常更偏向 SVG/组件库方案。
