盒模型：content-box vs border-box

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • box-sizing: content-box 是标准默认值，width/height 只表示内容区尺寸，最终渲染宽度还要加上左右 padding 和 border。
  • box-sizing: border-box 让 width/height 包含内容区、内边距和边框，但不包含 margin；本例中同样写 width: 180px，border-box 的最终占宽仍是 180px。
  • 现代工程常用 *, *::before, *::after { box-sizing: border-box; }，因为栅格、卡片和表单控件更容易按容器宽度计算；需要精确让定位偏移相对内容区时，仍可能保留 content-box。
  • 垂直 margin 折叠只发生在普通文档流中的块级盒之间，常见于相邻兄弟、父元素和第一个/最后一个流内子元素、空块；浮动、绝对定位、flex/grid 容器内项目不会发生普通 margin 折叠。
  • 阻止父子 margin 折叠可以给父元素加 padding、border、内联内容，或创建新的块格式化上下文；本例使用 display: flow-root 是现代、语义较清晰的 BFC 写法。

面试考点

  • content-box 和 border-box 的尺寸计算有什么区别？ 结论是 content-box 的 width 只算内容区，border-box 的 width 包含内容、padding 和 border。原因是 box-sizing 改变浏览器计算总宽高的方式。项目里卡片、网格和输入框多用 border-box 防止 padding 把元素撑破容器，但 margin 始终在盒子外部，不会被纳入这两种计算。
  • 为什么现代项目常把全局 box-sizing 设为 border-box？ 结论是它能让“元素占多宽”更接近代码里写的宽度。原因是 padding 和 border 被包含在声明的宽度内，响应式列布局不需要反复手算额外尺寸。边界是第三方组件或需要按内容区定位的场景可能依赖默认 content-box，全局重置时要覆盖伪元素并留意兼容样式。
  • margin 折叠发生在哪些场景，如何阻止？ 结论是垂直 margin 折叠主要发生在普通流块级盒的相邻兄弟、父子边界和空块中。原因是 CSS 规范会把这些外边距合并成一个外边距，避免连续段落间距被简单相加。项目中如果父元素背景没有包住子元素上边距，可给父元素加 padding/border，或用 display: flow-root 创建 BFC。
  • BFC 在盒模型和 margin 折叠问题里有什么作用？ 结论是 BFC 是一个独立的块级布局区域，可以隔离内部布局影响。原因是新的 BFC 会阻止父子 margin 折叠，并能包含内部浮动。项目里清除浮动和隔离折叠优先用 display: flow-root；老代码可能用 overflow: hidden，但它会带来裁剪内容的副作用。
