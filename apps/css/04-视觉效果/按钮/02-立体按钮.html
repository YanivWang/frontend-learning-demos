拟物 3D 按钮的按下动效

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，按住按钮时脚本会读取 :active 阴影变化。

知识点要点

  • 拟物按钮的核心是把 box-shadow 的垂直偏移当作“厚度”，默认状态阴影更长，按下时阴影变短。
  • :active 表示元素正在被激活，鼠标按住、触摸按下或键盘触发时会短暂匹配，适合做即时点击反馈。
  • 位移推荐用 transform: translateY() ，它不会改变文档流位置；修改 top 也能看到效果，但更容易牵涉布局和定位细节。
  • 真实交互控件应使用 <button> ，因为它天然支持键盘、焦点、禁用状态和表单语义；用 div 需要额外补键盘事件和 ARIA，容易漏边界。
  • 动效要保留可见焦点，并尊重 prefers-reduced-motion ；按钮文案也要避免只靠颜色或阴影表达状态。

面试考点

  • 立体按钮为什么通常用 box-shadow 模拟厚度？ 结论是阴影偏移可以低成本制造按钮离开背景的错觉。原因是人眼会把下方较深的投影理解成厚度或光照差。本页按下时缩短阴影并下移按钮，就是把“凸起”变成“压下”。
  • :active 点击反馈应该改 top 还是 transform？ 结论是现代写法优先用 transform 。原因是 transform 更适合做视觉位移，通常不会改变布局位置； top 依赖定位上下文，维护成本更高。边界是静态小 demo 改 top 也能演示，但项目组件应优先选择 transform。
  • 为什么不能用 div 假装按钮？ 结论是能点击的控件应优先使用 button 。原因是原生按钮自带键盘触发、焦点、禁用和辅助技术语义， div 需要手工补齐且容易遗漏。项目里只有在语义不是按钮时，才考虑其他元素加相应角色和事件处理。
  • 按钮动效有哪些无障碍边界？ 结论是动效不能替代语义、焦点和文本状态。原因是部分用户依赖键盘或开启减少动效，过强位移也可能造成不适。真实项目里要提供 :focus-visible 、明确文字/状态，并用 prefers-reduced-motion 降低非必要动画。

box-shadow 模拟厚度

默认 box-shadow: 0 4px 0 表示凸起； :active 时阴影缩短并 translateY(2px) 模拟按下（MDN :active）。
