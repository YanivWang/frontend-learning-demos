用正则匹配 URL 中的语言段与路径段

下方为 console.log 同步输出；也可打开 DevTools Console 查看。

知识点要点

  • [course|unit] 是字符类，匹配 c/o/u/r/s/e/|/n/i/t 中任一字符，不是「course 或 unit」。
  • 多选分支用分组：(?:course|unit) 非捕获，(course|unit) 捕获。
  • g 全局匹配时 lastIndex 会递增；同一 RegExp 对象多次 test 可能因 lastIndex 得到不同结果。
  • 单次校验建议不用 g，或每次 new RegExp / lastIndex=0。

面试考点

  • [] vs ()？ [] 字符类/范围；() 分组与捕获。(a|b) 是或；[ab] 是 a 或 b 单字符。
  • 贪婪/非贪婪？ * + 默认贪婪；*? +? 非贪婪。URL/query 长串用非贪婪或限定字符集防回溯。
  • g 对 test 的影响？ test 从 lastIndex 开始；匹配成功后 lastIndex 前进。循环 test 同一对象需重置或不用 g。
