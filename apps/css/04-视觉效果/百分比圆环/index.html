CSS3 transform 百分比圆环组件

下方圆环用两个半圆、overflow:hidden 和 transform: rotate() 表达 12%、25%、50%、72%、88% 等进度，只依赖 CSS3 常见能力。

知识点要点

  • 圆环由左右两个半圆容器组成，半圆容器本身负责裁剪；内部 .ring-fill 是真正旋转的彩色半圆。
  • 一个完整圆是 360 度，所以 1% 对应 3.6 度；例如 25% 是 90 度，50% 是 180 度。
  • 0-50% 只需要旋转右半圆；超过 50% 后，右半圆固定到 180 度，左半圆再旋转剩余角度，例如 72% 的左半圆旋转 (72 - 50) * 3.6 = 79.2deg。
  • 这种方案比旧式 clip 属性更适合 CSS3 教学，但如果要做连续动态进度、圆角描边、刻度或复杂图表，SVG 的 stroke-dasharray 往往更精确。
  • 进度环不能只靠颜色表达信息。本页每个圆环都有可见数值，并提供 role="progressbar"、aria-valuenow、aria-valuemin 和 aria-valuemax，方便辅助技术读取。

面试考点

  • CSS3 百分比圆环如何用两个半圆和 transform 表示进度？ 结论是把圆环拆成左右两个可裁剪半圆，再用 rotate() 映射进度角度。原因是 CSS3 transform 可以稳定旋转元素，配合 border-radius 和 overflow:hidden 就能模拟环形进度。
  • 为什么 50% 以上需要左右半圆分段处理？ 结论是单个半圆最多只覆盖 180 度。原因是右半圆旋转到 180 度后已经到达半圈，再继续转会超出它的裁剪区域；超过部分必须交给左半圆补足。项目里要特别检查 50% 临界值是否跳变。
  • transform 方案和 SVG / Canvas 圆环怎么选？ 结论是静态展示和 CSS3 教学可以用 transform 半圆方案，数据型组件优先 SVG，大量图形或高频绘制再考虑 Canvas。原因是 CSS 轻量但状态类较多，SVG 更容易控制描边、圆角、动画和语义，Canvas 则需要额外补可访问性。
  • 进度环如何补齐可访问性语义？ 结论是要同时给人看和给机器读。原因是颜色和圆环形状本身不能保证所有用户都能理解进度，辅助技术也无法从旋转角度推断数值。项目里应提供文本百分比、role="progressbar" 和 aria 数值；如果进度会变化，还要考虑状态更新提示。
