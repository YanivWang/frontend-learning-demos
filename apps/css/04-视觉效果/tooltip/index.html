纯 CSS tooltip（上下左右四方向）

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出伪元素与 attr() 的实测信息。

知识点要点

  • 本页用 ::before 画小三角，用 ::after 生成提示气泡， content: attr(data-tip) 会读取元素上的 data-tip 文案。
  • 定位机制是让触发元素 position: relative ，伪元素 position: absolute ，再用 top 、 bottom 、 left 、 right 和 margin 控制四个方向。
  • 三角形来自透明边框技巧：元素尺寸为 0 时，四条边框会形成四个三角区域，只给某一侧设置颜色就能得到箭头。
  • 交互上不能只写 :hover ，因为键盘和触屏用户可能无法触发；本页额外支持 :focus-visible ，按钮获得焦点时也能显示提示。
  • 纯 CSS tooltip 适合短文本装饰说明；如果提示内容很长、需要关闭逻辑、动态定位、防溢出或读屏器稳定朗读，应使用 JS 管理状态并补充 aria-describedby / role="tooltip" 。

面试考点

  • 纯 CSS tooltip 如何生成提示内容？ 结论是用伪元素生成结构，用 attr(data-tip) 读取提示文案。原因是 ::before / ::after 可以通过 content 插入生成内容，并相对触发元素定位。边界是生成内容不适合承载复杂交互，也不能替代可靠的无障碍语义。
  • 为什么 tooltip 不能只依赖 :hover？ 结论是只依赖 hover 会漏掉键盘和部分触屏场景。原因是键盘用户通过焦点导航，触屏设备没有稳定的悬停状态。本页补了 :focus-visible ，真实项目还应根据交互复杂度加入 JS、Escape 关闭和 aria-describedby 。
  • CSS 三角形的原理是什么？ 结论是利用透明边框拼出的斜切区域。原因是零尺寸盒子的四条 border 会分别形成三角形，只给目标方向的边框着色，其余保持透明，就能得到箭头。项目里要注意箭头尺寸、气泡背景色和方向边框颜色保持一致。
  • 纯 CSS tooltip 适合什么场景？ 结论是适合短提示、静态方向和低风险装饰说明。原因是 CSS 缺少碰撞检测、异步内容、关闭时机和完整可访问性状态管理。复杂表单错误提示、动态菜单或移动端长文本说明应使用组件化 tooltip/popover。
