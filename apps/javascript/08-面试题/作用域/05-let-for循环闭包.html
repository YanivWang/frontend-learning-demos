let + for 循环的块作用域闭包

先看本复习区，再查看下方输出是否与标准答法一致；细节可打开 DevTools Console。

知识点要点

  • for (let i = 0; ...) 在每次迭代都会创建新的词法环境，闭包保存的是该轮迭代自己的 i 绑定。
  • 本例把 5 个函数存入数组，循环结束后再执行；由于每个函数捕获不同的 i ，所以依次输出 0 到 4 。
  • 如果把循环头改成 var i ， var 只有函数作用域，5 个函数共享同一个 i ，循环结束时 i === 5 ，常见输出就是 5 个 5 。
  • 老环境或老代码中不用 let 时，可以用 IIFE、函数参数或数组方法回调参数为每轮迭代创建快照，但现代代码优先用 let 。
  • let / const 还有暂时性死区和禁止重复声明等语义，能让循环回调、事件绑定、异步任务中的变量边界更清晰。
  • 项目里如果不需要手动控制索引， forEach 、 map 等数组方法的回调参数也能天然提供每次调用的局部参数，但需要注意异步并发和提前退出能力不同。

面试考点

  • for (let i...) 中的闭包为什么能分别输出 0 到 4 ？ 结论：因为每次迭代都有独立的 i 绑定。原因是规范为 for 循环头中的 let 创建 per-iteration 词法环境，闭包捕获的是对应迭代的绑定。项目里这能直接修复循环中绑定事件、创建异步回调时的索引错乱问题。
  • 把 let 换成 var 为什么常见结果会变成 5 个 5 ？ 结论：所有闭包共享同一个函数作用域变量 i 。原因是 var 没有块级作用域，也不会为每次迭代创建新绑定；等回调执行时循环已经结束， i 变成 5 。边界是如果回调在每轮同步立即执行，看到的值可能仍是当轮值。
  • 老代码中不用 let 如何修复循环闭包问题？ 结论：用 IIFE 或函数参数把当前索引作为新作用域里的参数保存。原因是参数绑定属于一次函数调用的局部环境，闭包会捕获这份局部参数。现代项目通常不再这样写，除非维护旧浏览器代码或解释历史面试题。
  • let 循环和数组方法回调怎么选？ 结论：需要传统循环控制时用 for + let ，数据转换遍历时优先用语义化数组方法。原因是 for 支持 break 、 continue 和更直接的性能控制， map / forEach 则让每次回调参数天然局部化。边界是异步回调仍要额外考虑并发顺序和错误处理。

运行结果

标准答法： for (let i...) 依次输出 0, 1, 2, 3, 4 ；若换成 var 则常见为 5 个 5 ；IIFE 可修复 var 版。
