多行文本省略（line-clamp）

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出省略与行高实测值。

知识点要点

  • 单行省略靠 white-space: nowrap + overflow: hidden + text-overflow: ellipsis ；多行省略则要限制 最大行数 ，常见写法是 line-clamp: N 配合 overflow: hidden 。
  • MDN 说明 line-clamp 是截断块级容器内行数的简写；旧写法还需 display: -webkit-box 与 -webkit-box-orient: vertical ，现代浏览器可直接写 line-clamp 。
  • 省略生效前提是容器有 可计算宽度 ；在 flex/grid 子项里如果省略失效，常见原因是默认 min-width: auto 让子项被内容撑开，需要补 min-width: 0 。
  • 被裁掉的文本仍然在 DOM 中，但视觉上不可见；标题、摘要如果承载关键信息，hover 展开、详情页或 title /ARIA 提示仍要考虑，不能把所有语义都压在 CSS 省略上。
  • text-overflow: ellipsis 只在块级/行内块且发生 行内方向溢出 时生效；多行场景应使用 line clamping，而不是强行 nowrap 单行截断。

面试考点

  • 多行省略和单行 ellipsis 的核心区别是什么？ 结论是多行省略要限制行数并截断盒子高度，单行 ellipsis 只处理一行不换行文本。原因是单行方案依赖 nowrap ，多行方案依赖 line-clamp 。项目里卡片摘要、评论预览常用 2–3 行 clamp，表格单元格标题仍可能用单行 ellipsis。
  • line-clamp 为什么常配合 overflow: hidden ？ 结论是没有隐藏溢出，超出行数的文本仍会显示出来。原因是 clamp 通过限制盒子展示区域实现“截断”，溢出部分必须被裁掉，省略号才会出现。
  • flex 子项里多行省略不生效通常是什么原因？ 结论是子项默认最小宽度受内容保护，容器宽度算不出来。原因是 flex item 的 min-width: auto 会阻止收缩。项目里文本区常见写法是 flex: 1; min-width: 0; overflow: hidden; line-clamp: 2 。
  • 多行省略有哪些可访问性和 SEO 边界？ 结论是 CSS 省略不会删除 DOM 文本，但视觉和部分交互场景下用户可能看不到完整内容。原因是被裁切内容不可见，屏幕阅读器是否完整朗读还取决于 DOM 结构和 aria 设计。项目里重要信息应保留完整文本入口，而不是只显示 clamp 后的摘要。

容器写 line-clamp: 3 + overflow: hidden ，超过 3 行后截断并显示省略号。

这是一段用于演示多行省略的长文本。单行 ellipsis 只能处理一行，而卡片摘要、评论预览、搜索结果描述通常需要保留两到三行后再截断。line-clamp 会在指定行数后显示省略号，同时配合 overflow hidden 裁掉后续内容。

flex 卡片中的多行省略

文本列必须 min-width: 0 ，否则 flex 子项会被长文本撑开，line-clamp 可能失效。

左侧缩略图 + 右侧摘要是非常常见的列表卡片结构。如果忘记给文本列设置 min-width: 0，flex 子项会被长文本撑开，多行省略就不会按预期工作。
