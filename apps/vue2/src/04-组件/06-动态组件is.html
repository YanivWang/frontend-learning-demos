动态组件 <component :is="...">

点 Home / About / User 切换 tab，在某 tab 的输入框输入文字后再切换回来，验证 keep-alive 是否保留输入内容。

知识点要点

  • <component> 是 Vue 的动态组件占位符， :is 可以绑定组件名字符串，也可以绑定组件选项对象。
  • 本页通过 currentTab 在 Home/About/User 之间切换，按钮只改变状态，渲染组件由 :is 决定。
  • 动态组件默认切换会卸载旧实例并创建新实例；放入 <keep-alive> 后，切走的组件实例被缓存，输入框内容可以保留。
  • 适合 tab、步骤页、插件面板这类“同一位置切换组件类型”的场景；如果只是控制同一个组件是否显示， v-if 或 v-show 更直接。
  • 被切换的组件必须可解析：要么已全局注册，要么在当前实例局部注册，要么直接把组件对象赋给 is 。

面试考点

  • component 的 is 属性可以接收哪些值？ 结论：可以接收已注册组件名字符串，也可以接收组件选项对象。原因是 Vue 在渲染动态组件时会根据该值解析出要渲染的组件定义。边界是 DOM 模板里组件名要注意大小写和注册名匹配。
  • 动态组件切换时组件实例默认会怎样变化？ 结论：默认旧组件会被销毁，新组件会被创建。原因是当前渲染的组件类型变了，Vue 需要替换对应组件实例。需要保留表单、滚动或缓存数据时，用 keep-alive 包住动态组件。
  • keep-alive 为什么能保留 tab 内输入状态？ 结论：它缓存的是组件实例，不只是 DOM。原因是被切走的组件不会走销毁流程，再次进入时复用原实例。边界是缓存过多会占内存，复杂场景要配置 include、exclude 或 max。
  • 动态组件、v-if 和路由切换如何取舍？ 结论：同位置互斥组件类型用动态组件，简单条件块用 v-if / v-show ，页面级导航用路由。原因是动态组件把组件类型变成数据，路由则管理 URL 与历史栈。边界是需要保留状态时配合 keep-alive 或路由 meta。
