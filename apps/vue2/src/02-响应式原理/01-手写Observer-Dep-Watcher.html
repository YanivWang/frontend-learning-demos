手写 Vue2 响应式核心（Observer / Dep / Watcher / Computed）

打开浏览器 DevTools → Console，刷新本页后观察日志：改 vm.message、读 vm.messageLength 与 watch 回调，对照 Observer / Dep / Watcher / Computed 链路；复习要点与面试答法见本区块。

知识点要点

  • Vue2 的响应式入口是观察 data：把普通对象的每个已有字段转换成 getter/setter。这样模板渲染、计算属性或侦听器读取字段时能被追踪，字段写入时能通知更新。
  • Dep 是依赖容器，保存“谁依赖了这个属性”；Watcher 是订阅者，代表一次渲染、一个 watch 回调或一个计算属性求值。读取时通过 Dep.target 建立当前 watcher 与属性 dep 的关系。
  • 普通 watcher 在依赖变化时立即重新取值并触发回调；computed watcher 是 lazy 的，依赖变化只把 dirty 设为 true，下次访问计算属性时才重新执行 getter。
  • 真实 Vue2 还要处理 watcher 栈、重复依赖去重、异步更新队列、数组方法拦截、删除属性、组件渲染 watcher 等细节；本页是教学版模型，不等价于完整源码。
  • 常见限制来自 Object.defineProperty：新增对象属性和数组索引/长度变化无法自动被拦截，需要 Vue.set、splice 或整体替换。Vue 3 改用 Proxy 后这类边界明显减少，但 Vue2 存量项目仍要掌握。

面试考点

  • Vue2 响应式为什么要在初始化时遍历 data？ 结论是因为 Object.defineProperty 只能拦截已经定义好的属性。Vue2 必须在实例创建时把 data 中已有字段转换成 getter/setter，后续读取和写入才能被追踪。边界是运行时新增属性默认不响应，必须用 Vue.set 或替换对象。
  • Dep 和 Watcher 分别解决什么问题？ 结论是 Dep 管“这个属性有哪些订阅者”，Watcher 管“某段逻辑依赖了哪些属性并如何更新”。原因是 getter 阶段只有当前正在执行的 watcher 才知道自己依赖谁，setter 阶段属性自己的 dep 才知道该通知谁。项目里 render、watch、computed 都可以理解成不同类型的 watcher。
  • computed 为什么可以缓存，watch 为什么适合副作用？ 结论是 computed 背后是 lazy watcher，依赖变化只标记 dirty，下次读取才重新求值并缓存；watch 在依赖变化时立即执行回调，适合异步请求、DOM 操作等副作用。边界是 computed getter 必须同步、无副作用；需要“数据变了就做事”时用 watch。
  • 手写响应式和真实 Vue2 实现有哪些边界差距？ 结论是教学版只演示核心链路，真实 Vue2 还包含依赖去重、异步批量更新、数组拦截、组件生命周期和错误处理。原因是框架要在复杂组件树中控制更新成本。面试回答时不要把几十行教学代码说成完整 Vue 源码。
  • Vue2 defineProperty 响应式相比 Vue3 Proxy 有什么限制？ 结论是 defineProperty 只能逐属性拦截，无法天然感知新增/删除属性和数组索引变化。原因是每个 key 都要单独定义描述符。Vue2 因此需要 Vue.set 和数组变更方法；Vue3 Proxy 能从对象层面拦截更多操作。
