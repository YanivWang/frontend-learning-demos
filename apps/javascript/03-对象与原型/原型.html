原型对象（prototype）与共享属性

请打开 DevTools Console 查看输出。

知识点要点

  • 共享挂载点： 每个可构造函数都有 prototype 对象； new Person() 创建的实例会把 [[Prototype]] 链到 Person.prototype 。
  • 查找机制： 访问实例属性时先查自有属性，找不到再沿 [[Prototype]] 查 prototype 上的属性或方法。
  • 标准 API： __proto__ 是访问 [[Prototype]] 的历史属性，可辅助理解；项目代码推荐 Object.getPrototypeOf() / Object.setPrototypeOf() 。
  • constructor： Person.prototype.constructor 默认指回 Person ；替换整个 prototype 后需手动恢复 constructor 。
  • 共享方法： 挂在 prototype 上的方法被所有实例共享， p1.sayHello === p2.sayHello 为 true。
  • 共享状态陷阱： 若把数组、对象等可变引用放在 prototype 上，所有实例会共享同一份状态，一个实例修改会影响其他实例。

面试考点

  • prototype 在构造函数和实例之间起什么作用？ 它是实例共享行为的对象。 new Person() 会让实例的 [[Prototype]] 指向 Person.prototype ，因此多个实例能共享同一个 sayHello 方法。
  • [[Prototype]]、__proto__、prototype 有什么区别？ [[Prototype]] 是规范内部槽； __proto__ 是访问它的历史属性； prototype 是 函数对象上 用于给实例链接原型链的普通属性。三者方向与所属对象不同。
  • constructor 属性为什么会指回构造函数？ 默认的 prototype 对象自带 constructor ，方便从原型回到构造函数。边界是它可被改写，替换原型后不自动保持正确。
  • 原型上为什么不建议放可变引用状态？ 原型属性被所有实例共享，一个实例改数组或对象会影响其他实例。实例状态放构造函数，原型上主要放方法或不可变常量。
  • 多个实例如何共享同一份方法？ 方法定义在 Person.prototype 上，实例通过 [[Prototype]] 委托访问，因此引用相等且节省内存。
