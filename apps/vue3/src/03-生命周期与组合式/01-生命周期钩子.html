Composition API 生命周期钩子

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • setup() 会在组件创建阶段同步执行，早于挂载；Composition API 中 beforeCreate / created 的大多数初始化逻辑（创建 ref、注册 watch）都应直接写在 setup 顶层。本页刷新后日志首行不会出现 mounted，说明 setup 已完成但 DOM 尚未就绪。
  • onBeforeMount / onMounted 对应挂载前后。需要访问真实 DOM、初始化依赖 DOM 的第三方库时，应放在 onMounted 之后；本页子组件 mounted 后日志区才出现 onMounted 行，控制台同步打印 [lifecycle] onMounted。
  • onBeforeUpdate / onUpdated 对应响应式更新引发的 DOM patch 前后。点子组件「+1」后日志会出现 update 对；读更新后的 DOM 可放在 onUpdated，但不要在其中无条件修改会触发本组件更新的状态，否则可能造成更新循环。
  • onBeforeUnmount / onUnmounted 用于组件销毁前后清理资源。点「挂载/卸载子组件」隐藏子组件后，日志末尾会出现 unmount 对；事件监听、定时器、订阅必须在 unmounted 中释放，异步副作用清理见 02-onWatcherCleanup与watch清理。
  • 生命周期注册函数必须在 setup 同步阶段调用，这样 Vue 才能把钩子绑定到当前组件实例；在 setTimeout 或 await 之后临时注册通常不是可靠模式，会丢失组件上下文。
  • 本页用 provide("lifecycleLogs") 让子组件写入同一份日志 ref，避免全局变量；composable 中注册钩子的写法见 02-composable复用。全量钩子与 KeepAlive 的 activated / deactivated 见 04-全量钩子与KeepAlive。

面试考点

  • Vue 3 中 created 逻辑放哪里？ 结论是大多直接放在 setup 顶层。原因是 setup 已经覆盖创建阶段初始化能力，并且可以创建响应式状态和注册副作用。边界是依赖 DOM 的逻辑仍要等到 onMounted；本页子组件 count 初始化为 0 写在 setup 里，DOM 相关日志在 mounted 才出现。
  • onMounted、onUpdated、onUnmounted 怎么分工？ 结论是 mounted 做 DOM 就绪后的初始化，updated 做更新后 DOM 读取或同步，unmounted 做资源释放。原因是它们分别对应渲染生命周期的不同阶段。项目里第三方图表初始化和销毁常成对写在 mounted / unmounted；本页「+1」按钮验证 update 钩子。
  • 为什么钩子要同步注册？ 结论是为了绑定当前组件实例。原因是 Vue 在 setup 同步执行期间维护当前实例上下文，钩子注册依赖这个上下文。异步之后再注册，可能没有可关联的实例，钩子不会按预期触发。
  • 卸载时要清理什么？ 结论是清理 Vue 无法自动接管的外部资源。包括 addEventListener、setInterval、WebSocket、第三方实例和手动订阅。否则隐藏或路由切换后回调仍可能执行；watch / watchEffect 的 stop 与 onWatcherCleanup 见 02-onWatcherCleanup。
  • setup 同步代码与 onMounted 的分工？ 结论是 setup 做不依赖 DOM 的初始化（状态、watch、provide），onMounted 做 DOM 就绪后的工作（测量、第三方库 init、focus）。本页刷新后日志区为空直到子组件挂载完成，即 mounted 是「DOM 已存在」的分界线。
