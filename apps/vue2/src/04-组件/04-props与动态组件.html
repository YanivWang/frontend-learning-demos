props 传值 + 自定义事件 + 动态组件

点子组件「发送消息」在 Console 查看 blow-event 负载；点击组件根区域验证 @click.native；点 next 按钮切换 <component :is> 动态组件。

知识点要点

  • 本页同时演示三件事：父组件通过 title/content props 下传数据，子组件通过 $emit('blow-event', payload) 通知父组件，父组件用监听器处理结果。
  • 在 DOM 模板中，浏览器会把特性名大小写归一；Vue 2 自定义事件不会自动大小写转换，所以事件名推荐写成 blow-event 这类 kebab-case。
  • @click.native 是 Vue 2 的组件事件修饰符，用来监听子组件根节点的原生点击。Vue 3 已移除该修饰符，迁移时通常改为显式 emits 或透传监听器。
  • <component :is="currentComp"> 根据字符串或组件对象决定渲染哪个组件。切换目标必须已经全局注册、局部注册，或直接传入组件选项对象。
  • 动态组件适合一组互斥的组件视图，例如 tab 或步骤页；如果只是条件显示少量 DOM，v-if 更直观。需要保留被切走组件状态时再配合 keep-alive。

面试考点

  • props 和 $emit 为什么常一起讲？ 结论：它们组成 Vue 2 父子通信的基本闭环。原因是 props 只负责父到子输入，子组件不能直接改父状态，只能 emit 事件请求父组件处理。项目里这样能保持状态来源清晰。
  • 自定义事件为什么推荐 kebab-case？ 结论：在 DOM 模板中最稳妥。原因是 HTML 特性大小写会被浏览器处理，camelCase 事件名可能监听不到。边界是在单文件组件字符串模板里限制较少，但团队仍应统一命名。
  • .native 是什么，迁移时注意什么？ 结论：Vue 2 的 .native 监听组件根元素上的原生事件。原因是普通 @click 监听的是子组件 emit 的自定义事件。迁移 Vue 3 时该修饰符已移除，应由子组件显式声明并触发事件，或通过 attrs 透传。
  • 动态组件和 v-if 怎么选？ 结论：一组同位置互斥组件用动态组件更清晰，简单条件块用 v-if。原因是 :is 把“当前组件类型”抽成状态。边界是动态组件默认切换会销毁旧实例，需要缓存状态时使用 keep-alive。
