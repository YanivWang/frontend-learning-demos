表头固定 + 表体滚动

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • 现代固定表头优先让表格外层成为滚动容器，并给 thead th 设置 position: sticky; top: 0，这样能保留原生 table 的列宽计算和语义结构。
  • 示例 A 的老方案把 tbody 变成 display:block 并单独滚动，同时让 thead 和 tbody tr 继续模拟 table 布局；它能工作，但列宽、滚动条宽度和响应式都要手动补。
  • sticky 必须有阈值（如 top:0），并依附最近的滚动祖先；本页的 .wrap-b { max-height; overflow:auto } 就是表头吸附的滚动边界。
  • 固定表头要给 th 设置不透明背景和合适 z-index，否则滚动内容会从表头下方透出；边框折叠、阴影和多行表头时还要单独检查层叠效果。
  • 真实数据表还常见横向滚动、固定首列、排序筛选和几千行性能问题；CSS sticky 解决的是表头视觉固定，不替代虚拟滚动、列宽测量或无障碍表格标注。

面试考点

  • 表头固定为什么优先用 position: sticky？ 结论是 sticky 能在保留原生表格布局的同时固定表头。原因是列宽仍由 table 算法统一计算，不需要把表头和表体拆开。边界是 sticky 依赖滚动容器和阈值，复杂多级表头仍要测试层叠和边框。
  • sticky 表头失效通常检查什么？ 结论是先查 top 是否设置、滚动祖先是否正确、祖先是否有影响显示的 overflow/transform、以及表头是否被其他层覆盖。原因是 sticky 是相对滚动容器的定位模型。项目里外层滚动容器变化后，吸附边界也会变化。
  • tbody display:block 的方案有什么风险？ 结论是能兼容旧做法，但维护成本高。原因是它破坏了表格头体共享列宽的天然布局，需要手动扣滚动条、固定列宽或同步宽度。响应式、字体变化、隐藏列和横向滚动都会放大这些问题。
  • 大数据表格只靠 CSS 固定表头够不够？ 结论是不够。原因是 CSS sticky 只解决视觉固定，不减少 DOM 数量。上千行数据还需要分页、虚拟滚动、懒渲染或专门的数据表组件，同时保留表头、单元格关联和键盘可访问性。
