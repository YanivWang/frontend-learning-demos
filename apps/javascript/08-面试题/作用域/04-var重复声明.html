var 重复声明：后值覆盖前值

先看本复习区，再查看下方输出是否与标准答法一致；细节可打开 DevTools Console。

知识点要点

  • var 是函数作用域或全局作用域声明，同一作用域内重复声明同名变量是允许的，不会像 let / const 那样抛语法错误。
  • var a = 234 可以拆成“声明 a”和“给 a 赋值 234”；重复声明不会清空已有值，执行阶段的赋值才会覆盖 1。
  • 这段代码按顺序执行后，第一次赋值让 a 为 1，第二次赋值让 a 为 234，所以 console.log(a) 输出 234。
  • let / const 是块级作用域声明，同一作用域重复声明会在解析阶段抛 SyntaxError，这也是现代代码更推荐它们的原因之一。
  • 浏览器普通脚本里的顶层 var a 会创建 window.a；顶层 let / const 不会，type="module" 中的顶层声明也属于模块作用域。
  • 真实项目里不要依赖 var 重复声明；它容易掩盖命名冲突，尤其在老脚本拼接、全局工具函数和条件分支中更难排查。

面试考点

  • var 重复声明为什么不会报错，最终为什么输出 234？ 结论：var 允许同一作用域重复声明，最终值来自最后一次赋值。原因是重复的 var a 声明不会创建第二个变量，也不会把已有值重置为 undefined。边界是如果第二句只有 var a; 没有赋值，输出仍是前一次赋值后的 1。
  • 声明提升和赋值执行在这题里怎么区分？ 结论：声明会在进入作用域时处理，赋值仍按代码顺序执行。原因是 var a = 1 的声明部分被提升，= 1 留在原位置执行。项目里分析老代码时，要把“有没有声明”与“什么时候赋值”分开看。
  • let / const 为什么能避免重复声明问题？ 结论：它们在同一词法作用域内禁止重复声明。原因是块级词法声明会在解析阶段建立约束，重复名字直接抛 SyntaxError。实际项目中这能更早发现变量名冲突，也是禁用 var 的重要理由。
  • 顶层 var 在普通脚本和模块里有什么区别？ 结论：普通脚本顶层 var 会成为全局对象属性，模块顶层声明不会。原因是 ES module 有独立的模块作用域。边界是面试题默认常是普通 <script>，答题时要主动说明运行环境。

运行结果

标准答法：console.log(a) 输出 234（重复 var 不报错，最后一次赋值覆盖）。
