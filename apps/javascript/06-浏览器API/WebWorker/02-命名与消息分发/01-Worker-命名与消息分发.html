给 Worker 命名 + 用消息体分发不同任务

下方为 console.log 同步输出；也可打开 DevTools Console 查看。

知识点要点

  • new Worker(path, { name }) 可标识不同 worker，便于 DevTools 调试。
  • 主线程发送 {task, data}，子线程依据 task 字段调用对应函数。
  • 易错点：直接在主线程跑大循环（startCount）会卡死 UI，应放到 worker。
