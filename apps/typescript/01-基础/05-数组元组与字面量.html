数组、元组与字面量类型

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • 数组适合变长同类集合，例如 number[] ；元组适合固定长度且每个位置有语义的数据，例如本页的 readonly [number, number] 坐标。
  • readonly 是类型层面的不可写约束，能阻止调用方在 TypeScript 中修改入参；它不会自动冻结运行时对象，本页用 Object.freeze 才让路由对象运行时也不可变。
  • 字面量类型把值限制为有限集合，适合状态机、路由键、事件名和组件 variant；比裸 string 更能提前发现拼写错误。
  • as const 会把对象和数组推断为最窄的 readonly 字面量；普通 const 只保证变量绑定不可重新赋值，不会让对象属性都变成只读字面量。
  • satisfies 是 TypeScript 4.9+ 能力；它校验表达式满足目标类型，同时尽量保留表达式自身的精确推断，适合配置表。本仓库校验使用 TypeScript 5.8.3，可直接使用。

面试考点

  • 数组和元组的建模边界是什么？ 结论是变长同类数据用数组，固定位置语义用元组。原因是元组能表达第 0 位、第 1 位分别代表什么，并限制长度和元素类型。项目里坐标、RGB、Hook 返回值常用元组，普通列表用数组。
  • readonly 类型为什么不等于 Object.freeze ？ 结论是 readonly 只影响 TypeScript 检查， Object.freeze 才是运行时冻结。原因是类型会被编译擦除。跨模块传配置时可以用 readonly 防误改，确实要运行时不可变时再冻结或深拷贝。
  • as const 和普通 const 声明有什么区别？ 结论是普通 const 固定变量绑定， as const 还会把属性和值收窄为 readonly 字面量。原因是对象属性默认仍可变，推断通常会放宽为 string 等类型。配置对象、路由表和事件名表常用 as const 。
  • satisfies 相比类型注解和 as 断言有什么优势？ 结论是它既检查结构，又保留更精确的推断。原因是类型注解可能把字面量放宽， as 断言可能绕过部分错误。TS 4.9+ 项目中，配置表常写成 as const satisfies Record<...> 。

// —— 数组 vs readonly 元组 ——
      const scores: number[] = [95, 88, 100];
      type Point = readonly [number, number];
      function distance([x, y]: Point): number { return Math.hypot(x, y); }
      // —— 字面量联合与 as const satisfies ——
      type Status = "idle" | "loading" | "success" | "error";
      const routes = {
      home: "/",
      user: "/user/:id",
      } as const satisfies Record<string, `/${string}`>;
