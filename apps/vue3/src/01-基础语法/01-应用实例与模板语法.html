createApp / mount / 模板语法

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Vue3 使用 createApp() 创建应用实例，应用级配置、插件和组件注册都挂在这个实例上；调用 mount("#app") 后才会渲染并接管目标容器。
  • 模板支持文本插值、JavaScript 表达式、指令和事件绑定，但表达式应保持轻量；复杂派生逻辑更适合放进 computed 或方法，避免模板难读且重复执行。
  • ref 在模板渲染上下文会自动解包，所以可以写 {{ title }}；在 setup() 的 JavaScript 代码里仍要通过 title.value 读写。
  • v-text 会写入文本内容，v-html 会把字符串作为 HTML 插入；本页只渲染固定可信字符串，真实项目绝不能把未消毒的用户输入交给 v-html。
  • Vue3 CDN 全局构建把 API 暴露在 Vue 对象上，本页用 const { createApp, reactive, ref } = Vue 方便直接运行；工程化项目通常从 vue 包按需导入。

面试考点

  • createApp 与 mount 分别解决什么问题？ 结论是 createApp 创建应用实例，mount 把应用渲染到具体 DOM 容器。原因是 Vue3 把应用级能力隔离在实例上，多个应用可以有各自的配置和插件。边界是同一个应用实例通常只挂载一次，挂载后应通过响应式状态驱动视图，而不是直接重建根实例。
  • 模板表达式和方法调用应该如何取舍？ 先结论：简单展示逻辑可以写在模板表达式里，复杂派生值用 computed，带副作用的操作放事件或 watch。原因是模板表达式会参与渲染求值，过重逻辑会降低可读性并可能重复执行。项目里常把格式化、过滤和权限判断抽到计算属性或函数中复用。
  • 模板里的 ref 自动解包为什么不等于 JS 里不用 .value？ 结论是自动解包只发生在模板渲染上下文等特定场景，setup() 代码仍要使用 .value。原因是 ref 本质是带 value 的响应式包装对象。边界是深层对象、数组元素和普通解构场景不要假设都会自动解包。
  • 为什么 v-html 不能渲染用户输入？ 结论是 v-html 会把字符串交给浏览器按 HTML 解析，未消毒内容可能造成 XSS。原因是它绕过了 Vue 默认的文本转义保护。项目中只有后端或前端明确消毒后的可信富文本才适合使用，并且要配合内容安全策略和白名单。
