模板 ref / nextTick

在浏览器中操作下方交互演示；复习要点与面试答法见本复习区。

知识点要点

  • 模板 ref="textEl" 与 setup 中同名 ref(null) 配对，挂载后 textEl.value 指向 DOM。本页 onMounted 日志区首行会显示 textEl= P；挂载前为 null。
  • 本页的 ref 指 DOM 引用，与响应式 ref() 包装值不同；响应式基础见 02-ref-reactive与解构。Vue 3.5+ 写法见 06-useTemplateRef。
  • Vue 异步批量更新 DOM：点「修改后读取 DOM」时，下方日志第一行同步读取仍是旧文本，await nextTick() 后第二行才是新文本。调度机制深讲见 08-调度器与nextTick。
  • 勾选「隐藏段落」时 v-if="showBlock" 为 false，textEl 变为 null；再次显示后 ref 重新绑定。访问 ref 前须判断空值。
  • nextTick 只保证 Vue 本轮 patch 完成，不等待图片 load、CSS 动画结束或接口返回；这些场景须用对应事件或 Promise。
  • 能由数据描述的 UI 优先用响应式状态；模板 ref 适合聚焦、测量、滚动、接入第三方 DOM 库等命令式需求。

面试考点

  • 数据变了为什么同步读取 DOM 还是旧值？ 结论是 DOM 更新异步批量执行。本页「修改后读取 DOM」按钮第一行日志即证据；测量/滚动应 await nextTick()。
  • nextTick 适合解决什么问题，不适合解决什么问题？ 适合等 Vue patch；不适合等图片、接口、动画。原因是它只关联 Vue 更新队列。
  • 模板 ref 什么时候可能是 null？ 挂载前、v-if 未渲染、节点卸载后。本页勾选隐藏段落可观察 textEl=null 日志。
  • 什么时候应该用响应式状态而不是直接操作 DOM？ 结论是 UI 由数据驱动；直接改 DOM 可能被下一次 patch 覆盖。ref 用于聚焦、测量等逃逸场景。
