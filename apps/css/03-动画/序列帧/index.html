序列帧动画（精灵图）

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • 精灵图序列帧把多张帧图横向或纵向拼到一张图里，容器只显示单帧尺寸，本页用 5 个 500px 宽的色块模拟 2500px 宽精灵图。
  • background-position 控制背景图在容器中的起始位置；横向精灵图通常把 x 坐标从 0 移到负方向，让下一帧进入可视区域。
  • steps(5, end) 把 1.5 秒拆成 5 个离散区间，避免背景位置连续滑动；如果使用 linear，用户会看到图像平移而不是逐帧切换。
  • animation-play-state: paused 会暂停当前进度，恢复为 running 时从暂停处继续，而不是从第一帧重新开始。
  • 序列帧比 GIF 更容易用 CSS 控制播放、暂停和速度，但大图可能增加首屏体积；真实项目要压缩图片、控制帧数，并在 reduced motion 下停止非必要播放。

面试考点

  • 精灵图序列帧为什么要用 steps()？ 结论是序列帧需要离散跳变，不需要连续插值。原因是每一帧都是独立画面，线性移动会看到中间滑动过程。项目里步数通常等于帧数或可见切换次数，写错会导致重复帧、漏帧或末帧停顿异常。
  • background-position 偏移量怎么计算？ 结论是偏移要按单帧宽度和帧数计算。原因是容器只显示一帧，背景图每次移动一个帧宽才能对齐下一帧。本页 5 帧、单帧 500px，关键帧移动到 -2000px，对应最后一帧左边缘进入容器。
  • animation-play-state 暂停后再播放会怎样？ 结论是会从暂停位置继续。原因是该属性只改变动画播放状态，不重置动画时间线。项目里如果要从第一帧重播，通常需要移除并重新添加动画类，或改变 animation-name 触发重建。
  • CSS 序列帧、GIF、视频和 Canvas 怎么选？ 结论是短小可控 UI 动效可用 CSS 精灵图，简单表情可用 GIF，长时长高压缩动画用视频，复杂实时绘制和交互用 Canvas。原因是它们在体积、控制能力、性能和可访问性上的取舍不同。边界是大尺寸序列帧会占内存和带宽，要谨慎用于首屏。
