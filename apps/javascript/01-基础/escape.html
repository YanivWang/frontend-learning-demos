字符串中反斜杠转义与 JSON 序列化

下方为 console.log 同步输出；也可打开 DevTools Console 查看。

知识点要点

  • JSON 规范要求 " \ / 及控制字符（\n \t 等）在字符串内用反斜杠转义。
  • JSON.stringify 自动加转义；JSON.parse 还原为实际字符。
  • 把 JSON 字符串再放进 HTML 属性或另一层 JSON 时，反斜杠本身也要再转义（双重 \\）。
  • JS 源码字符串、正则字面量、JSON 三者的转义规则各自独立，勿混用。
  • escape 是 URI 时代的 API，与 JSON 转义无关，已废弃。

面试考点

  • JSON 哪些字符要转义？ 双引号、反斜杠、以及 U+0000～U+001F 控制字符；unicode 可用 \uXXXX。
  • 为何双重反斜杠？ 第一层 JSON 转义；第二层是字符串字面量或 HTML 属性转义。调试时看清「存储形式」vs「解析后形式」。
  • escape vs JSON？ 不同标准。JSON 用 \n \" 等；escape 产出 %uXXXX，已废弃。URL 编码用 encodeURIComponent。
