定位与层叠上下文

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出定位与层叠上下文实测信息。

知识点要点

  • position: absolute 会脱离普通流，偏移量参考它的包含块；常见包含块是最近的非 static 定位祖先，所以父级常写 position: relative 。
  • position: fixed 通常相对视口固定，适合悬浮按钮；但包含 transform 、 filter 等属性的祖先可能改变固定定位的包含块，真实项目要特别检查。
  • position: sticky 先按普通流占位，滚动到阈值后吸附；它依赖最近滚动祖先，缺少 top 等阈值或祖先滚动/裁剪设置不当时会看似失效。
  • z-index 不是全局数字排行榜，只在同一个层叠上下文中比较； transform 、 opacity < 1 、 position 配合 z-index 等都可能创建新的层叠上下文。
  • 弹窗、下拉菜单、吸顶栏和 fixed 按钮要统一规划层级，并检查滚动容器、裁剪和新层叠上下文，否则会出现“z-index 写到 9999 仍被盖住”的问题。

面试考点

  • absolute 的包含块如何确定？ 结论是常见情况下相对最近的非 static 定位祖先。原因是绝对定位需要一个包含块来解析 top/right/bottom/left 。项目里父级写 position: relative 通常不是为了移动父级，而是为了给子级建立定位参考。
  • fixed 和 sticky 有什么区别？ 结论是 fixed 脱离文档流并固定在视口附近，sticky 仍保留原位置并在滚动到阈值后吸附。原因是 sticky 是普通流与相对定位的混合模型。边界是 transformed ancestor、滚动祖先和 overflow 裁剪都会影响实际表现。
  • 为什么 z-index: 9999 仍可能被盖住？ 结论是元素可能处在较低的层叠上下文里。原因是 z-index 先比较层叠上下文，再比较上下文内部的层级。项目里要沿祖先查 transform 、 opacity 、 position + z-index 、 isolation 等创建上下文的属性。
  • transform 创建层叠上下文有什么风险？ 结论是它能隔离内部层级，也可能让子弹层无法覆盖外部元素。原因是内部 z-index 不会跳出父上下文。动画优化、3D 变换和弹层同时出现时，通常应把弹层挂到更高层容器或统一 portal/root。
