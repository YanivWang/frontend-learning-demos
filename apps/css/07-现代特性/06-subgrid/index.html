subgrid 卡片对齐

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • subgrid 是 CSS Grid Level 2 的能力，让嵌套网格在某个轴上复用父网格的轨道尺寸。普通嵌套 grid 会独立计算自己的行列，subgrid 则让子网格和父网格在同一套轨道上对齐。
  • 本页演示的是行方向 subgrid：父级 .cards 定义 grid-template-rows: auto 1fr auto，每张 .card 通过 grid-row: span 3 跨过三条父级行轨道，再用 grid-template-rows: subgrid 让标题、正文、按钮行共享这些轨道。
  • subgrid 可以用于列，也可以用于行；只声明 grid-template-rows: subgrid 时，列仍由子网格自己决定。需要跨列对齐表单标签、价格列或卡片内容时，再考虑 grid-template-columns: subgrid。
  • 常见坑是忘记让子网格跨越父网格轨道范围。子元素如果没有用 grid-row 或 grid-column 覆盖足够轨道，subgrid 没有可继承的多条轨道，效果就不明显。
  • 兼容性上应保留普通 Grid 作为基础布局，再用 @supports (grid-template-columns: subgrid) 或同类检测增强。这样旧浏览器仍能展示卡片，只是跨卡片的行列对齐不如支持 subgrid 的浏览器精确。

面试考点

  • subgrid 和普通嵌套 grid 的核心区别是什么？ 结论是普通嵌套网格独立计算轨道，subgrid 在指定轴上复用父网格轨道。原因是很多布局需要嵌套内容跨卡片或跨表单项对齐，而独立网格只能在各自卡片内部对齐。项目里商品卡、表单标签列、详情页参数表都适合评估它。
  • 为什么卡片列表的标题、正文、按钮对齐适合用 subgrid？ 结论是卡片内部有相同结构但内容高度不同，按钮行需要共享父级行轨道才能稳定对齐。原因是单张卡片用 grid-template-rows: auto 1fr auto 只能保证卡片内部贴底，不能保证多张卡之间每一行一致。使用 subgrid 后，标题、正文和操作区可以跨卡片对齐。
  • grid-template-rows: subgrid 和 grid-row: span 3 为什么要配合使用？ 结论是子网格必须先跨过父级的多条行轨道，才有轨道可以继承。原因是 subgrid 继承的是子网格占据范围内的父网格轨道，而不是凭空创建三行。项目里如果发现 subgrid 无效，先检查子项是否跨到了预期行列范围。
  • subgrid 的兼容性和降级策略怎么设计？ 结论是把普通 Grid 写成可用基础布局，再用 @supports 启用 subgrid 增强。原因是 subgrid 是较新的 Grid 能力，旧环境可能不支持。项目里降级目标通常是“内容可读、操作可用”，精确跨卡对齐作为增强体验。
