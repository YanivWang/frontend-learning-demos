条件渲染中的「短路求值」陷阱

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • && 是 JavaScript 短路表达式：左侧为 falsy 时返回左侧本身，左侧为 truthy 时返回右侧 JSX；React 不会额外把返回值强制转成布尔值。
  • React 会把 false、null、undefined 当作空节点跳过，但数字 0 和非空字符串会作为文本子节点渲染。本 demo 的 arr.length 为 0，因此页面会出现一个文本 0。
  • 条件来自数量、字符串或可能为 0 的表达式时，先显式得到布尔值：arr.length > 0 && <List /> 或 Boolean(arr.length) && <List />。
  • 需要二选一展示时优先使用三元表达式，例如 items.length > 0 ? <List /> : <Empty />；只在“满足条件才展示一段 UI”时使用 && 更清晰。
  • React 18 的客户端入口应使用 ReactDOM.createRoot(container).render(element)，这样才能进入 React 18 root 行为；旧 ReactDOM.render 只适合维护旧项目。

面试考点

  • 为什么 arr.length && <List /> 会在空数组时渲染出 0？ 结论是 0 && <List /> 的结果就是数字 0，React 会渲染数字子节点。原因是 && 返回操作数本身，而不是布尔值；项目里所有数量判断都应写成 arr.length > 0 或显式 Boolean(...)。
  • React 会忽略哪些子节点值？ 结论是 false、null、undefined 会被当作空节点，数字和字符串会渲染。原因是 React children 支持文本节点；边界是空字符串通常不可见但仍是字符串，不能把“不可见”等同于“被忽略”。
  • 条件渲染中何时用 &&，何时用三元表达式？ 结论是“有就展示、没有就什么都不展示”适合 &&，“A/B 二选一”适合三元表达式。原因是三元表达式能显式写出 fallback；空态、错误态、加载态这些产品状态不要隐藏在短路表达式里。
  • React 18 页面入口为什么仍应使用 createRoot？ 结论是 React 18 新代码使用 createRoot。原因是它创建 React 18 root 并启用新行为；维护旧页面时可能保留 ReactDOM.render，但不应和 React 18 推荐写法混用。
