同步组件 vs 异步组件（工厂函数形式）

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • 同步组件注册时直接传组件选项对象，模板编译后可以立即渲染；异步组件传工厂函数，只有组件第一次真正需要渲染时才触发解析。
  • 基础工厂函数接收 resolve 与 reject：成功时把组件定义交给 resolve，失败时调用 reject 让 Vue 走失败处理。
  • Vue 2.3+ 支持高级异步组件对象，可配置 component、loading、error、delay 和 timeout。本页同时演示成功加载和失败兜底。
  • 异步组件解析成功后会缓存组件定义，后续重新渲染同一个异步组件不会重复执行完整加载流程；这适合大组件、低频面板和路由级代码分割。
  • 涉及异步加载必须补失败路径：网络失败、超时、重复点击和加载中空白都要处理。旧 Vue 2 项目常配合 webpack 动态导入；新项目应优先 Vue 3 与现代构建工具。

面试考点

  • Vue 2 异步组件什么时候加载？ 结论：第一次渲染到该组件时才调用工厂函数。原因是异步组件的目标是按需拆分和延迟加载。边界是已经解析成功的结果会被缓存，再次渲染不会重新下载同一组件定义。
  • resolve 和 reject 分别做什么？ 结论：resolve 交付组件定义，reject 表示加载失败。原因是 Vue 需要根据异步结果决定渲染真实组件还是错误组件。项目里不要只写成功路径，应给失败状态和重试入口。
  • 高级异步组件配置有什么价值？ 结论：可以声明 loading、error、delay、timeout 等状态。原因是真实网络加载会有等待、失败和超时。边界是简单 demo 可用工厂函数，生产项目更常用动态 import 与路由懒加载。
  • 异步组件和路由懒加载是什么关系？ 结论：路由懒加载通常就是把路由组件写成异步组件工厂或动态导入。原因是路由页天然是按访问路径拆分的代码块。边界是首屏关键组件不宜过度异步化，否则会增加等待和错误兜底成本。
