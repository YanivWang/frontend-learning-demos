用正则识别 URL 与图片格式

请打开 DevTools Console 查看输出。

知识点要点

  • URL 匹配通常分：协议 https? 、主机、路径、query；复杂 URL 用 URL API 更可靠。
  • 图片扩展名： \.(jpg|jpeg|png|gif|webp)(?=\?|$) ； i 标志忽略大小写。
  • 贪婪 .* 匹配长 query 时可能大量回溯；改用 [^?#]+ 或非贪婪 .*? 。
  • 生产环境 URL 解析优先 new URL(str) ，正则仅做轻量格式提示。

面试考点

  • 如何写 URL 校验正则？ 结论是可以写基础格式提示，但不应把复杂 URL 合法性完全交给正则。原因是协议、IPv6、国际化域名、编码和 query 都很复杂；项目里优先用 new URL() 解析，再用白名单限制协议和域名。
  • 贪婪量词有什么风险？ .* 或嵌套量词在长字符串上可能导致大量回溯。边界是简单后缀匹配应收紧为 \.(jpg|png)(?=\?|$) 这类规则，而不是整串贪婪吞掉再回退。
  • 正则校验 URL 的局限是什么？ 正则只能说明“看起来像”，不能说明资源可信或可访问。安全场景必须配合 URL API、协议白名单、域名白名单和服务端校验，不能只靠前端正则。
