Unicode 编码与字符串互转

JS 字符串以 UTF-16 存储；BMP 字符用 charCodeAt，4 字节字符（emoji）用 codePointAt。

知识点要点

  • JS 内部 UTF-16；基本字符 1 个 code unit；增补平面字符（emoji）占 2 个 code unit（代理对）。
  • charCodeAt 返回 16 位单元；codePointAt 返回完整 Unicode 码点。
  • str.length 是 code unit 数，不是「字符」数；'😀'.length === 2。
  • for...of、展开运算符 [...str] 按码点迭代；传统 for 按索引可能切半代理对。
  • String.fromCodePoint / fromCharCode 由码点/单元建串。

面试考点

  • emoji 长度不是 1？ UTF-16 代理对占 2 个 code unit。用 [...s].length 或 for...of 数「字符」。
  • charCodeAt vs codePointAt？ 前者 16 位单元；后者完整码点，适合 BMP 外字符。
  • 正确遍历？ for...of、Array.from(str)、match(/./gu)。slice 截取 emoji 需注意不要从代理对中间切。
