严格模式综合

请打开 DevTools Console 查看输出。

知识点要点

  • "use strict"; 必须是脚本/函数体第一条有效语句（前面只能有注释）。
  • 禁止未声明变量赋值（防意外全局）；禁止 delete 变量/函数；禁止形参重名。
  • 禁止八进制字面量 010 ； with 禁用； eval 不创建外层作用域变量。
  • 独立调用函数时 this === undefined ；非严格模式为全局对象。
  • ES module 和 class 默认严格模式；现代项目整体开启。

面试考点

  • 严格模式限制？ 未声明变量、delete 变量、重复形参、八进制、with、arguments.callee、静默失败变抛错等。
  • 如何开启？ 文件/函数顶部 "use strict" ；或写 ES module（ type="module" ）。
  • 对 this 的影响？ 非严格：独立调用 fn() 时 this 为 window；严格：undefined。不影响 call/apply 显式绑定。
