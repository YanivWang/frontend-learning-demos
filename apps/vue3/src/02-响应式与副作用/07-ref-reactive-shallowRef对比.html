ref / shallowRef / reactive 三种响应式 API 对比

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • ref() 返回带 .value 的响应式包装。JavaScript 中读写必须使用 .value，模板里顶层 ref 会自动解包，所以示例中模板直接写 {{ message }}。
  • ref() 包裹对象时，默认会把对象深层转为响应式；示例中的 objRef.form.name 改动会更新视图。
  • reactive() 返回深层响应式 Proxy，访问字段不需要 .value。它适合形状稳定的对象，但直接解构属性会断开代理访问，跨函数传递字段时应使用 toRef() / toRefs()。
  • shallowRef() 只追踪 .value 本身的替换，不递归代理内部对象。示例中改内部 form.name 不会稳定更新，整体替换 .value 才会触发视图。
  • 现代 Vue 3 项目常用 ref 表达可替换状态，用 reactive 表达聚合对象，用 shallowRef 包裹大对象、外部实例或需要手动控制刷新的状态。

面试考点

  • ref 和 reactive 最大差异是什么？ 结论是 ref 是值容器，reactive 是对象代理。原因是 ref 通过 .value 追踪读写，reactive 通过 Proxy 拦截属性访问。项目里基本类型、需要整体替换的对象常用 ref，稳定对象模型常用 reactive。
  • shallowRef 为什么改内部字段不更新？ 结论是它只追踪 .value 的引用变化。原因是内部对象不会递归转成响应式代理。项目里如果用它存大对象或第三方实例，应通过整体替换、triggerRef 或额外版本号驱动视图。
  • reactive 解构后如何保留响应式？ 结论是不要直接解构普通属性，应该使用 toRef 或 toRefs。原因是直接解构得到的是当前值，不再经过原代理的 get/set。组合式函数返回 reactive 字段时尤其容易踩坑。
  • 业务状态如何选型？ 结论是先看状态形态和更新方式。需要替换就用 ref，对象字段稳定且整体作为模型使用可用 reactive，不希望深层代理或对象很大时用 shallowRef。边界是 shallow API 会牺牲自动深层更新能力。
