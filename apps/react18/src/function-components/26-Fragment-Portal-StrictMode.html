Fragment、Portal 与 StrictMode

请打开 DevTools Console 查看 Portal 挂载与 StrictMode 日志；下方 #root 为 React 挂载/交互演示区（弹窗 Portal 挂载在 #modal-root）。

知识点要点

  • Fragment 允许组件返回多个兄弟节点而不额外创建 DOM 包裹层，适合列表项、表格结构和不希望影响 CSS 布局的场景。
  • createPortal(children, domNode) 会把子节点挂到指定 DOM 节点；它只改变真实 DOM 位置，不改变 React 组件树关系。
  • Portal 中触发的 React 事件仍按 React 组件树冒泡，而不是按真实 DOM 层级冒泡；弹窗、Tooltip、浮层要特别留意事件关闭逻辑。
  • StrictMode 只在开发环境生效，会额外执行一些检查，并可能重复调用渲染逻辑和 Effect setup/cleanup，以暴露不纯渲染和遗漏清理的问题。
  • 本 demo 的 Portal 只演示挂载位置。生产弹窗还应处理焦点陷阱、Esc 关闭、滚动锁定、遮罩点击、 aria-modal 等交互和无障碍边界。

面试考点

  • Fragment 与普通 div 包裹有什么区别？ 结论是 Fragment 不会产生真实 DOM 节点。原因是它只是 React 组件返回多个子节点的语法容器。边界是需要 class、样式、语义或布局容器时仍应使用真实元素。
  • Portal 渲染到 root 外时事件为什么仍能回到 React 父组件？ 因为 Portal 保留在原 React 组件树里，只是提交到另一个 DOM 容器。本 demo 弹窗显示在 #modal-root ，但状态仍由父组件控制。项目里不要把 DOM 层级和 React 事件路径混为一谈。
  • StrictMode 为什么会让开发环境出现重复执行？ 结论是这是有意的开发检查。原因是 React 通过重复调用暴露不纯渲染、遗漏 Effect 清理和废弃 API。边界是生产构建不会因为 StrictMode 自动重复执行这些检查，但代码仍应按可重复执行来写。
  • 用 Portal 做弹窗有哪些边界？ 结论是 Portal 只解决挂载位置，不解决完整弹窗体验。原因是焦点管理、键盘关闭、滚动锁定和无障碍语义属于应用逻辑。真实项目应补 role="dialog" 、焦点回收、Esc 关闭和遮罩策略。
