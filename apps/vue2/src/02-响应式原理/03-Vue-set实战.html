Vue.$set 与 Object.assign 实战

点击下方 Click 按钮：先用 $set 补 age，再用 Object.assign 整体替换 person 观察视图变化；打开 Console 可看 deerObj.weight 访问器日志；复习要点与面试答法见本区块。

知识点要点

  • Vue2 只能在初始化时把已有字段转成 getter/setter，所以 this.person.age = 18 这种运行时新增属性不会自动触发视图更新。
  • this.$set(this.person, "age", 18) 会给目标对象新增响应式属性并通知更新，适合只补一个明确字段的场景；全局写法是 Vue.set(target, key, value)。
  • Object.assign(this.person, source) 是原地合并，新增 key 不会自动拥有 Vue2 的响应式 getter/setter；如果要批量补字段，推荐 this.person = Object.assign({}, this.person, source) 整体替换。
  • 整体替换对象会让 Vue 观察新对象结构，但也会改变对象引用；如果外部代码持有旧引用，需要评估引用失效和状态同步问题。
  • 本页的 deerObj.weight 访问器演示了 defineProperty 的读取/写入拦截；它帮助理解 Vue2 响应式边界，但普通业务新增字段不要手写访问器，优先使用 Vue API 或替换状态。

面试考点

  • Vue2 给对象新增属性为什么页面不更新？ 结论是新增属性没有在初始化时被转换成 getter/setter。Vue2 的依赖收集依赖 Object.defineProperty，它不能自动拦截后来出现的 key。项目里如果后端返回动态字段，要么预先声明默认结构，要么用 Vue.set 或整体替换对象。
  • Vue.set 和整体替换对象分别适合什么场景？ 结论是少量明确字段用 Vue.set，批量合并模型更适合整体替换。原因是 Vue.set 逐个 key 处理更清晰，整体替换能让 Vue 重新观察完整新对象。边界是整体替换会产生新引用，可能影响依赖旧对象引用的代码。
  • Object.assign 原地合并为什么可能破坏响应式预期？ 结论是原地给响应式对象新增字段并不会让这些字段自动响应。原因是 Object.assign 只是普通属性赋值，不会替 Vue 调用 defineReactive。项目里看到 Object.assign(this.form, payload) 时，要确认 payload 只更新已有 key 还是新增 key。
  • 数组索引赋值和对象新增属性的修复思路有什么相似点？ 结论是二者都是 Vue2 defineProperty 模型的检测盲区。对象新增 key 用 Vue.set 或整体替换，数组索引用 Vue.set(arr, index, value) 或 splice。边界是 Vue3 Proxy 下这些限制减少，但 Vue2 老项目仍常见。
