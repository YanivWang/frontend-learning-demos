defineModel（Vue 3.4+）

在浏览器中操作下方交互演示；复习要点与面试答法见本复习区。

知识点要点

  • defineModel 是 Vue 3.4+ 的 SFC <script setup> 编译宏 ，声明一个 prop 和对应的 update:xxx 事件，并返回可读写的模型 ref 。本页标题旁 Vue 3.4+ 标记 API 版本要求。
  • 默认 defineModel() 对应 modelValue / update:modelValue ， defineModel('title') 对应 title / update:title ；本页用 computed getter / setter（ useVModel 辅助函数）演示等价运行时行为，手写 v-model 见 04-组件v-model 。
  • 宏只能在 SFC 编译阶段被识别，浏览器全局构建没有 SFC 编译器，所以 HTML demo 不能直接调用 defineModel ，只能展示等价实现和代码片段。下方 pre 区为真实 SFC 写法。
  • 带 trim 修饰符的输入组件用 defineModel({ get, set }) 或 setter 内 value.trim() 演示格式化；改输入时控制台打印 [defineModel equiv] set → emit ，证明赋值仍走 emit 路径。
  • 注意官方文档提示：子组件模型设置 default 而父组件绑定值是 undefined 时，可能出现父子初始值不同步；组件库中要明确默认值策略。点「父组件清空 title」可观察父 ref 变为空字符串后子输入同步。
  • Vue 3.5+ 的响应式 props 解构是另一编译宏能力，见 06-props解构（3.5+） 。

面试考点

  • defineModel 和手写 modelValue / emits 是什么关系？ 结论是语义等价， defineModel 是编译期语法糖。它减少重复声明 prop、emit 和 computed setter 的样板代码，但运行时仍是父组件传值、子组件发出 update 事件。边界是 Vue 3.4+ 才能使用。
  • 为什么说 defineModel 返回的是可写 ref？ 因为模板或脚本给这个 ref 赋值时，编译结果会 emit 对应的 update:xxx 事件。本页 TrimInput 的 setter 日志即证明。不要把它误解成子组件私有状态。
  • defineModel 的 default 有什么坑？ 结论是可能造成父子值短暂不一致。原因是子组件 default 会让本地模型有初始值，而父组件绑定的 ref 仍可能是 undefined 。表单组件应让父组件初始化值，或在文档中明确 default 的同步边界。
  • 什么时候继续手写 props / emits？ 当项目还在 Vue 3.3 或更早版本、组件不是 SFC <script setup> 、或需要非常显式的运行时声明时继续手写。组件库若面向多个 Vue 版本，也要确认构建链是否支持该宏。
  • 修饰符如何在 defineModel 中处理？ 结论是读取 defineModel().modelModifiers 或在 set 选项中统一处理。例如 v-model.trim 可在 setter 内 trim。本页 TrimInput 模拟 trim 行为，与原生 input 修饰符语义对照。
