Mustache 插值 / v-html / v-bind / v-once

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • {{ msg }} 是文本插值，Vue 会把内容作为普通文本输出并进行 HTML 转义。本页 {{ htmlStr }} 会显示字符串本身，而不会把其中的标签当成 DOM 执行。
  • v-html 会把表达式结果作为真实 HTML 写入元素，相当于绕过文本转义。它只适合渲染可信、已清洗的 HTML；不要把用户输入、接口返回的富文本或评论直接交给 v-html，否则容易产生 XSS 风险。
  • HTML 属性不能用 Mustache 插值，title="{{ msg }}" 这类写法会被当成普通字符串。属性、布尔属性、class、style 等都应使用 v-bind:title="msg" 或缩写 :title="msg"。
  • Vue 2 模板表达式支持 JavaScript 表达式，但不是完整语句：可以写 count + 1、三元表达式和方法调用，不能写 if、for 或变量声明。表达式应尽量保持纯净，避免在渲染时赋值修改状态。
  • v-once 会让节点及其子节点只渲染一次，后续数据变化不再更新。它适合稳定的静态内容或性能优化点，不适合展示会随状态变化的业务字段。

面试考点

  • Mustache 插值和 v-html 的安全边界有什么不同？ 结论是 Mustache 输出转义文本，v-html 输出真实 HTML。原因是文本插值不会把字符串解析成标签，而 v-html 会写入元素的 HTML 内容。项目里默认用 Mustache，只有可信富文本才考虑 v-html，并且要先做服务端或可信库清洗。
  • 为什么 HTML 属性不能写 {{ msg }}，而要用 v-bind？ 结论是 Mustache 只用于文本内容，属性绑定必须用 v-bind。原因是浏览器先按 HTML 解析属性，Vue 不会把属性里的双大括号当成动态绑定。项目里动态 title、disabled、class、style 都应使用 : 缩写。
  • Vue 2 模板表达式为什么只能写单个表达式？ 结论是模板表达式用于声明“这个位置的值如何计算”，不是写业务流程控制。原因是渲染函数需要可重复执行的表达式，语句、循环和副作用会让模板难维护。项目里复杂逻辑放到 computed、methods 或预处理后的数据里。
  • v-once 适合什么场景，不适合什么场景？ 结论是 v-once 适合渲染后不会变化的内容，不适合响应式业务数据。原因是它会跳过后续更新以节省渲染成本。项目里可用于静态说明、固定标题或一次性快照，不能用于需要随状态刷新的一般字段。
