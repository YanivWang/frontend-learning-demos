transition 与 animation 对比

左侧是 hover 触发的 transition，右侧是自动循环的 animation；下方表格总结选型差异。

知识点要点

  • transition 描述元素从起始状态到结束状态的过渡，必须有一次属性变化作为触发，例如 :hover、focus、class 切换；它不会自己开始播放。
  • animation 通过 @keyframes 定义完整时间线，并由 animation-name、时长、次数、方向等控制播放；它可以自动开始、无限循环、往返或暂停。
  • 两者都适合优先动画 transform 和 opacity；按钮 hover、卡片浮起、颜色渐变反馈通常用 transition，loading 旋转、呼吸强调、序列动效通常用 animation。
  • transition 只能过渡可插值的值，所以 height: auto 不能直接平滑过渡；这类场景见 `展开折叠` demo，常用固定值、max-height 或 grid-template-rows 方案。
  • animation 简写若出现两个时间值，第一个是 duration，第二个是 delay；例如 animation: 2s 0.5s linear infinite spin 表示延迟 0.5s 后开始，每轮 2s，线性无限循环。

面试考点

  • transition 和 animation 的核心区别是什么？ 结论是 transition 需要状态变化触发，在两个值之间过渡；animation 基于关键帧时间线，可自动播放和控制循环。原因是 transition 是“从 A 到 B”，animation 是“沿时间线走完整段表演”。项目里按钮/卡片交互优先 transition，loading/徽标脉冲优先 animation。
  • 为什么 transition 不能直接过渡 height: auto？ 结论是 auto 不是可插值长度，浏览器不知道中间帧该怎么算。原因是 transition 需要明确的起始值和结束值。项目里折叠面板常用 max-height、grid 0fr→1fr 或 JS 测量高度，而不是写 transition: height 0.3s 搭配 auto。
  • animation 简写里两个时间值分别表示什么？ 结论是第一个是 duration，第二个是 delay。原因是 animation 简写会把时间值按顺序分配给这两个子属性。边界是只有一个时间值时它表示 duration；名称建议放末尾，避免和关键字混淆。
  • 什么时候用 transition，什么时候用 @keyframes？ 结论是交互反馈、状态切换、hover/focus 这类“从当前样式过渡到目标样式”用 transition；需要自动播放、多段状态、无限循环或复杂节奏时用 @keyframes。原因是 transition 更简单，animation 表达力更强但维护成本也更高。非必要无限动画要配合 prefers-reduced-motion 降级。

选型对照

transition animation + @keyframes
触发方式 需要属性值变化（hover、class、JS 改样式） 可自动播放，也可配合 class 控制
表达能力 两个状态之间过渡 多关键帧、循环、反向、延迟
典型场景 按钮 hover、菜单展开、颜色/位移反馈 loading、骨架屏、序列帧、强调动效
性能建议 优先 transform / opacity；避免动画 width / height / top / left
