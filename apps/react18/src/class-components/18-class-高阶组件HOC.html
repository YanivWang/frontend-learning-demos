高阶组件 HOC（Higher-Order Component）

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • HOC 是一个函数：接收组件，返回增强后的新组件。它是 React 组合模型上的模式，不会修改原组件本身，典型签名是 const Enhanced = withSomething(Wrapped)。
  • class 维护项目中，HOC 常用于注入 props、权限校验、日志埋点、错误兜底、复用生命周期逻辑。本 demo 的 withLoading 注入加载态分支，withLogger 复用挂载和更新日志。
  • 不要在 render 里临时调用 HOC 创建新组件。每次 render 都返回新的组件类型会导致 React 卸载旧树、挂载新树，丢失内部 state，也破坏 diff 复用。
  • ref 不是普通 prop，HOC 不会自动透传 ref；静态属性也不会自动复制。需要时要用 React.forwardRef 或手动提升静态属性，并设置 displayName 保持调试可读性。
  • Hooks 时代，很多“复用状态逻辑”的 HOC 可改为自定义 Hook；但权限包裹、路由守卫、错误边界、兼容 class 组件的横切逻辑仍可能保留 HOC。

面试考点

  • HOC 为什么说是组件逻辑复用模式而不是组件本身？ 结论是 HOC 是接收组件并返回组件的函数。原因是它复用的是包装逻辑和 props 注入方式，而不是一段固定 UI；项目中常把鉴权、埋点、加载态写成 HOC。
  • HOC 和 render props、自定义 Hook 的区别是什么？ 结论是 HOC 通过包装组件复用逻辑，render props 通过函数子节点交出渲染控制，自定义 Hook 复用函数组件的状态逻辑。原因是三者组合层级不同；新函数组件通常优先 Hook，class 维护代码常见 HOC。
  • 为什么不要在 render 中创建 HOC？ 结论是会生成新的组件类型。原因是 React 会把前后两次渲染看成不同组件并卸载重建；边界是 HOC 应在模块作用域或 render 外创建。
  • HOC 包装后 ref、displayName 和静态属性有哪些坑？ 结论是 ref 和静态属性不会自动传给被包裹组件，displayName 也可能丢失。项目里需要显式 forwardRef、提升静态属性并设置包装组件名称。
