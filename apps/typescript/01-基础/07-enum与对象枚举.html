enum 与对象枚举

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • enum 是 TypeScript 少数会生成 JavaScript 的类型语法；字符串 enum 保留键到值的对象，数字 enum 还会生成值到键的反向映射。
  • const enum 默认在编译期内联为字面量，运行时代码更少，但跨包发布、isolatedModules、Babel/SWC 单文件转译时容易遇到不一致或无法内联的问题。
  • as const 对象枚举保留普通 JavaScript 对象，再用 typeof Obj[keyof typeof Obj] 得到联合类型，调试、序列化和 tree-shaking 都更符合现代前端工程习惯。
  • 如果后端协议、URL 参数或埋点值本身就是字符串，优先用字符串字面量联合或 as const 对象；只有需要运行时命名空间、兼容历史代码或与第三方 enum API 对齐时再选 enum。
  • 常见坑是把 enum 当成纯类型使用，导致输出多余运行时代码；另一个坑是把 const enum 暴露给库消费者，消费者的编译方式可能和发布方不一致。

面试考点

  • enum 和 as const 对象枚举怎么选？ 结论是业务前端状态值通常优先用 as const 对象加联合类型，已有运行时 enum API 再用 enum。原因是对象枚举是普通 JS，输出直观、可 tree-shaking，也不会引入数字 enum 反向映射。边界是需要在运行时按枚举名访问、或维护历史 TS enum 代码时，保留 enum 更稳。
  • const enum 为什么有争议？ 结论是应用内部可谨慎使用，公共库声明里不建议暴露。原因是它依赖编译期内联，消费者若使用 isolatedModules 或不同转译链路，可能无法安全解析或出现版本错配。项目里更常见的替代是字符串字面量联合和 as const 对象。
  • 数字 enum 的反向映射有什么影响？ 结论是它会让运行时对象同时包含名称和数字键。原因是 TS 为数字 enum 生成双向映射，方便从值查名，但也带来额外体积和遍历时的意外键。项目里如果值要落库、传接口或展示，字符串 enum 或对象枚举通常更清晰。
  • 为什么联合字面量常能替代 enum？ 结论是联合字面量能在类型层约束可选值，不生成额外运行时代码。原因是很多状态只需要编译期检查，不需要一个真实枚举对象。边界是需要集中维护值表、渲染下拉选项或做运行时校验时，要配合 as const 对象或数组保存值。
