映射类型与模板字面量类型

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • 映射类型通过 [K in keyof T] 遍历对象键，能批量改变属性的可选、只读和值类型，是 Partial、Readonly、Pick 等工具类型的基础。
  • 索引访问类型 T[K] 根据键取得对应值类型，本页的 FieldGetter 保证 getName 返回 User["name"]，不会和字段类型脱节。
  • 模板字面量类型可以把字符串字面量组合成新类型，例如 ${keyof User & string}Changed 生成字段变化事件名。
  • TypeScript 4.1 起映射类型支持 as 键名重映射，配合 Capitalize 等内置字符串工具类型，可以从数据模型派生 getter、setter 或事件 API。
  • 常见坑是模板字面量类型过度嵌套后难以阅读，或让类型层 API 和真实运行时对象不一致；示例中运行时 getters 和 events 仍需要手动实现或由代码生成保证同步。

面试考点

  • 映射类型为什么重要？ 结论是它能把“逐字段重复声明”变成“按键批量变换”。原因是 keyof 提供键集合，in 遍历键，T[K] 保留每个字段原本的值类型。项目里它适合派生表单补丁、只读模型和接口 DTO。
  • keyof、T[K] 和模板字面量类型如何配合？ 结论是 keyof 取字段名，T[K] 取字段值类型，模板字面量类型生成受字段名约束的新字符串。原因是这样能把事件名、getter 名和原模型绑定起来。示例里的 idChanged、nameChanged 就来自 User 的键。
  • as 键名重映射适合什么场景？ 结论是适合从已有模型派生一套有规则的新 API。原因是它能在映射时重命名、过滤或格式化键名。边界是重映射只改变类型，运行时对象仍要按同样规则创建，否则类型和实现会脱节。
  • 复杂模板字面量类型有什么风险？ 结论是过度组合会降低可读性并拖慢类型检查。原因是多个联合字符串会产生笛卡尔积式组合。项目里只应把它用于稳定、规则清晰的命名体系，例如事件名、路由名和设计系统 token。
