原生 DOM 拖拽调整侧栏宽度

请在下方面板拖拽右侧手柄调整宽度；本页纯原生 DOM，不依赖 Vue/React。细节可打开 DevTools Console。

知识点要点

  • 拖拽三段式：pointerdown 记录起点与初始宽度 → pointermove 计算新宽度 → pointerup / pointercancel 解绑并恢复样式。
  • mousemove / pointermove 应绑在 document 上，否则指针移出手柄后无法继续拖动。
  • 拖拽期间给侧栏加 transition: none 和 user-select: none，避免 CSS 过渡与文字选中干扰操作。
  • requestAnimationFrame 把同一帧内多次 move 合并为一次 DOM 写入，减少 layout thrashing。
  • Pointer Events 统一鼠标、触屏、笔输入；手柄设置 touch-action: none 阻止浏览器默认滚动。对比 Vue 版见 贴边侧栏拖拽。

面试考点

  • 为什么 move/up 绑 document？ 结论：指针会移出拖拽手柄元素。原因是事件目标随指针移动变化，只在手柄上监听会中断拖拽。项目里 document/window 级监听是侧栏、列宽、面板分割线的常见写法。
  • rAF 在拖拽里做什么？ 结论：合并高频 move，每帧最多写一次宽度。原因是 pointermove 可能一帧触发多次，直接改 style 会造成多余 reflow。边界是最终仍须 pointerup 时提交 pending 值。
  • Pointer Events 的优势？ 结论：一套 API 覆盖鼠标与触屏。原因是 Mobile 上仅 Mouse Events 可能不触发或行为不一致。新项目优先 pointer*，Legacy 再补 mouse*。
  • 为什么要禁用 transition？ 结论：过渡动画会延迟宽度视觉反馈，拖拽 feels 滞后。原因是 transition 在每次 width 变化时插值。拖拽结束后再恢复 transition 即可。
