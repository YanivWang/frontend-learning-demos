createApp / mount / 模板语法

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Vue3 用 createApp() 创建应用实例，插件、全局组件和应用级配置都挂在这个实例上；调用 mount("#app") 后才会渲染并接管容器。本页加载后标题与下方区块出现，控制台会打印 [mounted] createApp 已接管 #app。
  • 模板里只能写单条 JavaScript 表达式，不能写 if、for、var 等语句。本页「年龄 + 1」「是否成年」用算术与三元表达式演示；分支列表与循环应交给 v-if、v-for（见下一页）。
  • ref 在模板中自动解包，可写 {{ title }}；在 setup() 的 JavaScript 里仍要通过 title.value 读写。点「在 setup 中改 title」后，标题与「setup 记录」同步更新，控制台可见 [setup] title.value = ...。
  • 本页的 ref 指响应式包装对象，与模板上的 DOM ref 属性不是同一概念；访问 DOM 或子组件见 05-模板ref与nextTick、06-useTemplateRef模板ref（3.5+）。响应式细节见 02-ref-reactive与解构。
  • v-text 与 {{ message }} 一样写入纯文本；v-html 会把字符串按 HTML 解析插入。本页 v-html 区域「v-html 会按 HTML 渲染」显示为粗体；点「改 v-text 文案」只更新 v-text 段落。
  • 模板里调用方法（如本页 formatAge()）会在每次渲染时重新执行。点「触发重渲染」且 age 不变时，「已执行 N 次」仍会增加，控制台会再打印 [method] formatAge 执行；可缓存的派生值对比见 02-computed-watch-watchEffect。
  • Vue3 CDN 全局构建把 API 暴露在 Vue 对象上，本页用 const { createApp, reactive, ref } = Vue 直接运行；工程化项目实践中通常从 vue 包按需导入。v-html 只能用于可信内容，绝不能渲染未消毒的用户输入。

面试考点

  • createApp 与 mount 分别解决什么问题？ 结论是 createApp 创建应用实例，mount 把应用渲染到具体 DOM 容器。原因是 Vue3 把应用级能力隔离在实例上，多个应用可以有各自的配置和插件。边界是同一个应用实例通常只挂载一次，挂载后应通过响应式状态驱动视图，而不是反复重建根实例。
  • 同一应用实例能否多次 mount？多个 createApp 如何隔离？ 结论是同一实例再次 mount 会先卸载再挂载，实践中通常只 mount 一次；需要多区域独立应用时分别 createApp(...).mount(...)。原因是每个实例有独立的配置、插件和组件注册表。本页刷新后控制台只出现一条 [mounted] 即说明 mount 已完成渲染。
  • 模板表达式和方法调用应该如何取舍？ 先结论：简单展示可写内联表达式，每次渲染都跑的格式化用方法，可缓存派生值用 computed。原因是表达式与方法都参与渲染求值，过重逻辑降低可读性；方法无缓存，computed 依赖不变时不重算。本页「触发重渲染」按钮可验证方法重复执行；完整缓存对比见 02-computed-watch-watchEffect。
  • 模板里的 ref 自动解包为什么不等于 JS 里不用 .value？ 结论是自动解包只发生在模板渲染上下文等特定场景，setup() 代码仍要使用 .value。原因是 ref 本质是带 value 的响应式包装对象。边界是深层对象、数组元素和普通解构场景不要假设都会自动解包；本页「在 setup 中改 title」按钮即验证 .value 写法。
  • 为什么 v-html 不能渲染用户输入？ 结论是 v-html 会把字符串交给浏览器按 HTML 解析，未消毒内容可能造成 XSS。原因是它绕过了 Vue 默认的文本转义保护。项目中只有后端或前端明确消毒后的可信富文本才适合使用，并配合内容安全策略和白名单。
