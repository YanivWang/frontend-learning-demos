Vue 2 八个生命周期钩子速记

打开页面后先看 #app 下方 <pre> 里自动出现的 created → beforeMount → mounted 顺序；点「触发更新」观察 beforeUpdate / updated ；点「销毁实例」观察 beforeDestroy ，并打开 Console 查看 destroyed 日志。

知识点要点

  • 创建阶段顺序是 beforeCreate → created 。 beforeCreate 时 data、computed、methods、watch 还没初始化； created 后可以访问响应式数据和方法，但还没有挂载真实 DOM。
  • 挂载阶段顺序是 beforeMount → mounted 。 mounted 后当前实例的 $el 已可访问；若要确保整个视图已根据后续状态变化更新，再配合 $nextTick 。
  • 更新阶段顺序是 beforeUpdate → updated 。 beforeUpdate 中数据已经是新值，DOM 可能还是旧值； updated 中 DOM 已 patch，但不要在这里无条件改状态，容易形成更新循环。
  • 销毁阶段顺序是 beforeDestroy → destroyed 。定时器、全局事件、第三方实例、订阅等副作用优先在 beforeDestroy 清理； $destroy() 会触发这两个钩子，但日常更推荐用 v-if 控制组件生命周期。
  • Vue 2 使用 beforeDestroy/destroyed ，Vue 3 Options API 改名为 beforeUnmount/unmounted ；维护 Vue 2 老项目时要用旧命名，新项目迁移时要注意生命周期名和组合式 API 写法差异。

面试考点

  • beforeCreate 和 created 最大区别是什么？ 结论是 beforeCreate 不能可靠访问 data/methods， created 可以。原因是 Vue 初始化选项、响应式数据和方法绑定发生在这两个钩子之间。项目里初始化纯 JS 数据请求通常从 created 起步，不要在 beforeCreate 调用 methods。
  • created 和 mounted 分别适合放什么逻辑？ 结论是 created 适合不依赖 DOM 的初始化，mounted 适合依赖 DOM 的第三方插件、尺寸读取或焦点处理。原因是 created 阶段实例数据已就绪但 DOM 未挂载，mounted 后才有 $el 。边界是服务端渲染和异步子组件场景下，DOM 相关逻辑还要更谨慎。
  • beforeUpdate 和 updated 中读 DOM 有什么边界？ 结论是 beforeUpdate 读到的 DOM 可能仍是旧状态， updated 后 DOM 已与本次状态同步。原因是更新流程先改响应式数据，再进行虚拟 DOM patch。项目里如果要在状态变化后读最新 DOM，也常用 $nextTick 聚焦到一次具体更新；在 updated 里无条件改 data 可能触发更新循环。
  • beforeDestroy / destroyed 应该如何清理副作用？ 结论是外部资源在 beforeDestroy 清理， destroyed 时实例已不可用。原因是 Vue 会清理自己的指令、事件和子组件，但不会自动取消你手动注册到 window 的监听、定时器、WebSocket 或第三方库实例。项目里通常在 beforeDestroy 做清理，避免内存泄漏和重复回调。
  • Vue2 生命周期和 Vue3 命名有什么迁移差异？ 结论是核心阶段类似，但销毁命名从 destroy 改为 unmount。原因是 Vue 3 API 统一表达「卸载组件」。维护 Vue 2 用 beforeDestroy/destroyed ，迁到 Vue 3 Options API 用 beforeUnmount/unmounted ，组合式 API 用 onBeforeUnmount/onUnmounted 。
