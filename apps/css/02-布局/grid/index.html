Grid 布局练习

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • display: grid 会让直接子元素参与网格布局，容器通过 grid-template-columns 和 grid-template-rows 定义列、行轨道。
  • 固定像素和百分比适合明确尺寸，但百分比轨道再叠加边框、内边距或 gap 容易超出容器；示例中的 fr 更适合按剩余空间分配。
  • repeat() 用来减少重复轨道书写，auto-fill 会按可用空间尽量填入更多轨道，常用于卡片列表的自适应列数。
  • minmax() 给轨道设置最小和最大边界，能避免列宽过窄；与 fr 配合时常见写法是 minmax(0, 1fr) 或业务最小宽度。
  • grid-template-areas 用命名区域表达页面骨架，可读性好；边界是区域名和 DOM/样式强绑定，频繁动态增删的列表更适合自动放置和显式轨道。

面试考点

  • Grid 为什么叫二维布局，和 Flex 的核心差异是什么？ 结论是 Grid 同时控制行和列，Flex 主要沿一条主轴分配空间。原因是 Grid 先定义网格轨道和单元格，再把项目放进去。项目里页面骨架、仪表盘、卡片矩阵优先 Grid，导航条和局部对齐优先 Flex。
  • fr、百分比、auto、minmax() 怎么区分？ 结论是百分比按容器尺寸计算，fr 分配剩余空间，auto 倾向内容尺寸，minmax() 给轨道加边界。原因是它们参与网格轨道尺寸算法的阶段不同。真实项目中不确定内容宽度时应避免纯百分比硬凑。
  • repeat(auto-fill, 100px) 适合什么场景？ 结论是适合固定卡片最小宽度、容器变宽就自动增加列数的布局。原因是 auto-fill 会创建尽可能多的轨道。边界是它不会自动拉伸卡片宽度，想兼顾最小宽度和铺满容器通常写 repeat(auto-fit, minmax(160px, 1fr))。
  • grid-template-areas 的优势和边界是什么？ 结论是它适合 header/sidebar/main/footer 这类语义明确的区域。原因是命名区域能让布局结构一眼可读。边界是区域必须形成矩形，且命名和 CSS 强耦合，动态列表不适合用它逐项维护。
