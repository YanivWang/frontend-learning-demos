keep-alive 与 activated / deactivated

切换 TabA / TabB，观察下方生命周期日志；在某 tab 输入框输入文字后再切换，验证 keep-alive 是否保留内容与 activated/deactivated 触发顺序。

知识点要点

  • <keep-alive> 是 Vue 2 的抽象组件，用来缓存包裹的动态组件实例；切走时实例不会销毁，所以本页输入框内容可以保留。
  • 首次渲染某个 tab 时会走 created 、 mounted 和 activated ；之后在 tab 间切换时主要观察 activated 与 deactivated 。
  • activated 表示缓存组件重新进入激活状态，适合恢复轮询、刷新可见数据； deactivated 表示离开但未销毁，适合暂停定时器或取消可见区监听。
  • 常见场景是 tab、列表返回详情、表单步骤缓存。风险是缓存过多会占用内存，且隐藏组件中的副作用如果不暂停，仍可能造成重复请求或性能问题。
  • 生产项目应结合组件 name 与 include 、 exclude 、 max 控制缓存边界；Vue 2 已 EOL，维护项目更要明确缓存策略。

面试考点

  • keep-alive 缓存的是什么？ 结论：缓存的是组件实例。原因是它让被切走的动态组件不进入销毁流程，再次显示时复用原实例。边界是缓存会占内存，不能无脑包所有页面。
  • activated/deactivated 和 mounted/destroyed 有什么区别？ 结论：mounted/destroyed 对应真实挂载和销毁，activated/deactivated 对应缓存组件进入和离开活跃状态。原因是 keep-alive 下组件离开时并未销毁。项目里恢复或暂停副作用应放在 activated/deactivated。
  • keep-alive 常用于哪些页面场景，风险是什么？ 结论：适合 tab、列表返回详情、表单步骤等需保留状态的互斥视图。原因是用户期望输入、滚动不被重置。风险是缓存过多占内存，隐藏组件中的副作用若不暂停可能造成重复请求或性能问题。
  • include/exclude/max 为什么重要？ 结论：它们控制缓存范围和数量。原因是缓存实例越多，内存和副作用管理成本越高。项目里应给需要缓存的组件明确 name ，再配置白名单或最大缓存数。
