Actions 与 useActionState

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • React 19 的 Action 是用于处理异步提交的函数，可绑定到表单 action，也可和 transition 体系配合，让提交状态进入 React 的调度模型。
  • useActionState(action, initialState) 返回 [state, formAction, isPending]；本页把 formAction 传给 <form action>。
  • Action 函数接收 previousState 和 formData，返回下一次 state 或 Promise；这让失败信息、成功结果和服务端返回值都能成为显式状态。
  • isPending 表示当前提交未完成，可用于禁用输入和按钮，避免重复提交；失败路径要返回可展示的错误，而不是只在控制台打印。
  • 本页校验空名称和长度不足两个失败路径；真实项目还要处理网络失败、服务端校验、重试与成功后是否清空表单。

面试考点

  • Action 和普通 onSubmit 有什么区别？ 结论是 Action 把异步提交结果纳入 React 的状态模型。原因是 useActionState 能直接返回下一次 state 和 pending 状态，减少手写 loading / error 样板。边界是高度定制的表单库仍可能保留自己的提交管线。
  • previousState 有什么用？ 结论是它让下一次提交能基于上一次结果计算。原因是表单提交常需要保留错误、计数、服务端返回 ID 或局部状态。本页只返回错误字符串，复杂项目可返回对象，例如 { ok, message, fieldErrors }。
  • isPending 应该怎么用？ 结论是用它禁用按钮、展示“提交中”，并防止重复提交。原因是异步 Action 期间用户可能连续点击或修改输入。边界是禁用输入会影响用户修正内容，实际产品要根据提交耗时和可取消性设计交互。
  • 失败路径应该放在哪里？ 结论是校验和服务端错误应作为 Action 返回 state 展示给用户。原因是表单失败是正常业务结果，不应只靠异常或控制台。项目里网络异常可以转成通用错误，字段校验则返回字段级错误。
