浏览器渲染流程演示

从 TTFB 到 LCP，每一步都有注释

渲染流程时间线

  1. TTFB 浏览器发出请求 → 收到服务器第一个字节
  2. HTML 解析开始 流式解析，边下载边构建 DOM；遇到阻塞资源暂停
  3. CSS 下载 + CSSOM 构建 渲染阻塞；必须完成才能进行第一次渲染
  4. FCP 第一个内容（文字/图片）出现在屏幕上
  5. LCP 视口内最大内容元素渲染完成（通常是主图或大标题）
  6. TTI / INP JS 执行完毕，页面可交互；之后持续监测交互延迟

CLS 说明：避免布局位移

✅ 正确：声明了 width/height，浏览器预留位置，图片加载不引起位移

🚫 错误做法：不给图片声明尺寸 → 图片加载后内容跳动 → CLS 增大

INP 说明：交互响应速度

INP 目标 < 200ms。主线程被长任务阻塞时，点击后页面"卡住不动"。

总结：影响各指标的关键因素

指标 主要影响因素 目标
TTFB 服务器响应速度、CDN、网络延迟 < 800ms
FCP 渲染阻塞 CSS/JS、关键资源大小 < 1.8s
LCP 最大元素（主图）加载速度、fetchpriority < 2.5s
CLS 图片/广告未声明尺寸、动态插入内容 < 0.1
INP 主线程长任务、JS 执行时间 < 200ms
