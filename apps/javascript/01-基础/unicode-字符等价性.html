Unicode 字符等价性（NFC / NFD）

请打开 DevTools Console 查看输出。

知识点要点

  • Unicode 允许多种编码表示同一「字符」（如 é 可以是单码点或 e+组合重音）。
  • NFC（组合形式）与 NFD（分解形式）是两种标准规范化；str.normalize('NFC') 统一后再比较。
  • 全角/半角、不同空格（NBSP）也会导致 === 失败。
  • 用户输入清洗流程：trim → 折叠连续空白 → normalize → 再比较/入库。

面试考点

  • 相同外观为何不相等？ 不同 Unicode 码点序列（组合字符 vs 预组合字符）。需 normalize 后比较。
  • normalize 作用？ 将字符串转为 NFC/NFD 等规范形式，便于搜索、去重、数据库唯一约束。
  • 用户名校验？ 不能只做 ===；要 normalize、考虑大小写/全半角、防 homoglyph 攻击（国际化产品）。
