class 组件 constructor 必须调用 super()

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • React class 组件本质上是继承 React.Component 的 ES class；只要显式声明 constructor，就必须先调用 super()，否则在访问 this 前会触发 JavaScript 子类初始化错误。
  • 在 React class 中推荐写 super(props)，因为这样 constructor 内的 this.props 与传入参数保持一致；如果 constructor 不读取 props，语法上 super() 也能完成父类初始化，但没有必要刻意省略参数。
  • constructor 只适合初始化 this.state 或绑定实例方法；网络请求、订阅、定时器等副作用应放在 componentDidMount，并在卸载时清理。
  • 如果不需要初始化 state 或绑定方法，class 组件可以完全省略 constructor；现代 React 新代码更推荐函数组件与 Hooks，class 更多用于维护历史代码和理解生命周期。
  • class 组件必须实现 render()，React 会调用它得到要渲染的元素；示例中的 render 返回一段 JSX，证明组件实例初始化成功后才能进入渲染阶段。

面试考点

  • 为什么 class 组件 constructor 里要先调用 super(props)？ 结论是：因为它既满足 ES class 子类必须先初始化父类的规则，也让 React 组件实例拿到 props。原因是子类在 super() 之前不能访问 this；项目中如果 constructor 里要初始化 state 或绑定方法，通常直接写 constructor(props) { super(props); ... }。
  • 什么时候可以不写 constructor？ 结论是：没有初始化 state、没有手动绑定实例方法时就不要写。原因是 React 会正常构造组件实例，额外的空 constructor 只增加样板代码；维护老 class 组件时再根据需要补充。
  • super() 与 super(props) 有什么区别？ 结论是：两者都能调用父类构造函数，但 React class 中应优先写 super(props)。原因是 constructor 内如果读取 this.props，传入 props 才最直观；边界是新代码通常用函数组件，不会再遇到这个 constructor 细节。
  • constructor 里适合做哪些事？ 结论是：只做同步初始化，例如设置初始 state、绑定方法。原因是 constructor 发生在挂载前，不能依赖 DOM，也不适合请求数据；副作用放到 componentDidMount，清理放到 componentWillUnmount。
  • class 组件为什么必须实现 render()？ 结论是：render() 是 class 组件描述 UI 的必需方法。原因是 React 需要调用它得到 React element；如果漏写，组件没有可渲染输出，开发环境会给出缺少 render 方法的错误。
