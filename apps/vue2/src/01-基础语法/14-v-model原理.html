v-model 原理

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • 普通文本 input 上的 v-model 可以理解为 :value="keyword" 加 @input="keyword = $event.target.value"。本页前两个输入框演示了语法糖和手写展开写法保持同一份状态。
  • textarea 也使用文本值同步；checkbox、radio、select 有各自表单语义。多个 checkbox 绑定数组时，选中会把对应 value 加入数组，取消会移除。
  • 自定义组件上的 v-model 默认约定是父组件传入 value prop，子组件通过 input 事件把新值发回父组件。本页 name-input 正是这个模式。
  • Vue 2 还支持 .lazy、.number、.trim：分别改为 change 时同步、把输入转成数字、自动去掉首尾空白。它们改变的是输入同步规则，不是替代表单校验。
  • 真实表单要处理空值、非法值、输入法组合、异步校验和提交失败。Vue 2 已 EOL，维护旧项目时理解这些约定即可，新项目应优先 Vue 3 当前写法。

面试考点

  • Vue 2 中普通 input 的 v-model 可以拆成什么？ 结论是可拆成 :value 绑定和 @input 赋值。原因是文本输入的核心就是从状态渲染 value，再在输入事件里把 DOM 新值同步回状态。项目里手写自定义输入组件时，要保证这两个方向都完整。
  • checkbox 的 v-model 为什么不能简单理解成 value + input？ 结论是 checkbox 有布尔值、数组收集和真实 value 三种语义。原因是多个 checkbox 绑定数组时，Vue 根据选中状态添加或移除 value，而不是把输入框文本写回状态。项目里多选标签、权限选择要特别注意绑定类型。
  • 自定义组件 v-model 默认约定是什么？ 结论是默认接收 value prop，触发 input 事件。原因是父组件的 v-model="x" 会编译成 :value="x" 和 @input="x = $event"。边界是 Vue 2 可以用组件 model 选项改成其他 prop/event。
  • .lazy、.number、.trim 分别改变什么行为？ 结论是 .lazy 改为 change 后同步，.number 尝试转数字，.trim 去掉首尾空白。原因是它们是 v-model 的输入修饰符。项目里修饰符不能替代表单校验，尤其数字转换失败时仍可能保留原值。
