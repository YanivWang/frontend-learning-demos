useState 异步合并与连续更新

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • setCount 不会立刻改变当前 render 中的 count 变量，而是请求下一次渲染。
  • 同一事件回调里多次 setCount(count + 1) 都基于当前 render 快照，连续 +1 三次通常只 +1。
  • 函数式更新 setCount(prev => ...) 每次基于队列最新值，可正确累加。
  • 对比 class 03：原理相同，只是 API 从 this.setState 换成 useState 返回的 setter。
