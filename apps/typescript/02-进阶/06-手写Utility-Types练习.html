手写 Utility Types（运行时等价演示）

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • MyPartial<T> 的核心是 { [K in keyof T]?: T[K] }：遍历所有键，保留原值类型，并给属性加上可选修饰符。
  • MyPick<T, K extends keyof T> 的核心是限制 K 必须来自 T 的键集合，再用 [P in K]: T[P] 只保留这些字段。
  • MyOmit<T, K> 常组合 Pick<T, Exclude<keyof T, K>> 实现：先从键集合里排除 K，再挑剩余键。
  • 本页按钮演示的是运行时 myPartial / myPick / myOmit 对对象做拷贝过滤；类型层工具只负责静态约束，两者需要同时理解。
  • 常见坑是忘记 K extends keyof T 导致可以挑不存在的键，或误以为 Partial 会递归处理嵌套对象；面试答题要说清“浅层”和“编译期”。

面试考点

  • 如何手写 MyPartial？ 结论是用映射类型遍历 keyof T，并给每个属性加 ?。原因是 T[K] 能保留原字段值类型，? 改变属性是否必填。边界是这是浅层变换，嵌套对象不会自动变成可选字段。
  • MyPick 里为什么要写 K extends keyof T？ 结论是为了限制只能挑选真实存在的字段。原因是没有这个约束时，T[P] 无法保证 P 是 T 的键，也失去了字段名拼错时报错的价值。项目里这能保护 DTO 和表单字段白名单。
  • Omit 为什么能由 Pick 和 Exclude 组合出来？ 结论是 Omit 本质是“先排除键，再挑剩余键”。原因是 Exclude<keyof T, K> 先在联合键里移除 K，Pick 再按剩余键重建对象类型。这个思路比死背源码更重要。
  • 运行时 pick/omit 和类型层 Pick/Omit 有什么区别？ 结论是运行时函数真的创建新对象，类型工具只改变编译期视角。原因是 TypeScript 类型会被擦除，不能自动删除对象属性。涉及脱敏、接口返回和安全边界时，必须写运行时代码真正移除敏感字段。

type MyPartial<T> = { [K in keyof T]?: T[K] };
