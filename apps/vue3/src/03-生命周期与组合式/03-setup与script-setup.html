setup / script setup

在浏览器中操作下方交互演示；复习要点与面试答法见本复习区。

知识点要点

  • setup() 是 Composition API 的组件入口，创建响应式状态、注册生命周期和返回模板可用数据都在这里完成。本页根组件用普通 setup 返回 title 、 count 等字段，模板可直接访问。
  • <script setup> 是单文件组件 SFC 的 编译时 语法糖，不是浏览器原生语法。它会在构建阶段编译成普通 setup() ，所以本仓库的全局构建 demo 只能直接运行普通 setup() ，下方 pre 区展示等价 SFC 片段。
  • <script setup> 中的顶层变量、函数和导入会 自动暴露 给模板，不需要手写 return ；本页子组件 CounterBlock 模拟这一行为：setup 内声明的 count 经 return 后在模板可用，与 script setup 编译结果等价。
  • defineProps 、 defineEmits 、 defineExpose 等是编译宏，不需要从 vue 导入，也不能当运行时函数在普通浏览器 <script> 中调用。本页子组件用手写 props / emits 选项对照，详见 04-props与emit 。
  • 点「子组件 +1」只改变子组件本地 count ，父组件 parentCount 不变，说明 setup 返回的状态默认按组件实例隔离；跨组件共享见 composable 或 provide-inject，见 02-composable复用 、 04-provide-inject 。
  • Vue 3.5+ 的 useTemplateRef() 进一步简化模板 ref 配对，见 01-useTemplateRef模板ref（3.5+） 。需要按条件创建组件选项、兼容老项目或直接在无构建环境运行时，普通 setup() 仍然有价值。

面试考点

  • setup 和 script setup 是什么关系？ 结论是 script setup 是编译为 setup() 的 SFC 语法糖。原因是它减少样板代码，让顶层绑定自动进入模板。边界是运行时能力仍来自 Composition API，不是另一套组件模型；本页 pre 区 SFC 片段与下方可运行 demo 对照。
  • 为什么普通 HTML 里不能直接写 script setup ？ 结论是因为它需要 Vue SFC 编译器处理。浏览器只认识普通 JavaScript，不认识 SFC 编译宏和模板绑定规则。本页因此用全局构建的普通 setup() 演示同一组概念。
  • defineProps / defineEmits 为什么不用导入？ 结论是它们是编译宏，不是普通运行时 API。原因是编译器会静态分析并生成组件 props / emits 选项。项目里不要把它们封装进普通函数，也不要期待运行时动态调用。
  • 什么时候不用 script setup ？ 结论是无构建环境、需要显式组件选项组织、迁移中的 Options API 文件可以继续用普通 setup 或 Options API。原因是 script setup 依赖 SFC 编译流程，且更适合现代构建链。
  • setup 返回值与 script setup 自动暴露有何差异？ 结论是无运行时差异，script setup 只是省掉手写 return。普通 setup 必须显式 return 模板需要的绑定；遗漏 return 会导致模板变量未定义。本页 CounterBlock 的 return 即 script setup 编译后会省略的部分。
