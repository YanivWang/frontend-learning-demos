Flex 一维布局

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出 flex 子项实测宽度。

知识点要点

  • display: flex 会让直接子元素成为 flex items，按一条主轴排布； flex-direction 决定主轴方向，交叉轴与主轴垂直。
  • justify-content 分配主轴上的剩余空间， align-items 处理整行项目在交叉轴上的对齐，单个项目可用 align-self 覆盖。
  • flex 是 flex-grow 、 flex-shrink 、 flex-basis 的简写；布局先看 flex-basis ，再按剩余空间 grow 或 shrink。
  • flex: 1 常用于“占满剩余空间”，实际会让项目可增长且基础尺寸趋近于 0；需要保留内容基础宽度时应显式写 flex: 1 1 auto 或具体 basis。
  • 常见坑是 flex item 默认 min-width: auto ，长单词、表格或代码块可能撑破布局；两栏/三栏自适应区域通常补 min-width: 0 。

面试考点

  • Flex 的主轴和交叉轴怎么判断？ 结论是主轴由 flex-direction 决定，默认横向从左到右；交叉轴与主轴垂直。原因是 Flex 是一维布局，所有空间分配都先围绕主轴展开。项目里不要死记“justify 是水平、align 是垂直”，换成 column 后两者方向会互换。
  • flex: 1 到底等价于什么？ 结论是浏览器通常按 flex: 1 1 0% 展开，用于把剩余空间按比例分掉。原因是 basis 为 0 时内容基础尺寸不再主导分配。边界是内容不能被压缩时仍可能溢出，需要配合 min-width: 0 或显式 basis。
  • Flex 子项为什么会把容器撑破？ 结论是默认最小尺寸可能来自内容本身，而不是可用空间。原因是 flex item 的 min-width:auto 会保护内容不被无限压缩。项目里自适应主栏、卡片标题、代码块区域通常要给子项加 min-width:0 。
  • Flex 和 Grid 怎么选？ 结论是一维排列、居中、导航栏、左右栏优先 Flex；同时控制行和列的二维结构优先 Grid。原因是 Flex 以内容流和剩余空间分配为核心，Grid 以显式网格轨道为核心。复杂页面可外层 Grid、局部 Flex 组合使用。
