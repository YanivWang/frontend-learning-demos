解构、rest/spread、模板字符串、可选链与空值合并

请打开 DevTools Console 查看输出。

知识点要点

  • 解构用于从数组或对象中按结构绑定变量，支持数组位置、对象属性名、重命名、默认值和嵌套模式。
  • 解构默认值只在取到 undefined 时生效，取到 null 不会使用默认值；解构 null / undefined 本身会抛错。
  • ... 在绑定位置是 rest，用来收集剩余元素或剩余自有可枚举属性；在调用、数组字面量、对象字面量中是 spread，用来展开已有数据。
  • 数组/对象 spread 只做浅层复制，嵌套对象仍共享引用；需要深拷贝时应使用结构化克隆或业务明确的复制逻辑。
  • 模板字符串支持多行、 ${expression} 插值和标签函数；标签函数可拿到字符串片段与插值值，不一定返回字符串。
  • ?. 只在左侧为 null / undefined 时短路， ?? 只在左侧为 null / undefined 时使用默认值，能保留 0 、空字符串和 false 。

面试考点

  • 解构默认值什么时候生效？ 只在解构结果为 undefined 时生效， null 会作为有效值保留下来。项目里处理接口数据时，如果字段可能为 null ，不能指望解构默认值兜底。
  • rest 和 spread 有什么区别？ 结论是 rest 收集，spread 展开。原因是它们出现在不同语法位置：函数参数或解构左侧的 ... 收集剩余值；调用参数、数组字面量、对象字面量里的 ... 展开已有值。
  • 对象 spread 为什么不是深拷贝？ 它只复制一层自有可枚举属性，属性值如果是对象，复制的仍是引用。项目里用 spread 更新状态时要逐层复制被修改的分支，不能误以为嵌套对象已经隔离。
  • ?. / ?? 与 && / || 有什么区别？ ?. 和 ?? 只把 null / undefined 当缺失； && / || 会受所有 falsy 值影响。需要保留 0 、 "" 、 false 时应优先用 ?? 。
