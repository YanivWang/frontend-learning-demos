defineModel（Vue 3.4+）

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • defineModel 是 Vue 3.4+ 的 SFC <script setup> 编译宏，声明一个 prop 和对应的 update:xxx 事件，并返回可读写的模型 ref。
  • 默认 defineModel() 对应 modelValue / update:modelValue，defineModel('title') 对应 title / update:title；本页用 computed getter / setter 演示等价运行时行为。
  • 宏只能在 SFC 编译阶段被识别，浏览器全局构建没有 SFC 编译器，所以 HTML demo 不能直接调用 defineModel，只能展示等价实现和代码片段。
  • 可以给模型声明 required、default、类型，以及通过 get / set 处理修饰符；例如输入组件可在 setter 中实现 trim 或格式转换。
  • 注意官方文档提示：子组件模型设置 default 而父组件绑定值是 undefined 时，可能出现父子初始值不同步；组件库中要明确默认值策略。

面试考点

  • defineModel 和手写 modelValue / emits 是什么关系？ 结论是语义等价，defineModel 是编译期语法糖。它减少重复声明 prop、emit 和 computed setter 的样板代码，但运行时仍是父组件传值、子组件发出 update 事件。边界是 Vue 3.4+ 才能使用，低版本要手写。
  • 为什么说 defineModel 返回的是可写 ref？ 因为模板或脚本给这个 ref 赋值时，编译结果会 emit 对应的 update:xxx 事件。这样子组件写起来像本地状态，实际仍让父组件更新数据源。项目中要避免把它误解成子组件私有状态。
  • defineModel 的 default 有什么坑？ 结论是可能造成父子值短暂不一致。原因是子组件 default 会让本地模型有初始值，而父组件绑定的 ref 仍可能是 undefined。表单组件中应让父组件初始化值，或在文档中明确 default 的同步边界。
  • 什么时候继续手写 props / emits？ 当项目还在 Vue 3.3 或更早版本、组件不是 SFC <script setup>、或需要非常显式的运行时声明时继续手写。组件库若面向多个 Vue 版本，也要确认构建链是否支持该宏。
