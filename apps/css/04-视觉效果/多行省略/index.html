多行文本省略（line-clamp）

对比单行 text-overflow: ellipsis 与多行 line-clamp；注意 flex 场景下的 min-width: 0。

知识点要点

  • 单行省略靠 white-space: nowrap + overflow: hidden + text-overflow: ellipsis；多行省略则要限制最大行数，常见写法是 line-clamp: N 配合 overflow: hidden。
  • 为兼容稍旧环境，实践中常同时写 line-clamp 和 -webkit-line-clamp，并保留 display: -webkit-box + -webkit-box-orient: vertical 作为回退；这里的旧语法是为截断行数服务，不是拿它做页面主布局。
  • 省略生效前提是容器有可计算宽度；在 flex/grid 子项里如果省略失效，常见原因是默认 min-width: auto 让子项被内容撑开，需要补 min-width: 0。
  • 被裁掉的文本仍然在 DOM 中，但视觉上不可见；标题、摘要如果承载关键信息，hover 展开、详情页或 title/ARIA 提示仍要考虑，不能把所有语义都压在 CSS 省略上。

面试考点

  • 多行省略和单行 ellipsis 的核心区别是什么？ 结论是多行省略要限制行数并截断盒子高度，单行 ellipsis 只处理一行不换行文本。原因是单行方案依赖 nowrap，多行方案依赖 line-clamp 或 -webkit-line-clamp。项目里卡片摘要、评论预览常用 2–3 行 clamp，表格单元格标题仍可能用单行 ellipsis。
  • line-clamp 为什么常配合 overflow: hidden？ 结论是没有隐藏溢出，超出行数的文本仍会显示出来。原因是 clamp 通过限制盒子展示区域实现“截断”，溢出部分必须被裁掉，省略号才会出现。边界是某些浏览器组合仍需要 -webkit-box 回退，部署前要按目标浏览器验证。
  • flex 子项里多行省略不生效通常是什么原因？ 结论是子项默认最小宽度受内容保护，容器宽度算不出来。原因是 flex item 的 min-width: auto 会阻止收缩。项目里文本区常见写法是 flex: 1; min-width: 0; overflow: hidden; line-clamp: 2。
  • 多行省略有哪些可访问性和 SEO 边界？ 结论是 CSS 省略不会删除 DOM 文本，但视觉和部分交互场景下用户可能看不到完整内容。原因是被裁切内容不可见，屏幕阅读器是否完整朗读还取决于 DOM 结构和 aria 设计。项目里重要信息应保留完整文本入口，而不是只显示 clamp 后的摘要。
