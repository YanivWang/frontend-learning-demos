默认插槽 / 具名插槽 / 作用域插槽

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • 插槽是 Vue 的内容分发机制：子组件声明 <slot> 出口，父组件把模板片段传进去，从而让结构由子组件控制、具体内容由调用方决定。
  • 默认插槽适合单个主要内容区域，具名插槽适合卡片的 header、footer 等多个出口；本页 BaseCard 同时演示了三种插槽形态。
  • 作用域插槽通过 slot props 让子组件把 item、total 等内部数据交给父组件渲染，常用于表格列、自定义列表项、表单项渲染。
  • 插槽内容在父组件作用域中编译，因此可以访问父组件变量，但不能直接访问子组件内部变量；需要子组件主动通过 slot props 暴露。
  • 组件库设计插槽时要稳定命名和 slot props 结构，并提供默认内容，否则调用方漏写插槽时页面可能空白或语义不完整。

面试考点

  • 默认插槽、具名插槽和作用域插槽分别解决什么问题？ 默认插槽解决单出口内容组合，具名插槽解决多区域内容分发，作用域插槽解决“子组件提供数据、父组件决定渲染”的问题。原因是插槽把组件外观扩展点暴露给调用方。边界是数据修改仍应通过 props / emit 或上下文 API 管理，不要把插槽当状态管理。
  • 为什么插槽内容访问的是父组件作用域？ 结论是插槽内容由父组件模板编译，所以变量解析发生在父组件上下文。子组件只决定插槽出口位置，不能让父组件直接读取子组件局部变量。需要暴露数据时必须通过 slot props，就像本页的 item 和 total。
  • 作用域插槽和传组件 props 有什么区别？ 作用域插槽更适合组件库提供渲染扩展点，例如表格单元格、列表项、卡片区域。传组件 props 更适合固定子组件类型和清晰输入。项目里如果调用方需要完全控制一块模板，作用域插槽通常比塞很多配置项更清楚。
  • 插槽 API 的常见坑是什么？ 结论是命名、默认内容和 slot props 契约要稳定。原因是插槽是调用方写模板，字段改名会直接破坏渲染。组件库升级时应避免随意改 slot props，必要时保留兼容或写清迁移说明。
