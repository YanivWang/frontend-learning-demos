类型注解与推断

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • TypeScript 是 JavaScript 的静态类型层：IDE 和 tsc 在编译期发现类型问题，但浏览器执行的仍是擦除类型后的 JavaScript；本页故意展示 add("1", 2) 在 JS 中会拼接。
  • 现代写法是让简单局部变量依赖推断，给函数参数、公共返回值、对象边界和难推断泛型补显式注解；过度注解会增加噪声，缺少边界注解会让 API 变得不稳定。
  • 常用原始类型是 string、number、boolean；另有 null、undefined，以及较少见的 bigint、symbol（Handbook 称 Less Common Primitives）。
  • strictNullChecks 默认关闭时，null 和 undefined 可赋给任意类型；开启后它们只属于各自类型，可空字段须写成 string | null、string | undefined 等联合类型，并在使用前判空。
  • any 会关闭后续类型检查，适合极少数迁移逃生口；unknown 表示“目前不知道”，使用前必须通过 typeof、判空、用户自定义类型守卫等方式收窄。
  • 常见坑是把 TypeScript 当运行时校验：外部接口、表单输入和本地存储读出的数据仍需要运行时验证，否则类型声明只能让调用处“看起来安全”。

面试考点

  • 类型注解和类型推断应该如何取舍？ 结论是局部简单值依赖推断，函数参数、公共 API 和复杂对象边界显式注解。原因是推断能减少重复，边界注解能稳定契约。项目中导出的函数、组件 props、接口返回值通常要写清，临时变量不必处处标注。
  • TypeScript 的类型检查为什么不能改变 JavaScript 运行时？ 结论是 TypeScript 类型在编译后会被擦除。原因是 TS 设计目标是给 JS 加静态检查，而不是改变 JS 语义；所以本页的 add("1", 2) 运行时仍按 JS 拼接。边界是外部数据必须做运行时校验。
  • any 和 unknown 的核心区别是什么？ 结论是 any 放弃检查，unknown 保留检查。原因是 any 可以任意访问属性和调用方法，unknown 必须先收窄。迁移旧代码可短期用 any，新边界更推荐 unknown 加解析逻辑。
  • strictNullChecks 打开后 null 和 undefined 有什么影响？ 结论是它们不再默认可赋给 string、number 等普通类型，须用联合类型显式声明可空。默认关闭时 TS 与 JS 一样宽松；开启后能把空值风险提前到编译期。项目里推荐开启，接口可空字段写成 T | null 或 T | undefined 并在使用前判空。
  • null 和 undefined 在 TypeScript 里有什么区别？ 结论是在类型系统中它们是两个独立类型，语义不同：undefined 多表示缺省/未赋值，null 多表示刻意空值；开启 strictNullChecks 后不能混用。运行时 null == undefined 为 true，=== 为 false。

TypeScript

// —— 显式注解 vs 类型推断 ——
let count: number = 0;
let title = "demo";              // 推断为 string

// —— strictNullChecks：可空须写联合类型 ——
let flag: boolean | null = null;
let missing: string | undefined = undefined;
// strictNullChecks 关闭时下面能通过；开启后须改成 string | null | undefined：
// let onlyString: string = null;
// let onlyString: string = undefined;

// —— 较少见的原始类型 ——
let big: bigint = 100n;
let key: symbol = Symbol("demo");

// —— any / unknown ——
let loose: any = JSON.parse('{"name":"Ada"}');
let input: unknown = "TypeScript";
if (typeof input === "string") {
  input.toUpperCase();           // unknown 必须先收窄
}
// input.toUpperCase();         // ❌ 编译错误：unknown 上不能直接调用方法

// —— 函数参数与返回值注解 ——
function add(a: number, b: number): number {
  return a + b;
}
// add("1", 2);                  // ❌ 编译错误：参数类型不匹配
// count = "x";                  // ❌ 编译错误：不能赋 string 给 number
