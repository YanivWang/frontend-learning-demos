vue-router 路由懒加载

点击 About Lazy 链接，首次进入会延迟约 0.5s 后显示「About 组件模拟异步加载完成」；再次切换可对比 Home 同步加载与 About 异步解析的差异。

知识点要点

  • 路由懒加载把不同页面组件拆成独立异步块，用户访问对应路由时才加载，常用于降低首屏 JavaScript 体积和解析成本。
  • Vue Router 3 的工程写法通常是 component: () => import('./views/About.vue')；构建工具会把动态 import 识别为代码分割点。
  • 当前仓库是可直接打开的 HTML，没有构建步骤，所以示例用返回 Promise 的异步组件工厂和 setTimeout 模拟加载延迟。
  • 懒加载不是“让页面一定更快”：首次进入懒加载路由会多一次 chunk 请求，弱网下应提供 loading、error、timeout 或重试提示，关键首屏路由不一定适合懒加载。
  • keep-alive 缓存已加载组件，懒加载减少尚未访问页面的初始下载量；二者可以配合，但分别解决“重复渲染”和“首屏体积”两个问题。

面试考点

  • 路由懒加载解决什么问题？ 结论是把非首屏页面延后加载，减少初始包体积。原因是大型 SPA 如果一次加载所有页面组件，会拖慢首屏下载、解析和执行。边界是用户首次进入懒加载页面会等待 chunk 下载，不能把所有性能问题都归结为懒加载。
  • Vue Router 3 里怎么写懒加载？ 结论是路由的 component 可以是返回 Promise 的异步组件工厂，工程中常写 () => import(...)。原因是 Vue Router 会在导航解析阶段等待异步组件 resolved。项目里还可用 webpack chunk name 把相关页面打进同一块。
  • 懒加载失败要怎么兜底？ 结论是要给用户 loading、失败提示、超时和重试入口。原因是 chunk 请求可能 404、网络超时或版本发布后缓存不一致。项目里常配合全局错误监控、刷新提示和路由级错误页。
  • 懒加载、预加载、keep-alive 怎么区分？ 结论是懒加载延后下载，预加载提前下载未来可能需要的资源，keep-alive 缓存已创建组件。原因是三者优化阶段不同。边界是移动端弱网下预加载可能浪费流量，应按路由热度和业务优先级取舍。
