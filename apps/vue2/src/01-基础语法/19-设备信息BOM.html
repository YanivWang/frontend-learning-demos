设备信息读取（与 Vue 无强关联，纯 BOM）

点「刷新设备信息」读取 DPR、视口宽度与 screen 对象；对比三者的数值差异； 缩放浏览器窗口后再次刷新观察变化。

知识点要点

  • window.devicePixelRatio 是物理像素与 CSS 像素的比例。高清屏、浏览器缩放和系统缩放都会影响它，Canvas 绘制时常按 DPR 放大画布位图尺寸，再用 CSS 尺寸显示。
  • window.innerWidth 表示布局视口宽度，包含滚动条占用； document.documentElement.clientWidth 表示根元素内容盒宽度，常用于布局计算。 window.clientWidth 不是标准属性。
  • window.screen 返回物理屏幕相关信息，例如 width 、 height 、 availWidth ；它不等于当前浏览器窗口，也不等于页面可用布局空间。
  • 设备信息读取属于 BOM 能力，与 Vue 没有直接关系；在 Vue 项目中也应封装成普通工具函数或 composable，而不是散落在模板逻辑里。
  • 风险边界：设备信息可能因外接屏、缩放、移动端浏览器 UI、隐私防护而变化，只适合做展示适配和性能提示，不适合当作用户身份、权限或安全判断。

面试考点

  • devicePixelRatio 表示什么？ 结论是它表示一个 CSS 像素对应多少物理像素。原因是浏览器用 CSS 像素参与布局，再映射到不同密度的屏幕像素。项目里常见场景是 Canvas 高清绘制、图片资源选择和调试缩放问题，但不要假设 DPR 永远是整数或固定不变。
  • innerWidth、clientWidth、screen.width 怎么区分？ 结论是 innerWidth 看浏览器视口， clientWidth 看文档根元素内容宽度， screen.width 看物理屏幕宽度。原因是它们属于不同坐标和盒模型层级。做响应式布局时优先 CSS 媒体查询或容器查询，JS 只在确实需要读数时使用。
  • 为什么不能用 screen 信息判断页面可用宽度？ 结论是 screen 不等于当前窗口。用户可以缩放、分屏、外接显示器或让浏览器只占屏幕一部分。项目里如果用 screen.width 判断移动端布局，桌面窄窗口和移动浏览器地址栏都会造成误判。
  • 读取设备信息有什么风险边界？ 结论是它只能做体验适配，不能做安全依据。原因是浏览器可能限制、模糊或改变这些值，用户环境也会动态变化。涉及风控、权限或用户识别时，应使用后端认证和明确授权数据，而不是依赖 BOM 指纹。
