变量声明、提升、null vs undefined、constructor

下方为 console.log 同步输出；也可打开 DevTools Console 查看。

知识点要点

  • var 是函数作用域，声明会提升到函数顶部，重复声明后者被忽略；for (var i…) 的 i 在循环外仍可访问。
  • let / const 是块级作用域，不可重复声明；const 必须初始化，但对象/数组内部属性仍可改。
  • 声明提升：在 var 声明前访问得到 undefined；let / const 有暂时性死区，声明前访问抛 ReferenceError（详见 变量进阶）。
  • undefined 表示「已声明未赋值」；null 表示开发者主动赋的「空值」；null == undefined 为 true，=== 为 false。
  • 原始类型 7 种：number、string、boolean、null、undefined、symbol、bigint；引用类型 object；typeof null === 'object' 是历史遗留 bug。
  • 不加声明直接赋值（非严格模式）会在全局对象上创建属性，污染全局命名空间。

面试考点

  • var / let / const 区别？ var 函数作用域、可重复声明、会提升；let/const 块级、不可重复声明；const 绑定不可重新赋值。业务代码优先 let/const。
  • 什么是暂时性死区（TDZ）？ let/const 在词法上也会提升，但在声明语句执行前处于 TDZ，访问会 ReferenceError。var 提升后可直接读到 undefined。
  • null 和 undefined 区别？ undefined 是缺省/未赋值；null 是刻意清空引用。判断空值推荐 value == null（同时覆盖两者）或 value === null || value === undefined。
  • typeof null 为什么是 object？ 早期 JS 用二进制低位 tag 表示类型，null 的 tag 与 object 相同。精准类型检测用 Object.prototype.toString.call（见 typeof）。
