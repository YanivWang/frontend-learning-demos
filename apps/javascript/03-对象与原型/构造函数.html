构造函数最基本用法

知识点要点

  • 构造调用：函数被 new 调用时，this 指向新实例；构造函数内部通常只初始化每个实例自己的数据。
  • 原型方法：多个实例共享的行为应放在 Person.prototype 上，避免每次 new 都创建一份相同函数。
  • 类型判断：instanceof 沿原型链查找 Person.prototype；constructor 只是来自原型的属性，原型被替换后可能不准。
  • class 关系：class 是构造函数和原型模式的语法糖，类方法仍然定义在原型上。
  • 常见坑：漏写 new、在构造函数里定义方法、把引用类型共享到原型上，都会导致难查的状态或内存问题。

面试考点

  • 构造函数和普通函数的区别是什么？ 结论是函数本身没有硬性区别，关键在调用方式；用 new 调用时才会创建实例并绑定 this。项目里通常用首字母大写或 class 表示“需要 new”。
  • 为什么实例方法通常放在 prototype 上？ 因为方法是共享行为，不需要每个实例都保存一份函数对象。放到原型上能节省内存，并让实例通过原型链访问同一份方法。
  • instanceof 和 constructor 判断类型有什么边界？ instanceof 看原型链，适合判断某个实例是否链接到某构造函数原型；constructor 可被改写，不能单独作为可靠依据。跨 realm 场景还要更谨慎。
  • 构造函数与 class 是什么关系？ class 让构造函数、原型方法和继承写法更清晰，但底层仍依赖原型链。维护旧代码时理解构造函数，有助于理解 class 的运行机制。
