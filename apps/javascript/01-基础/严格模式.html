严格模式综合

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • "use strict"; 必须是脚本/函数体第一条有效语句（前面只能有注释）。
  • 禁止未声明变量赋值（防意外全局）；禁止 delete 变量/函数；禁止形参重名。
  • 禁止八进制字面量 010；with 禁用；eval 不创建外层作用域变量。
  • 独立调用函数时 this === undefined；非严格模式为全局对象。
  • ES module 和 class 默认严格模式；现代项目整体开启。

面试考点

  • 严格模式限制？ 未声明变量、delete 变量、重复形参、八进制、with、arguments.callee、静默失败变抛错等。
  • 如何开启？ 文件/函数顶部 "use strict"；或写 ES module（type="module"）。
  • 对 this 的影响？ 非严格：独立调用 fn() 时 this 为 window；严格：undefined。不影响 call/apply 显式绑定。

严格模式通过 "use strict"; 启用， 可声明在整个脚本顶部，或函数内部第一行（局部生效）。 

"use strict"; // 必须是脚本/函数体的第一条语句，前面只能有注释

1. 不允许使用未声明的变量（防止意外创建全局变量）

"use strict";
// testStr = "abc";   // ❌ ReferenceError: testStr is not defined
let testStr = "abc";  // ✅

2. 禁用八进制字面量（避免 010 被误以为是 8）

"use strict";
// var x = 010;       // ❌ SyntaxError
var y = 0x10;         // ✅ 十六进制依旧可用
console.log(y);       // 16

3. 不能 delete 变量 / 函数（只允许 delete 对象属性）

"use strict";
var a = 1;
// delete a;          // ❌ SyntaxError

const person = { name: "fighting!", age: 20 };
delete person.name;   // ✅
console.log(person);  // { age: 20 }

4. 函数形参不允许重名

"use strict";
// function fn(a, a) { return a; } // ❌ SyntaxError

5. 函数内独立的 this 指向 undefined

非严格模式下，独立调用的函数 this 指向全局对象（window）；严格模式下变成 undefined。

"use strict";
function fa() {
  return this;        // undefined
}
console.log(fa());    // undefined（非严格下是 Window）

function fb() {
  return !this;       // !undefined === true
}
console.log(fb());    // true

6. 部分参数对象（arguments）不再随形参联动

严格模式下修改 arguments[0] 不会改变形参 a，反之亦然。

常见踩坑

  • 'use strict' 必须放在第一条语句，否则被当作普通字符串忽略
  • ES6 模块（type="module"）和 class 内部默认就是严格模式
  • 用 IIFE 包裹的 'use strict' 只在该 IIFE 内生效，不会污染外部
