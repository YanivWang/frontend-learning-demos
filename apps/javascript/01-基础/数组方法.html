数组方法与常见面试题

请打开 DevTools Console 查看输出。

知识点要点

  • 不改原数组：map、filter、reduce、some、every、find、findIndex、slice、concat、flat、includes 等（返回新数组或布尔值）。
  • 会改原数组：push、pop、shift、unshift、splice、sort、reverse、fill、copyWithin。
  • sort() 无比较函数时，元素先转字符串再按 UTF-16 码元 排序；数字需 (a,b)=>a-b。
  • includes(NaN) 为 true（SameValueZero）；indexOf(NaN) 为 -1（严格相等 ===）。
  • 去重：Set 按 SameValueZero 可去 NaN；filter+indexOf 依赖 indexOf，找不到 NaN 会误删；对象按 key 用 Map。
  • flat() 默认深度为 1；完全扁平用 flat(Infinity) 或 reduce 递归。

面试考点

  • 哪些方法改原数组？ 结论：push/pop/shift/unshift/splice/sort/reverse/fill/copyWithin 会原地修改。原因是这些 API 设计为就地更新；在 React/Vue 状态或 reducer 中要先复制再调用，或使用 toSorted/toReversed 等现代不可变 API。
  • 数组去重怎么选？ 结论：基础值用 [...new Set(arr)] 最直接，对象数组按 key 用 Map。原因是 Set 按 SameValueZero 比较值；filter+indexOf 内部用 indexOf（严格相等），含 NaN 时去重结果不可靠。
  • map vs forEach？ 结论：map 返回等长新数组，forEach 返回 undefined。原因是 map 用于映射数据，forEach 用于副作用；链式计算用 map/filter/reduce，日志或 DOM 操作用 forEach。
  • [1,2,3].map(parseInt)？ 结论：结果是 [1, NaN, NaN]。原因是 map 传入 (value, index)，parseInt 把 index 当 radix；项目中应写 .map(Number) 或 .map(v => parseInt(v, 10))。
