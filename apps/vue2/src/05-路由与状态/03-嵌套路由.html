嵌套路由 + 默认子路由 + 编程式导航 push / replace

点击 push /user/foo、push profile、replace course 按钮或下方 router-link，观察地址栏与嵌套 router-view 内容；用浏览器后退对比 push（可回退）与 replace（替换当前记录）的差异。

知识点要点

  • 嵌套路由由两部分组成：父路由配置 children，父组件模板内部提供一个 <router-view> 作为子路由出口。
  • 子路由 path 默认相对父路径拼接，例如父路径 /user/:id 下的 profile 会匹配 /user/foo/profile。
  • path: '' 是默认子路由，访问父路径 /user/foo 时也能显示一个明确的子页面，避免父页面下方空白。
  • $router.push 会新增一条历史记录，用户点击后退能回到上一个页面；$router.replace 会替换当前记录，更适合登录后跳转、默认页修正等不希望回退的场景。
  • 真实项目要补未知子路径兜底，例如在子路由末尾配置 404/空状态；否则用户输入不存在的嵌套路径时，父组件可能渲染但内容区为空。

面试考点

  • 嵌套路由为什么必须在父组件里放 router-view？ 结论是 children 只声明匹配关系，真正渲染位置由父组件里的 <router-view> 决定。原因是嵌套路由本质是布局组件内部再放一个出口。项目里常用它实现用户中心、设置页、后台菜单内容区。
  • 子路由 path 什么时候写相对路径，什么时候写绝对路径？ 结论是大多数嵌套页面写相对路径，不以斜杠开头。原因是相对路径会拼接父路径，能体现层级关系。边界是子路由以 / 开头会作为根路径匹配，适合少数需要共享父布局但 URL 不体现层级的场景。
  • 默认子路由 path: '' 解决什么问题？ 结论是它让访问父路径时也能渲染一个默认子页面。原因是没有默认子路由时，父组件中的子出口没有匹配内容。项目里常把概览页、欢迎页、默认 tab 写成 path: ''。
  • push 和 replace 对浏览器历史记录有什么区别？ 结论是 push 入栈，replace 替换当前条目。原因是它们分别对应浏览器 history 的新增和替换语义。比如列表到详情用 push，登录成功后从登录页去首页常用 replace，避免后退回登录页。
  • 嵌套路由页面如何处理未知子路径？ 结论是在 children 末尾配置 catch-all 或 404 子路由。原因是父组件可能已渲染但子出口无匹配时会空白。项目里常用 { path: '*', component: NotFound } 或 redirect 到默认子页。
