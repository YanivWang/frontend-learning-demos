Vue Router 4 核心面试点

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Vue Router 4 是 Vue 3 官方路由库，核心入口是 createRouter({ history, routes })；安装到 app 后，<router-view> 渲染匹配组件，<router-link> 生成可导航链接。
  • createWebHistory() 生成干净 URL，但刷新任意子路径时服务端必须回退到入口 HTML；createWebHashHistory() 把路径放在 # 后面，静态托管和 file:// demo 更稳。
  • 动态路由参数放在 /users/:id，页面组件应监听同一组件复用时的参数变化；鉴权、标题、权限码等跨页面信息通常放在 route.meta。
  • 导航守卫可以返回路径、路由对象、false 或 Promise；Vue Router 4 仍兼容 next，但项目里应避免混用返回值和 next，否则容易重复放行或卡住导航。
  • 路由懒加载用 () => import('./views/Home.vue') 让打包器拆分页面代码；真实项目还要考虑加载失败、权限重定向循环和未知路由 fallback。

面试考点

  • createWebHistory 和 createWebHashHistory 怎么选？ 结论是正式 SPA 更常用 createWebHistory，静态 demo 或无法配置服务器回退时选 hash。原因是 history 模式 URL 更自然，但刷新 /users/1 需要服务端返回入口页；hash 模式不会把 # 后路径发给服务器，边界是 SEO、分享 URL 形态和旧系统托管能力。
  • beforeEach 守卫里为什么推荐 return？ 结论是 Vue Router 4 推荐通过返回值控制导航，比如返回 "/login" 重定向或 false 取消。原因是返回值语义清晰且能配合 async/await；只有兼容旧写法时才用第三个参数 next，边界是不要在同一个守卫里既 return 又多次 next()。
  • 路由 meta 鉴权的边界在哪里？ 结论是 meta.requiresAuth 适合做前端体验层拦截，不等于安全授权。原因是前端路由和状态都可被用户篡改；项目里必须由后端接口再次校验权限，前端只负责减少误入、保存跳转来源和展示无权限页。
  • 路由懒加载如何影响首屏？ 结论是页面级懒加载能降低首屏 bundle，但会把首次进入页面的成本转为异步加载。原因是动态 import() 会生成独立 chunk；边界是核心首屏不要过度拆分，异步组件加载失败要有重试、错误提示或兜底页。
