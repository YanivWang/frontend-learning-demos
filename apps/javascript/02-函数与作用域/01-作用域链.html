作用域链与变量查找

请打开 DevTools Console 查看输出。

知识点要点

  • 函数执行时会创建活动对象（AO）/ 词法环境，并与外层环境引用串联成作用域链。
  • 标识符查找沿作用域链由内向外：当前函数 → 外层函数 → … → 全局；找不到则 ReferenceError。
  • 内层函数可访问外层与全局变量（闭包的基础）；外层无法访问内层局部变量。
  • JS 采用词法作用域：作用域在写代码时确定，与调用位置无关（对比动态作用域）。
  • 在 DevTools 中对 debugger 断点查看 Scope 面板，可直观看到当前链上的各层绑定。

面试考点

  • 什么是作用域链？变量查找顺序是什么？ 执行函数时沿词法环境外层引用向外查找；先当前环境记录，再外层，直到全局。与原型链不同：作用域链管标识符，原型链管属性访问。
  • 词法作用域和动态作用域的区别？ JS 是词法作用域，函数写在哪儿就继承哪儿的外层；动态作用域（如部分 Bash）按调用栈决定，JS 没有。
  • 闭包与作用域链的关系？ 内层函数引用外层变量时，外层已返回，内层仍通过作用域链上的环境记录保持访问权，即闭包。常考 for 循环 + var/let，见 let for 循环闭包。
