Vuex Module 演示（按模块拆分 store）

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Vuex 使用单一状态树，但大型应用把所有 state、mutation、action 放在一个文件会变臃肿；module 允许按用户、商品、计数器等业务域拆分。
  • 模块 state 挂在根 state 的模块名下面，示例中是 store.state.moduleA.count 和 store.state.moduleB.name。
  • namespaced: true 会让 mutation、action、getter 带上模块路径，例如 commit('moduleA/increment')，避免不同模块都叫 increment 时冲突。
  • 模块 state 推荐写成函数返回对象，尤其是模块会被复用或动态注册时，可以避免多个实例共享同一个 state 对象。
  • mapState('moduleA', ...)、mapGetters('moduleA', ...)、mapActions('moduleA', ...) 能把命名空间访问收敛到组件 computed / methods 中。

面试考点

  • Vuex 为什么需要 module？ 结论是 module 用来把大型 store 按业务域拆分。原因是 Vuex 仍是单一状态树，但代码组织可以分模块维护。项目里用户、权限、购物车、订单通常各自一个模块。
  • namespaced: true 有什么影响？ 结论是 mutation、action、getter 访问要带模块路径。原因是命名空间把局部类型注册到 moduleName/type 下。边界是模块内部 action 默认拿到局部 context，访问根状态要用 rootState 或根路径选项。
  • module state 为什么建议写函数？ 结论是为了每次创建模块状态时返回新对象。原因是模块对象可能被复用，直接写对象会共享引用。项目里动态注册模块或多实例模块尤其要注意这个边界。
  • 模块化 store 有哪些坑？ 结论是命名空间路径、根模块访问和辅助函数参数最容易写错。原因是 state 访问路径、commit/dispatch 类型、getter key 三者规则不同。项目里应约定模块名常量或封装 action 名，减少字符串散落。
