React 元素与渲染（React 18 createRoot）

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • JSX 是 JavaScript 的语法扩展，构建时通常会被 Babel 转成 React.createElement(type, props, ...children)。浏览器不能直接理解 JSX，本 demo 用 babel.min.js 做运行时转译只适合学习演示。
  • React.createElement 返回的是 React element：一个描述 UI 的普通不可变对象，而不是真实 DOM 节点。ReactDOM 根据这些描述创建、更新和删除浏览器 DOM。
  • React 18 客户端入口使用 ReactDOM.createRoot(container).render(element)。旧的 ReactDOM.render 属于 React 17 兼容入口，不会启用 React 18 的新根特性。
  • 自定义组件名必须大写，例如 <App />。小写 JSX 会被当作内置 HTML 标签名；本 demo 注释中的 <ele1 /> 就是反例。
  • JSX 花括号中写的是 JavaScript 表达式，引用变量前必须确保它已经定义；未定义变量会抛 ReferenceError，导致当前渲染失败。

面试考点

  • JSX、React.createElement 和 React element 是什么关系？ 结论是 JSX 通常被转译为 React.createElement 调用，调用结果是 React element。原因是 React 用 element 描述 UI，再由渲染器负责落到具体平台；浏览器 DOM 只是 ReactDOM 的目标之一。
  • React 18 为什么使用 createRoot 而不是 ReactDOM.render？ 结论是 createRoot 是 React 18 新客户端根入口。原因是它启用 React 18 的并发能力和自动批处理等新行为；维护旧代码时仍可能见到 ReactDOM.render，但新代码不应继续使用。
  • React element 和真实 DOM 节点有什么区别？ React element 是不可变描述对象，真实 DOM 是浏览器节点。原因是 React 需要先比较描述，再最小化更新 DOM；项目里不要把 element 当成 DOM 调用 appendChild。
  • 自定义组件为什么必须以大写字母开头？ 结论是 React 用大小写区分自定义组件和内置标签。原因是小写 <div> 会被当作字符串标签名，大写 <App> 才会引用 JavaScript 变量。
