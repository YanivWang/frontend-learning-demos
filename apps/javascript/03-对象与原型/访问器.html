访问器属性 get/set

请打开 DevTools Console 查看输出。

知识点要点

  • 访问器属性：get 在读取属性时执行，set 在赋值时执行；通过函数计算或同步后备字段，不直接保存 value。
  • 描述符互斥：属性描述符要么是数据描述符（value / writable），要么是访问器描述符（get / set），不能混在同一个属性上。
  • defineProperty：Object.defineProperty(obj, key, descriptor) 可动态添加访问器，并控制 enumerable、configurable 等属性行为。
  • 递归陷阱：在 get name() 内部返回 this.name 会再次触发 getter，导致栈溢出；应使用 _name、私有字段 #name 或闭包变量作为后备存储。
  • 默认值：defineProperty 新增属性时 enumerable、configurable 默认均为 false，可能导致 Object.keys 遍历不到。
  • 项目场景：访问器适合派生属性、输入归一化、兼容旧字段或轻量响应式；复杂拦截多个属性时通常用 Proxy 更合适（见 05-元编程/proxy.html）。

面试考点

  • 数据属性和访问器属性有什么区别？ 数据属性直接保存值，访问器属性通过 get/set 函数响应读取和赋值。访问器没有自己的 value；项目里常用它做派生字段、校验或兼容旧 API。
  • Object.defineProperty 定义访问器时描述符要注意什么？ get/set 不能和 value/writable 同时出现。还要明确 enumerable 和 configurable，否则默认都是 false，可能导致遍历和后续重定义行为不符合预期。
  • getter/setter 为什么可能无限递归？ getter 读取同名属性，或 setter 给同名属性赋值，就会再次触发自己。需要独立后备字段，例如 _name，或使用私有字段 #name。
  • 访问器属性适合哪些项目场景？ 适合少量字段的派生计算、懒加载、输入规范化和兼容层。若要观察大量属性增删改，Proxy 能覆盖更多操作，访问器则需要逐个字段定义。
  • enumerable/configurable 默认值对访问器有何影响？ 通过 defineProperty 新增时默认不可枚举、不可配置；Object.keys 看不到，且后续无法再次 defineProperty 修改描述符（除非最初 configurable: true）。
