use() 读取 Promise 与 Context

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • use(usable) 是 React 19 的读取 API，可在渲染阶段读取 Promise 或 Context；读取 Promise 时组件会挂起。
  • 挂起需要外层 Suspense 提供 fallback，否则用户会看不到明确加载态；本页点击“加载 Promise”时会先显示挂起文案。
  • Promise reject 不应通过包住 use(promise) 的 try/catch 处理，而是交给 Error Boundary；本页失败按钮演示错误边界兜底。
  • use() 可在条件分支或循环中调用，这是它和 useState、useEffect 等传统 Hooks 的重要区别，但调用它的组件仍必须是组件或 Hook。
  • 读取 Context 时可写 use(ThemeContext)，语义接近 useContext；普通客户端交互请求仍不应为了追新而全部改成 use(Promise)。

面试考点

  • use(Promise) 为什么需要 Suspense？ 结论是读取未完成 Promise 会让组件挂起。原因是 React 需要暂停当前子树，等 Promise 完成后再继续渲染。项目里必须给用户 fallback，并避免每次渲染都新建 Promise 造成反复挂起。
  • Promise 失败怎么处理？ 结论是 reject 会抛给 Error Boundary，而不是在 use(promise) 周围写 try/catch。原因是挂起和错误都由 React 渲染流程接管。边界是错误边界要提供重试或重新创建 Promise 的入口。
  • use() 和 useContext 有什么关系？ 结论是读取 Context 时 use(Context) 类似 useContext(Context)。原因是两者都会订阅最近的 Provider 值。差异是 use() 还支持 Promise，并允许在条件分支中调用。
  • use() 能否替代 useEffect 请求？ 结论是不能简单替代。原因是 use(Promise) 适合和 Suspense、缓存、服务端组件或框架数据层配合；事件驱动请求、订阅和命令式副作用仍常放在事件处理或 effect 中。
