Vue2 源码阅读前置：Symbol 与源码标记

点「比较两个 Symbol」观察下方日志区与控制台：两次 Symbol() 互不相等； 点「查看枚举行为」对比 Object.keys 与 Object.getOwnPropertySymbols 的返回差异。

知识点要点

  • Symbol() 每次都会创建唯一值，即使描述字符串相同也不相等，常用于避免对象属性名与业务字符串发生冲突。
  • Symbol 属性默认不会出现在 for...in 、 Object.keys() 和 JSON 序列化结果中，但仍可通过 Object.getOwnPropertySymbols() 或 Reflect.ownKeys() 读取，所以它不是安全意义上的私有字段。
  • 阅读 Vue2 源码时，先把语言前置和框架机制分开：Symbol、数组方法、闭包、原型链属于 JavaScript 能力； Observer 、 Dep 、 Watcher 才是 Vue2 响应式链路。
  • 同目录 vue.js 是官方 2.x 运行时源码副本，按第三方运行时文件处理；本页只演示前置概念，实际响应式行为应结合 Vue2 官方响应式文档和源码阅读。
  • Vue2 已进入历史版本语境，新项目通常优先 Vue3；但维护老项目、排查响应式限制或阅读第三方库兼容代码时，仍需要理解 Vue2 源码中的能力检测与内部标记写法。

面试考点

  • Symbol 为什么适合作为源码内部标记？ 结论是它能生成唯一属性键，降低与用户字段或普通字符串常量冲突的概率。原因是每次调用 Symbol() 都返回新的唯一值，并且 Symbol 键不会被常规枚举 API 遍历到。边界是 Symbol 不是安全私有，调试工具和反射 API 仍能看到它。
  • 阅读 Vue2 源码时如何区分语言能力和框架机制？ 先结论：语言能力是源码的表达工具，框架机制才是 Vue 行为本身。比如 Symbol、闭包、原型链帮助组织代码，而 Vue2 的响应式核心仍是基于 Object.defineProperty 的依赖收集和派发更新。项目排障时不要把语言语法误判为 Vue 的响应式限制。
  • Symbol 属性为什么不等于真正私有字段？ 结论是它只能降低命名冲突和常规枚举暴露，不提供访问控制。原因是 Object.getOwnPropertySymbols() 与 Reflect.ownKeys() 可以取到 Symbol 键。需要强私有语义时，现代 JavaScript 更常用类私有字段或模块闭包。
  • 维护 Vue2 项目时如何说明 Vue2 与 Vue3 的选型边界？ 先说明 Vue2 已是历史版本，新增项目通常优先 Vue3。继续维护 Vue2 时，源码阅读重点放在响应式限制、渲染更新和兼容性行为上；如果要迁移，需要特别核对 Vue2 与 Vue3 在响应式实现、生命周期和全局 API 上的差异。
