Composition API 生命周期钩子

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • setup() 会在组件创建阶段执行，早于挂载，Composition API 中 beforeCreate / created 的大多数初始化逻辑都应直接写在 setup 顶层。
  • onBeforeMount / onMounted 对应挂载前后。需要访问真实 DOM、初始化依赖 DOM 的第三方库时，应放在 onMounted 之后。
  • onBeforeUpdate / onUpdated 对应响应式更新引发的 DOM patch 前后。读更新后的 DOM 可放在 onUpdated，但不要在其中无条件修改状态，否则可能造成更新循环。
  • onBeforeUnmount / onUnmounted 用于组件销毁前后清理资源，比如事件监听、定时器、订阅、外部实例和未完成任务。
  • 生命周期注册函数必须在 setup 同步阶段调用，这样 Vue 才能把钩子绑定到当前组件实例；异步回调里临时注册通常不是可靠模式。

面试考点

  • Vue 3 中 created 逻辑放哪里？ 结论是大多直接放在 setup 顶层。原因是 setup 已经覆盖创建阶段初始化能力，并且可以创建响应式状态和注册副作用。边界是依赖 DOM 的逻辑仍要等到 onMounted。
  • onMounted、onUpdated、onUnmounted 怎么分工？ 结论是 mounted 做 DOM 就绪后的初始化，updated 做更新后 DOM 读取或同步，unmounted 做资源释放。原因是它们分别对应渲染生命周期的不同阶段。项目里第三方图表初始化和销毁常成对写在 mounted / unmounted。
  • 为什么钩子要同步注册？ 结论是为了绑定当前组件实例。原因是 Vue 在 setup 同步执行期间维护当前实例上下文，钩子注册依赖这个上下文。异步之后再注册，可能没有可关联的实例。
  • 卸载时要清理什么？ 结论是清理 Vue 无法自动接管的外部资源。包括 addEventListener、setInterval、WebSocket、第三方实例和手动订阅。否则隐藏或路由切换后回调仍可能执行。
