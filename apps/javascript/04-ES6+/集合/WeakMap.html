WeakMap 弱引用映射

请打开 DevTools Console 查看输出。

知识点要点

  • WeakMap 是键值映射，但键必须是可被垃圾回收的对象或非注册 Symbol；它不会因为键存在于 WeakMap 中就阻止键对象被回收。
  • WeakMap 不提供 keys、values、entries、forEach 或 size，因为可枚举性会暴露垃圾回收时机并引入不确定性。
  • 典型场景是把元数据挂到已有对象外部：类私有数据、DOM 节点状态、对象输入的缓存结果，都能随键对象生命周期自动清理。
  • 如果业务需要列出所有键、统计数量、导出调试快照或序列化，应使用 Map，不要使用 WeakMap。
  • 弱引用不等于立刻回收，也不能预测回收时间；代码不应依赖“多久后键值对消失”。

面试考点

  • WeakMap 为什么不能枚举键？ 因为键是弱引用，某个键是否仍存在取决于垃圾回收状态；如果允许枚举，就等于让程序观察 GC 时机。规范因此不提供 keys、entries、size 等 API。
  • WeakMap 和 Map 的核心区别是什么？ Map 强引用键、可枚举、有 size；WeakMap 不强引用键、不可枚举、键类型受限。需要缓存对象派生数据且不想造成内存泄漏时选 WeakMap，需要完整遍历时选 Map。
  • WeakMap 如何用于私有数据或 DOM 元数据？ 把实例或 DOM 节点作为键，把内部状态作为值；外部没有 WeakMap 引用就拿不到数据，节点或实例释放后元数据也可回收。项目里常用于不污染对象本身的附加状态。
  • 弱引用是否意味着可以预测 GC 时间？ 不能。弱引用只表示不阻止回收，具体何时回收由引擎决定。业务逻辑不能依赖某个 WeakMap 条目在固定时间内消失。
