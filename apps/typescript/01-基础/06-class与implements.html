class、implements 与 abstract

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • implements 只在编译期检查 class 的实例侧是否满足接口形状，不会把接口方法注入到类中，也不会生成运行时代码。
  • public、private、protected 是 TypeScript 访问控制；编译后是否保留限制取决于目标语法，而 JavaScript #private 字段是运行时私有，本页运行示例使用 #users 演示真实私有字段。
  • abstract class 可以放公共实现和抽象成员，不能直接实例化；子类必须实现抽象方法，适合有共享逻辑的类层级。
  • interface 只描述形状，abstract class 提供可继承实现，普通 class 同时产生类型和值；是否需要运行时代码是选型关键。
  • 常见坑是以为 implements 会检查静态成员或影响运行时。它默认检查实例侧；构造函数和静态属性需要单独建模。

面试考点

  • implements 会不会改变 class 的运行时行为？ 结论是不会，它只做编译期形状检查。原因是接口和 implements 都会在输出 JS 时被擦除。项目中它能保证服务类实现约定方法，但不能替代运行时依赖注入或接口校验。
  • TS private 和 JavaScript #private 有什么区别？ 结论是 TS private 主要是类型检查语义，#private 是 JavaScript 运行时私有字段。原因是类型会被擦除，而 #users 会成为语言级私有槽。需要运行时强私有时优先用 #private，只需类型边界时 TS 修饰符足够。
  • abstract class 和 interface 应该如何选？ 结论是只要形状用 interface，需要共享实现和模板方法时用 abstract class。原因是 interface 没有运行时代码，抽象类会生成可继承的 JS 类。项目中仓储契约可用接口，带通用日志、缓存或校验流程的基类可用抽象类。
  • implements 检查的是实例侧还是静态侧？ 结论是默认检查实例侧。原因是 class 类型有实例侧和静态侧两部分，implements 面向 new 出来的对象。若要约束构造函数或静态方法，需要单独定义构造签名或工厂函数类型。

TypeScript

// —— interface 形状 + abstract class ——
interface Repository<T> {
  findById(id: number): T | undefined;
}
abstract class BaseService {
  protected prefix = "service";
  abstract label(): string;
}

// —— implements + 访问修饰符 ——
class UserService extends BaseService implements Repository<User> {
  private users: User[] = [];  // TS private；运行时强私有用 #users
  findById(id: number) { return this.users.find((u) => u.id === id); }
  label() { return this.prefix + ":user"; }
}
