generator 基础：yield 与 next

请打开 DevTools Console 查看输出。

知识点要点

  • function* 定义 generator 函数；调用它只会返回一个 Generator 对象，函数体会停在开头，直到第一次 next() 才开始执行。
  • 每次 next() 都把函数推进到下一个 yield、return 或函数结束位置，并返回迭代器结果对象 { value, done }。
  • yield 会把右侧值交给调用方并暂停执行；下一次 next(value) 传入的值会成为当前暂停的 yield 表达式结果。
  • return 会结束 generator，返回 { value, done: true }；之后继续 next() 只会得到 { value: undefined, done: true }。
  • 可迭代对象必须实现 Symbol.iterator 并返回带 next() 的迭代器；数组、Map、Set、字符串内置可迭代，普通对象默认不可 for...of。
  • generator 仍适合惰性序列、自定义迭代和可暂停流程；现代异步控制流通常优先用 async/await，可读性和错误处理更直接。

面试考点

  • generator 函数调用后为什么不会立即执行函数体？ 因为调用 function* 返回的是 Generator 对象，执行权由调用方通过 next() 驱动。这样函数可以在多个 yield 点暂停和恢复，适合惰性产生数据或表达可暂停流程。
  • yield、return 和 next() 的结果如何理解？ yield 产生一个中间值，done 为 false；return 或函数结束表示迭代完成，done 为 true。项目里消费迭代器时不能只看 value，还要看 done。
  • 普通对象如何支持 for...of？ 给对象实现 [Symbol.iterator]()，返回包含 next() 的迭代器即可。边界是对象属性枚举用 Object.keys 或 for...in，不是所有对象都天然可迭代。
  • generator 和 async/await 如何取舍？ generator 适合自定义迭代和惰性序列；异步请求、串行流程和错误处理优先使用 async/await。历史上 generator 可配合执行器模拟异步流程，但现代项目很少需要手写。
