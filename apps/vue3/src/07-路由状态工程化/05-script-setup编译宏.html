script setup 编译宏与工程约定

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • <script setup> 是 SFC 编译期语法糖，会被编译为组件的 setup()；顶层变量、函数和导入可直接在模板中使用，不需要手动 return。
  • defineProps、defineEmits、defineExpose、withDefaults 都是编译宏，无需导入，且必须在 <script setup> 顶层使用；它们不是浏览器全局函数。
  • defineModel 在 Vue 3.4+ 可声明组件的 v-model 绑定，默认对应 modelValue 与 update:modelValue，也支持命名 model 和修饰符；本 demo 的子组件用运行时 props/emits 展示等价行为。
  • TypeScript props 默认值可用 withDefaults(defineProps<Props>(), defaults)；Vue 3.5+ 还支持响应式 props 解构并可直接写默认值，低版本需要注意解构会丢响应式语义。
  • 常见坑是把宏写进条件分支、函数内部或普通 <script>，以及在非 SFC 的 CDN HTML 中直接调用宏；真实工程必须交给 Vue SFC 编译器处理。

面试考点

  • script setup 和普通 setup 有什么差异？ 结论是运行语义仍是组件 setup，但 <script setup> 在编译期减少样板代码并自动暴露顶层绑定给模板。原因是 SFC 编译器会改写这段代码；边界是它只存在于 .vue SFC 编译流程，不能在普通 HTML script 中直接运行。
  • defineProps/defineEmits 为什么不能随便调用？ 结论是它们是编译宏，不是运行时 API。原因是编译器需要静态分析 props、emits 和类型声明，所以要求位于顶层且不能依赖运行时条件；项目里如果需要动态逻辑，应在声明后用普通 JS 处理。
  • defineModel 适合替代哪些样板代码？ 结论是 Vue 3.4+ 中可用它替代手写 modelValue prop 与 update:modelValue emit。原因是组件 v-model 的协议固定，编译宏能生成等价声明；边界是低于 3.4 的项目仍要手写 props/emits，复杂校验仍需清楚声明模型名称、类型和默认值。
  • withDefaults 与响应式 props 解构怎么说明版本边界？ 结论是 Vue 3.4 及更早版本的 TS props 默认值常用 withDefaults，Vue 3.5+ 的响应式 props 解构可直接写默认值。原因是 3.5 编译器会把同一 <script setup> 内的解构访问改写为 props.xxx；边界是跨版本教程和库代码要标明最低 Vue 版本。
