盒模型：content-box vs border-box

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出实际测量值。

知识点要点

  • MDN 定义盒模型由四层组成： content edge → padding edge → border edge → margin edge 。其中 margin 在最外层，不参与 box-sizing 对 width / height 的计算。
  • box-sizing: content-box 是初始默认值， width / height 只表示内容区尺寸，最终 border box 宽度还要加上左右 padding 和 border 。
  • box-sizing: border-box 让 width / height 包含内容区、内边距和边框，但不包含 margin ；本例同样写 width: 180px 时，border-box 的 border box 宽度仍是 180px，内容区会被压缩。
  • MDN 建议在布局中常用 border-box ；工程里常见 *, *::before, *::after { box-sizing: border-box; } 全局重置。需要定位偏移相对内容区时，仍可能保留 content-box 。
  • 垂直 margin 折叠只发生在普通文档流的块级盒之间，MDN 列出三种基本情形： 相邻兄弟 、 父子（无分隔内容） 、 空块 ；浮动、绝对定位元素的 margin 不折叠； display: flex 或 grid 容器内 margin 不折叠。
  • 阻止父子 margin 折叠可给父元素加 border-top/bottom 、 padding-top/bottom 、内联内容，或给父元素设 height / min-height （针对底部折叠）；也可创建 BFC，本例用 display: flow-root 。

面试考点

  • content-box 和 border-box 的尺寸计算有什么区别？ 结论是 content-box 的 width 只算内容区， border-box 的 width 包含 content、padding 和 border。原因是 box-sizing 改变浏览器计算 border box 的方式。项目里卡片、网格和输入框多用 border-box 防止 padding 把元素撑破容器，但 margin 始终在盒子外部，两种模式都不纳入 margin。
  • 为什么现代项目常把全局 box-sizing 设为 border-box？ 结论是它让“元素占多宽”更接近代码里写的宽度。MDN 指出 padding 和 border 被包含在声明宽度内，响应式列布局不必反复手算。边界是第三方组件或需要按内容区定位的场景可能依赖默认 content-box ，全局重置时要覆盖伪元素并留意兼容样式。
  • margin 折叠发生在哪些场景，如何阻止？ 结论是垂直 margin 折叠主要发生在相邻兄弟、父子边界和空块中，且仅垂直方向。MDN 对父子折叠的例外包括 border、padding、inline content、 height / min-height ，以及在父上创建 BFC。项目中若父背景包不住子元素上边距，可加 padding / border ，或用 display: flow-root 。
  • BFC 在盒模型和 margin 折叠问题里有什么作用？ 结论是 BFC 是独立的块级布局区域，可隔离内部布局。MDN 写明新 BFC 会 suppress margin collapsing，并 contain internal floats。项目里优先用 display: flow-root ；老代码可能用 overflow: hidden ，但可能带来 scrollbars 或 clipped shadows 等副作用。
