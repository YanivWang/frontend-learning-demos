类的字段、方法与原型对象关系

请打开 DevTools Console 查看输出。

知识点要点

  • class 是更严格、更清晰的原型语法：constructor 负责实例初始化，类体中的实例方法定义在 ClassName.prototype 上，由所有实例共享。
  • 公有实例字段和私有实例字段会在每次 new 时初始化到当前实例；静态字段和静态方法定义在类构造器本身，实例默认访问不到。
  • 在 constructor 里写 this.sayHello = function () {} 会给每个实例创建一个新的函数对象；除非需要闭包捕获构造参数，否则更推荐写成原型方法。
  • 私有字段 #field 是语法级私有，只能在声明它的类体内部通过 this.#field 访问；_field 只是命名约定，外部仍能读写。
  • 推荐用 Object.getPrototypeOf(instance) 验证原型关系；__proto__ 是历史访问器，不适合作为现代示例的首选写法。
  • 理解字段与原型的存放位置，有助于判断内存占用、方法共享、静态工具方法和实例状态的边界。

面试考点

  • class 方法、实例字段、静态字段分别存在哪里？ 结论是实例方法在 prototype 上共享，实例字段在每个实例上，静态成员在类构造器本身上。原因是 class 仍建立在原型机制之上；项目里要根据“每实例状态”还是“类级能力”选择字段或方法位置。
  • 为什么不建议在 constructor 里重复创建方法？ 因为每次 new 都会创建新的函数对象，实例之间无法共享，实例数量大时会浪费内存。边界是如果方法必须闭包捕获 constructor 的局部变量，才考虑放到实例上。
  • 私有字段和下划线属性有什么区别？ #field 是语言语法级私有，类外访问会直接语法错误；_field 只是团队约定，运行时仍是普通属性。库代码需要隐藏内部状态时，优先使用真正的私有字段或模块闭包。
  • 如何验证实例和类原型的关系？ 可以用 Object.getPrototypeOf(p1) === Person.prototype 验证实例原型，用 Object.getPrototypeOf(Sub.prototype) === Base.prototype 验证继承链。现代代码更推荐 Object.getPrototypeOf，不要依赖 __proto__。
