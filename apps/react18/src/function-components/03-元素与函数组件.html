React 元素、JSX 嵌入表达式与 createRoot

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • 本文件名保留历史命名「元素与函数组件」，但当前示例真正演示的是 JSX 花括号表达式、React element 和 createRoot 渲染流程；函数组件系统讲解从 05 开始。
  • JSX 花括号中写的是 JavaScript 表达式，例如变量、函数调用、算术表达式或 console.log(...) 的调用结果；if、for 这类语句不能直接放在花括号里。
  • {fun} 只是把函数对象作为 child 传给 React，不能作为可渲染文本；{fun()} 才会执行函数并把返回值作为 child。本 demo 通过 {fun()} 渲染字符串。
  • React element 是描述 UI 的不可变对象。createRoot(container).render(element) 会把这棵描述交给 ReactDOM，后续由 React 比较描述差异并提交真实 DOM 更新。
  • React 18 新代码使用 createRoot；旧 ReactDOM.render 属于 legacy 入口，不要作为 React 18 当前推荐写法。

面试考点

  • JSX 花括号里为什么只能放表达式？ 结论是花括号位置需要产生一个值，React 才能把它作为 child 或 attribute。原因是 JSX 会被转译成函数调用参数；项目里复杂分支应先在外部计算变量，再放进 JSX。
  • {fun} 与 {fun()} 有什么区别？ 结论是前者传入函数引用，后者执行函数并传入返回值。原因是 JSX 不会自动调用普通函数；边界是事件处理里恰好需要函数引用，例如 onClick={handleClick}，不要写成立即调用。
  • React element 和真实 DOM 是什么关系？ 结论是 element 是 UI 描述对象，真实 DOM 是 ReactDOM 提交后的浏览器节点。原因是 React 先构建和比较描述，再把必要变更落到 DOM；项目里不要尝试把 React element 当 DOM 节点操作。
  • React 18 的 createRoot 有什么意义？ 结论是它是 React 18 客户端根入口。原因是它启用 React 18 root 行为；旧 ReactDOM.render 只用于维护历史代码，新 demo 和新项目应使用 createRoot。
