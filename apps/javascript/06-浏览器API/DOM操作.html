DOM 基础：选择 / 属性 / 数据集 / 内容

请阅读本复习区，并在下方演示区点击按钮观察 DOM 变化；同时打开 DevTools Console 查看选择器对比输出。

知识点要点

  • getElementById 返回单个元素或 null；getElementsByClassName 返回实时 HTMLCollection，DOM 变化会反映到集合里。
  • querySelectorAll 返回静态 NodeList，它是当次查询快照；如果后续插入新匹配元素，需要重新查询。
  • dataset 是读写 data-* 属性的现代接口，适合把少量声明式配置挂在 HTML 上；getAttribute/setAttribute 更通用，能处理任意属性。
  • classList.add/remove/toggle 以 token 方式操作类名，比拼接 className 字符串更不容易重复、误删或覆盖已有类。
  • innerHTML 会解析 HTML，只有在内容可信或经过严格净化时才应使用；渲染用户输入、日志和错误信息时优先用 textContent。

面试考点

  • HTMLCollection 和 querySelectorAll 返回的 NodeList 有什么差异？ 结论是前者通常是实时集合，后者是静态快照。原因是传统 DOM 集合会随文档变化自动更新，而选择器查询返回的是当次匹配结果。项目里遍历实时集合时要小心边遍历边修改 DOM 导致索引变化。
  • dataset、getAttribute 和直接挂 JS 属性分别适合什么场景？ 结论是 dataset 适合 data-* 配置，getAttribute 适合读取真实 HTML 属性，直接挂 JS 属性只存在于当前 DOM 对象实例上。原因是属性 attribute 和 JS property 是两套相关但不完全相同的模型。需要被 CSS、选择器、序列化或服务端模板感知时应写 attribute。
  • classList 为什么比拼接 className 更可靠？ 结论是 classList 按类名 token 操作，不需要手动处理空格、重复和覆盖。原因是浏览器提供了专门的 DOMTokenList API。项目里做状态类切换、校验高亮或组件开关时优先用 classList.toggle(name, force)。
  • innerHTML 和 textContent 在安全边界上如何选择？ 结论是可信 HTML 模板才用 innerHTML，用户输入和接口文本优先用 textContent。原因是 innerHTML 会把字符串作为 HTML 解析，可能引入 XSS。项目里如果必须渲染富文本，要经过可信的 HTML sanitizer，并避免直接拼接未过滤数据。
