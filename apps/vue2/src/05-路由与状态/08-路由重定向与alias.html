vue-router 路由重定向 与 alias 别名

点击 to a 观察地址栏变为 /b （redirect）；点击 to /我是 对比 to c ：alias 下地址栏保持别名但渲染 C 组件；「当前」行显示 $route.fullPath 。

知识点要点

  • redirect 表示命中某个路径后改去另一个目标路径，地址栏最终显示目标 URL，例如访问 /a 后进入 /b 。
  • alias 表示给同一个路由记录增加另一个访问入口，地址栏保持别名 URL，但渲染的是原路由对应的组件。
  • 旧路径迁移、默认首页跳转、登录后入口修正通常使用 redirect；同一页面需要多个可读 URL 或兼容短链接时可以用 alias。
  • redirect 会触发到目标路由的新导航，权限、异步加载和守卫逻辑要以目标路由为准；alias 则更像同一组件的另一个匹配路径。
  • 真实项目要注意权限和 SEO 边界：不要用 alias 绕过需要鉴权的路径，也要避免多个 URL 长期指向同一内容却没有规范化策略。

面试考点

  • redirect 和 alias 的核心区别是什么？ 结论是 redirect 改变最终 URL，alias 保留访问 URL。原因是 redirect 会发起到目标位置的新导航，alias 是让另一个路径匹配同一个路由记录。项目里旧地址永久迁移更像 redirect，多个入口展示同一页更像 alias。
  • 旧路径迁移应该用 redirect 还是 alias？ 结论是多数迁移用 redirect。原因是用户和搜索引擎最终看到的是新路径，团队也能把旧路径逐步下线。alias 更适合保留业务别名或短路径，但长期多 URL 指向同内容要考虑规范化。
  • alias 为什么地址栏不变但组件会复用原路由？ 结论是 alias 只是额外匹配入口，不会把 location 改成原 path。原因是 Vue Router 在匹配别名时仍使用原路由记录的组件配置。边界是业务中读取 $route.path 时会看到别名路径，要避免把 path 当唯一业务身份。
  • redirect 是否会执行目标路由的守卫？ 结论是会按最终目标导航流程处理。原因是 redirect 产生的是一次新的路由解析，目标路由的匹配、组件和守卫才是最终结果。项目里鉴权和埋点应关注最终到达的路由，而不是只看被 redirect 的入口。
  • 配置 redirect / alias 时有哪些 SEO 和权限边界？ 结论是 redirect 利于 URL 规范化与旧链迁移；alias 多 URL 同内容需考虑 canonical 与鉴权一致性。原因是搜索引擎可能索引多个入口，alias 路径读到的 $route.path 是别名。项目里勿用 alias 绕过需鉴权的路径。
