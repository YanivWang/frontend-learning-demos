原型链与组合继承

请打开 DevTools Console 查看输出。

知识点要点

  • 属性查找： 访问属性时先查自有属性，找不到再沿 [[Prototype]] 指向的对象逐级查找，直到 null （原型链顶端）。
  • 组合继承： Parent.call(this, …) 把父构造函数里的实例属性复制到子实例； Object.create(Parent.prototype) 让子 prototype 链接父 prototype 以共享方法。
  • constructor 修复： 把 Child.prototype 整体替换为新对象后，默认 constructor 会来自父原型，需手动设回 Child 。
  • hasOwnProperty vs in： hasOwnProperty 只查自有； in 沿原型链查找，能发现继承的方法。
  • 静态属性边界： Object.create(Parent.prototype) 只连接 Child.prototype → Parent.prototype ，不会复制 Parent.someStatic 这类构造函数自身属性。
  • 现代写法： class extends 封装了实例原型链接、 super() 和静态继承等细节；理解组合继承有助于解释 class 底层机制。

面试考点

  • 原型链属性查找规则是什么？ 先查对象自身，再沿 [[Prototype]] 一层层向上查。JS 复用行为依赖委托而不是类复制；可用 hasOwnProperty / Object.hasOwn 区分自有与继承属性。
  • 组合继承为什么要同时用 Parent.call 和 Object.create？ Parent.call(this) 解决每个实例自己的状态， Object.create(Parent.prototype) 解决共享方法。只用原型链会共享引用状态，只借构造函数又拿不到父原型方法。
  • 为什么要重置 Child.prototype.constructor？ 替换子原型后， constructor 可能沿链找到 Parent 。重置能恢复调试和约定上的指向，但它不是类型判断的唯一依据。
  • 父类静态属性为什么不会通过原型链继承？ 连接的是 Child.prototype → Parent.prototype ，不是 Child → Parent 。静态属性挂在构造函数对象本身，需显式复制或使用 class extends 的静态继承语义。
  • class extends 和组合继承是什么关系？ class extends 是现代语法封装，处理实例原型、 super() 和静态继承等细节。面试可先讲组合继承机制，再说明现代项目通常直接用 class。
