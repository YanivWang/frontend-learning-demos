mouseenter / leave vs mouseover / out 差异

在 parent 与 son 之间移动鼠标，观察下方日志： mouseenter/leave 各触发一次， mouseover/out 因冒泡可能多次触发；打开 Console 也可查看输出。

知识点要点

  • mouseenter / mouseleave 不冒泡，表示指针进入或离开当前元素这个整体。鼠标从父元素移动到子元素时，不会因为经过子元素边界而再次触发父级 mouseenter 。
  • mouseover / mouseout 会冒泡。鼠标在父子元素之间移动时，子元素的事件会冒泡到父元素，因此父级日志可能比 mouseenter 更频繁。
  • 如果做卡片、菜单、悬浮面板这类“整体 hover”，优先用 mouseenter / mouseleave 或 CSS :hover 。如果需要事件委托到父容器处理很多子项，才更常用 mouseover / mouseout 。
  • relatedTarget 表示鼠标来自哪个元素或要去哪个元素。使用 mouseover / mouseout 做复杂判断时，常用它判断是否仍在同一个容器内，避免误判离开。
  • 鼠标事件只覆盖鼠标输入；现代项目如果要统一鼠标、触控笔等指针输入，可以了解 pointerenter / pointerleave 。本页是 Vue 2 基础目录里的 DOM 事件对比，重点仍是事件冒泡差异。

面试考点

  • mouseenter 和 mouseover 的根本区别是什么？ 结论是 mouseenter 不冒泡， mouseover 会冒泡。原因是鼠标进入子元素时， mouseover 可以从子元素冒泡到父元素，而 mouseenter 只关心是否进入当前元素整体。项目里整体悬停用 mouseenter 更直观。
  • 为什么从父元素移到子元素时 mouseover / mouseout 可能重复触发？ 结论是因为父子元素之间也存在事件目标切换。原因是进入子元素会触发子元素的 mouseover ，离开父元素内部某个目标也可能触发 mouseout 并冒泡。边界判断要看 relatedTarget 是否仍在容器内。
  • 做卡片整体 hover 应该选哪组事件？ 结论是优先 mouseenter / mouseleave 或 CSS :hover 。原因是它们不会因为子元素层级变化而频繁触发。项目里如果只是切换样式，用 CSS 更简单；如果要触发 JS 状态或埋点，再用事件。
  • relatedTarget 能解决什么边界判断？ 结论是用 mouseover/out 时可判断鼠标是否仍在容器内。原因是 relatedTarget 指向关联元素，可区分「进入子元素」与「真正离开容器」。项目里复杂 hover 菜单常用此属性避免误触发 leave 逻辑。
  • 鼠标事件和 pointer 事件在现代项目里如何取舍？ 结论是只关心鼠标时可用 mouse 事件，需要统一鼠标、触控笔等输入时优先了解 pointer 事件。原因是 pointer 事件覆盖更广的输入设备。边界是老项目或 Vue 2 静态 demo 里仍常见 mouse 事件，需要先理解其冒泡语义。
