用正则匹配 URL 中的语言段与路径段

请打开 DevTools Console 查看输出。

知识点要点

  • [course|unit] 是 字符类 ，匹配 c/o/u/r/s/e/|/n/i/t 中任一字符，不是「course 或 unit」。
  • 多选分支用分组： (?:course|unit) 非捕获， (course|unit) 捕获。
  • g 全局匹配时 lastIndex 会递增；同一 RegExp 对象多次 test 可能因 lastIndex 得到不同结果。
  • 单次校验建议不用 g，或每次 new RegExp / lastIndex=0 。

面试考点

  • [] 和 () 在正则里有什么区别？ 结论是 [] 是字符类，只匹配其中一个字符； () 是分组，可配合 | 表示分支。写 [course|unit] 不会匹配单词，只会匹配字符集合，这是 URL 规则里常见错误。
  • 什么是贪婪/非贪婪量词？ 默认量词会尽可能多吃字符，加 ? 后变成尽可能少吃。URL 路径匹配中更推荐限定字符集或明确段边界，避免 .* 把后续段也吞掉。
  • g 标志对 test/exec 有什么影响？ 带 g 的同一个 RegExp 会维护 lastIndex ，下一次匹配从上次结束位置继续。做单次校验时不要加 g ，或每次重置 lastIndex 。
