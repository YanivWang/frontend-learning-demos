KeepAlive

在浏览器中操作下方交互演示；复习要点与面试答法见本复习区。

知识点要点

  • KeepAlive 是 Vue 内置组件，用来缓存被包裹的动态组件实例；本页 Tab 输入框切走再回来，文本仍会保留。
  • 缓存命中时组件不会重新执行普通挂载流程，而是在重新显示和离开缓存树时触发 activated / deactivated，适合恢复定时器、订阅或滚动位置。
  • include、exclude 按组件名筛选缓存对象，max 控制最大缓存数量；大型页面应避免无上限缓存导致内存上涨。
  • 适合缓存 tab、详情页返回、搜索条件、表单草稿等局部 UI 状态；不适合缓存强实时数据、权限敏感页面或必须每次重新拉取的流程。
  • KeepAlive 保留的是组件实例状态，不等于 HTTP 缓存或全局 store；跨路由共享状态见 07-Router与Pinia可运行示例。

面试考点

  • KeepAlive 缓存的到底是什么？ 结论是缓存组件实例及其响应式状态，而不是简单把 DOM 隐藏起来。原因是 Vue 会把离开的组件实例放入缓存，回来时复用实例。边界是外部数据是否重新请求取决于你的生命周期和路由逻辑。
  • activated / deactivated 和 mounted / unmounted 有什么区别？ 缓存组件第一次创建仍会 mounted，后续切入切出触发 activated / deactivated，而不是反复 mounted / unmounted。适合在 activated 恢复订阅，在 deactivated 暂停副作用。真正离开缓存或缓存被淘汰时才会 unmounted。
  • 怎么控制 KeepAlive 缓存范围？ 使用 include、exclude 按组件 name 匹配，用 max 限制数量。原因是缓存越多占用越高，且 stale 状态越难管理。项目里常结合路由 meta 决定哪些页面缓存。
  • 什么时候不应该用 KeepAlive？ 实时行情、权限状态敏感页面、一次性提交流程和必须重新初始化的页面不适合盲目缓存。否则用户可能看到过期数据或残留表单。更稳的做法是缓存 UI 状态，同时在 activated 中按需刷新关键数据。
