BFC 与清除浮动

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出容器高度实测值。

知识点要点

  • BFC （Block Formatting Context）是块级盒子的独立布局上下文，内部块盒按普通流垂直排列，内部浮动不会逃出该上下文的高度计算。
  • 常见触发方式包括 display: flow-root 、非 visible / clip 的 overflow 、浮动元素、绝对/固定定位元素、 inline-block 和部分表格/弹性/网格容器。
  • 现代清浮动优先用 display: flow-root ，语义就是“让这个盒子建立新的块格式化上下文”。
  • overflow: hidden 虽然也能创建 BFC，但它会裁剪溢出内容，遇到阴影、弹层、焦点轮廓或内部滚动时容易变成布局副作用。
  • BFC 还能隔离外部浮动，阻止父元素与子元素之间的垂直 margin 折叠；但相邻兄弟块之间的 margin 折叠不是靠给其中一个元素创建 BFC 就一定消失。

面试考点

  • BFC 为什么能解决浮动父元素高度塌陷？ 结论是父元素创建 BFC 后，内部浮动盒会参与该 BFC 的高度计算。原因是 BFC 是独立格式化上下文，浮动不再只影响外部普通流。项目里应优先写 display: flow-root 。
  • flow-root 和 overflow:hidden 清浮动怎么选？ 结论是现代布局优先 flow-root ，它表达的是布局意图。 overflow:hidden 的原因是副作用触发 BFC，但边界是会裁剪内容。
  • BFC 对 margin 折叠和浮动环绕有什么影响？ 结论是 BFC 可以阻止父子垂直 margin 折叠，并让内容块避开外部浮动。原因是新的格式化上下文隔离了内部布局计算。边界是它不是万能清除器，兄弟元素之间的 margin 折叠和 stacking 问题要分别处理。
  • overflow:hidden 清浮动有什么项目风险？ 结论是能用但不推荐作为默认方案。原因是它把“创建 BFC”和“裁剪溢出”绑定在一起。下拉菜单、阴影、焦点环、绝对定位提示层被截断时，应改用 flow-root 。
