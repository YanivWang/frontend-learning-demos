静态初始化块在继承场景的特性

请打开 DevTools Console 查看输出。

知识点要点

  • 在 class Child extends Parent 中，父类静态字段和 static 块会先完成初始化，子类再按自己的声明顺序初始化静态字段和静态块。
  • 子类静态块里的 this 指向子类构造器，super.xxx 会从父类构造器查找静态属性或静态方法，不会访问父类实例原型。
  • 静态成员定义在类本身上，适合放工厂方法、固定配置、缓存或工具函数；实例只能通过构造器访问，不能直接调用 instance.staticMethod()。
  • static 块中不能调用 super()，因为它不是实例构造流程；派生类实例构造器中的 super() 才负责初始化实例 this。
  • 静态工厂方法可以统一创建实例、校验参数或隐藏构造细节，但真正需要每个实例状态时仍应放到 constructor 或实例字段里。

面试考点

  • 父类和子类的静态初始化顺序是什么？ 结论是父类先、子类后，同一个类内部再按声明顺序执行静态字段和 static 块。原因是子类静态逻辑可能依赖父类静态状态；项目里初始化配置时应避免子类静态块反向依赖尚未执行的子类下方字段。
  • static 块里的 super.xxx 查找什么？ 它查找父类构造器上的静态成员，不查找父类实例方法。边界是实例方法中的 super.method() 走父类原型，静态块中的 super.method() 走父类构造器。
  • 为什么静态方法要通过类名调用？ 因为静态方法挂在构造器对象上，不在实例原型链上。项目中用它表达“与类相关但不依赖某个实例”的能力，例如 fromJSON、create 或缓存读取。
  • 静态工厂方法和 constructor 如何分工？ constructor 负责初始化一个实例的必要状态，静态工厂方法负责命名化创建流程、参数归一化或返回已有缓存。需要异步创建时更适合静态工厂返回 Promise，而不是让 constructor 做异步工作。
