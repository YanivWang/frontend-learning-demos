解构赋值（数组按位、对象按名）

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • 数组解构按照可迭代对象的取值顺序匹配变量，let [a, b] = [1, 2] 中的 a、b 对应第 1、2 个元素。
  • 对象解构按照属性名匹配，顺序不重要；如果本地变量名会冲突，可以写成 const { state: localState } = obj。
  • 默认值只在匹配结果严格等于 undefined 时生效，属性值为 null、0、空字符串时都会保留原值。
  • Vue/Vuex 中常见写法是把 actions(context) 写成 actions({ commit, state, dispatch })，让函数参数直接暴露当前逻辑需要的能力。
  • 注意 Vue 2 响应式对象被解构成普通局部变量后，变量本身不会再自动追踪原属性变化；需要保持响应式读取时，应继续通过对象属性访问。

面试考点

  • 数组解构和对象解构的匹配规则有什么不同？ 结论是数组看位置，对象看属性名。原因是数组解构消费的是迭代顺序，对象解构读取的是同名属性。项目中接口返回对象字段较多时，对象解构更适合取少数字段；元组或固定顺序结果才适合数组解构。
  • 解构默认值为什么遇到 null 不生效？ 结论是默认值只在结果为 undefined 时使用。原因是规范把缺失或值为 undefined 视为需要默认值，null 是一个明确传入的值。接口字段可能返回 null 时，要额外用空值合并或业务判断处理。
  • Vuex actions 为什么常写成 ({ commit, state })？ 结论是为了把上下文对象中真正用到的能力直接声明出来。原因是 action 的第一个参数包含 commit、dispatch、state、getters 等属性，解构能减少重复的 context.。边界是参数过多时可读性会下降，应只解构当前函数需要的部分。
  • 解构会不会保留 Vue2 响应式对象的能力？ 结论是解构出来的普通值不再等同于原对象属性访问。原因是 Vue 2 的响应式依赖收集发生在 getter 读取上，局部变量保存的是当时的值或引用。对象引用内部属性仍可能响应式，基础类型解构后则容易变成快照。
