CommonJS 与 ES Module 对比

请打开 DevTools Console 查看输出。

知识点要点

  • CommonJS 主要用于 Node.js 早期生态，运行时 require，同步加载。
  • ES Module 是语言标准，静态 import/export，支持 tree-shaking。
  • 面试重点是语法、加载时机、值拷贝/实时绑定、浏览器支持差异。

面试考点

  • CommonJS 主要用于 Node.js 早期生态？ CommonJS 主要用于 Node.js 早期生态，运行时 require，同步加载
  • ES Module 是语言标准？ ES Module 是语言标准，静态 import/export，支持 tree-shaking
  • 面试重点是语法、加载时机、值拷贝/实时绑定、浏览器支持差异？ 面试重点是语法、加载时机、值拷贝/实时绑定、浏览器支持差异
维度 CommonJS ES Module
语法 const mod = require("./mod")
module.exports = value		 import mod from "./mod.js"
export default value
加载时机 运行时加载，可以写在条件语句里 静态解析，顶层 import 会先于模块代码建立依赖关系
加载方式 同步加载为主 浏览器中异步加载，天然适合静态分析
导出值 导出对象的值拷贝/引用组合，取决于导出内容 实时绑定，导出变量变化后 import 方能观察到新值
优化 动态性强，静态优化较难 支持 tree-shaking，现代前端默认选择

代码形态

// CommonJS
const math = require("./math");
module.exports = { add };

// ES Module
import { add } from "./math.js";
export { add };
