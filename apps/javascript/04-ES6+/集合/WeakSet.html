WeakSet 弱引用集合

请打开 DevTools Console 查看输出。

知识点要点

  • WeakSet 是弱引用值集合，只能保存可被垃圾回收的对象或非注册 Symbol，不能保存数字、字符串等普通原始值。
  • WeakSet 对成员是弱引用，不会阻止对象被垃圾回收；对象没有其他强引用后，集合中的记录会随之变得可回收。
  • WeakSet 没有 size、没有 keys/values/entries、不能 forEach、for...of 或展开，因为内容受 GC 影响不可稳定观察。
  • 典型场景是临时标记对象是否已处理，例如递归遍历对象图时记录访问过的对象，避免循环引用导致无限递归。
  • 需要枚举成员、统计数量或保存原始值时应使用 Set；只需要“这个对象是否被标记过”且不想阻止回收时才用 WeakSet。

面试考点

  • WeakSet 和 Set 的核心区别是什么？ Set 可保存任意值、可枚举、有 size；WeakSet 只能保存对象类成员、不可枚举、没有 size，且不强引用成员。需要普通集合能力用 Set，需要弱标记对象用 WeakSet。
  • WeakSet 为什么没有 size 和迭代能力？ 因为成员可能在任意 GC 时机消失，暴露 size 或迭代结果会让程序观察垃圾回收状态。规范选择不提供这些 API，保持弱集合语义稳定。
  • WeakSet 如何用于递归遍历防循环？ 每访问一个对象就加入 WeakSet，下次遇到同一引用先用 has 判断并跳过。这样既能阻止循环引用无限递归，也不会长期强引用这些对象。
  • 什么时候应该用 Set 而不是 WeakSet？ 需要存原始值、遍历成员、统计数量、序列化或调试完整列表时都应使用 Set。WeakSet 只适合不需要枚举的对象临时标记。
