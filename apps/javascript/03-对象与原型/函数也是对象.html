函数也是对象

请打开 DevTools Console 查看输出。

知识点要点

  • 可调用对象：函数既有 [[Call]] 能力可以执行，也能像对象一样挂属性；typeof fn 返回 "function" 是运算符特例，但函数仍参与对象原型链。
  • 函数自身 [[Prototype]]：普通函数对象的 [[Prototype]] 指向 Function.prototype，而 Function.prototype 的 [[Prototype]] 又指向 Object.prototype。
  • 两个 prototype 概念：Object.getPrototypeOf(fn) 描述函数对象自己从哪里继承；fn.prototype 是函数作为构造函数时给实例链接的原型对象。两者方向与含义完全不同。
  • instanceof 结果：fn instanceof Function 和 fn instanceof Object 都为 true，因为原型链上分别能找到 Function.prototype 和 Object.prototype。
  • 静态属性：可直接给函数对象挂属性（如 Person.version = "1.0"），对应 class 的静态方法/字段，挂在构造函数本身而非 prototype。
  • 历史访问器：fn.__proto__ 等价于 Object.getPrototypeOf(fn)，仅作理解；项目代码优先标准 API。

面试考点

  • 为什么说函数也是对象？ 函数是一种特殊对象：能被调用，也能拥有属性并沿原型链继承。JS 中函数对象有内部调用能力，同时仍链接到 Function.prototype 和 Object.prototype。
  • fn 的 [[Prototype]] 和 fn.prototype 有什么区别？ [[Prototype]]（经 getPrototypeOf(fn) 读取）是函数对象自己的原型，通常等于 Function.prototype；fn.prototype 是它作为构造函数时给实例使用的原型对象。两者不能混淆。
  • Function instanceof Object 为什么是 true？ 因为 Function 本身也是函数对象，它的 [[Prototype]] 链最终会到达 Object.prototype。这体现了「函数是对象」的统一模型。
  • 理解函数对象对原型链有什么帮助？ 它能解释构造函数为什么能挂静态属性、实例为什么能共享原型方法，以及 class 静态方法和实例方法为什么落在不同对象上。
  • typeof fn 为什么是 function 而不是 object？ typeof 对 callable 对象返回 "function" 是历史设计特例，便于区分可调用与普通对象；但函数仍继承 Object.prototype 上的方法如 toString。
