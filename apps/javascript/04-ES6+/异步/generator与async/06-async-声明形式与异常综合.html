async 函数的多种声明形式与异常综合

请打开 DevTools Console 查看输出。

知识点要点

  • async 可用于函数声明、函数表达式、箭头函数、对象方法和类方法；每次调用都会返回一个 Promise。
  • async 函数内部 return 的值会成为返回 Promise 的 fulfilled value；内部未捕获的 throw 或 await rejection 会让返回 Promise rejected。
  • try...catch 包住 await 可以捕获 await 产生的异常；如果在 await 前对 Promise 先接了 .catch 并吞掉错误，外层 try...catch 就接不到原始 rejection。
  • 局部 catch 适合把某一步失败转为默认值；外层 try...catch 适合让一组步骤共享失败处理。两者混用时要清楚错误是否继续抛出。
  • 调用 async 函数时仍要在外层 await try...catch 或 .catch 兜底，否则未处理 rejection 会进入全局错误处理。

面试考点

  • async 可以声明在哪些函数形态上？ 可以用于函数声明、函数表达式、箭头函数、对象方法和类方法。无论哪种形态，调用结果都是 Promise；区别主要在 this、提升和命名调试信息等普通函数语义上。
  • async 函数内部 throw 和 await rejection 如何传到调用方？ 未捕获的 throw 或 await rejection 会让 async 函数返回的 Promise rejected。调用方可以用 await 外层 try...catch 捕获，也可以用 .catch 捕获。
  • await promise.catch(...) 和 try/catch 包 await 有什么区别？ 前者先在 Promise 链上局部处理错误，如果 catch 返回正常值，外层 await 看到的是 fulfilled；后者统一捕获 await 抛出的异常。项目里局部降级用前者，整体失败流程用后者。
  • 为什么调用 async 函数时外层仍要写 catch？ 因为 async 内部未处理错误不会同步抛给调用栈，而是表现为返回 Promise rejected。没有外层 catch 会形成未处理 rejection，生产环境应统一兜底、提示用户并上报监控。
