Composition API 中的全部生命周期钩子

在浏览器中操作下方交互演示；复习要点与面试答法见本复习区。

知识点要点

  • setup() 承担创建阶段初始化，Composition API 生命周期统一以 onXxx 函数注册，并且要在 setup 同步阶段调用。基础钩子演示见 01-生命周期钩子。
  • 挂载阶段是 onBeforeMount / onMounted，更新阶段是 onBeforeUpdate / onUpdated，卸载阶段是 onBeforeUnmount / onUnmounted。本页根组件 mounted 后日志会显示 elRef 已挂载：true，证明模板 ref 在 mounted 后才可用。
  • 点缓存子组件「+1」会触发 onBeforeUpdate / onUpdated；点「切换缓存页」在 A/B 之间切换时，日志出现 onDeactivated / onActivated 而不会重复 mounted，说明实例被 KeepAlive 保留。
  • <KeepAlive> 缓存的组件被切走时不会立即销毁，而是触发 onDeactivated；再次显示时触发 onActivated。点「显示/隐藏 KeepAlive 区域」把整个 keep-alive 卸载时，才会看到 unmount 钩子。动态组件 + KeepAlive 详见 05-动态组件与异步组件。
  • 本页用 :key="currentTab" 区分 A/B 两个缓存实例；切换 tab 时各自 count 会保留（切回 A 可见之前 +1 的结果）。若去掉 key，A/B 会复用同一实例，行为不同。
  • 常见坑包括把 onUnmounted 写成 onUnMounted、忘记从 Vue 解构钩子、在 onUpdated 中无条件改状态造成循环更新。Vue 3.5+ 可用 useTemplateRef 替代手动 ref 配对，见 01-useTemplateRef（3.5+）。

面试考点

  • Vue 3 生命周期如何按阶段记忆？ 结论是创建逻辑在 setup，之后按挂载、更新、卸载三组 before / after 钩子记忆；KeepAlive 额外有 activated / deactivated。原因是 Composition API 用函数注册替代 Options API 的同名选项。本页按钮操作可对照日志验证各阶段。
  • KeepAlive 为什么触发 activated / deactivated？ 结论是缓存组件切换时实例被保留，不会每次销毁重建。本页切换 A/B 只出现 activated/deactivated，不出现第二次 mounted。项目里计时器、轮询应在 deactivated 暂停，在 activated 恢复。
  • onUpdated 里能改状态吗？ 结论是不应无条件修改会触发本组件更新的状态。原因是 updated 已经在 DOM patch 之后，继续改同一依赖可能形成更新循环。需要响应 DOM 变化时，优先明确触发条件或使用 watch / nextTick。
  • onUnmounted 和 onDeactivated 怎么分工？ 结论是 unmounted 做最终释放，deactivated 做缓存组件临时停用。本页隐藏 KeepAlive 区域才出现 unmounted；仅切换 tab 只触发 deactivated。WebSocket 可在 unmounted 关闭，页面不可见时的轮询可在 deactivated 暂停。
  • 切换 KeepAlive 内组件 key 与切换 tab 名有何差异？ 结论是不同的 key 会创建不同缓存槽位，各自保留状态；同一 key 复用实例。本页 A/B 用不同 key，因此 count 独立保留。若业务需要强制重建实例，应改 key 或不用 KeepAlive。
