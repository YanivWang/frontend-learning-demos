async/await 串行流程最小演示

知识点要点

  • async 函数调用后会立即返回一个 Promise；函数体先同步执行到第一个 await 或结束位置。
  • await 会等待右侧表达式按 Promise 规则决议，并把决议值作为表达式结果；非 Promise 值也会被当成已 fulfilled 的值处理。
  • await 后面的代码会被安排到后续微任务中恢复执行，因此不会阻塞主线程。
  • 多个 await 连续写默认是串行流程，适合后一步依赖前一步结果；互不依赖的任务应先启动，再用 Promise.all 并发等待。
  • async/await 主要改善 Promise 链的可读性，错误处理可用普通 try...catch 包裹 await。

面试考点

  • async 函数返回值为什么总是 Promise？ 结论是 async 函数调用结果永远是 Promise，普通 return 值会成为 fulfilled 值，未捕获异常会变成 rejected。这样调用方可以统一用 then/catch 或 await 消费。
  • await 如何让异步代码看起来像同步串行流程？ await 暂停当前 async 函数，等右侧 Promise fulfilled 后再恢复下一行。原因是语言把后续代码接到 Promise 反应流程里；项目里可用它消除层层嵌套的回调。
  • 多个 await 什么时候会造成不必要的串行等待？ 如果任务互不依赖，逐个 await 会让后一个任务等前一个结束后才启动，浪费时间。应先创建多个 Promise，再用 Promise.all 等待；只有存在数据依赖时才串行。
  • async/await 和 then 链如何互相改写？ await timeout(ms); console.log(value) 等价于 timeout(ms).then(() => console.log(value))。固定业务流程用 async/await 更清晰，动态任务数组可用 Promise 链或 reduce 组织。
