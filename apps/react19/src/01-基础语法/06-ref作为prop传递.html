ref 作为 prop 传递（React 19）

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • React 19 允许函数组件直接接收名为 ref 的 prop，本页 TextField 把父组件传入的 ref 绑定到内部 input。
  • React 18 及更早版本中，函数组件不能直接接收特殊 ref，需要 forwardRef；所以该写法有明确 React 19 版本边界。
  • ref 适合聚焦、选中文本、滚动、测量尺寸、整合第三方 DOM 库等命令式场景；普通表单值仍应优先使用受控或非受控表单模式。
  • 组件库迁移时要看消费者支持的 React 版本：若还支持 React 18，就不能直接移除 forwardRef 入口。
  • 不要把 ref 当成状态管理替代品；用 ref 改 DOM 不会触发 React 重新渲染，容易和声明式 UI 脱节。

面试考点

  • React 19 的 ref as prop 改了什么？ 结论是函数组件可以直接从 props 接收 ref。原因是 React 19 放宽了之前需要 forwardRef 才能透传 ref 的限制。边界是 React 18 项目仍不能用这个写法，必须保留 forwardRef。
  • 什么时候应该用 ref？ 结论是只在需要命令式访问 DOM 或组件实例能力时使用。原因是聚焦、选中、滚动、测量不是纯 props 能表达的状态。本页两个按钮分别演示 focus() 和 select()。
  • 组件库如何迁移到 ref as prop？ 结论是先确认最低 React 版本。原因是直接把 forwardRef 改成 ref prop 会破坏 React 18 消费者。项目里可在主版本升级时切换，或继续保留兼容层直到不再支持 React 18。
  • 为什么不要滥用 ref？ 结论是 ref 不是声明式状态。原因是修改 ref 或直接改 DOM 不会自动驱动 React 渲染，容易产生 UI 与数据不一致。表单值、显示隐藏、列表数据仍应优先由 state / props 管理。
