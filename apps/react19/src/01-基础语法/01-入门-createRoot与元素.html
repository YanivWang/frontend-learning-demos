React 19 元素与 createRoot 渲染

请打开 DevTools Console 查看 element 结构输出；下方 #root 为 React 挂载与交互演示区。

知识点要点

  • JSX 是 JavaScript 的语法扩展。Babel 转译有两种 runtime： classic 生成 React.createElement(type, props, ...children) ；React 19 真实项目要求的 automatic 生成 react/jsx-runtime 中的 jsx() 调用。本 demo 用 babel.min.js 做浏览器运行时转译（classic），只适合学习演示。
  • React.createElement 返回 React element：描述 UI 的不可变普通对象（含 type 、 props 、 key 、 ref ），不是真实 DOM。创建 element 不会立刻渲染组件或创建 DOM 节点。
  • React 19 客户端 CSR 入口为 ReactDOM.createRoot(container).render(element) 。旧版 ReactDOM.render 在 React 18 已废弃， React 19 已移除 ，必须迁移到 createRoot 。
  • 全 React 应用通常只在启动时 createRoot 一次并 root.render(<App />) 一次；之后 UI 更新靠 setState / useState 触发组件 re-render ，组件返回新 JSX，React 经 Trigger → Render → Commit 三阶段更新 DOM， 不需要 每次 state 变化都再调 root.render 。
  • 自定义组件名必须大写，例如 <Greeting /> 。小写 <ele1 /> 等价于 createElement('ele1') ，会渲染为未知 HTML 标签， 不会 抛 ReferenceError。花括号 {ele1} 在变量未定义时才会 ReferenceError 。
  • 挂载时须传入 React element： root.render(<App />) 正确； root.render(App) 传入函数引用会报 Functions are not valid as a React child 。
  • 服务端渲染或静态生成的 HTML 需用 hydrateRoot 接管已有 DOM，而不是 createRoot （后者首次 render 会清空容器内 HTML）。
  • React 19 官方移除了 UMD 构建，本仓库通过本地 React 19 运行时文件教学演示；真实项目应优先使用构建工具或 ESM CDN（如 esm.sh），并与 React 18 demo 区分版本。

面试考点

  • React 元素是什么？ 结论是 React 元素是描述 UI 的不可变对象（含 type 、 props 、 key 、 ref ），不是真实 DOM。原因是 React 用 element 树表达「此刻 UI 应该长什么样」，再由协调（Render）与提交（Commit）决定 DOM 更新。项目里通常用 JSX 创建元素，而不是手写大量 React.createElement 。
  • createRoot 和 render 分别做什么？ 结论是 createRoot 创建并管理 React 根对象， render 把 React element 提交进该根。原因是根对象负责后续调度、协调和卸载（ root.unmount() ）。边界：一个 DOM 容器通常只 createRoot 一次；全 React 应用启动后一般不再重复 root.render ，更新交给 state。
  • state 更新后 UI 如何变化？ 结论是 setState / useState 的 setter 触发组件 re-render，组件根据最新 state / props 返回新 JSX（新 element 描述），React 比较前后结果并最小化 DOM 更新。本页点击「改名」演示的是 state → re-render → 新 props， 不是 再次调用 root.render 。
  • JSX、React.createElement 和 React element 是什么关系？ 结论是 JSX 被 Babel 转译为 createElement （classic）或 jsx() （automatic），调用结果是 React element。React 19 要求现代 JSX transform；本页 classic 写法便于对照学习。
  • React 19 为什么必须使用 createRoot？ 结论是 ReactDOM.render 已在 React 19 移除，客户端挂载只能走 createRoot 。SSR 场景对应 hydrateRoot ，不是 ReactDOM.hydrate （同样已移除）。
  • root.render(<App />) 和 root.render(App) 有什么区别？ 结论是前者传入 React element，后者传入函数引用。原因是 render 需要的是描述 UI 的 element/node，不是组件函数本身。
  • 自定义组件为什么必须以大写字母开头？ 结论是 React 用大小写区分组件与内置标签。小写 <div> 的 type 是字符串 'div' ；大写 <App> 的 type 是组件函数本身。
  • React 19 的浏览器脚本引入有什么版本边界？ 结论是官方不再发布 UMD 构建，真实项目应使用构建工具或 ESM CDN。本仓库为教学保留本地运行时文件，不代表生产项目推荐继续用 script 标签直连 UMD。
