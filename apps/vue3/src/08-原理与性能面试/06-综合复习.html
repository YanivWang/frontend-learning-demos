知识点要点

  • 应用入口： createApp() 创建实例，app.use() 安装插件，mount() 只能调用一次。In-DOM 模板见 01-应用实例与模板语法。
  • 响应式： 基本类型 ref，对象 reactive；解构丢响应用 toRefs；派生 computed，副作用 watch / watchEffect。详见 02-computed-watch-watchEffect。
  • 组件： props / emit、provide/inject、组件 v-model、插槽与动态组件。通信汇总见 04-props与emit。
  • 工程化： Vite + Pinia + Vue Router 4；本仓库 HTML 用 CDN 全局构建，Router/Pinia 可运行见 06-Router与Pinia、08-路由懒加载。
  • 原理与性能： Proxy 响应式、虚拟 DOM diff + 稳定 key、调度器与 nextTick、shallowRef / markRaw 等优化。见本章 01–05 页。

面试考点

  • Vue 3 相对 Vue 2 的核心变化？ 结论是 Composition API、Proxy 响应式、Fragment/Teleport、更小的运行时与更好的 TS 支持。Options API 仍可用。详见 04-Vue2与Vue3差异。
  • ref / reactive / computed / watch 如何选型？ 结论是基本类型 ref，对象 reactive，展示派生 computed，异步/日志/同步外部系统 watch。下方 demo 改关键词可观察 computed 缓存与 watch 日志。
  • 组件通信与全局状态如何分层？ 结论是父子 props/emit，跨层 provide/inject，多页面共享 Pinia；不要把所有状态都塞进 composable 或 props drilling。
  • 列表与性能必答点？ 稳定 key、v-memo / 异步组件 / 路由懒加载、避免模板无副作用方法、大对象用 shallow。见 05-性能优化。
  • HTML demo 与 Vite 工程的边界？ 结论是 CDN demo 验证 API 行为；SFC 编译宏、Vitest、history 路由部署、真实 chunk 拆分需在 Vite 工程验证。本库 07 章概念页已标注边界。
