嵌套函数 - 返回内层函数执行结果

先看本复习区，再查看下方输出是否与标准答法一致；细节可打开 DevTools Console。

知识点要点

  • f 在 checkscope 内部定义，它的外层词法环境包含局部变量 scope = "local" ，所以执行 f() 时优先读取局部 scope 。
  • return f() 返回的是内层函数的执行结果，调用 checkscope() 得到字符串 "local" ，而不是得到函数本身。
  • JavaScript 使用词法作用域：函数能访问哪些外层变量由“定义位置”决定，不会因为未来在哪里调用而改成调用方的局部变量。
  • 本题中 f() 在 checkscope 还没返回前执行，结果已经计算完成；调用结束后，如果没有返回函数或其他引用，局部环境即可被回收。
  • 与下一题 return f 的区别是：返回函数会把 f 连同它引用的外层词法环境一起带出，形成闭包，因此之后再调用也能读到 "local" 。
  • 现代代码中更推荐用 let / const 表达块级作用域；面试老题常用 var ，答题时要说明这是函数作用域语境。

面试考点

  • checkscope() 为什么输出 local 而不是 global ？ 结论：因为 f 的词法作用域优先找到 checkscope 内的局部 scope 。原因是函数定义时就记录了外层词法环境，变量查找从当前函数环境向外层逐级查找。边界是如果局部没有声明 scope ，才会继续向外找到全局变量。
  • return f() 和 return f 有什么本质区别？ 结论： return f() 返回执行结果， return f 返回函数引用。原因是前者在外层函数返回前已经调用内层函数，后者把内层函数带到外部等待之后调用。项目里封装工厂函数、权限校验器、事件回调时，返回函数通常意味着要依赖闭包保存外层状态。
  • 词法作用域为什么不是按调用位置查找变量？ 结论：JavaScript 是词法作用域语言，不是动态作用域语言。原因是函数创建时的外层环境决定了自由变量解析路径，调用位置只影响调用栈和普通函数的 this 。边界是 eval 、 with 等低频特性会让静态分析变复杂，现代项目应避免使用。
  • 闭包什么时候会让外层局部变量继续存活？ 结论：当内层函数被外部引用且仍访问外层变量时，相关词法环境会继续存活。原因是垃圾回收不能释放仍可达的数据。本题返回的是 f() 的字符串结果，不会把函数引用带出；下一题返回 f 才是典型闭包持久化场景。
