class 声明的关键特性

请打开 DevTools Console 查看输出。

知识点要点

  • class 声明用于定义可被 new 调用的构造器模板，语法底层仍基于原型：实例方法放在 ClassName.prototype，静态成员放在类构造器本身。
  • 类声明的名字是词法绑定，声明前处于暂时性死区，不能像函数声明那样提前调用；同一作用域内不能和 let、const、class 等再次声明同名标识符。
  • class 主体默认严格模式，因此方法被解构后独立调用时 this 会是 undefined，不会在非严格模式下自动指向全局对象。
  • 静态字段在类求值时创建在类本身；实例字段在每次 new 时为当前实例初始化。字段声明不是变量声明，不能写 const userAge = 18。
  • 私有字段以 # 开头，必须在类体中预先声明，只能在类内部访问；这和 TypeScript 的 private 类型修饰不同，运行时也有语法级限制。
  • 只有派生类构造器需要并且必须先调用 super() 才能使用 this；没有 extends 的基类构造器中写 super() 是语法错误。

面试考点

  • class 声明和函数构造器有什么区别？ 结论是 class 是更严格的原型构造语法：不能在声明前访问，类体自动严格模式，必须用 new 调用。原因是规范给 class 加了暂时性死区、方法不可枚举等语义；维护老代码时仍能把它理解为原型链上的构造器与方法集合。
  • 静态字段、实例字段、私有字段何时初始化？ 静态字段在类定义被求值时初始化一次，实例字段在每次创建实例时初始化，私有字段也随实例创建但只能在类内部访问。项目中常把工具方法、常量缓存放静态成员，把每个对象自己的状态放实例字段。
  • class 名能不能重新声明或重新赋值？ 同一作用域不能再次声明同名 class，因为它是词法声明，会和 let/const/class 产生声明冲突；但 class 声明创建的是可变绑定，声明后可以重新赋值。项目里不建议这么做，因为会破坏构造器身份、静态成员和原型方法引用的一致性。
  • super() 的边界是什么？ 基类构造器里不能调用 super()；派生类如果写了 constructor，则必须先调用 super() 才能访问 this。这来自派生类实例初始化顺序，实际排查继承报错时先看是否遗漏或提前使用了 this。
