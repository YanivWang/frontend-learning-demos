lds-ring 圆环旋转 loading

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • 这个圆环 loading 由 4 个同尺寸、绝对定位的圆形边框叠放而成，每个圆环播放同一个 lds-ring 旋转关键帧。
  • border-color: #FF6A00 transparent transparent transparent 按上、右、下、左设置颜色，只保留上边框可见；圆角边框旋转后就形成一段弧线。
  • 第 1 到第 3 个圆环使用负 animation-delay，含义是动画立即开始，但从周期中间的进度切入，因此首屏就能看到追逐状态。
  • cubic-bezier(0.5, 0, 0.5, 1) 是缓动函数，两个控制点的 x 坐标必须在 0 到 1 之间；y 坐标可超出 0 到 1，用来制造回弹等效果。
  • 旋转动效通常用 transform: rotate()，但 transform 会创建新的 stacking context；loading 放在复杂遮罩或弹层里时要检查层叠顺序。

面试考点

  • 圆环 loading 为什么能用边框做出来？ 结论是圆形元素只显示一侧边框，再整体旋转，就会看起来像弧线在转。原因是四值 border-color 可以分别控制上、右、下、左边框颜色。项目里这种写法适合轻量 loading，但多色进度环、真实百分比进度更适合 SVG 或 conic-gradient。
  • 负 animation-delay 有什么用？ 结论是它让动画立即播放，但从指定偏移后的进度开始。原因是负延迟相当于动画已经运行了一段时间。项目里多个 spinner 层叠时用负 delay，可以避免首次渲染时所有圆环都从同一角度起步。
  • cubic-bezier() 参数怎么判断合法？ 结论是四个参数代表两个控制点，两个 x 坐标必须落在 [0, 1]。原因是 CSS 缓动函数需要保证输入进度到输出进度可计算。边界是 y 坐标可以小于 0 或大于 1，用来制造超调或回弹，但 loading 场景通常选择平滑曲线。
  • 无限旋转 loading 的风险是什么？ 结论是要控制动效强度并提供 reduced-motion 降级。原因是持续旋转可能造成眩晕或注意力干扰。项目里还要给异步任务设置超时、失败提示和重试入口，不能让 spinner 无限挂起。
