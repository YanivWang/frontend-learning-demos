插槽 slot（默认 / 具名 / 作用域）

观察 my-card 三种插槽：具名 header、默认正文、作用域 item 插槽如何渲染列表项。

知识点要点

  • 插槽用于“组件提供结构，使用方提供内容”。本页 my-card 决定卡片外壳和列表循环，父组件决定 header、默认正文和每个列表项怎么展示。
  • 默认插槽接收没有名称的内容；具名插槽通过子组件的 <slot name="header"> 和父组件的 <template v-slot:header> 对齐。
  • 作用域插槽把子组件内部数据暴露给父模板。本页子组件循环 list，把 item/index 作为 slot props 传出，父组件用解构语法渲染。
  • Vue 2.6+ 推荐 v-slot 统一默认、具名和作用域插槽写法；旧的 slot、slot-scope 在维护项目里还会看到，但新写法更一致。
  • 插槽适合内容形态由调用方决定的组件，如布局、表格列、列表项和弹窗底部操作区；如果只是传普通数据，应优先用 props。

面试考点

  • 默认插槽、具名插槽和作用域插槽怎么区分？ 结论：默认插槽放普通内容，具名插槽放多个命名区域，作用域插槽让子组件把数据交给父组件渲染。原因是它们解决的是内容分发粒度不同的问题。项目里常见于卡片、布局、表格和列表组件。
  • 作用域插槽为什么叫“子给数据，父管渲染”？ 结论：数据来源在子组件，模板决定权在父组件。原因是 slot props 由子组件绑定到 <slot> 上，父组件通过 v-slot 接收。边界是不要把大量业务状态都塞进 slot props，复杂逻辑应抽成组件或状态层。
  • Vue 2.6 后为什么推荐 v-slot？ 结论：它统一了具名插槽和作用域插槽语法。原因是旧的 slot 与 slot-scope 分散在不同位置，阅读成本更高。维护旧项目要能看懂旧写法，新代码建议统一 v-slot。
  • 插槽和 props 怎么选？ 结论：传数据用 props，传模板结构用插槽。原因是 props 适合值，插槽适合可组合 UI。边界是表格列、弹窗 footer 这类“内容由外部决定”的场景，插槽比配置对象更直观。
