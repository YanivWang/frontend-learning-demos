@keyframes 与 animation 旋转动画

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • @keyframes 只定义动画序列本身，元素是否播放取决于 animation-name 或 animation 简写是否引用了同名关键帧。
  • from/to 等价于 0%/100%；未写起点或终点时，浏览器会使用元素当前样式作为缺失端点。
  • animation 简写可同时设置时长、缓动、延迟、次数、方向、填充模式、播放状态和名称；写两个时间值时，第一个是 duration，第二个是 delay。
  • 旋转 loading 这类视觉动效优先动画 transform，通常比动画 width、height、margin 更少触发布局计算。
  • 无限循环动画属于非必要动效时，应考虑 prefers-reduced-motion，给对动效敏感的用户提供暂停、减少或替代体验。

面试考点

  • @keyframes 和 animation 的分工是什么？ 结论是 @keyframes 定义时间线上的样式片段，animation 把这条时间线应用到元素。原因是关键帧规则本身不会自动执行，必须由 animation-name 匹配名称。项目里通常把可复用的关键帧命名清楚，再在不同元素上用不同 duration、delay 或 iteration-count 复用。
  • animation: 2s linear infinite spinnerAni 怎么解析？ 结论是 2s 是单次时长，linear 是等速缓动，infinite 是无限次数，spinnerAni 是关键帧名称。原因是 animation 简写会按可解析的子属性分配值。边界是出现两个时间值时第二个会变成 delay，名称建议放最后，避免和其他关键字混淆。
  • 为什么 loading 旋转常用 transform？ 结论是旋转、位移、缩放优先用 transform。原因是它修改视觉坐标空间，通常不需要重新计算周围元素布局。边界是 transform 会创建新的 stacking context，定位和层叠关系复杂时要额外检查。
  • 无限 CSS 动画有哪些可访问性风险？ 结论是非必要的无限动画要提供降低动效的路径。原因是闪烁、缩放或大幅运动可能影响注意力、前庭敏感或偏头痛用户。项目里可用 @media (prefers-reduced-motion: reduce) 暂停动画或换成更弱的透明度变化。
