parseInt(string, radix) 全面演示

请打开 DevTools Console 查看输出。

知识点要点

  • parseInt(str, radix) 从左解析整数，遇到非法字符停止；忽略前导空白。
  • 务必传 radix（通常 10）；不传时以 0x 开头按 16，某些旧行为对 0 开头按 8（现代浏览器多按 10）。
  • Number(str) 要求整个字符串是合法数字，否则 NaN；与 parseInt 部分解析不同。
  • parseInt('') → NaN；Number('') → 0。
  • map(parseInt) 坑：map 回调是 (value, index)，parseInt 第二参是 radix。

面试考点

  • parseInt 和 Number 有什么区别？ 结论是 parseInt 从左到右读取整数，遇到非法字符就停止；Number 要求整体能转换为数字。边界是空字符串、布尔值、科学计数法和小数在两者下表现不同，业务校验不要混用。
  • [1,2,3].map(parseInt) 为什么出坑？ 因为 map 会传 (value, index)，而 parseInt 第二参正好是 radix。结果等价于 parseInt("1",0)、parseInt("2",1)、parseInt("3",2)；项目里用 map(v => parseInt(v, 10)) 或 map(Number)。
  • radix 不传会怎样？ 现代规范下未传或为 0 时会根据前缀推断，0x 走 16 进制，否则通常按 10 进制。为了可读性和避免历史兼容坑，生产代码仍建议显式传 10 或目标进制。
