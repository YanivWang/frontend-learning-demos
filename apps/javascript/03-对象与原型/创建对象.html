创建对象的多种方式

请打开 DevTools Console 查看输出。

知识点要点

  • 对象字面量优先：{} 是创建普通对象最常见、最清晰的写法；new Object() 行为相近，但在业务代码里通常没有必要。
  • 构造函数场景：需要批量创建同类对象时可用构造函数或 class；实例属性写在构造函数中，方法通常挂到 prototype 以便共享。
  • constructor 边界：constructor 通常来自原型，原型被替换或手动改写后可能不准；类型判断更常用 instanceof、品牌字段或业务判别字段。
  • 属性删除：delete obj.key 会移除自有属性，obj.key = undefined 只是把值变成 undefined，"key" in obj 仍为 true。
  • 现代推荐：数据对象用字面量，带行为的对象用 class 或工厂函数；不要为了“像类”而在构造函数中重复创建大量相同方法。

面试考点

  • 对象字面量、new Object、自定义构造函数如何取舍？ 结论是普通配置/数据用对象字面量，同类实例用构造函数或 class。new Object() 更多是理解语言模型，不是项目首选；需要复用行为时要考虑原型方法共享。
  • constructor 属性为什么不适合作为唯一类型判断？ 因为 constructor 来自原型链，原型对象可以被替换或篡改。面试中可以说它能辅助理解实例来源，但生产判断更依赖 instanceof、Array.isArray 或明确的业务类型字段。
  • delete 和 obj.key = undefined 有什么区别？ delete 移除属性本身，in 和 Object.keys 都会反映变化；赋值 undefined 只是清空值，键仍存在。项目里需要表达“没有这个字段”时用 delete 或构造新对象更清晰。
  • 为什么构造函数中的方法不建议每个实例都创建一份？ 因为函数也是对象，每次 new 都创建新函数会浪费内存，也让方法引用不相等。共享行为应放到 prototype 或 class 方法上，实例只保存各自状态。
