background-blend-mode 与 mix-blend-mode 混合模式

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会读取混合模式实测值。

知识点要点

  • background-blend-mode 作用于同一个元素的多层背景图像，本页用图片层叠加纯色或渐变层，再用 lighten 改变整体色调。
  • mix-blend-mode 作用于元素自身内容与其后方已绘制内容，本页的文字会和深色渐变背景做 difference 或 screen 混合。
  • 混合模式来自图形合成模型，不是简单透明度； multiply 会压暗， screen 会提亮， difference 常用于反色效果。
  • 常见坑是把两者混用：想给一张卡片背景统一调色用 background-blend-mode ，想让文字或图标与下方内容融合才用 mix-blend-mode 。
  • 混合结果依赖底图亮度和颜色，线上需要检查文字对比度；本页图片来自外链，离线或网络失败时只会剩渐变/纯色背景，视觉效果会退化。

面试考点

  • background-blend-mode 和 mix-blend-mode 的作用范围有什么区别？ 结论是前者混合同一元素的背景层，后者混合元素内容和背后的页面内容。原因是它们处在不同的绘制对象上：background 先在元素背景内合成，mix 则参与元素与 backdrop 的合成。项目里给封面图统一加品牌色通常用前者，做反白标题或特殊海报效果才考虑后者。
  • 为什么用渐变层给图片调色，而不是直接改图片资源？ 结论是渐变层 + background-blend-mode 更适合运行时主题化和批量复用。原因是 CSS 可以按主题、暗色模式或状态动态调整，不需要生成多套图片。边界是混合结果依赖原图，品牌色严格一致或无网络外链时仍应准备可靠图片资源或纯色降级。
  • 混合模式做文字反白有什么风险？ 结论是可以做视觉效果，但不能默认保证可读性。原因是 difference 、 screen 的结果取决于后方像素，背景换图后对比度可能突然变差。真实项目里应限制背景范围、提供普通颜色兜底，并在关键文本上做可访问性对比测试。
  • 外链背景图加载失败时页面会怎样？ 结论是混合仍会执行，但没有图片层时效果会退化为渐变或纯色。原因是 CSS 背景列表中失败的图片不会提供像素参与混合。项目里应设置基础背景色、避免把关键内容只放在背景图里，并为离线或弱网场景准备降级视觉。
