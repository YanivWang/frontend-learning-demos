Vue 指令简写形式（: 与 @）

悬停「正确」区块看 :title 动态提示；点「点我更新 msg」刷新； 对比「错误 :bind:title」与静态 title="msg" 的差异。

知识点要点

  • : 是 v-bind: 的简写，所以 :title="msg" 等价于 v-bind:title="msg" 。它把表达式结果绑定到指定 attribute、prop 或 class/style 等特殊处理目标。
  • @ 是 v-on: 的简写，所以 @click="onClick" 等价于 v-on:click="onClick" 。它只负责监听事件，不等于方法调用本身。
  • :bind:title 不是正确简写。因为冒号已经代表 v-bind: ，后面的完整参数会被当成 bind:title 这个 attribute 名，而不是 title 。
  • 静态 title="msg" 是普通字符串，不会读取 data；动态 :title="msg" 才会把 msg 当表达式求值。本页可通过 hover 标题观察差异。
  • Vue 2.6+ 支持动态参数，例如 :[attrName]="msg" 或 @[eventName]="onClick" 。动态参数表达式要保持简单，并注意 DOM 模板里的大小写限制。Vue 2 已 EOL，新项目优先 Vue 3。

面试考点

  • :title 和 v-bind:title 是什么关系？ 结论是完全等价， : 是 v-bind: 的官方简写。原因是 Vue 指令语法允许对高频指令使用更短前缀。项目里推荐统一团队风格，常见代码基本都用简写。
  • @click 和 v-on:click 是什么关系？ 结论是完全等价， @ 是 v-on: 的官方简写。原因是事件监听是模板中高频操作。边界是事件参数、修饰符、动态参数规则不因简写而改变。
  • 为什么 :bind:title 不是正确简写？ 结论是因为冒号已经代表 v-bind: ，后面的 bind:title 会成为要绑定的参数名。原因是 Vue 不会把 :bind:title 再二次解析成 v-bind:title 。项目里看到这种写法应改成 :title 。
  • 动态参数和普通简写有什么区别？ 结论是普通简写的参数名固定，动态参数会先计算方括号里的表达式再决定绑定目标。原因是 :[attrName] 或 @[eventName] 允许运行时选择 attribute 或事件名。边界是动态参数过度使用会降低可读性，应只用于确实动态的封装组件。
  • 维护 Vue 2 项目时如何说明指令简写与 Vue 3 选型边界？ 结论是 Vue 2/3 的 : / @ 简写完全一致。原因是 Vue 2 已 EOL，新项目应优先 Vue 3；存量项目的简写写法无需改动。边界是 Vue 3 新增了 v-memo 等指令，但简写规则不变。
