Canvas 2D 基础

先看本复习区，再观察下方 canvas 中的矩形、路径、圆弧、文字与渐变绘制效果。

知识点要点

  • <canvas> 本身只是一个位图绘制区域，真正绘图要通过 getContext("2d") 拿到 CanvasRenderingContext2D，再调用矩形、路径、文字、渐变等 API。
  • 基础流程是先设置绘图状态，例如 fillStyle、strokeStyle、font，再创建形状或路径，最后调用 fill() 或 stroke() 把当前状态绘制到位图上。
  • 路径是有状态的：moveTo()、lineTo()、arc()、arcTo() 会加入当前路径；绘制互不相关的形状前要调用 beginPath()，否则后续 fill()/stroke() 可能影响旧路径。
  • Canvas 绘制后不保留 DOM 节点语义，浏览器无法像操作 SVG/HTML 那样单独选择某个圆或文字；如果需要事件命中、无障碍文本或结构化导出，要自己维护模型，或改用 SVG/HTML。
  • 常见坑包括忘记处理 getContext() 失败、只改 CSS 尺寸导致画面拉伸、未按 devicePixelRatio 缩放导致高 DPI 屏模糊，以及重新绘制时忘记清屏或恢复绘图状态。

面试考点

  • Canvas 2D 绘图的基本调用顺序是什么？ 结论是获取 canvas、拿 2D 上下文、设置绘图状态、构建路径或形状，最后调用 fill() 或 stroke()。原因是 Canvas 是命令式位图 API，每次绘制都会把当前上下文状态写入像素。项目里通常会把数据模型和绘制函数分开，状态变化后清屏并重绘，而不是期待浏览器帮你维护图形节点。
  • beginPath、fill 和 stroke 分别解决什么问题？ 结论是 beginPath() 开始一条新路径，fill() 填充当前路径内部，stroke() 描边当前路径边界。原因是路径命令会累积在上下文里，不隔离路径会让多个图形被同一次填充或描边影响。边界是 fillRect()/strokeRect() 这类矩形快捷 API 会直接绘制，不需要先手写路径。
  • Canvas 和 SVG 在图表或可视化项目里如何选型？ 结论是大量点、频繁动画、像素处理优先 Canvas，少量结构化图形、需要 CSS/DOM 事件和无障碍语义优先 SVG。原因是 Canvas 绘制结果是位图，重绘成本由你控制；SVG 是保留模式图形，每个图形都是节点。项目里折中方案常见于图表库：主图层用 Canvas 提性能，tooltip、坐标轴或交互层用 HTML/SVG。
  • Canvas 基础绘图有哪些尺寸、清晰度和可访问性边界？ 结论是 Canvas 不应只看页面上显示的 CSS 尺寸。原因是 canvas.width/height 决定绘图缓冲区，CSS 宽高只决定显示尺寸，两者不一致会缩放像素；高 DPI 屏还要考虑 devicePixelRatio。此外 canvas 内文字不是可选择文本，重要信息要提供旁路 DOM 文本或 ARIA 说明。
