Unicode 编码与字符串互转

请打开 DevTools Console 查看输出。

知识点要点

  • JS 内部 UTF-16；基本字符 1 个 code unit；增补平面字符（emoji）占 2 个 code unit（代理对）。
  • charCodeAt 返回 16 位单元；codePointAt 返回完整 Unicode 码点。
  • str.length 是 code unit 数，不是「字符」数；'😀'.length === 2。
  • for...of、展开运算符 [...str] 按码点迭代；传统 for 按索引可能切半代理对。
  • String.fromCodePoint / fromCharCode 由码点/单元建串。

面试考点

  • emoji 长度为什么不是 1？ 结论是 JS 字符串长度统计 UTF-16 code unit，不是用户感知字符。BMP 外字符通常由代理对组成，所以 "😀".length 是 2；项目里统计展示字符要考虑码点甚至 grapheme cluster。
  • charCodeAt 和 codePointAt 区别是什么？ charCodeAt 返回某个 UTF-16 单元，codePointAt 在高代理项位置能返回完整码点。处理 emoji、罕见汉字等增补平面字符时优先用 codePointAt 和 fromCodePoint。
  • 如何正确遍历含 emoji 的字符串？ 使用 for...of、Array.from(str) 或带 u 标志的正则，避免普通索引把代理对切开。边界是国旗、肤色、组合符这类 grapheme cluster 仍可能由多个码点组成。
