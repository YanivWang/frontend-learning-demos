attrs / 属性透传

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Vue 3 把传给组件但未被 props 或 emits 声明接收的内容称为 fallthrough attributes，可通过 $attrs 访问，包括 class、style、普通属性和未声明事件监听器。
  • 单根组件会默认把 $attrs 透传到根元素。示例中的 DefaultButton 没有显式写 v-bind="$attrs"，但 class、data-id 和 @click 仍会落到内部 button。
  • inheritAttrs: false 可以关闭自动透传，让封装组件决定属性落点。示例中的 BaseButton 手动把 $attrs 绑定到内部 button，这是按钮、输入框、表单项等基础组件常用写法。
  • 多根组件没有唯一根元素，Vue 不会自动判断 attrs 应该落到哪里，必须显式 v-bind="$attrs"，否则容易出现运行时警告或属性没有生效。
  • Vue 3 中监听器也属于 attrs 的一部分；声明 emits 可以告诉 Vue 哪些是组件自定义事件，避免它们被误透传到 DOM 根元素。

面试考点

  • 哪些内容会进入 $attrs？ 结论是未被 props 或 emits 消费的属性和监听器会进入 $attrs。原因是 Vue 要把调用方传来的额外信息透传给组件内部 DOM。项目里常见的是 class、style、aria-*、data-* 和原生事件监听。
  • 单根和多根组件透传有什么差异？ 结论是单根组件会自动透传到根元素，多根组件需要显式选择目标。原因是多根组件没有唯一根节点，Vue 无法替你判断属性语义。封装布局组件时要主动把 $attrs 绑定到正确元素。
  • inheritAttrs: false 解决什么问题？ 结论是解决“调用方属性不应该落在组件根元素”这一类封装问题。原因是基础组件常有外层 wrapper，真正的可点击或可输入元素在内部。项目里按钮、输入框、弹窗触发器常用它控制属性落点。
  • emits 和 attrs 有什么关系？ 结论是声明过的自定义事件不会作为 fallthrough listener 继续透传。原因是 Vue 3 需要区分组件事件和原生 DOM 监听。组件库若不声明 emits，可能导致监听器落到根 DOM 上，产生重复触发或语义混乱。
