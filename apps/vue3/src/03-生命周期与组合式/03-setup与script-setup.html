setup / script setup

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • setup() 是 Composition API 的组件入口，创建响应式状态、注册生命周期和返回模板可用数据都在这里完成。返回对象中的字段会暴露给模板。
  • <script setup> 是单文件组件 SFC 的编译时语法糖，不是浏览器原生语法。它会在构建阶段编译成普通 setup()，所以本仓库的全局构建 demo 只能直接运行普通 setup()。
  • <script setup> 中的顶层变量、函数和导入会自动暴露给模板，不需要手写 return，模板也能直接使用。
  • defineProps、defineEmits、defineExpose 等是编译宏，不需要从 vue 导入，也不能当运行时函数在普通浏览器 <script> 中调用。
  • 需要按条件创建组件选项、兼容老项目或直接在无构建环境运行时，普通 setup() 仍然有价值；SFC 新项目通常优先使用 <script setup>。

面试考点

  • setup 和 script setup 是什么关系？ 结论是 script setup 是编译为 setup() 的 SFC 语法糖。原因是它减少样板代码，让顶层绑定自动进入模板。边界是运行时能力仍来自 Composition API，不是另一套组件模型。
  • 为什么普通 HTML 里不能直接写 script setup？ 结论是因为它需要 Vue SFC 编译器处理。浏览器只认识普通 JavaScript，不认识 SFC 编译宏和模板绑定规则。本页因此用全局构建的普通 setup() 演示同一组概念。
  • defineProps / defineEmits 为什么不用导入？ 结论是它们是编译宏，不是普通运行时 API。原因是编译器会静态分析并生成组件 props / emits 选项。项目里不要把它们封装进普通函数，也不要期待运行时动态调用。
  • 什么时候不用 script setup？ 结论是无构建环境、需要显式组件选项组织、迁移中的 Options API 文件可以继续用普通 setup 或 Options API。原因是 script setup 依赖 SFC 编译流程，且更适合现代构建链。
