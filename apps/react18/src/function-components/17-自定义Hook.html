自定义 Hook 抽离复用逻辑

请打开 DevTools Console 查看两个 useToggle 实例的独立 state；下方 #root 为 React 挂载/交互演示区。

知识点要点

  • 自定义 Hook 是以 use 开头的函数，内部可以调用 useState 、 useEffect 等 Hook，并且同样必须在组件或其他 Hook 的顶层调用。
  • 自定义 Hook 复用的是状态逻辑，不是共享同一份状态；示例里每个组件调用 useToggle 都会得到独立的 value 。
  • 把副作用抽进 Hook 时，也要一起封装 cleanup。示例中的 useWindowWidth 在挂载时监听 resize ，卸载时移除监听，避免重复绑定和泄漏。
  • Hook 返回值应围绕调用方需求设计：单值可直接返回，多个值可返回对象或数组；团队里要保持一致，避免调用方猜测顺序或字段语义。
  • 浏览器 API Hook 有运行环境边界。这个 demo 是浏览器静态页，可以直接读 window ；SSR 或测试环境中通常要延后到 effect 或先判断 typeof window 。

面试考点

  • 自定义 Hook 复用的是逻辑还是状态本身？ 结论是复用状态逻辑，不共享状态实例。原因是 Hook 每次调用都会走一套独立的 Hook 状态槽位。
  • 为什么必须以 use 开头？ 结论是为了让 React 和 lint 规则识别这是 Hook。原因是只有识别出来，工具才能检查“只能在顶层调用”和“依赖数组写全”等规则。
  • 抽离窗口 resize Hook 时要处理哪些边界？ 结论是要处理初始值、事件监听清理和非浏览器环境。原因是 resize 是外部系统，绑定后不清理会泄漏。
  • 自定义 Hook、组件和普通函数怎么区分？ 结论是复用 UI 用组件，复用有状态逻辑用 Hook，纯计算用普通函数。原因是组件返回元素，Hook 返回状态和操作。
