NProgress 顶部进度条

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见上方复习区。

知识点要点

  • NProgress 是轻量顶部进度条库，常用于页面跳转、动态导入或全局异步任务；NProgress.start() 显示进度，NProgress.done() 完成并隐藏。
  • 它展示的是感知进度，不是网络真实百分比；如果需要上传/下载百分比，应使用请求进度事件或服务端进度接口，而不是只靠 NProgress。
  • SPA 中通常在路由开始前 start()，在成功、失败、重定向和取消路径里统一 done()，避免异常时顶部条一直挂着。
  • 全局进度条适合“整页正在切换”，局部按钮 loading 或骨架屏适合局部内容更新；不要让一个全局条掩盖具体错误信息。
  • 本页第三方源码位于 lib/，按仓库约定不修改库文件；业务层只封装调用时机、错误处理和可访问性提示。

面试考点

  • NProgress 表示的是真实进度吗？ 结论是不是，它主要表示“任务正在进行”的感知进度。原因是很多路由切换和接口请求没有稳定的总进度，NProgress 只能用递增动画缓解等待感。边界是文件上传、下载或长任务需要真实百分比时，应接入 progress 事件或后端任务状态。
  • SPA 路由里怎么保证进度条不挂死？ 结论是在所有结束路径调用 done()。原因是路由守卫、懒加载组件、接口预取都可能成功、失败、重定向或取消。项目里常在 router beforeEach 里 start，在 afterEach 和 onError/finally 里 done，并处理重复导航。
  • 顶部进度条、骨架屏和局部 loading 怎么选？ 结论是整页切换用顶部进度条，内容占位用骨架屏，按钮提交或局部列表刷新用局部 loading。原因是反馈范围要和任务影响范围一致。边界是全局条不能替代错误提示，失败后必须给出可见状态和重试入口。
  • 第三方进度条库接入有哪些风险？ 结论是要封装调用点，不要把库 API 散落到业务里。原因是异步竞态、重复请求和页面卸载都可能让进度条状态错乱。项目里还要给屏幕阅读器提供文本状态，例如本页用 aria-live 更新任务结果。
