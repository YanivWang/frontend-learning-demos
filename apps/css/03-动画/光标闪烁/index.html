命令行式闪烁光标

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出 animation-timing-function 与 prefers-reduced-motion 状态。

知识点要点

  • steps(1, start) 会把一段动画拆成 1 个离散区间，并在区间开始处跳变；这比 linear 渐变更接近终端光标“亮/灭”切换。
  • 本页用 @keyframes 在 #000 与 transparent 之间切换 background-color，所以光标宽度和布局位置保持稳定。
  • 这只是一个视觉伪光标，不是输入框或 contenteditable 中的真实 insertion caret；真实文本插入光标的颜色应使用 caret-color。
  • opacity 也能做闪烁，但会影响整个光标元素；visibility 是离散切换；background-color 适合只改变竖线填充色。
  • 持续闪烁可能干扰阅读或触发不适，真实产品应在 prefers-reduced-motion 下停用或降低频率。

面试考点

  • steps() 为什么适合做闪烁光标？ 结论是它能让动画按离散步骤跳变，而不是连续插值。原因是终端光标通常只有可见和不可见两个状态。项目里做打字机、精灵图或帧动画时也常用 steps()，但要确认步数和状态数量匹配。
  • 伪光标和 caret-color 有什么区别？ 结论是伪光标是普通元素或伪元素，caret-color 控制的是可编辑文本中的插入光标。原因是 insertion caret 只出现在可输入、可编辑并获得焦点的元素里。项目里展示命令行视觉可用伪元素，真实输入框只改 caret 颜色时用 caret-color。
  • 闪烁动画有哪些可访问性风险？ 结论是要降低频率或提供关闭方式。原因是持续闪烁可能干扰注意力，对部分用户造成不适。项目里可用 @media (prefers-reduced-motion: reduce) 停止动画，并避免大面积高频闪烁。
  • background-color、opacity、visibility 怎么选？ 结论是只想让竖线颜色消失可用 background-color，整块元素淡入淡出用 opacity，完全离散显示隐藏可用 visibility。原因是它们影响的绘制范围和插值方式不同。边界是不要让闪烁影响文本本身的可读性。
