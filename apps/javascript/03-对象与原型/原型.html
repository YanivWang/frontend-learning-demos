原型对象（prototype）与共享属性

请打开 DevTools Console 查看输出。

知识点要点

  • 共享挂载点：每个可构造函数都有 prototype 对象，通过 new 创建的实例会把内部 [[Prototype]] 链到它。
  • 查找机制：访问实例属性时先查自有属性，找不到再沿原型链查 prototype 上的属性或方法。
  • API 选择：__proto__ 是历史访问器，学习时能帮助理解；项目代码推荐用 Object.getPrototypeOf() 和 Object.setPrototypeOf()。
  • constructor：prototype.constructor 默认指回构造函数，但它只是普通属性，替换整个原型对象后需要手动恢复。
  • 共享状态陷阱：原型适合放方法；若把数组、对象这类可变引用放在原型上，所有实例会共享同一份状态，容易互相影响。

面试考点

  • prototype 在构造函数和实例之间起什么作用？ 结论是它是实例共享行为的对象。new Person() 会让实例的 [[Prototype]] 指向 Person.prototype，因此多个实例能共享同一个 sayHello 方法。
  • __proto__、[[Prototype]]、prototype 有什么区别？ [[Prototype]] 是规范里的内部槽，__proto__ 是访问它的历史属性，prototype 是函数对象上用于创建实例原型链的普通属性。项目里优先用标准 API 查看原型。
  • constructor 属性为什么会指回构造函数？ 默认的 prototype 对象自带 constructor，方便从原型回到构造函数。边界是它可被改写，替换原型后不自动保持正确。
  • 原型上为什么不建议放可变引用状态？ 因为原型属性被所有实例共享，一个实例改数组或对象会影响其他实例。项目里实例自己的状态放构造函数，原型上主要放方法或不可变常量。
