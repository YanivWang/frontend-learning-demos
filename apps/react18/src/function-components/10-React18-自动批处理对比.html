知识点要点

  • React 18 createRoot 下，事件、setTimeout、Promise 等中的多次 setState 默认都会批处理。
  • 批处理减少重复渲染，最终 state 仍按更新队列合并。
  • flushSync 可退出批处理，强制同步提交。
  • 对比 class 05：原理一致，演示函数组件在异步回调中的批处理行为。
