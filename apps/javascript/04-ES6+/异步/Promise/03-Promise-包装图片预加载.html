用 Promise 把图片预加载封装成同步风格

请打开 DevTools Console 查看输出；图片加载需网络，失败时页面会显示错误提示。

知识点要点

  • new Image() 会创建一个 HTMLImageElement ，本质上等价于创建 <img> ；资源成功加载会触发 load ，失败会触发 error 。
  • 把 load / error 事件封装成 Promise 后，可以用 then 、 catch 、 await 或 Promise.all 组织预加载流程。
  • 推荐先绑定 onload / onerror ，再设置 img.src ，避免缓存命中、Data URL 或极快响应时事件先于监听注册完成。
  • 失败路径不能只打印日志：404、403、网络断开、图片格式错误都可能触发 error ，应 reject 并在 UI 上显示兜底信息。
  • 如果加载用户可控 URL，不要把 URL 拼进 innerHTML ；本示例使用 DOM API 和 textContent 展示错误，避免把输入当 HTML 执行。

面试考点

  • 如何把 Image 的 load / error 回调封装成 Promise？ 结论是在 Promise executor 里创建 Image ，成功事件 resolve 图片元素，失败事件 reject 错误。原因是图片加载结果天然是一次性异步结果，Promise 能把回调式 API 接入链式或 async/await 流程。
  • 为什么要先绑定事件再给 img.src 赋值？ 因为设置 src 会立即启动资源加载，缓存命中或本地资源可能很快完成。先绑监听再赋值可以避免事件发生时处理器尚未注册；这是封装事件型异步 API 的通用习惯。
  • 图片预加载失败时应该如何处理？ Promise 必须 reject，并且调用方要在 catch 中给出 UI 兜底，例如占位图或错误文案。项目里不要只在控制台打印错误，否则用户会看到空白区域，也不利于监控失败率。
  • 多张图片预加载时如何选 Promise.all 和 allSettled ？ 如果任一图片失败就不能进入下一步，用 Promise.all ；如果允许部分图片失败并展示占位图，用 allSettled 收集每张结果。边界是 all fail-fast，失败后不会给你成功/失败的完整清单。
