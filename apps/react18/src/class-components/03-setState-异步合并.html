setState 异步合并（同步触发多次只会更新一次）

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • setState 是异步的；连续多次同步调用，React 会做「合并优化」最终一次更新。
  • 因此 for(0~9) 内每次都基于「当前 state.count」自增，结果只会得到 1，不是 10。
  • 想要逐次叠加，需要用函数式 setState(prev => ({ count: prev.count + 1 }))。
  • React 18 使用 createRoot 渲染，ReactDOM.render 已废弃。
