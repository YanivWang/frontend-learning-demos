CSS 继承与层叠

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出继承与关键字实测值。

知识点要点

  • CSS 继承指子元素会取得父元素某些属性的计算值，常见可继承属性包括 color、font-*、line-height、text-align、visibility 等；width、height、margin、padding、border 等盒模型属性通常不继承。
  • 本页父容器设置了蓝色文字和 18px 字号，内部未声明 color/font-size 的子元素会自动继承；但子元素不会因为没有写 width 就和父级同宽，因为宽度不是继承属性。
  • inherit 显式让属性继承父级计算值，例如 border-color: inherit 可以让边框颜色和父级文字颜色保持一致。
  • initial 把属性重置为 CSS 规范定义的初始值，不一定等于“看起来像没设置”；例如 color: initial 通常会回到浏览器默认文字色，而不是继续继承父级颜色。
  • unset 对可继承属性等价于 inherit，对不可继承属性等价于 initial；它是“撤销当前元素上的声明，回到继承/初始规则”的通用写法。
  • “层叠（cascade）”和“继承（inheritance）”不是一回事：层叠解决多条规则冲突时选哪条；继承解决子元素没写值时从哪里取值。冲突比较顺序通常是来源 → 重要性 → 层 → specificity → 顺序。

面试考点

  • CSS 哪些属性会继承，哪些不会？ 结论是文字和排版类属性（如 color、font-family、line-height）常继承，盒模型和定位类属性（如 width、margin、position）通常不继承。原因是继承适合表达“同一文本环境”，而尺寸和布局应由每个元素自己决定。项目里表单、按钮、链接常依赖继承统一字体，但宽高仍要单独声明。
  • inherit、initial、unset 分别是什么意思？ 结论是 inherit 强制继承父值，initial 回到规范初始值，unset 在可继承属性上等价 inherit、否则等价 initial。原因是三者解决的是“当前元素这个属性该从哪里取值”。项目里组件 reset 常用 unset 或 revert，而不是把所有属性都写死成 initial。
  • 继承和 CSS 层叠有什么区别？ 结论是继承是“没写值时向上找”，层叠是“写了多条规则时挑一条生效”。原因是它们发生在级联过程的不同阶段。面试里不要混成“优先级继承”；样式冲突应回到 cascade 顺序和 specificity 排查。
  • 为什么父元素改 color 会影响子元素，子元素改 color 不会影响父元素？ 结论是继承是单向从祖先传给后代，不会反向传播。原因是 CSS 继承树只向下传递计算值。项目里如果某个子组件不想继承主题色，应显式声明 color 或使用独立作用域/token。

inherit / initial / unset 对照

inherit强制使用父级计算值，例如 border-color: inherit。
initial使用 CSS 规范初始值，不等于“继承父级”。
unset可继承属性 → inherit；不可继承属性 → initial。
