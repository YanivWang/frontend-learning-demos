:style 三种写法（行内对象 / 对象变量 / 数组）

在 app1 点按钮切换颜色/字号； app3 观察数组合并时后者 fontSize 覆盖前者；打开 Elements 查看内联 style 变化。

知识点要点

  • :style 是 v-bind:style 的简写。Vue 2 对 style 绑定做了增强，表达式可以返回 JavaScript 对象，也可以返回多个样式对象组成的数组。
  • 行内对象语法适合少量动态值，例如 :style="{ color: activeColor, fontSize: fontSize }" 。对象 key 是 CSS 属性名，value 来自响应式数据，数据变了会更新元素的内联样式。
  • 对象变量语法 :style="styleObject" 更适合样式项较多的场景，让模板保持干净。CSS 属性名可以写 fontSize ，也可以写带引号的 'font-size' ；不要写错大小写，也不要在同一个对象里混入重复同义 key。
  • 数组语法 :style="[baseStyle, overrideStyle]" 会把多个样式对象合并到同一个元素上；当同一 CSS 属性重复出现时，后面的对象通常覆盖前面的值，因此适合表达“基础样式 + 覆盖样式”。
  • 项目里优先用 :class 表达可复用的视觉状态，把颜色、间距、主题等交给 CSS； :style 更适合运行时计算出的宽度、高度、坐标、进度、用户配置色等必须内联的值。Vue 2 已 EOL，新项目应优先 Vue 3。

面试考点

  • Vue 2 的 :style 对象语法为什么像 CSS 但本质是 JavaScript 对象？ 结论是模板里的对象会被 Vue 计算成元素的内联样式。原因是 v-bind 表达式先按 JavaScript 求值，再由 Vue 把对象 key/value 写入 DOM style。边界是 kebab-case 属性名必须加引号，否则会被当成非法的 JavaScript 标识符。
  • :style 数组语法合并多个对象时冲突如何处理？ 结论是多个对象会应用到同一个元素，后面的同名样式通常覆盖前面的结果。原因是最终写入的是同一个内联 style 声明，后赋值的属性会成为最终值。项目里可以用它表达基础样式和局部覆盖，但不要滥用成复杂主题系统。
  • 动态样式应该优先用 :class 还是 :style ？ 结论是可枚举、可复用的状态优先用 :class ，运行时连续变化的数值才用 :style 。原因是 class 更利于复用、缓存、媒体查询和维护，而内联样式优先级高且分散。项目里按钮状态、错误态用 class，拖拽坐标、进度条宽度、用户选择色可用 style。
  • camelCase 和 kebab-case 属性名在 :style 中有什么注意点？ 结论是两者都能用，但写法不同： fontSize 是普通对象 key， 'font-size' 必须加引号。原因是模板表达式是 JavaScript 对象字面量，不是 CSS 文件。维护旧代码时看到 'font-Size' 这类大小写混乱写法要修正为标准 CSS 属性名。
  • 维护 Vue 2 动态样式时如何说明版本边界？ 结论是 Vue 2/3 的 :style 对象/数组语法基本一致。原因是 Vue 2 已 EOL，新项目应优先 Vue 3；存量项目的动态样式写法可直接迁移。边界是可枚举状态仍优先 :class ，运行时数值才用 :style 。
