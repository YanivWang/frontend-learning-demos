Suspense

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Suspense 是 Vue 内置组件，用来等待默认插槽中的异步依赖；等待期间渲染 #fallback，依赖完成后再显示默认内容。
  • 本页 AsyncPanel 使用 async setup() 并等待 800ms，因此首次渲染会先显示“加载中...”，再显示异步内容；点“重新加载”可再次观察 fallback。
  • async setup、异步组件都可能成为 Suspense 依赖；多个依赖在同一个 Suspense 边界内时，fallback 会覆盖整个边界。组件级异步见 05-动态组件与异步组件。
  • Vue 官方仍将 Suspense 标记为实验性特性，API 未来可能调整；业务项目应确认当前 Vue 版本、SSR 框架支持和团队约定。
  • Suspense 主要处理 pending，不负责吞掉错误；加载失败应通过组件自身错误状态、defineAsyncComponent 的 errorComponent，或 onErrorCaptured / errorCaptured 处理。

面试考点

  • Suspense 解决的 pending 边界是什么？ 结论是把一组异步组件或 async setup 的等待过程收敛到一个 fallback UI。原因是没有 Suspense 时，每个组件要自己管理 loading。边界是 fallback 覆盖的是整个 Suspense 默认插槽，不适合所有局部小 loading 都集中处理。
  • async setup 如何触发 Suspense？ 当组件 setup 返回 Promise 时，父级 Suspense 会等待它 resolve 后再展示默认内容。本页通过定时 Promise 模拟数据准备。项目中如果没有 Suspense 包裹，异步 setup 的渲染体验和框架集成要额外确认。
  • Suspense 能不能当错误边界？ 不能，Suspense 主要处理等待状态，不是 React Error Boundary 那类错误边界。异步失败要由组件捕获、异步组件配置 errorComponent，或通过错误捕获钩子处理。否则用户可能只看到控制台错误或空白区域。
  • Suspense 和组件内部 loading 怎么选？ 需要统一包住一组异步依赖时用 Suspense，单个按钮、局部列表刷新或需要细粒度重试时用组件内部 loading 更灵活。SSR、路由数据预取和实验性 API 风险也会影响选型。
