自定义指令 / 过滤器 / mixin / 插件

输入框自动聚焦（v-focus）；修改 price 看过滤器格式化；点按钮调用 mixin 方法与 $toast 插件；日志区显示 created 顺序与调用结果。

知识点要点

  • 自定义指令适合封装底层 DOM 行为，例如自动聚焦、滚动监听、尺寸测量或第三方 DOM 库接入。钩子包括 bind 、 inserted 、 update 、 componentUpdated 、 unbind ，参数里最常用的是 el 和 binding 。
  • 过滤器用于模板中的轻量文本格式化，例如金额、日期、大小写。本页 currency 只负责展示格式；复杂派生状态应放到 computed，交互动作放到 methods。Vue 3 已移除过滤器。
  • mixin 会把选项合并进组件：生命周期钩子都会执行，data 会合并，同名 methods / components / directives 等对象选项通常由组件自身覆盖。风险是来源隐蔽、命名冲突和调试困难。
  • 插件暴露 install(Vue, options) ，通常用于统一注册全局组件、指令、过滤器、mixin 或 Vue.prototype 方法。本页 ToastPlugin 在原型上挂了 $toast 。
  • 维护 Vue 2 项目时可以识别这些 API 的边界：DOM 行为用指令，展示格式用过滤器，横切复用谨慎用 mixin，全局能力用插件。新项目优先 Vue 3 和组合式 API，避免继续扩大过滤器和 mixin 的使用面。

面试考点

  • 自定义指令适合解决什么问题？ 结论是适合封装需要直接操作 DOM 的低层行为。原因是组件负责数据和视图结构，指令可以拿到真实元素 el 并在插入、更新、解绑等时机处理 DOM。项目里自动聚焦、权限埋点、拖拽和第三方库接入可以考虑指令，但业务组件复用优先用组件。
  • Vue 2 过滤器为什么不适合承载复杂业务逻辑？ 结论是过滤器只适合显示层格式化。原因是它只能在插值和 v-bind 表达式中使用，表达力和可测试性都不适合复杂状态派生。项目里金额展示可以用过滤器，带依赖的计算和复用逻辑应改为 computed、methods 或普通工具函数；Vue 3 已移除过滤器。
  • mixin 的合并规则和风险是什么？ 结论是 mixin 会按 Vue 选项合并策略合并进组件，生命周期钩子都会执行，同名对象选项可能冲突或被组件覆盖。原因是 mixin 本质是把一份组件选项混入另一份组件选项。项目里大量 mixin 会让属性和方法来源不清，维护时要谨慎。
  • 插件 install 通常用来注册哪些全局能力？ 结论是插件用于集中安装全局组件、指令、过滤器、mixin 或原型方法。原因是 Vue.use(plugin) 会调用插件的 install 方法，把全局扩展挂到 Vue 构造器上。边界是全局能力会影响整个应用，命名和副作用要非常克制。
  • 维护 Vue 2 项目时如何说明这些 API 与 Vue 3 选型边界？ 结论是 Vue 3 移除 filters、弱化 mixin，自定义指令与插件仍可用但写法有变。原因是 Vue 2 已 EOL，新项目应优先 Composition API 与 Vite。存量项目识别这些 API 边界，迁移时 filters→computed/methods，mixin→composables。
