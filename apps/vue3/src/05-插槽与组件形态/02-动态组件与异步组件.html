动态组件 / 异步组件

在浏览器中操作下方交互演示；复习要点与面试答法见本复习区。

知识点要点

  • <component :is="current"> 会根据字符串组件名或组件对象渲染当前组件，适合 tab、步骤面板、运行时可配置区域，比多段 v-if 更集中。本页四个按钮切换四种面板组件。
  • 动态组件默认切走会卸载实例；本页包上 <keep-alive> 后，资料面板的 visits 和设置面板的 theme 在 tab 间切换后保留。先在资料面板 +1 再切到设置，切回 visits 不会归零。
  • defineAsyncComponent 接收 loader，把异步加载结果包装成普通组件；真实项目常配合 import() 做代码分割，本页用 Promise + setTimeout(800ms) 模拟 chunk 加载。点「异步面板」先见 loading，再见加载时间。
  • 点「失败面板」会触发 loader reject，展示 errorComponent 而非白屏； asyncStatus 区记录当前异步状态。生产环境还应配置 timeout 、 onError 重试与上报。
  • 动态切换时要确认组件名、缓存策略和 key 是否符合预期；缓存太多占内存，不缓存则丢本地表单/滚动状态。KeepAlive 生命周期见 03-全量钩子与KeepAlive 。
  • 多个异步依赖要统一 loading 边界时可引入 Suspense ；单组件懒加载用 defineAsyncComponent 的 loading/error 选项通常足够。插槽组合见 01-默认具名作用域插槽 。

面试考点

  • component :is 和 v-if 切换组件有什么区别？ 结论是两者都能切换 UI，但 component :is 更适合把「当前组件类型」建模成状态。组件名可来自配置或 tab 列表。简单两三段条件用 v-if 更直观。
  • 为什么 KeepAlive 常配合动态组件？ 因为动态组件切换会触发卸载和重建，本地状态会丢失。本页 visits / theme 在 KeepAlive 下保留即证明。边界是用 include/exclude/max 控制缓存范围。
  • defineAsyncComponent 要处理哪些失败边界？ 结论是加载中、加载失败、超时、重试和重复切换。本页「失败面板」演示 errorComponent；「异步面板」演示 loading → 成功。真实 chunk 可能因网络或版本不一致失败。
  • 异步组件和 Suspense 怎么配合？ 异步组件定义加载器和错误策略，Suspense 协调组件树异步依赖和 fallback。局部懒加载只需 defineAsyncComponent；多异步依赖统一 loading 时再引入 Suspense。
  • 动态切换时 key 有什么影响？ 结论是改 key 会强制销毁重建实例，即使包在 KeepAlive 内也会按新实例处理。需要保留状态时不要随意改 key；需要强制刷新表单时应改 key 或不用 KeepAlive。
