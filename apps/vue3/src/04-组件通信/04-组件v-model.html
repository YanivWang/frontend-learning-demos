组件 v-model

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Vue 3 组件上的默认 v-model 等价于传入 :modelValue="title"，并监听 @update:modelValue；本页 SmartForm 用 @input 触发 update:modelValue，父组件 title 同步更新。
  • v-model:subtitle 会展开为 subtitle prop 和 update:subtitle 事件，因此同一组件可以暴露多个独立模型。改两个输入框时，下方「父组件 title / subtitle」与 pre 区更新日志同步变化。
  • 子组件接收的 prop 是只读输入，不能直接修改；正确做法是声明 emits 并发出 update:xxx，让父组件作为单一数据源决定是否接受新值。props / emit 基础见 01-props与emit。
  • 组件 v-model 只是父子同步协议，不会自动处理输入校验、空字符串、数字转换、去空格或 IME 输入边界；这些应通过组件 props、修饰符或内部校验明确约定。点「父组件重置」可验证单向数据流：父改值后子输入框同步。
  • Vue 3.4+ 可用 defineModel 减少 prop + emit + computed 样板代码，见 05-defineModel（3.4+）。用户输入通过模板插值显示时会被 Vue 转义为文本，v-html 需防 XSS。

面试考点

  • Vue 3 组件 v-model 编译后等价于什么？ 结论是默认等价于 :modelValue="x" 加 @update:modelValue="x = $event"。原因是 Vue 把双向绑定拆成单向数据输入和事件输出，仍保持父组件拥有状态。本页 pre 区展示等价展开写法。
  • 多个 v-model 怎么实现？ 结论是使用带参数写法，例如本页的 v-model:subtitle。它会匹配 subtitle prop 与 update:subtitle 事件。项目里要统一命名，避免 prop 名、事件名和文档不一致。
  • 为什么子组件不能直接修改 modelValue？ 结论是 prop 属于父组件数据源，子组件直接写会破坏单向数据流并触发警告。通过 emit 更新能让父组件参与校验、拒绝或转换数据。
  • 什么时候用 v-model，什么时候用普通 props / emit？ 可编辑表单值、开关状态、选择器值优先用 v-model。一次性动作、保存、删除、加载失败等事件用普通 emit 更清晰。只读展示仍用 props。
  • 表单组件如何处理空值和类型边界？ 结论是明确 prop 类型、空值语义（null vs ''）和 emit 前的转换。v-model 不替你做校验；数字输入应在 setter 或父组件侧 Number() 并处理 NaN。修饰符需组件显式读取 modelModifiers。
