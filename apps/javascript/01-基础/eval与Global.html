eval 与 Global 对象（window）

请打开 DevTools Console 查看输出。

知识点要点

  • 浏览器全局对象是 window；Node 是 global；ES2020 标准 globalThis 跨环境统一。
  • 顶层 var 和 function 声明会成为 window 属性；let/const 不会。
  • eval(str) 在运行时把字符串当 JS 执行，能访问外层词法变量。
  • 严格模式 + 直接 eval 有独立词法环境；indirect eval（如 (0,eval)(s)）在全局执行。
  • 风险：XSS 注入、CSP 禁止、引擎难以 JIT 优化、调试困难。

面试考点

  • 全局对象？ 宿主提供的根对象；浏览器 window。全局This 用 globalThis。模块顶层变量不在 window 上。
  • eval 风险？ 执行不可信字符串 → XSS；性能差；违反 CSP。替代：JSON.parse、Function 构造器（仍慎用）、配置 DSL。
  • eval vs new Function？ eval 共享调用处闭包；Function 只有全局作用域（+自身参数），不闭包外层 let。两者都不推荐在生产使用。
