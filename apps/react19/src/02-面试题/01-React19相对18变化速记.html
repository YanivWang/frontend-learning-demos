React 19 相对 18 变化速记

下方 #root 为对照表与高频追问速记区，无需 Console；对照上方知识点复习后可直接背诵表格内容。

知识点要点

  • React 19 的高频变化可以按四组记：Actions 表单提交流程、乐观 UI、渲染中读取资源、ref 传递简化；另加破坏性变化 ReactDOM.render / hydrate 移除与 UMD 构建移除。
  • useActionState 管理 Action 返回状态和 pending， useFormStatus 读取当前表单提交状态， useOptimistic 负责提交完成前的临时 UI。
  • use() 可读取 Promise 或 Context；读取 Promise 需要 Suspense ，失败交给 Error Boundary，不是把所有客户端请求都替换掉。
  • React 19 允许 ref 作为函数组件 prop，但 React 18 消费者仍依赖 forwardRef ；组件库升级要看最低支持版本。
  • 升级还要关注函数组件 defaultProps 等废弃项、文档元数据和资源预加载 API，以及 RSC 通常通过 Next.js 等框架落地，本仓库 UMD demo 只讲概念边界。

面试考点

  • React 19 相对 React 18 最值得记住的变化有哪些？ 结论是 Actions / useActionState / useOptimistic / use() 增强异步交互与资源读取，ref 可作为 prop，同时 ReactDOM.render 、 hydrate 、官方 UMD 已移除。原因是 React 19 聚焦表单、乐观 UI 和 Suspense 数据读取标准化。项目里要按版本边界逐步迁移。
  • Actions、 useActionState 、 useFormStatus 分别负责什么？ 结论是 Action 是异步提交函数， useActionState 拿提交结果和 pending， useFormStatus 在表单子组件里读取所属表单状态。原因是它们分别对应「做提交」「保存结果」「显示提交状态」。复杂表单仍可能结合路由框架或表单库。
  • use() 、Suspense、Error Boundary 的边界怎么答？ 结论是它能读取 Promise / Context，但读取 Promise 要配合 Suspense 和 Error Boundary。原因是 pending 会挂起，reject 会进入错误边界。边界是事件触发的请求、订阅和副作用仍常用事件处理或 useEffect 。
  • ref as prop 升级会影响哪些组件库代码？ 结论是封装 input、button、modal 等需透传 ref 的组件库可直接用 ref prop，但 React 18 消费者仍依赖 forwardRef 。原因是 ref 在 React 18 是特殊 prop。组件库需保留 forwardRef 兼容层或提高最低 React 版本到 19。
  • React 18 升级 19 前应该检查哪些破坏性变化？ 结论是先跑官方 upgrade guide / codemod，再查依赖兼容。原因是 ReactDOM.render / hydrate 移除、UMD 构建移除、函数组件 defaultProps 、字符串 ref、旧测试工具等都可能影响项目。组件库还要确认是否继续支持 React 18 的 forwardRef 消费方式。
