React 元素、JSX 嵌入表达式与 createRoot

请打开 DevTools Console 查看 JSX 表达式输出；下方 #root 为 React 挂载演示区。

知识点要点

  • 本文件名保留历史命名「元素与函数组件」，但当前示例真正演示的是 JSX 花括号表达式、React element 和 createRoot 渲染流程；函数组件系统讲解从 05 开始。
  • JSX 花括号中写的是 JavaScript 表达式，例如变量、函数调用、算术表达式； if 、 for 这类语句不能直接放在花括号里，应在外部计算后再嵌入。
  • {fun} 只是把函数对象作为 child 传给 React，不能作为可渲染文本； {fun()} 才会执行函数并把返回值作为 child。本 demo 通过 {fun()} 渲染字符串。
  • React element 是描述 UI 的不可变对象（含 type 、 props 等）。 createRoot(container).render(element) 会把描述交给 ReactDOM，经协调（reconciliation）后提交真实 DOM 更新。
  • React 18 新代码使用 createRoot ；旧 ReactDOM.render 已废弃，在 React 18 中会以 React 17 模式运行。

面试考点

  • JSX 花括号里为什么只能放表达式？ 结论是花括号位置需要产生一个值，React 才能把它作为 child 或 attribute。原因是 JSX 会被 Babel 转译为 createElement 的参数；项目里复杂分支应先在外部计算变量，再放进 JSX。
  • {fun} 与 {fun()} 有什么区别？ 结论是前者传入函数引用，后者执行函数并传入返回值。原因是 JSX 不会自动调用普通函数；边界是事件处理里恰好需要函数引用，例如 onClick={handleClick} ，不要写成立即调用。
  • React element 和真实 DOM 是什么关系？ 结论是 element 是 UI 描述对象，真实 DOM 是 ReactDOM 提交后的浏览器节点。原因是 React 先比较前后 element 描述，再把必要变更落到 DOM；不要把 element 当 DOM 节点去操作。
  • React 18 的 createRoot 有什么意义？ 结论是它是 React 18 客户端根入口，替代已废弃的 ReactDOM.render 。原因是它创建 React 18 root 并启用新行为；并发渲染需 opt-in，不是 createRoot 一调用就全面开启。
  • 为什么文件名是「函数组件」但示例重点是 JSX 表达式？ 结论是历史命名保留，本页聚焦 element 与 JSX 嵌入；函数组件（props/state/事件）从 05 起系统讲解。原因是学习路径上需先理解 element 与渲染，再进入组件通信。
