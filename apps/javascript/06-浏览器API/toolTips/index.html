Vue3 实现悬停与键盘可访问的交互浮层

在浏览器中打开本页，用鼠标悬停或 Tab 聚焦触发区，观察浮层显示、延迟隐藏和焦点离开边界。

知识点要点

  • mouseenter / mouseleave 不冒泡，适合把触发区和浮层分别作为进入、离开的边界；如果用 mouseover / mouseout ，子元素切换也会触发，通常要额外判断 relatedTarget 。
  • 可交互浮层不能只依赖 CSS :hover ，因为用户从触发区移动到浮层按钮时会短暂离开触发区；示例用短延迟和清理定时器，减少闪烁和“刚移入又被隐藏”的竞态。
  • 键盘用户无法触发鼠标悬停，因此触发区应是可聚焦控件，并用 focusin / focusout 维护显示状态； focusout 通过 event.relatedTarget 判断焦点是否仍在浮层容器内。
  • 包含按钮、链接等可交互内容时，不应把它当作纯 ARIA tooltip 。纯 tooltip 通常只提供说明文本；本例更接近非模态 popover / dialog，需要在真实项目里继续补充 Escape 关闭、焦点管理和定位防溢出。
  • Vue 这里只负责把事件和响应式状态绑定起来： ref 保存显示状态，模板类名与 ARIA 状态从响应式值派生；不要把 DOM 可见状态和 Vue 状态拆成两套来源。
  • 运行环境： 悬停与键盘焦点在 file:// 与 http(s):// 均可； alert 需用户交互。不涉及需 HTTPS 的 Notification 等 API。

面试考点

  • 可交互 Tooltip 为什么不能只靠 :hover ？ 结论是不能只靠 :hover 。原因是浮层里有按钮时，鼠标从触发区移动到浮层会离开触发元素，CSS hover 容易闪烁；项目里通常用状态机或延迟关闭，并在浮层移入时取消隐藏定时器。
  • mouseenter / mouseleave 与 mouseover / mouseout 怎么选？ 结论是简单边界用前者，事件委托或需要冒泡时才考虑后者。原因是 mouseenter 不冒泡，子元素之间移动不会重复触发；但大量节点逐个绑定会有成本，列表型场景可用冒泡事件加 relatedTarget 判断。
  • 为什么交互浮层要支持键盘焦点？ 结论是必须支持，否则键盘和辅助技术用户无法打开或操作浮层。原因是 hover 只覆盖鼠标路径；本例用可聚焦按钮、 focusin 和 focusout 让 Tab 进入时显示，焦点离开整个容器时再隐藏。
  • ARIA tooltip 与带按钮的 popover 有什么边界？ 结论是带按钮的浮层不应简单标成 role="tooltip" 。原因是 ARIA tooltip 是说明文本，不接收焦点；真实项目中带交互控件的浮层更接近 popover 或非模态 dialog，需要补 Escape 关闭、焦点返回和屏幕阅读器标签。
  • 延迟隐藏定时器有什么坑？ 结论是定时器必须可取消并只保留一个有效任务。原因是鼠标快速进出会产生多个待执行回调；如果不清理，旧回调可能覆盖新状态，导致浮层闪烁或点击按钮前被隐藏。
