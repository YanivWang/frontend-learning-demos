ref / reactive / 解构丢响应

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • ref() 用 .value 保存响应式值，适合基本类型，也可以包装对象；模板里会自动解包，所以本页显示 {{ count }} 而不是 {{ count.value }}。
  • reactive() 返回对象的 Proxy，只能接收对象、数组、Map、Set 等对象类型；它的响应性依赖代理访问，通常应保持同一个代理引用。
  • 直接从 reactive 对象解构属性会得到当前值快照，本页 nameSnapshot 不会随 user.name 更新，因为后续访问不再经过代理的 getter。
  • toRefs(user) 会把每个属性转换成与源对象属性联动的 ref，本页 name 会跟着 user.name 变化，适合在返回对象或解构时保留响应性。
  • 现代 Vue3 项目常用“独立基本状态用 ref、成组对象状态用 reactive、跨函数返回时用 toRef/toRefs 保持连接”的组合。

面试考点

  • ref 和 reactive 如何选型？ 结论是基本类型和需要整体替换的值优先 ref，成组对象状态可用 reactive。原因是 ref 通过 .value 持有值，替换天然清晰；reactive 通过 Proxy 拦截对象属性读写。项目里表单对象常用 reactive，计数、开关、当前 id 常用 ref。
  • reactive 直接解构为什么会丢响应？ 先结论：解构会把属性当前值取出来，后续读写不再经过原代理。原因是 Vue 的依赖收集发生在代理 getter 上，而普通变量没有 getter。需要解构给模板或调用方时，用 toRefs 保留每个属性与源对象的连接。
  • 为什么不要整体替换 reactive 代理对象？ 结论是替换变量会断开已有引用和依赖关系。原因是其他地方持有的是旧代理对象，重新赋值只改变当前局部变量指向。项目中要么修改代理对象的属性，要么一开始用 ref({ ... }) 承载可整体替换的数据。
  • composable 返回对象时如何避免调用方解构丢响应？ 结论是返回多个 ref，或对 reactive 对象使用 toRefs 后返回。原因是调用方很常见会写对象解构。边界是如果希望调用方保持对象整体语义，也可以返回整个 reactive 对象并文档化不要直接解构。
