fetch 基础与 CORS 跨域

请打开 DevTools Console 查看输出。

知识点要点

  • fetch 只有网络错误才会 reject；HTTP 404/500 需要检查 response.ok。
  • 跨域由浏览器同源策略限制，CORS 需要服务端返回 Access-Control-* 响应头。
  • 简单请求直接发出；非简单请求会先发送 OPTIONS 预检请求。
  • mode: 'no-cors' 不是万能跨域，响应会变成不可读的 opaque。

面试考点

  • fetch 错误处理？ 只有网络失败才 reject；404/500 需手动检查 response.ok 或 status。
  • CORS 预检何时触发？ 非简单请求（自定义头、PUT/DELETE、Content-Type 非表单三类型）先发 OPTIONS。
  • 跨域怎么解决？ 服务端返回 Access-Control-Allow-Origin 等头；开发环境可用代理，生产靠后端配置。
fetch 示例依赖网络；如果失败，仍可从代码里学习错误处理与 CORS 规则。
