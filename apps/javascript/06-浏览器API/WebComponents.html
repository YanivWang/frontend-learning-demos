Web Components 自定义元素

知识点要点

  • Web Components 是一组浏览器原生组件化能力，常见组合包括 Custom Elements、Shadow DOM 和 HTML templates；本页重点演示自定义元素和 Shadow Root。
  • customElements.define("user-card", UserCard) 注册自定义标签，名称必须包含连字符，类通常继承 HTMLElement 并使用生命周期回调。
  • attachShadow({ mode: "open" }) 创建 Shadow Root，组件内部样式默认不会泄漏到外部，外部普通选择器也不能直接选中 shadow tree 内部节点。
  • observedAttributes 声明要监听的 HTML 属性，属性变化会触发 attributeChangedCallback ；如果要同步复杂对象，通常需要属性序列化或直接暴露 DOM property。
  • 常见坑是忽略无障碍语义、slot 内容、事件 composed 边界和框架集成差异；跨框架设计系统适合 Web Components，业务强状态组件仍常用 React/Vue。

面试考点

  • Web Components 由哪些标准能力组成？ 结论是它不是单一 API，而是 Custom Elements、Shadow DOM、templates 等浏览器标准的组合。原因是自定义标签负责生命周期，Shadow DOM 负责封装边界，模板负责声明可复用 DOM。项目里通常用它封装跨框架 UI 基础组件。
  • Shadow DOM 的样式隔离和普通组件 CSS 作用域有什么区别？ 结论是 Shadow DOM 是浏览器级封装边界，普通框架 scoped CSS 多是编译或命名约定。原因是 shadow tree 对外部选择器不可见，样式默认不会互相污染。边界场景是主题定制需要用 CSS custom properties、parts 或 slots 显式开放。
  • observedAttributes 为什么只监听属性而不是属性对象？ 结论是它监听的是 HTML attribute 的字符串变化。原因是 Custom Elements 需要和声明式 HTML 兼容，而 attribute 天然只能表达字符串。复杂数据应通过 property、事件或序列化协议传递，并在 attribute/property 之间保持同步规则。
  • Web Components 与 React/Vue 组件如何选型和集成？ 结论是跨技术栈、生命周期长的基础组件适合 Web Components，强依赖框架状态和生态的业务组件适合框架组件。原因是 Web Components 提供标准分发形式，但表单绑定、事件、SSR 和类型体验在不同框架中仍有差异。实际项目要提前约定 props、events 和样式定制接口。
