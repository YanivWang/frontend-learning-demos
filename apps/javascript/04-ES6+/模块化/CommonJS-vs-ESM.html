CommonJS 与 ES Module 对比

请打开 DevTools Console 查看输出。

知识点要点

  • CommonJS：require / module.exports，运行时加载，同步为主，可写在条件语句里。
  • ES Module：import / export，静态解析，异步加载，支持 tree-shaking。
  • CJS 导出值拷贝/引用组合；ESM 导出实时绑定，源模块变量变化后 import 方能观察到新值。
  • 浏览器原生 ESM 需 <script type="module">；Node.js 现代项目默认 ESM 或 .mjs。

面试考点

  • CJS vs ESM 加载时机？ CJS 运行时 require，可动态；ESM 静态 import 编译期建立依赖图，动态场景用 import()。
  • 导出值有何不同？ CJS 导出快照/引用组合；ESM 实时绑定，export 的变量变化 import 方同步可见。
  • 现代前端选型？ 浏览器与 bundler 默认 ESM；Node 遗留 CJS 可用 createRequire 或 .cjs 混用。
