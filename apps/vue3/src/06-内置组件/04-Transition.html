Transition

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Transition 包裹单个条件渲染元素、v-show 元素、动态组件或组件根节点，为进入和离开过程添加过渡能力。
  • 设置 name="fade" 后，Vue 会自动添加 fade-enter-from、fade-enter-active、fade-enter-to、fade-leave-from、fade-leave-active、fade-leave-to 等阶段 class。
  • 本页用 opacity 和 transform 做进出场，因为它们通常可由合成线程处理，比频繁动画 height、top 这类布局属性更稳；性能总览见 08-性能优化。
  • 如果同时使用 CSS transition 和 animation，或过渡嵌套在内部元素上，可能需要显式设置 type 或 duration，避免 Vue 过早判定过渡结束。
  • 列表增删、排序动画应改用 TransitionGroup，并为每项提供稳定 key；key 与 diff 见 08-虚拟DOM-diff-key。

面试考点

  • Transition 的 enter / leave class 生命周期是什么？ 结论是 Vue 在插入和移除元素时自动添加不同阶段的 class。进入阶段从 *-enter-from 到 *-enter-to，活跃期间保留 *-enter-active；离开阶段同理。边界是 name、CSS 类名和实际过渡属性必须匹配，否则看不到动画。
  • v-if、v-show、动态组件使用 Transition 有什么差异？ v-if 会触发真实插入和移除，v-show 切换 display，动态组件切换组件实例。Transition 都能介入，但状态保留、生命周期和 DOM 是否销毁不同。项目中弹窗常用 v-if 或 v-show，tab 组件切换还要考虑 KeepAlive。
  • 为什么动画优先用 transform 和 opacity？ 因为这两个属性通常不触发布局重排，浏览器更容易合成优化。动画 height、width、top 可能引发布局和绘制成本。边界是仍要控制元素数量和阴影、滤镜等昂贵效果。
  • Transition 和 TransitionGroup 怎么选？ 单个元素、单个组件或路由页面切换用 Transition。列表增删、排序和多个节点移动用 TransitionGroup，并且每项必须有稳定 key。不要用 Transition 包多个兄弟节点来做列表动画。
