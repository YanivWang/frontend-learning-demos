Vue2 数组响应式限制

依次点击五个按钮，对比「索引赋值 / 改 length」与 Vue.set、splice 正确写法下 items 与下方日志是否同步更新；复习要点与面试答法见本区块。

知识点要点

  • Vue2 无法用 Object.defineProperty 可靠拦截数组索引和 length 变化，因此改写了 7 个会改变原数组的方法：push、pop、shift、unshift、splice、sort、reverse。
  • this.items[1] = "B" 会改变数组本身，但不会触发 Vue2 的依赖通知；页面可能要等下一次其他响应式更新才“顺带”显示新值。
  • this.items.length = 1 同样不会触发视图更新。要删除或截断数组，使用 splice，例如 this.items.splice(1)。
  • 修复索引替换有两种常用写法：Vue.set(this.items, index, value) 表意明确，this.items.splice(index, 1, value) 更贴近数组变更方法。
  • 大数组更新还要考虑性能：批量更新时减少多次响应式通知，必要时先在普通数组中整理数据，再一次性替换响应式数组；列表渲染还要配合稳定 key。

面试考点

  • Vue2 为什么能监听 push 却不能监听索引赋值？ 结论是 Vue2 对数组方法做了包装，但没有为每个可能的索引都建立拦截。原因是 Object.defineProperty 的逐属性模型不适合动态数组索引和 length。项目里用变更方法或 Vue.set，不要直接写 arr[i] = value 后期待页面刷新。
  • Vue.set(arr, index, value) 和 splice 替换怎么选？ 结论是二者都能触发更新，单点替换时 Vue.set 语义直观，涉及插入、删除、截断时用 splice。原因是 splice 属于 Vue2 包装过的数组变更方法。边界是索引必须有效，越界或负数索引要先明确业务含义。
  • 直接修改 length 为什么不可靠？ 结论是它会改变数组长度但不会通知 Vue2 视图更新。原因是 length 不是 Vue2 包装的变更方法调用。项目里清空数组可用 this.items = [] 或 this.items.splice(0)，截断可用 splice(newLength)。
  • 大列表数组更新在 Vue2 中要注意什么性能边界？ 结论是减少高频逐项响应式写入。原因是每次变更都可能触发依赖通知和后续渲染调度。项目里可先构造普通数组再整体赋值，并配合分页、虚拟列表和稳定 key 控制渲染成本。
