组件 v-model

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • Vue 3 组件上的默认 v-model 等价于传入 modelValue，并监听 update:modelValue；本页 SmartForm 用输入事件触发父组件更新。
  • v-model:subtitle 会展开为 subtitle prop 和 update:subtitle 事件，因此同一组件可以暴露多个独立模型，适合表单组件拆分主值、副值或配置项。
  • 子组件接收的 prop 是只读输入，不能直接修改；正确做法是声明 emits 并发出 update:xxx，让父组件作为单一数据源决定是否接受新值。
  • 组件 v-model 只是父子同步协议，不会自动处理输入校验、空字符串、数字转换、去空格或 IME 输入边界；这些应通过组件 props、修饰符或内部校验明确约定。
  • 用户输入通过模板插值显示时会被 Vue 转义为文本；如果把输入拼进 v-html 或第三方 DOM API，就需要额外处理 XSS 风险。

面试考点

  • Vue 3 组件 v-model 编译后等价于什么？ 结论是默认等价于 :modelValue="x" 加 @update:modelValue="x = $event"。原因是 Vue 把双向绑定拆成单向数据输入和事件输出，仍保持父组件拥有状态。边界是组件必须按约定声明 prop 和 emit，否则绑定不会形成预期同步。
  • 多个 v-model 怎么实现？ 结论是使用带参数写法，例如本页的 v-model:subtitle。它会匹配 subtitle prop 与 update:subtitle 事件，适合一个组件暴露多个可编辑字段。项目里要统一命名，避免 prop 名、事件名和文档不一致。
  • 为什么子组件不能直接修改 modelValue？ 结论是 prop 属于父组件数据源，子组件直接写会破坏单向数据流并触发警告。通过 emit 更新能让父组件参与校验、拒绝或转换数据。复杂表单中还要定义空值、非法值、提交失败时的回滚策略。
  • 什么时候用 v-model，什么时候用普通 props / emit？ 可编辑表单值、开关状态、选择器值优先用 v-model，因为语义是“父子同步一个模型”。一次性动作、保存、删除、加载失败等事件用普通 emit 更清晰。只读展示仍用 props，不要为了省事件把所有通信都包装成 v-model。
