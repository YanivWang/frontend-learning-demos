动态路径参数 :id 与 $route.params

在浏览器中操作下方交互演示；复习要点与面试答法见上方区块。

知识点要点

  • vue-router 3 在 path 中用冒号声明动态片段，例如 /user/:id/post/:postid，匹配后可以通过 this.$route.params.id 读取。
  • params 是路径结构的一部分，适合资源标识；query 是问号后的附加参数，适合筛选、分页、来源标记等可选信息。
  • 从 /user/wangcheng/post/123 切到 /user/wangcheng/post/456 时，匹配的是同一个路由记录，组件实例会复用，created 不会再次执行。
  • 参数变化后的数据刷新应放在 watch: { $route() {} } 或组件内守卫 beforeRouteUpdate 中，避免只在初始化钩子里请求一次数据。
  • 真实项目还要校验参数是否存在、是否符合格式、资源是否可访问；失败时显示空状态、错误提示或跳转 404，不能假设 URL 永远正确。

面试考点

  • vue-router 3 动态路径参数如何声明和读取？ 结论是在路由 path 中用 :name 声明，在组件中通过 $route.params.name 读取。原因是路由匹配器会把动态片段解析成 params。边界是 query 不会进入 params，二者不能混用。
  • params 变化为什么不会重新触发 created？ 结论是同一个路由记录只是参数变化时，Vue Router 会复用原组件实例。原因是复用比销毁重建更高效，也能保留局部状态。项目里如果要重新请求详情，需要 watch $route 或使用 beforeRouteUpdate。
  • params 和 query 应该如何区分？ 结论是资源身份用 params，可选条件用 query。原因是 params 通常是路径模式的一部分，query 可以增删而不改变路由层级。比如 /users/42 表示用户 42，?tab=orders 表示当前展示订单页签。
  • 动态路由参数缺失或格式错误时怎么兜底？ 结论是要校验并处理失败路径。原因是用户可以手动改 URL，参数也可能过期或没有权限。项目里常做格式判断、接口 404 展示、路由守卫拦截或跳转统一错误页。
