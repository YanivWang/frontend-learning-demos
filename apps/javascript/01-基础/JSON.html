JSON.stringify 进阶用法

请打开 DevTools Console 查看输出。

知识点要点

  • JSON.stringify(value, replacer?, space?) 仅支持 object、array、string、number、boolean、null。
  • replacer 为数组时作属性白名单；为函数时按 (key, value) 返回值，返回 undefined 则忽略该属性。
  • undefined、function、Symbol 在对象中会被忽略；数组中变成 null；Date 变 ISO 字符串。
  • 循环引用会抛 TypeError；NaN/Infinity 变成 null。
  • JSON.parse 不能解析 trailing comma、单引号等非标准 JSON。

面试考点

  • JSON.stringify 有哪些限制？ 结论是它只表达 JSON 数据模型，不是 JS 对象完整克隆器。对象中的 undefined、函数和 Symbol 会被忽略，数组中这些值会变 null，循环引用会抛错；项目里要先确认数据是否适合序列化。
  • replacer 怎么用？ 结论是数组形式用于白名单字段，函数形式用于逐个属性转换或过滤。原因是 stringify 会把每个键值传给 replacer，返回 undefined 表示省略该对象属性；边界是数组元素返回 undefined 会序列化为 null。
  • 能用 JSON 做深拷贝吗？ 简单纯数据对象可以，但它会丢失 Date、Map、Set、函数、undefined、Symbol 和循环引用等信息。现代项目更推荐 structuredClone() 或按业务模型显式拷贝。
