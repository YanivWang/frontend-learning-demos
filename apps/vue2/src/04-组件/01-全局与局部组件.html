Vue.component 全局组件 + data 与 vm 代理同步

查看 #app_1 的 todo-item 列表渲染；在 #app_2 点击文字切换 msg，打开 Console 观察 data.a 与 app_2.a、 app_2.$data === data 的同步日志。

知识点要点

  • Vue 2 组件是可复用的 Vue 实例。Vue.component('todo-item', options) 会做全局注册，当前根实例及其子组件树在注册之后编译模板时都能识别这个组件。
  • props 是父组件向子组件传值的声明入口；本页的 todo-item 接收 todo，并在模板里读取 todo.name 与 todo.age。子组件不应直接修改 prop，而应通过事件请求父组件更新。
  • new Vue({ data }) 会把 data 对象上的既有字段代理到实例上，所以示例里 app_2.a 与 data.a 互相同步，且 app_2.$data === data。
  • Vue 2 响应式初始化发生在实例创建阶段；创建后给根级 data 或实例动态添加新字段，不会自动变成响应式。维护旧项目时用 Vue.set 或预声明字段，新项目应优先使用 Vue 3。
  • 同一元素上重复写同名事件监听没有教学价值且容易误导维护者；实际项目应合并处理逻辑或拆成语义明确的方法。

面试考点

  • Vue.component 注册的组件什么时候能在模板里使用？ 结论：全局注册完成后、且早于根实例模板编译时即可使用。原因是 Vue 在编译模板前需先解析标签对应的组件定义。边界是注册顺序必须在前，局部注册则只对声明它的组件模板可见。
  • props、data 代理和 vm.$data 的关系怎么解释？ 结论：props 是父到子的外部输入，data 字段会代理到 vm 上，vm.$data 指向原始 data 对象。原因是 Vue 初始化时把 data 既有属性代理到实例。项目里改 vm.a 与改 data.a 等价，但动态新增根级字段不会自动响应式。
  • Vue 2 里为什么不建议运行时给根 data 动态加新字段？ 结论：实例创建后新增的根级字段不会自动具备响应式能力。原因是 Vue 2 基于 Object.defineProperty 在初始化时拦截已有属性。边界是嵌套对象可用 Vue.set，根级字段应预声明在 data 中。
  • 全局组件适合什么场景，什么时候应该改为局部组件？ 结论：跨页面复用的基础组件可全局注册，业务组件优先局部注册。原因是局部注册让依赖更显式、便于按需加载。Vue 2 已是历史版本语境，新项目通常应直接使用 Vue 3 的组件组织方式。
