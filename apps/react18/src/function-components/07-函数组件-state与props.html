函数组件 state 与 props 综合

请打开 DevTools Console 查看 props/state 变化日志；下方 #root 为 React 挂载/交互演示区。

知识点要点

  • props 是父组件拥有并传入的只读输入，state 是组件自己拥有的可变记忆。本 demo 中 title 来自父组件 props，localCount 是子组件 state。
  • 父组件输入框更新 parentTitle 后，父组件重新渲染并把新的 title props 传给子组件，所以子组件展示会同步变化。
  • 子组件点击“本地 +1”只更新自己的 localCount，不会直接改父组件 state。React 的数据流默认从父到子，避免子组件偷偷改外部数据。
  • 子组件需要影响父组件时，应调用父组件通过 props 传下来的回调。本 demo 的 onNotify 由父组件提供，子组件只负责在事件发生时调用。
  • 当多个兄弟组件需要共享同一份数据，或父组件必须控制子组件输入值时，应把 state 提升到最近共同父组件；纯局部交互则留在组件内部。

面试考点

  • props 和 state 的所有权分别属于谁？ 结论是 props 属于父组件，state 属于声明它的组件。原因是 React 通过所有权让数据流可预测；项目里不要在子组件里修改 props，也不要把同一份数据维护成两份 state。
  • 父组件输入框变化为什么会让子组件同步更新？ 结论是父 state 变化触发父组件重新渲染，新 props 会传给子组件。原因是子组件的 title 没有自己的副本；边界是如果子组件把 props 复制进 state，就可能出现不同步。
  • 子组件本地 state 会不会影响父组件？ 结论是不会自动影响。原因是 localCount 的 setter 只更新子组件自己的状态；如果业务需要通知父级，应通过回调、Context 或状态管理显式传递。
  • 子组件为什么通过 props 回调通知父组件？ 结论是子组件不拥有父级 state，只能通过父级提供的函数请求变更。原因是单向数据流；本 demo 的 onNotify 本质是父组件把 setMessage 下发给子组件。
  • 什么时候应把 state 提升到父组件？ 结论是当多个组件需要读取或修改同一份数据时提升 state。原因是单一数据源能避免状态分叉；项目里表单值、筛选条件、选中项通常会提升到共同父级。
