v-if / v-show 条件渲染

点击下方按钮，观察 v-if 节点被创建/销毁，v-show 节点始终存在但切换 display。

知识点要点

  • v-if 是真实条件渲染：条件为假时，对应元素、子组件和事件监听器都不会创建；条件从假变真时才创建 DOM。
  • v-show 不会销毁节点，它始终渲染在 DOM 中，只是在条件为假时添加类似 display: none 的内联样式。它适合 Tab、下拉层、提示块等频繁切换场景。
  • v-if 支持和 v-else、v-else-if 组成互斥分支。本页按钮切换 showFlag，可以看到 v-if 与 v-else 两块内容互相替换。
  • <template v-if> 可以条件渲染一组兄弟节点，但不会在最终 DOM 中增加额外包裹元素。它适合需要保持语义结构干净的列表、表格或多个并列节点。
  • 选型口诀是：初始不一定渲染、切换不频繁、涉及权限或昂贵子组件，用 v-if；需要频繁显示隐藏且保留内部状态，用 v-show。同时避免在同一个元素上混用复杂 v-if 和 v-for，可先用 computed 过滤列表。

面试考点

  • v-if 和 v-show 的根本区别是什么？ 结论是 v-if 控制是否创建 DOM，v-show 控制已创建 DOM 的显示隐藏。原因是 v-if 条件为假时不会渲染节点，v-show 只改 display。项目里权限块、重组件用 v-if，高频切换 UI 用 v-show。
  • 为什么频繁切换的弹层更适合 v-show，权限控制更适合 v-if？ 结论是频繁切换要降低反复创建销毁成本，权限控制要避免不该存在的节点被渲染。原因是 v-show 初始渲染成本更高但切换便宜，v-if 初始惰性但切换成本高。项目里还要结合内部状态是否需要保留来判断。
  • <template v-if> 解决了什么结构问题？ 结论是它能条件渲染一组节点而不额外生成包裹元素。原因是 template 是编译时容器，最终不会出现在 DOM 中。项目里表格行、列表项片段、多个并列控件需要一起出现或消失时很有用。
  • v-if 和 v-for 同用时为什么要谨慎？ 结论是同元素上混用会让条件判断和列表渲染耦合，既影响可读性也可能带来不必要的遍历。原因是 Vue 2 中 v-for 优先级高于 v-if，每一项都会先进入循环再判断条件。项目里通常用 computed 先过滤数组，再用 v-for 渲染结果。
