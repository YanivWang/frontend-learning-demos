class 表达式（匿名/具名）

请打开 DevTools Console 查看输出。

知识点要点

  • class 表达式会产生一个类值，可赋给变量、作为函数返回值或作为参数传递；匿名 class 的可访问名称通常来自接收它的变量。
  • 具名 class 表达式会额外创建一个只在类体内部可见的名称，适合在方法里引用当前类或让调试栈更清晰；外层作用域只能访问接收表达式结果的变量。
  • class 表达式和 class 声明共享 class 语义：类体默认严格模式，构造器必须通过 new 调用，实例方法定义在 prototype 上。
  • 表达式不会像函数声明那样提前可用；如果赋给 const 或 let，访问时还要遵守外层变量自己的暂时性死区。
  • 实际项目中，声明式 class 更适合稳定、顶层的领域模型；class 表达式常用于工厂函数、根据条件组合基类、测试里临时构造替身类等局部场景。
  • 没有 extends 的 class 是基类，constructor 内不能调用 super()；只有派生类构造器才需要先调用 super() 再使用 this。

面试考点

  • 匿名 class 表达式和具名 class 表达式有什么区别？ 结论是匿名表达式只把类值交给外部变量，具名表达式还提供一个类体内部可见的名字。原因是具名表达式的名称绑定被限制在 class 自身作用域内；项目里常用它改善调试栈或在类内部稳定引用当前类。
  • class 表达式和 class 声明如何区分？ 声明式 class 直接在当前作用域创建词法绑定，适合顶层模型；表达式是一个值，适合赋值、返回或按条件创建。边界是两者都有 class 的严格语义，不能把 class 表达式当成普通函数表达式直接调用。
  • 什么时候选择 class 表达式？ 当类定义是局部实现细节、需要从工厂函数返回，或要根据条件选择继承来源时，class 表达式更自然。业务长期维护的公开模型更建议用 class 声明，提高可读性和静态搜索能力。
  • 为什么具名 class 表达式的内部名外部访问不到？ 因为这个名字只绑定在 class 表达式自身作用域，用于类体内部引用，不会泄漏到外层。这样可以避免污染外层作用域；外部如果需要使用这个类，应通过接收它的变量访问。
