自定义组件上 v-bind:class 行为

用 DevTools 检查 load-comp 根节点 <p> 是否同时拥有 foo deer active 三个 class（父级 :class 合并到组件根节点）。

知识点要点

  • Vue 2 自定义组件必须有单一根节点；父组件写在组件标签上的 class 或 :class 会被合并到这个根节点上。
  • 本页 load-comp 的根节点是 <p class="foo deer"> ，父组件传入 activeClass 后，最终根节点会同时拥有 foo deer active 。
  • 组件根节点已有 class 不会被父级传入的 class 覆盖；它们会合并。这一点适合做“组件默认样式 + 使用方状态样式”的组合。
  • v-bind:class 可传字符串、对象或数组：简单固定类名用字符串，按布尔条件切换用对象，多组来源组合用数组。
  • Vue 3 支持多根节点组件，class/attrs 透传需要显式落到某个元素上；维护 Vue 2 内容时不要把 Vue 3 多根组件规则反套回来。

面试考点

  • Vue 2 组件标签上的 class 最终加到哪里？ 结论：加到组件模板的根节点上。原因是 Vue 2 组件只有一个根元素，外部传入的非 prop attribute 会落到该根元素。项目里常用它给通用组件追加状态类。
  • 根节点已有 class 会被覆盖吗？ 结论：不会，会合并。原因是 Vue 对 class 做了专门合并处理。边界是同名 CSS 规则仍会按层叠顺序生效，所以样式冲突要看选择器和加载顺序。
  • v-bind:class 三种写法怎么选？ 结论：字符串适合一个类名变量，对象适合按条件开关，数组适合组合多个来源。原因是 Vue 最终都会归一成 class 列表。项目里复杂条件建议放到 computed，保持模板可读。
  • Vue 2 和 Vue 3 的 class 透传有什么边界？ 结论：Vue 2 单根组件默认落到根节点，Vue 3 多根组件需要更明确地处理 attrs。原因是多根组件没有唯一目标元素。维护旧项目时要先确认版本，再判断透传行为。
