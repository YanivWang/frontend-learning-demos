5 条竖条交错弹跳的加载动画

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出 nth-child 对应 animation-delay 实测值。

知识点要点

  • 这个弹跳 loading 的核心不是写 5 套动画，而是让 5 个 span 复用同一个 @keyframes load 。
  • :nth-child() 为第 2 到第 5 个竖条设置递增 animation-delay ，从而让同一周期内的缩放状态错开，形成波浪感。
  • 本页用 transform: scaleY() 做竖向弹跳；相比动画 height 和 margin ，它更适合持续循环的装饰性动效。
  • 现代项目通常写标准 animation ，兼容旧 WebKit 时由 Autoprefixer 等工具按 browserslist 产出前缀；只有无构建流程且明确要兼容旧设备时才手写前缀。
  • 加载动画只说明“还在等待”，真实交互要补失败路径：请求失败、超时、用户取消和重复点击都需要明确反馈。

面试考点

  • 交错 loading 为什么常用 :nth-child() 加 delay？ 结论是它可以让多个相同元素复用同一套关键帧，只改变播放起点。原因是视觉差异来自时间相位，而不是每个元素的动画逻辑不同。项目里元素数量固定时可直接写 nth-child，数量动态时更适合用 CSS 变量传入 delay。
  • transform 弹跳和 height 弹跳怎么选？ 结论是装饰性 loading 优先 transform 。原因是 height 会改变盒子尺寸并影响布局流，而 transform 改变的是视觉变换。边界是如果页面需要真实撑开空间，就不能只用 transform，需要重新设计布局和动画范围。
  • 现在还需要手写 -webkit-animation 吗？ 结论是大多数现代项目不手写，交给构建工具按兼容目标生成。原因是标准 animation 已广泛可用，手写前缀容易和标准声明不一致。边界是离线 demo、老旧 WebView 或无构建环境可以按目标设备补前缀，但要成对维护。
  • loading 动效如何兼顾用户体验？ 结论是动画只是反馈，必须配合状态管理。原因是异步操作可能失败、超时或被重复触发。项目里应禁用重复提交、给出错误与重试入口，并在 prefers-reduced-motion 下减少循环运动。
