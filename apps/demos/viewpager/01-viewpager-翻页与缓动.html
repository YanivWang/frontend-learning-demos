ViewPager · 内置多种插值器（缓动函数）

这个 demo 用 ViewPager 和多种插值器演示翻页动画，重点观察线性进度如何被映射成不同手感，以及页面容器如何用 transform 移动。

知识点要点

  • 进度与插值：翻页动画先计算 0..1 的线性进度，再交给插值器映射成弹性、回弹或超调曲线，最后换算为位移。
  • 位移渲染：onPageScroll 返回 totalOffset，示例用 translate3d() 移动 .pager_items，避免每帧修改 left 这类更容易触发布局的属性。
  • 动画时钟：实践中动画循环通常使用 requestAnimationFrame()，并根据回调时间戳计算进度，避免高刷新率屏幕上动画速度变快。
  • CSS 边界：transform 不改变文档流中的占位，但非 none 值会创建 stacking context；复杂组件里要注意层叠、定位和点击命中。
  • 体验边界：弹性、回弹这类动效能增强反馈，但过强动画可能影响晕动敏感用户；项目里应考虑 prefers-reduced-motion 或提供关闭动效的策略。

面试考点

  • ViewPager 翻页动画为什么要把进度和位移分开？ 结论是进度描述动画完成比例，位移描述页面实际位置。这样同一套位移逻辑可以换不同插值器，项目里能复用成匀速、回弹、弹性等多种手感。
  • 插值器和 CSS transition timing-function 有什么相似点和区别？ 两者都把线性时间映射成非线性进度。区别是 JS 插值器可以写任意函数并和组件状态联动，CSS timing-function 更声明式，适合简单属性过渡。
  • 为什么翻页位移常用 transform 而不是 left？ transform 通常不会改变布局流，浏览器更容易把它交给合成阶段处理。边界是它会创建 stacking context，并不自动解决所有性能问题；元素过多或绘制内容复杂仍可能卡顿。
  • requestAnimationFrame 动画为什么要用时间戳计算进度？ 因为不同设备刷新率可能是 60Hz、120Hz 或更高，按帧累加会导致高刷新率设备动画更快。用时间戳计算 elapsed / duration，才能让动画时长稳定。
  • 动效组件需要考虑哪些边界和降级？ 至少要考虑快速连续点击、越界页码、窗口尺寸变化、后台标签暂停动画，以及用户开启减少动态效果偏好。这个 demo 聚焦插值器，不覆盖完整生产级状态机。
