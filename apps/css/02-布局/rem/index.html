rem 实现移动端等比适配

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出根字号与 rem 换算实测值。

知识点要点

  • rem 总是相对根元素 html 的 font-size 计算，示例把 750px 设计稿约定为 1rem = 100px ，所以 0.88rem 对应设计稿 88px。
  • 根字号由 calc(100vw / 7.5) 驱动时，视口变窄或变宽会让所有 rem 尺寸等比缩放；本页用 clamp() 增加上下限，避免桌面端字号过大或超窄屏过小。
  • viewport 的 width=device-width 让布局视口贴近设备宽度；现代页面不建议写 maximum-scale=1 禁止缩放，因为会影响低视力用户放大页面。
  • rem 适合整套视觉稿按比例换算， em 适合相对当前元素字体， vw 适合直接按视口比例表达尺寸，固定 px 适合边框、图标线条等不需要缩放的细节。
  • 真实项目里 rem 方案要配合最大内容宽度、断点或 clamp() ，并验证长文本、横屏、系统字体放大和固定底栏遮挡等边界。

面试考点

  • rem 适配的换算原理是什么？ 结论是所有 rem 尺寸都乘以根元素字号。原因是 rem 的参考值固定在 html ，只要按设计稿宽度动态改根字号，页面就能整体等比缩放。边界是图片、长文本和固定定位元素仍要单独验证。
  • 用 vw 计算根字号时为什么要加 clamp() ？ 结论是为了限制极端视口下的最小和最大字号。原因是纯 100vw / 7.5 在桌面窗口会把整页放得过大，在超窄屏又可能小到不可读。项目里通常还会加最大宽度容器或响应式断点。
  • rem、em、vw、px 怎么选？ 结论是整页设计稿换算用 rem，局部跟随字体用 em，随视口变化的容器尺寸可用 vw，精细边框和不可缩放视觉细节可用 px。原因是它们的参考基准不同。现代响应式页面常把 rem 与 flex/grid、媒体查询、 clamp() 组合使用。
  • maximum-scale=1 为什么不应作为默认写法？ 结论是不应默认禁止用户缩放。原因是缩放是重要的可访问性能力，移动端浏览器也会对禁用缩放有不同处理。只有非常特殊的沉浸式交互才评估限制缩放，普通业务页应保留用户放大能力。
