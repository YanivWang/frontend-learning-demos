useEffect 副作用、依赖数组与清理函数

请打开 DevTools Console 查看 effect 建立/cleanup 与 document.title 同步；下方 #root 为 React 挂载/交互演示区。

知识点要点

  • useEffect 表达“这次渲染结果需要和哪个外部系统同步”，常见外部系统包括定时器、DOM API、浏览器事件、网络连接和第三方库。
  • 依赖数组不是手动优化开关，而是 effect 读取的响应式值清单；示例中定时器依赖 step ，所以 step 变化时必须重新建立定时器。
  • cleanup 会在组件卸载前执行，也会在依赖变化导致下一次 setup 前执行；定时器、订阅和事件监听如果不清理，会造成重复执行或内存泄漏。
  • 空依赖数组表示 effect 不读取组件内会变化的响应式值；如果实际读取了却强行省略，容易产生旧闭包和线上难复现的状态不同步。
  • React 18 开发环境 Strict Mode 会额外做一次 setup + cleanup + setup，用来检查 cleanup 是否完整；生产环境不会因此重复挂载业务逻辑。

面试考点

  • useEffect 到底同步什么？ 结论是同步 React 渲染结果与外部系统。原因是渲染本身应保持纯净，DOM 标题、定时器、订阅和网络连接这类副作用必须在提交到页面后处理。
  • 依赖数组应该怎么写？ 结论是 effect 中读取的 props、state 和组件内声明的响应式值都应列入依赖。原因是 React 依赖这份清单决定何时重新同步外部系统。
  • 为什么 cleanup 会在下一次 effect 前执行？ 结论是 React 要先撤销上一轮渲染对应的外部副作用，再建立新副作用。原因是每次 effect 都属于某一次渲染快照。
  • React 18 Strict Mode 下 effect 执行两次是不是 bug？ 结论是不是 bug，这是开发环境的检查。原因是 React 通过额外 setup + cleanup 暴露不可重复执行或清理不完整的副作用。
