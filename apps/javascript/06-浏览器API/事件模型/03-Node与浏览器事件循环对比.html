Node.js vs 浏览器事件循环

在浏览器中操作下方按钮；要点与面试答法见上方复习区。

知识点要点

  • 浏览器宏任务：setTimeout、setInterval、I/O、UI 渲染；微任务：Promise.then、MutationObserver。
  • Node.js 宏任务：timers、poll、check、setImmediate；微任务：Promise.then、queueMicrotask。
  • Node 特有 process.nextTick，优先级高于 Promise 微任务。
  • Node 11+ 微任务在阶段之间清空，与浏览器更接近。

面试考点

  • 浏览器执行顺序？ 同步 → 微任务（Promise/queueMicrotask）→ 宏任务（setTimeout）。
  • Node 与浏览器差异？ Node 多 process.nextTick（最先）和 setImmediate；Node 11+ 微任务清空时机更接近浏览器。
  • Node 典型输出？ sync 1,5 → nextTick 4 → microtask 3 → macrotask 2（浏览器无 nextTick 则为 1,5,3,2）。
