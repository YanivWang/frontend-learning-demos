手写 new

先看本复习区，再查看下方输出是否与标准答法一致；细节可打开 DevTools Console。

知识点要点

  • new Constructor(...args) 的核心步骤是：创建新对象、连接原型、用新对象作为 this 执行构造函数、按返回值规则决定最终结果。
  • Object.create(Constructor.prototype) 用来创建一个原型指向构造函数 prototype 的对象，因此实例可以访问原型方法，并满足 instanceof Constructor 。
  • 构造函数执行时， this 指向新实例，像 this.name = name 这样的赋值会写入实例自身属性。
  • 如果构造函数显式返回对象或函数， new 表达式会返回这个对象；如果返回原始值或没有返回值，则返回前面创建的新实例。
  • 真实规范还涉及 new.target 、派生类构造器、绑定函数、不可构造函数等细节；本 demo 是面试版核心模拟，不等同完整 polyfill。
  • 现代代码如果需要以参数数组动态构造对象，优先使用 Reflect.construct() 或展开语法 new Ctor(...args) ，不要在业务里自定义 new 。

面试考点

  • new 操作符具体做了哪几步？ 结论：创建对象、连接原型、绑定 this 执行构造函数、处理返回值。原因是实例既要拥有构造函数写入的自身属性，也要通过原型链访问方法。边界是构造函数必须可构造，箭头函数和普通非构造调用不能直接套用这套语义。
  • 构造函数显式 return 对 new 结果有什么影响？ 结论：返回对象或函数会覆盖默认实例，返回原始值会被忽略。原因是规范对构造调用有专门的返回值处理规则，只有对象类型结果才会成为最终构造结果。项目里构造函数通常不显式返回对象，避免实例语义变得难读。
  • Object.create(Constructor.prototype) 解决了什么？ 结论：它让新对象的原型指向构造函数原型。原因是原型链决定方法查找和 instanceof 判断；如果只写 {} ，实例就访问不到 Constructor.prototype 上的方法。边界是如果 prototype 不是对象，原生 new 会退回默认对象原型，手写实现要考虑兜底。
  • 手写 new 和 Reflect.construct 有什么差异？ 结论：手写版只模拟核心流程， Reflect.construct 更接近规范级构造。原因是 Reflect.construct 能处理 newTarget 、类构造器和内置构造语义；而 Constructor.apply(instance, args) 不能直接调用 class 构造器。面试答核心步骤即可，生产动态构造应使用语言内置能力。
