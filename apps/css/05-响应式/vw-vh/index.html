rem、@media 与 vw/vh 的组合响应式布局

在浏览器中打开本页，结合下方演示区观察效果；要点与面试答法见本复习区，脚本会输出 vw/vh/rem 实测值。调整浏览器宽高观察断点与视口单位变化。

知识点要点

  • rem 是相对根元素 html 的字体大小。本页在 1600px 、 1280px 、 960px 等区间切换 html { font-size: ... } ，正文 1.3rem 和侧栏 1rem 会跟着根字号等比变化。
  • vw / vh 是视口百分比单位， 1vw 等于视口宽度的 1%， 1vh 等于视口高度的 1%。顶部演示块的 50vw 和 12vh 会直接随窗口宽高变化，不依赖父元素字号。
  • width: min(980px, 96vw) 表示内容宽度取 980px 与 96vw 中较小值：大屏限制最大行宽，小屏保留约 2vw 的左右边距，比固定 980px 更不容易横向溢出。
  • @media 、 rem 和 vw/vh 适合解决不同问题：断点负责结构变化， rem 适合字号和间距体系， vw/vh 适合随视口变化的容器或舞台尺寸。混用时要避免所有尺寸都跟视口线性缩放，导致极端屏幕上过大或过小。
  • 移动浏览器的地址栏、工具栏会改变可见视口，传统 vh 可能导致“100vh 被遮挡”或高度跳动；企业主线示例里应保留内容可滚动、避免关键操作贴死底部，必要时再用脚本计算可用高度。

面试考点

  • rem 和 vw/vh 的参照物分别是什么，为什么不能混为一谈？ 结论是 rem 参照根元素字号， vw/vh 参照视口宽高。原因是 rem 更适合统一文字、间距和组件比例， vw/vh 更适合做随屏幕尺寸变化的容器。项目里如果把字号全部设成 vw ，极窄或超宽屏会出现可读性问题，通常要配合 clamp() 或断点限制。
  • 用 @media 改 html font-size 做 rem 适配有哪些收益和风险？ 结论是收益是少量规则就能联动一批 rem 尺寸，风险是全局根字号变化会影响所有依赖 rem 的组件。原因是 rem 链路很短但作用域是全局的。项目里如果有第三方组件、用户浏览器字号设置或无障碍放大需求，应避免过度覆盖根字号，必要时用局部 CSS 变量控制比例。
  • min(980px, 96vw) 相比固定宽度或单纯百分比有什么优势？ 结论是它同时表达最大宽度和窄屏流式宽度。原因是固定 980px 会在小屏溢出，单纯 96vw 又会在超宽屏形成过长行宽； min() 能让容器在两种边界下都保持合理。项目里也常见等价写法 width: 96vw; max-width: 980px 。
  • 移动端使用 vh 做整屏或大图高度时要注意什么？ 结论是要注意浏览器动态工具栏导致的可见高度变化。原因是传统 vh 与移动端可见区域的关系在不同浏览器历史实现中并不完全一致，可能出现内容被底栏遮挡或滚动时跳动。企业主线里不要把关键内容完全依赖固定视口高度，必要时用 JS 兜底并保留内容可滚动。

前端对于网站来说，通常是指，网站的前台部分包括网站的表现层和结构层。因此前端技术一般分为前端设计和前端开发，前端设计一般可以理解为网站的视觉设计，前端开发则是网站的前台代码实现，包括基本的HTML和CSS以及JavaScript/ajax，现在最新的高级版本HTML5、CSS3，以及SVG等。

html指的是超文本标记语言 (Hyper Text Markup Language)，这个也是我们网页最常用普通的语言了，经历了多个版本的发展，已经发展到5.0版了，得力于W3C建立的标准和规范，已普遍升级到了XHTML，XHTML 指可扩展超文本标签语言（EXtensible HyperText Markup Language）， XHTML 于2000年的1月26日成为 W3C 标准，是更严格更纯净的 HTML 代码，XHTML 的目标是取代 HTML。XHTML 与 HTML 4.01 几乎是相同的，XHTML 是作为一种 XML 应用被重新定义的 HTML，是一个 W3C 标准。W3C 将 XHTML 定义为最新的HTML版本。所有新的浏览器都支持 XHTML

css级联样式表（Cascading Style Sheet）简称“CSS”，通常又称为“风格样式表（Style Sheet）”，它是用来进行网页风格设计的。比如，如果想让链接字未点击时是蓝色的，当鼠标移上去后字变成红色的且有下划线，这就是一种风格。通过设立样式表，可以统一地控制HTML中各标志的显示属性。级联样式表可以使人更能有效地控制网页外观。使用级联样式表，可以扩充精确指定网页元素位置，外观以及创建特殊效果的能力

javascript是一种由Netscape的LiveScript发展而来的原型化继承的面向对象的动态类型的区分大小写的客户端脚本语言，主要目的是为了解决服务器端语言，比如Perl，遗留的速度问题，为客户提供更流畅的浏览效果。当时服务端需要对数据进行验证，由于网络速度相当缓慢，只有28.8kbps，验证步骤浪费的时间太多。于是Netscape的浏览器Navigator加入了Javascript，提供了数据验证的基本功能
